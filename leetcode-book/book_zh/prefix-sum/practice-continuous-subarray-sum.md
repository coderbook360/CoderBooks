# 实战：连续的子数组和

这道题来自 LeetCode 523，是上一题"和可被 K 整除"的变体，但多了一个长度限制。

## 题目描述

给你一个整数数组 `nums` 和一个整数 `k`，如果 `nums` 有一个**好的子数组**，返回 `true`，否则返回 `false`。

一个**好的子数组**是：
- 长度**至少为 2**
- 子数组元素的总和为 `k` 的倍数

**示例 1**：

```
输入：nums = [23,2,4,6,7], k = 6
输出：true
解释：[2, 4] 是一个大小为 2 的子数组，和为 6。
```

**示例 2**：

```
输入：nums = [23,2,6,4,7], k = 6
输出：true
解释：[23, 2, 6, 4, 7] 是大小为 5 的子数组，和为 42，42 是 6 的倍数。
```

**示例 3**：

```
输入：nums = [23,2,6,4,7], k = 13
输出：false
```

## 与上一题的区别

上一题要求统计**个数**，这一题只需要判断**是否存在**。

更关键的区别是：这一题要求子数组长度**至少为 2**。

## 思路分析

沿用上一题的同余思路：
- 如果 `prefix[j] % k == prefix[i] % k`，则子数组 `nums[i..j-1]` 的和能被 k 整除
- 该子数组的长度是 `j - i`

我们需要 `j - i >= 2`，也就是说，当我们在位置 j 遇到余数 r 时，需要检查**是否存在 i <= j-2 使得 prefix[i] % k == r**。

关键技巧：**记录每个余数第一次出现的位置**，而不是记录计数。

这样当同一个余数再次出现时，只要间隔足够大（>= 2），就找到了答案。

## 代码实现

```javascript
function checkSubarraySum(nums, k) {
    // remainderIndex[r] 记录余数 r 第一次出现的索引
    const remainderIndex = new Map();
    remainderIndex.set(0, -1); // 空前缀在索引 -1 处
    
    let prefixSum = 0;
    
    for (let i = 0; i < nums.length; i++) {
        prefixSum += nums[i];
        
        const remainder = ((prefixSum % k) + k) % k;
        
        if (remainderIndex.has(remainder)) {
            // 检查长度是否 >= 2
            if (i - remainderIndex.get(remainder) >= 2) {
                return true;
            }
            // 不更新索引，保持第一次出现的位置
        } else {
            remainderIndex.set(remainder, i);
        }
    }
    
    return false;
}
```

## 为什么保持第一次出现的位置

考虑这种情况：

```
nums = [1, 2, 3], k = 6
前缀和：1, 3, 6
余数：  1, 3, 0
```

如果余数 3 在索引 1 第一次出现，后来又在某个更大的索引处出现，我们希望得到**最大的间隔**。

保持第一次出现的位置，可以让后续的配对有更大的机会满足长度要求。

## 初始化 remainderIndex[0] = -1 的含义

这处理的是"从数组开头到某位置的子数组和恰好是 k 的倍数"的情况。

举例：`nums = [6, 1], k = 6`

- 遍历到索引 0，prefixSum = 6，余数 = 0
- `remainderIndex[0] = -1`，间隔 = 0 - (-1) = 1 < 2，不满足
- 遍历到索引 1，prefixSum = 7，余数 = 1，记录
- 没找到

但如果是 `nums = [0, 6], k = 6`：

- 遍历到索引 0，prefixSum = 0，余数 = 0，间隔 = 0 - (-1) = 1 < 2
- 遍历到索引 1，prefixSum = 6，余数 = 0，间隔 = 1 - (-1) = 2 >= 2，返回 true

## 特殊情况：k = 0？

题目约束 `k >= 1`，所以不需要处理 k = 0 的情况。如果需要处理，当 k = 0 时，"和为 k 的倍数"就变成了"和为 0"。

## 图解执行过程

以 `nums = [23, 2, 4, 6, 7], k = 6` 为例：

```
初始：remainderIndex = {0: -1}

i = 0, num = 23:
  prefixSum = 23
  remainder = 23 % 6 = 5
  5 不在 map 中，记录 remainderIndex[5] = 0

i = 1, num = 2:
  prefixSum = 25
  remainder = 25 % 6 = 1
  1 不在 map 中，记录 remainderIndex[1] = 1

i = 2, num = 4:
  prefixSum = 29
  remainder = 29 % 6 = 5
  5 在 map 中，位置 0
  间隔 = 2 - 0 = 2 >= 2，返回 true!
```

对应子数组 `[2, 4]`，和为 6，长度为 2。

## 复杂度分析

**时间复杂度**：O(n)，最坏情况遍历整个数组

**空间复杂度**：O(min(n, k))，最多存储 k 个不同的余数

## 与上一题的对比

| 题目 | 目标 | 存储内容 | 额外约束 |
|-----|------|---------|---------|
| 和可被K整除 | 统计个数 | 余数出现次数 | 无 |
| 连续的子数组和 | 判断存在 | 余数首次出现位置 | 长度 >= 2 |

## 小结

这道题的技巧是：

1. 用同余原理转化问题
2. 记录余数**第一次出现的位置**（而非计数）
3. 检查配对时验证长度约束

当问题从"统计个数"变成"是否存在"，并且有额外约束时，记录位置比记录计数更合适。
