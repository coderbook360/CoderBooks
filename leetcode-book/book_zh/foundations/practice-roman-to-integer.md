# 实战：罗马数字转整数

这道题来自 LeetCode 第 13 题，考察规则理解和模式识别能力。

## 题目描述

罗马数字包含以下七种字符：`I`、`V`、`X`、`L`、`C`、`D`、`M`。

| 字符 | 数值 |
|-----|-----|
| I | 1 |
| V | 5 |
| X | 10 |
| L | 50 |
| C | 100 |
| D | 500 |
| M | 1000 |

例如，罗马数字 `2` 写做 `II`，即为两个并列的 `1`。`12` 写做 `XII`，即为 `X + II`。`27` 写做 `XXVII`，即为 `XX + V + II`。

通常情况下，罗马数字中小的数字在大的数字右边。但也存在特例，如 `4` 写作 `IV`（5-1=4），`9` 写作 `IX`（10-1=9）。

**六种特殊情况**：
- `IV` = 4，`IX` = 9
- `XL` = 40，`XC` = 90
- `CD` = 400，`CM` = 900

给定一个罗马数字，将其转换成整数。

**示例**：

```
输入：s = "III"
输出：3

输入：s = "IV"
输出：4

输入：s = "IX"
输出：9

输入：s = "LVIII"
输出：58
解释：L = 50, V= 5, III = 3

输入：s = "MCMXCIV"
输出：1994
解释：M = 1000, CM = 900, XC = 90, IV = 4
```

## 分析规则

观察罗马数字的规律：

1. **一般情况**：从左到右，大数在前，小数在后，直接累加
2. **特殊情况**：小数在大数前面时，表示减法

换句话说：**如果当前字符代表的值比下一个字符小，就减去当前值；否则加上当前值**。

## 实现代码

```javascript
function romanToInt(s) {
    const values = {
        'I': 1,
        'V': 5,
        'X': 10,
        'L': 50,
        'C': 100,
        'D': 500,
        'M': 1000
    };
    
    let result = 0;
    
    for (let i = 0; i < s.length; i++) {
        const current = values[s[i]];
        const next = values[s[i + 1]] || 0;
        
        if (current < next) {
            // 特殊情况：小数在大数前，做减法
            result -= current;
        } else {
            // 一般情况：直接累加
            result += current;
        }
    }
    
    return result;
}
```

## 图解执行过程

以 `"MCMXCIV"` 为例（结果应为 1994）：

```
位置 0：M = 1000，下一个 C = 100
        1000 > 100，累加 → result = 1000

位置 1：C = 100，下一个 M = 1000
        100 < 1000，减去 → result = 900

位置 2：M = 1000，下一个 X = 10
        1000 > 10，累加 → result = 1900

位置 3：X = 10，下一个 C = 100
        10 < 100，减去 → result = 1890

位置 4：C = 100，下一个 I = 1
        100 > 1，累加 → result = 1990

位置 5：I = 1，下一个 V = 5
        1 < 5，减去 → result = 1989

位置 6：V = 5，没有下一个
        累加 → result = 1994
```

以 `"IV"` 为例（结果应为 4）：

```
位置 0：I = 1，下一个 V = 5
        1 < 5，减去 → result = -1

位置 1：V = 5，没有下一个
        累加 → result = 4
```

## 另一种思路：先处理特殊情况

```javascript
function romanToInt(s) {
    const values = {
        'I': 1, 'V': 5, 'X': 10, 'L': 50,
        'C': 100, 'D': 500, 'M': 1000
    };
    
    // 先替换特殊组合
    s = s.replace('IV', 'a')
         .replace('IX', 'b')
         .replace('XL', 'c')
         .replace('XC', 'd')
         .replace('CD', 'e')
         .replace('CM', 'f');
    
    const special = {
        'a': 4, 'b': 9, 'c': 40, 'd': 90, 'e': 400, 'f': 900
    };
    
    let result = 0;
    for (const char of s) {
        result += values[char] || special[char];
    }
    
    return result;
}
```

这种方法更直观，但多了字符串替换的开销。

## 复杂度分析

**方法一（比较相邻）**：
- 时间复杂度：O(n)，遍历字符串一次
- 空间复杂度：O(1)，只用了固定的映射表

**方法二（替换特殊）**：
- 时间复杂度：O(n)
- 空间复杂度：O(n)，创建了新字符串

## 边界情况

1. **单个字符**：如 `"I"`，直接返回其值
2. **全是相同字符**：如 `"III"`，直接累加
3. **最大值**：题目保证输入在 1 到 3999 之间

## 小结

这道题的核心是识别罗马数字的规则：
- 正常情况：大数在前，小数在后，累加
- 特殊情况：小数在大数前，做减法

通过比较相邻字符的大小关系，可以简洁地处理所有情况。
