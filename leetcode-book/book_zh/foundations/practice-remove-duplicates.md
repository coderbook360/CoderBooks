# 删除排序数组中的重复项

> LeetCode 26. Remove Duplicates from Sorted Array

给你一个**升序排列**的数组，请你**原地**删除重复出现的元素，使每个元素只出现一次，返回删除后数组的新长度。

这道题的关键词是"原地"——不能创建新数组，只能在原数组上操作。这正是双指针大显身手的舞台。

## 问题描述

```javascript
输入：nums = [0, 0, 1, 1, 1, 2, 2, 3, 3, 4]
输出：5
解释：函数返回新长度 5，nums 的前五个元素为 [0, 1, 2, 3, 4]
```

**注意**：不需要考虑数组中超出新长度后面的元素。

## 思路分析

### 利用有序性质

这道题有一个重要的前提：**数组已经排好序了**。

这意味着什么？**重复的元素一定是相邻的**。

```
[0, 0, 1, 1, 1, 2, 2, 3, 3, 4]
    ↑     ↑ ↑     ↑     ↑
    重复   重复   重复   重复
```

如果数组是无序的，比如 `[1, 0, 1, 2, 0]`，重复元素就可能分散在各处，处理起来会复杂得多。但有序数组的这个特性，让我们可以用一次遍历解决问题。

### 双指针思想

想象你在整理一排书架：

- 你用左手（慢指针）标记"已经整理好的区域"的末尾
- 你用右手（快指针）去扫描后面的书
- 每发现一本不重复的新书，就放到左手的下一个位置

具体来说：

- **慢指针 `slow`**：指向"不重复序列"的最后一个位置
- **快指针 `fast`**：负责扫描数组，寻找新的不重复元素

当 `nums[fast]` 和 `nums[slow]` 不相等时，说明发现了一个新元素，把它放到 `slow + 1` 的位置。

## 解法详解

### 代码实现

```javascript
function removeDuplicates(nums) {
    // 空数组直接返回
    if (nums.length === 0) return 0;
    
    // slow 指向不重复序列的末尾
    let slow = 0;
    
    // fast 从第二个元素开始扫描
    for (let fast = 1; fast < nums.length; fast++) {
        // 发现新元素
        if (nums[fast] !== nums[slow]) {
            slow++;  // 扩展不重复区域
            nums[slow] = nums[fast];  // 放入新元素
        }
        // 如果相等，说明是重复元素，fast 继续前进，slow 不动
    }
    
    // slow 是索引，长度要 +1
    return slow + 1;
}
```

### 执行过程图解

以 `[0, 0, 1, 1, 1, 2, 2, 3, 3, 4]` 为例：

```
初始状态：slow=0, fast=1
[0, 0, 1, 1, 1, 2, 2, 3, 3, 4]
 ↑  ↑
 s  f

第1步：nums[1]=0 == nums[0]=0，重复，跳过
       fast++ → fast=2

第2步：nums[2]=1 != nums[0]=0，新元素！
       slow++ → slow=1
       nums[1] = 1
[0, 1, 1, 1, 1, 2, 2, 3, 3, 4]
    ↑     ↑
    s     f
       fast++ → fast=3

第3步：nums[3]=1 == nums[1]=1，重复，跳过
       fast++ → fast=4

第4步：nums[4]=1 == nums[1]=1，重复，跳过
       fast++ → fast=5

第5步：nums[5]=2 != nums[1]=1，新元素！
       slow++ → slow=2
       nums[2] = 2
[0, 1, 2, 1, 1, 2, 2, 3, 3, 4]
       ↑           ↑
       s           f
       fast++ → fast=6

... 继续执行 ...

最终状态：slow=4
[0, 1, 2, 3, 4, 2, 2, 3, 3, 4]
             ↑              ↑
             s              f

返回 slow + 1 = 5
```

前 5 个元素 `[0, 1, 2, 3, 4]` 就是去重后的结果。后面的元素是"垃圾数据"，题目说不用管它们。

## 复杂度分析

**时间复杂度：O(n)**

- 快指针从头到尾扫描一遍数组
- 每个元素最多被访问一次

**空间复杂度：O(1)**

- 只使用了两个指针变量
- 没有创建额外的数组

这就是"原地算法"的魅力——用最少的空间完成任务。

## 边界情况

写代码时，边界情况往往是 bug 的温床：

```javascript
// 空数组
removeDuplicates([])  // 返回 0

// 单元素
removeDuplicates([1])  // 返回 1，数组不变

// 全部重复
removeDuplicates([1, 1, 1])  // 返回 1，结果 [1, ...]

// 没有重复
removeDuplicates([1, 2, 3])  // 返回 3，数组不变
```

我们的代码已经正确处理了这些情况：
- 空数组在开头就返回 0
- 单元素时 `fast` 不会进入循环
- 全部重复时 `slow` 始终为 0
- 没有重复时每个元素都会被"复制"到原位置

## 常见误区

**误区一：使用额外空间**

```javascript
// ❌ 错误：违反了"原地修改"的要求
function removeDuplicates(nums) {
    const unique = [...new Set(nums)];
    return unique.length;
}
```

虽然这段代码能得到正确的长度，但它：
1. 创建了新数组，空间复杂度 O(n)
2. 没有修改原数组

**误区二：忘记处理空数组**

```javascript
// ❌ 错误：空数组会导致问题
function removeDuplicates(nums) {
    let slow = 0;
    for (let fast = 1; fast < nums.length; fast++) {
        // 当 nums.length = 0 时，循环不会执行
        // 但返回 slow + 1 = 1，这是错的！
    }
    return slow + 1;
}
```

所以我们需要在开头检查空数组。

## 小结

这道题教会我们：

1. **利用有序性**：有序数组的重复元素一定相邻
2. **双指针技巧**：快指针扫描，慢指针维护结果区域
3. **原地操作**：不创建新数组，在原数组上修改

双指针是数组问题的核心技巧之一，你会在后续的题目中反复遇到它。每次使用时，想想这道题的思路：一个指针负责"探索"，一个指针负责"守护"。
