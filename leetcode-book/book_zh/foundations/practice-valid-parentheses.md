# 实战：有效的括号

这是你的第一道算法实战题，来自 LeetCode 第 20 题。虽然是"简单"难度，但它引出了一个非常重要的数据结构——**栈**。

## 题目描述

给定一个只包括 `'('`、`')'`、`'{'`、`'}'`、`'['`、`']'` 的字符串 `s`，判断字符串是否有效。

有效字符串需满足：
1. 左括号必须用相同类型的右括号闭合
2. 左括号必须以正确的顺序闭合
3. 每个右括号都有一个对应的相同类型的左括号

**示例**：

```
输入：s = "()"
输出：true

输入：s = "()[]{}"
输出：true

输入：s = "(]"
输出：false

输入：s = "([)]"
输出：false

输入：s = "{[]}"
输出：true
```

## 分析问题

乍一看，你可能会想：统计每种括号的数量不就行了？

```javascript
// 错误思路
function isValid(s) {
    let round = 0;  // ()
    let square = 0; // []
    let curly = 0;  // {}
    
    for (const char of s) {
        if (char === '(') round++;
        if (char === ')') round--;
        if (char === '[') square++;
        if (char === ']') square--;
        if (char === '{') curly++;
        if (char === '}') curly--;
    }
    
    return round === 0 && square === 0 && curly === 0;
}
```

这个方法对 `"([)]"` 会返回 true——三种括号数量都平衡了。但实际上 `"([)]"` 是无效的！

问题在哪？**顺序**。

`"([)]"` 中，`[` 和 `)` 相遇时，它们不匹配。正确的嵌套应该是 `"([])"` 或 `"([])"`。

## 关键观察：后进先出

观察有效的括号串 `"{[()]}"`：

```
{  [  (  )  ]  }
↓  ↓  ↓  ↓  ↓  ↓
入 入 入 出 出 出
```

最后进入的 `(`，需要最先被 `)` 匹配。这是一个典型的**后进先出（LIFO）**模式——栈的特性！

## 栈解法

**算法思路**：

1. 从左到右遍历字符串
2. 遇到左括号 `(`、`[`、`{`：压入栈
3. 遇到右括号 `)`、`]`、`}`：
   - 检查栈是否为空（空说明没有左括号可匹配）
   - 检查栈顶是否是对应的左括号
   - 如果匹配，弹出栈顶
   - 如果不匹配，返回 false
4. 遍历结束后，栈必须为空

```javascript
function isValid(s) {
    // 奇数长度必定无效
    if (s.length % 2 !== 0) {
        return false;
    }
    
    // 定义括号配对关系
    const pairs = {
        ')': '(',
        ']': '[',
        '}': '{'
    };
    
    const stack = [];
    
    for (const char of s) {
        // 如果是右括号
        if (char in pairs) {
            // 栈为空，或者栈顶不匹配
            if (stack.length === 0 || stack[stack.length - 1] !== pairs[char]) {
                return false;
            }
            // 匹配成功，弹出栈顶
            stack.pop();
        } else {
            // 左括号，入栈
            stack.push(char);
        }
    }
    
    // 栈为空才是有效的
    return stack.length === 0;
}
```

## 图解执行过程

以 `"{[()]}"` 为例：

```
字符: {  →  栈: ['{']
字符: [  →  栈: ['{', '[']
字符: (  →  栈: ['{', '[', '(']
字符: )  →  栈顶 '(' 匹配 ')' ✓  →  栈: ['{', '[']
字符: ]  →  栈顶 '[' 匹配 ']' ✓  →  栈: ['{']
字符: }  →  栈顶 '{' 匹配 '}' ✓  →  栈: []

栈为空，返回 true
```

以 `"([)]"` 为例：

```
字符: (  →  栈: ['(']
字符: [  →  栈: ['(', '[']
字符: )  →  栈顶 '[' 不匹配 ')' ✗

立即返回 false
```

以 `"((("` 为例：

```
字符: (  →  栈: ['(']
字符: (  →  栈: ['(', '(']
字符: (  →  栈: ['(', '(', '(']

遍历结束，栈不为空，返回 false
```

## 为什么用对象存储配对关系

你可能注意到我们用了一个对象来存储括号的配对关系：

```javascript
const pairs = {
    ')': '(',
    ']': '[',
    '}': '{'
};
```

为什么不直接用 if-else？两个原因：

1. **可读性**：配对关系一目了然
2. **可扩展性**：如果要支持新的括号类型，只需修改对象

用 if-else 写的话：

```javascript
if (char === ')') {
    if (stack.pop() !== '(') return false;
} else if (char === ']') {
    if (stack.pop() !== '[') return false;
} else if (char === '}') {
    if (stack.pop() !== '{') return false;
}
```

功能相同，但代码更冗长，也更容易出错。

## 边界情况

在写算法题时，一定要考虑边界情况：

1. **空字符串**：按题意，空字符串是有效的（栈为空）
2. **单个字符**：如 `"("` 或 `")"`，必定无效
3. **奇数长度**：必定无效，可以提前返回
4. **全是左括号**：如 `"((([[["`，遍历结束栈不为空
5. **以右括号开头**：如 `")()"`，一开始栈就是空的，无法匹配

## 复杂度分析

**时间复杂度**：O(n)
- 遍历字符串一次
- 每个字符的入栈/出栈操作都是 O(1)

**空间复杂度**：O(n)
- 最坏情况：字符串全是左括号，如 `"((((("`
- 此时栈需要存储 n/2 个元素

## 小结

这道题的核心是识别**后进先出**的模式，然后用栈来实现。

关键点：
1. 遇到左括号就入栈
2. 遇到右括号就和栈顶匹配
3. 最后栈必须为空

栈是一种简单但非常有用的数据结构。在后面的章节中，你会看到更多栈的应用，比如表达式求值、单调栈等。
