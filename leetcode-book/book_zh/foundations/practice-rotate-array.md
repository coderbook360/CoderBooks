# 旋转数组

> LeetCode 189. Rotate Array

给定一个数组，将数组中的元素向右移动 `k` 个位置。

这道题看似简单，却有多种解法，从暴力到优雅，体现了算法设计的不同层次。其中"三次翻转"的解法尤为精妙，值得细细品味。

## 问题描述

```javascript
输入：nums = [1, 2, 3, 4, 5, 6, 7], k = 3
输出：[5, 6, 7, 1, 2, 3, 4]

解释：
向右移动 1 步: [7, 1, 2, 3, 4, 5, 6]
向右移动 2 步: [6, 7, 1, 2, 3, 4, 5]
向右移动 3 步: [5, 6, 7, 1, 2, 3, 4]
```

**要求**：尽可能想出更多的解决方案，至少有三种不同的方法可以解决这个问题。

## 解法详解

### 解法一：额外数组

最直观的思路：创建一个新数组，把每个元素放到旋转后的正确位置。

元素 `nums[i]` 旋转后的位置是 `(i + k) % n`。

```javascript
function rotate(nums, k) {
    const n = nums.length;
    k = k % n;  // 处理 k > n 的情况
    
    const temp = new Array(n);
    
    // 把每个元素放到新位置
    for (let i = 0; i < n; i++) {
        temp[(i + k) % n] = nums[i];
    }
    
    // 把结果复制回原数组
    for (let i = 0; i < n; i++) {
        nums[i] = temp[i];
    }
}
```

**为什么要 `k % n`**？

如果 `k = 10`，数组长度 `n = 7`，旋转 10 次等于旋转 3 次（10 % 7 = 3）。旋转 n 次会回到原位，所以只需要考虑 `k % n`。

**复杂度**：
- 时间：O(n)
- 空间：O(n)，需要额外数组

### 解法二：翻转数组（推荐）

这是最优雅的解法：**三次翻转**实现旋转。

```javascript
function rotate(nums, k) {
    const n = nums.length;
    k = k % n;
    
    // 三次翻转
    reverse(nums, 0, n - 1);     // 1. 整体翻转
    reverse(nums, 0, k - 1);     // 2. 翻转前 k 个
    reverse(nums, k, n - 1);     // 3. 翻转后 n-k 个
}

function reverse(nums, start, end) {
    while (start < end) {
        [nums[start], nums[end]] = [nums[end], nums[start]];
        start++;
        end--;
    }
}
```

**复杂度**：
- 时间：O(n)，每个元素被翻转两次
- 空间：O(1)，原地操作

### 翻转过程图解

以 `[1, 2, 3, 4, 5, 6, 7]`，`k = 3` 为例：

```
原数组:
[1, 2, 3, 4, 5, 6, 7]

第一步：整体翻转
[1, 2, 3, 4, 5, 6, 7]  →  [7, 6, 5, 4, 3, 2, 1]
 ↑                 ↑       ↑                 ↑
 交换              交换     交换              交换

第二步：翻转前 k=3 个
[7, 6, 5, 4, 3, 2, 1]  →  [5, 6, 7, 4, 3, 2, 1]
 ↑     ↑                   ↑     ↑
 翻转这部分

第三步：翻转后 n-k=4 个
[5, 6, 7, 4, 3, 2, 1]  →  [5, 6, 7, 1, 2, 3, 4]
          ↑--------↑              ↑--------↑
          翻转这部分

最终结果：[5, 6, 7, 1, 2, 3, 4] ✓
```

## 为什么三次翻转有效？

这个技巧看起来像魔法，但背后有严谨的数学原理。

设原数组为 `[A | B]`，其中：
- `A` 是前 `n-k` 个元素
- `B` 是后 `k` 个元素

我们的目标是得到 `[B | A]`。

**观察翻转的性质**：
- 翻转 `A` 得到 `A'`（A 的逆序）
- 翻转 `B` 得到 `B'`
- 翻转 `A'` 得到 `A`（翻转两次回到原样）

**三次翻转的过程**：

```
原数组:          [A | B]
整体翻转:        [B' | A']    （整体逆序）
翻转前k个(B'):   [B | A']     （B'翻转回B）
翻转后n-k个(A'): [B | A]      （A'翻转回A）
```

最终得到 `[B | A]`，正是我们想要的结果！

用具体例子验证：

```
原数组:   [1,2,3,4 | 5,6,7]    A=[1,2,3,4], B=[5,6,7]
整体翻转: [7,6,5 | 4,3,2,1]    B'=[7,6,5], A'=[4,3,2,1]
翻转B':   [5,6,7 | 4,3,2,1]    B=[5,6,7]
翻转A':   [5,6,7 | 1,2,3,4]    A=[1,2,3,4]
```

### 解法三：环状替换

每个元素最终要到达的位置是确定的：`(i + k) % n`。我们可以直接把元素"传递"到目标位置：

```javascript
function rotate(nums, k) {
    const n = nums.length;
    k = k % n;
    
    let count = 0;  // 已移动的元素数量
    
    for (let start = 0; count < n; start++) {
        let current = start;
        let prev = nums[start];
        
        do {
            const next = (current + k) % n;  // 目标位置
            const temp = nums[next];         // 保存目标位置的值
            nums[next] = prev;               // 放入当前值
            prev = temp;                     // 更新要放入的值
            current = next;                  // 移动到下一个位置
            count++;
        } while (start !== current);  // 直到回到起点
    }
}
```

**为什么会形成"环"**？

从位置 0 出发，跳到 `k`，再跳到 `2k`...最终会回到 0，形成一个环。

但有时候一个环不能覆盖所有元素（当 `n` 和 `k` 有公约数时），需要从下一个起点开始新的环。

**例子**：`n = 6, k = 2`

```
环1: 0 → 2 → 4 → 0（回到起点）
环2: 1 → 3 → 5 → 1（回到起点）
两个环，覆盖全部6个元素
```

**复杂度**：
- 时间：O(n)
- 空间：O(1)

## 复杂度对比

| 解法 | 时间复杂度 | 空间复杂度 | 特点 |
|------|-----------|-----------|------|
| 额外数组 | O(n) | O(n) | 最直观 |
| 三次翻转 | O(n) | O(1) | 最优雅 |
| 环状替换 | O(n) | O(1) | 数学思想 |

**推荐**：三次翻转法，代码简洁，原地操作，而且原理一旦理解就不会忘记。

## 边界情况

```javascript
// k = 0，不需要旋转
rotate([1, 2, 3], 0)  // [1, 2, 3]

// k = n，旋转回原位
rotate([1, 2, 3], 3)  // [1, 2, 3]

// k > n
rotate([1, 2], 3)     // 等价于 k = 1，结果 [2, 1]

// 单元素数组
rotate([1], 5)        // [1]

// 负数元素
rotate([-1, -100, 3, 99], 2)  // [3, 99, -1, -100]
```

## 常见误区

**误区一：忘记处理 k > n**

```javascript
// ❌ k = 10, n = 3 时，(i + 10) % 3 虽然结果正确
// 但如果先 k = k % n，可以避免不必要的计算
```

**误区二：翻转边界搞错**

```javascript
// ❌ 错误：翻转 k 个，但边界是 k
reverse(nums, 0, k);    // 应该是 k-1

// ✅ 正确：索引 0 到 k-1 共 k 个元素
reverse(nums, 0, k - 1);
```

## 小结

这道题展示了解决同一问题的多种思路：

1. **额外空间**：最直观，空间换简单
2. **三次翻转**：最优雅，利用翻转的数学性质
3. **环状替换**：最直接，追踪每个元素的最终位置

三次翻转法是一个经典技巧，它的核心思想是：**两次翻转可以抵消，但翻转的顺序不同会产生"移位"效果**。

掌握这个技巧后，类似的"循环移位"问题都可以迎刃而解。
