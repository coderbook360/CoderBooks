# 数组拆分

> LeetCode 561. Array Partition

给定长度为 2n 的整数数组，将这些数分成 n 对，使得每对的较小值之和最大。

这道题的解法出奇的简单，但理解**为什么这样做是对的**才是关键。

## 问题描述

```javascript
输入：nums = [1, 4, 3, 2]
输出：4

解释：
- 可能的配对方式：
  - (1,4), (2,3) → min(1,4) + min(2,3) = 1 + 2 = 3
  - (1,3), (2,4) → min(1,3) + min(2,4) = 1 + 2 = 3
  - (1,2), (3,4) → min(1,2) + min(3,4) = 1 + 3 = 4 ✓ 最大
```

## 思路分析

### 问题转化

我们要最大化：`min(a1,b1) + min(a2,b2) + ... + min(an,bn)`

每对取较小值，意味着每对中较大的那个数被"浪费"了。

**目标**：让浪费最小 = 让每对的差距最小

### 贪心策略

**排序后相邻配对**。

如果数组排序后是 `[a1, a2, a3, a4, ...]`（升序），那么：
- 配对 `(a1, a2)`，贡献 `a1`
- 配对 `(a3, a4)`，贡献 `a3`
- ...

答案就是所有**偶数索引**元素之和。

## 为什么排序后相邻配对是最优的？

**直觉理解**：

假设数组是 `[1, 2, 3, 4]`。

如果配对 `(1, 4)` 和 `(2, 3)`：
- 贡献：1 + 2 = 3
- 浪费：4-1=3 和 3-2=1，总浪费 4

如果配对 `(1, 2)` 和 `(3, 4)`：
- 贡献：1 + 3 = 4
- 浪费：2-1=1 和 4-3=1，总浪费 2

相邻配对的浪费更小！

**数学证明**（简化版）：

设排序后数组为 `s[0] ≤ s[1] ≤ ... ≤ s[2n-1]`。

对于任意配对方案：
- 总和 = 所有数之和 - 每对的"浪费"
- 浪费 = 每对较大值 - 较小值 = 每对的差

要最大化总和，就要最小化总浪费。

当相邻配对时，每对的差是 `s[1]-s[0]`, `s[3]-s[2]`, ...，这是最小可能的差之和。

## 解法详解

```javascript
function arrayPairSum(nums) {
    // 排序
    nums.sort((a, b) => a - b);
    
    let sum = 0;
    // 取偶数索引的元素
    for (let i = 0; i < nums.length; i += 2) {
        sum += nums[i];
    }
    
    return sum;
}
```

### 执行过程

以 `[1, 4, 3, 2]` 为例：

```
排序后: [1, 2, 3, 4]

配对: (1,2), (3,4)

取偶数索引:
  i=0: sum += 1  → sum = 1
  i=2: sum += 3  → sum = 4

返回 4
```

### 更简洁的写法

```javascript
function arrayPairSum(nums) {
    return nums
        .sort((a, b) => a - b)
        .filter((_, i) => i % 2 === 0)
        .reduce((sum, num) => sum + num, 0);
}
```

或者用 `reduce` 一行搞定：

```javascript
function arrayPairSum(nums) {
    nums.sort((a, b) => a - b);
    return nums.reduce((sum, num, i) => i % 2 === 0 ? sum + num : sum, 0);
}
```

## 复杂度分析

**时间复杂度：O(n log n)**
- 排序是主要开销
- 遍历求和是 O(n)

**空间复杂度：O(log n)**
- 排序的递归栈空间
- 如果用原地排序，额外空间就是排序的开销

## 边界情况

```javascript
// 只有一对
arrayPairSum([1, 2])  // 1

// 有负数
arrayPairSum([-1, 2, -3, 4])  
// 排序: [-3, -1, 2, 4]
// 配对: (-3,-1), (2,4)
// 答案: -3 + 2 = -1

// 有重复
arrayPairSum([1, 1, 1, 1])  // 1 + 1 = 2
```

## 小结

这道题的核心洞察：

1. **问题转化**：最大化最小值之和 = 最小化浪费
2. **贪心策略**：相邻配对浪费最小
3. **实现简单**：排序后取偶数索引求和

贪心算法的关键是找到"局部最优"的判断标准。在这道题中，"相邻配对"就是那个简单而正确的策略。
