# 实战：四数相加II

这道题展示了哈希表在处理"多数组求和"问题中的分治思想——把四数之和拆成两个两数之和。

## 问题描述

给你四个整数数组`nums1`、`nums2`、`nums3`和`nums4`，数组长度都是n。请你计算有多少个元组`(i, j, k, l)`满足：

`nums1[i] + nums2[j] + nums3[k] + nums4[l] == 0`

**示例**：
```
输入: 
nums1 = [1,2]
nums2 = [-2,-1]
nums3 = [-1,2]
nums4 = [0,2]

输出: 2

解释:
(0,0,0,1) → 1 + (-2) + (-1) + 2 = 0
(1,1,0,0) → 2 + (-1) + (-1) + 0 = 0
```

## 思路分析

### 暴力方法：O(n⁴)

四重循环枚举所有组合，太慢。

### 分治思想

`a + b + c + d = 0`等价于`a + b = -(c + d)`

把四个数组分成两组：
1. 枚举nums1和nums2的所有和，存入Map
2. 枚举nums3和nums4的所有和，查找相反数

## 完整实现

```javascript
/**
 * @param {number[]} nums1
 * @param {number[]} nums2
 * @param {number[]} nums3
 * @param {number[]} nums4
 * @return {number}
 */
function fourSumCount(nums1, nums2, nums3, nums4) {
    const map = new Map();
    
    // 存储前两个数组所有可能的和
    for (const a of nums1) {
        for (const b of nums2) {
            const sum = a + b;
            map.set(sum, (map.get(sum) || 0) + 1);
        }
    }
    
    let count = 0;
    
    // 查找后两个数组和的相反数
    for (const c of nums3) {
        for (const d of nums4) {
            const target = -(c + d);
            if (map.has(target)) {
                count += map.get(target);
            }
        }
    }
    
    return count;
}
```

## 执行过程图解

以示例为例：

```
nums1 = [1,2], nums2 = [-2,-1]
nums3 = [-1,2], nums4 = [0,2]

第一步：存储 nums1 + nums2 的所有和
  1 + (-2) = -1
  1 + (-1) = 0
  2 + (-2) = 0
  2 + (-1) = 1
  
map = {-1→1, 0→2, 1→1}

第二步：枚举 nums3 + nums4，查找相反数
  (-1) + 0 = -1, target = 1, map.get(1) = 1, count += 1
  (-1) + 2 = 1, target = -1, map.get(-1) = 1, count += 1
  2 + 0 = 2, target = -2, 不存在
  2 + 2 = 4, target = -4, 不存在

count = 2
```

## 为什么存储计数而非布尔值？

因为可能有多个组合产生相同的和：

```
nums1 = [0,0], nums2 = [0,0]

0 + 0 = 0 出现4次
map = {0 → 4}
```

如果只存储"是否存在"，会丢失计数信息。

## 为什么两组各两个数组？

分组方式的选择影响时间复杂度：

| 分组方式 | 第一步 | 第二步 | 总复杂度 |
|----------|--------|--------|----------|
| 1+3 | O(n) | O(n³) | O(n³) |
| 2+2 | O(n²) | O(n²) | O(n²) |
| 3+1 | O(n³) | O(n) | O(n³) |

2+2的分组使两步复杂度平衡，达到最优O(n²)。

## 变体：如果是同一个数组？

如果四个数都来自同一个数组（LeetCode 454的变体），需要注意索引不能重复使用。

但本题是四个独立的数组，索引自然不会冲突。

## 复杂度分析

**时间复杂度：O(n²)**
- 第一步：枚举n²个组合
- 第二步：枚举n²个组合，每次O(1)查找

**空间复杂度：O(n²)**
- Map最多存储n²个不同的和

## 与"两数之和"的联系

| 问题 | 方法 |
|------|------|
| 两数之和 | 遍历时存储，查找配对 |
| 四数相加 | 先存前两数之和，再查找后两数之和的相反数 |

四数相加就是"两数之和"的升级版——把两组数的和看作"两个数"。

## 为什么这题不能用双指针？

双指针需要**排序**数组，但这里是四个独立数组的组合问题，排序后索引信息丢失，且无法在四维空间高效应用双指针。

哈希表的"查找配对"能力在这里完美适用。

## 小结

四数相加II的核心：

1. **分治思想**：`a+b+c+d=0` → `a+b=-(c+d)`
2. **平衡分组**：2+2分组达到O(n²)最优
3. **存储计数**：Map存储`和 → 出现次数`
4. **查找相反数**：第二步查找`-(c+d)`

这道题的思想可以推广到任意偶数个数组的求和问题——分成两半，分别计算所有可能的和，然后配对。
