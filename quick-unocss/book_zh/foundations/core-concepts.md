# 核心概念：规则、预设与变体

欢迎来到 UnoCSS 的核心。如果你曾对原子化 CSS 的魔法感到好奇——那些看似随机的字符是如何转化为精确样式——那么这一章将为你揭开谜底。UnoCSS 的强大之处并非源于海量的预定义类，而是其优雅且可扩展的架构。这个架构建立在三个基本概念之上：**规则 (Rules)**、**预设 (Presets)** 和 **变体 (Variants)**。

理解这三者如何协同工作，是你从“使用”UnoCSS 到“精通”UnoCSS 的关键一步。它们共同构成了一个动态的样式生成引擎，能即时响应你的每一个类名需求，赋予你前所未有的灵活性和控制力。

---

### 1. 规则 (Rules)：样式的基本原子

规则是 UnoCSS 中最基本的构建单元。它的职责非常纯粹：**将一个类名映射到一段具体的 CSS 代码**。当你写下一个工具类时，正是规则在幕后将其“翻译”为浏览器能够理解的样式。

规则分为两种：

- **静态规则 (Static Rules)**：提供一对一的精确匹配。例如，`flex` 这个类名总是生成 `display: flex;`。
- **动态规则 (Dynamic Rules)**：通过正则表达式捕捉模式，并从中提取数值来动态生成样式。这是 UnoCSS 实现“按需生成”的魔力所在。

思考一下 `m-4` 这个类名。它并非存在于某个巨大的 CSS 文件中，而是被一条动态规则所捕捉。这条规则的设计模式大致是 `m-(\d+)`，它会：
1. 匹配以 `m-` 开头的类名。
2. 捕获后面的数字 `4`。
3. 将数字 `4` 转换为 `margin: 1rem;`（假设 `1` 单位等于 `0.25rem`）。

正是这种动态机制，让你能够写出 `m-10`、`p-3.5`、`w-1/2` 等无穷无尽的工具类，而无需担心打包体积。

### 2. 预设 (Presets)：规则与配置的集合

如果说规则是砖块，那么**预设 (Presets)** 就是精心设计的建筑图纸和材料包。预设是规则、变体、快捷方式（Shortcuts）甚至主题（Theme）的集合。

它的核心价值在于：
- **开箱即用**：像 `preset-wind` 和 `preset-uno` 这样的官方预设，为你提供了大量基于流行框架（如 Tailwind CSS）的默认规则和工具类，让你无需从零配置即可快速启动项目。
- **封装设计系统**：你可以将公司的设计规范——包括颜色、间距、字体等——封装成一个独立的预设。这使得在多个项目中复用和同步设计系统变得异常简单。
- **社区共享**：开发者可以分享自己的预设，极大地丰富了 UnoCSS 的生态。

在配置文件中，你只需简单地引入并声明预设，即可立即获得其提供的所有能力：

```typescript
// uno.config.ts
import { defineConfig, presetUno } from 'unocss'

export default defineConfig({
  presets: [
    presetUno() // 引入官方默认预设
  ]
})
```

### 3. 变体 (Variants)：样式的“条件状语”

如果说规则定义了“做什么”（例如，`bg-blue-500` 设置蓝色背景），那么**变体 (Variants)** 就定义了“在什么条件下做”。变体是强大的修饰符，它为你的工具类添加了上下文或条件。

最常见的变体包括：
- **响应式设计**：如 `md:`、`lg:`，允许样式仅在特定的屏幕尺寸下生效。
- **伪类与伪元素**：如 `hover:`、`focus:`、`first:`、`::before`，为元素的不同状态或部分施加样式。
- **媒体查询**：如 `print:`、`dark:`，根据打印预览、暗黑模式等条件应用样式。
- **属性选择器**：如 `disabled:`、`aria-checked:`，根据元素的属性状态应用样式。

变体的强大之处在于其**组合能力**。你可以像搭积木一样将它们串联起来，创造出精确的、复杂的条件：

```html
<!-- 
  仅在中等屏幕 (md) 及以上尺寸、
  且鼠标悬停 (hover) 时，
  背景变为蓝色 (bg-blue-500)
-->
<button class="md:hover:bg-blue-500">
  Click me
</button>
```

在这个例子中，`md:` 和 `hover:` 两个变体共同作用于 `bg-blue-500` 这条规则，生成了如下高度精确的 CSS：

```css
@media (min-width: 768px) {
  .md\:hover\:bg-blue-500:hover {
    --un-bg-opacity: 1;
    background-color: rgba(59, 130, 246, var(--un-bg-opacity));
  }
}
```

### 总结：三位一体的动态引擎

规则、预设、变体，这三个核心概念共同构成了一个优雅而强大的系统：
- **预设**提供了一套基础的**规则**集和**变体**集。
- 你在代码中写下一个工具类，它可能来自**预设**中的某条**规则**。
- 你为这个工具类加上**变体**前缀，用于指定它生效的条件。
- UnoCSS 引擎实时解析这一切，动态生成最终的 CSS。

这个“三位一体”的架构，是 UnoCSS 实现极致灵活性、高性能和零体积打包的基石。掌握了它，你就掌握了创造无限样式可能性的钥匙。在接下来的章节中，我们将分别深入这三个概念，探索它们的更多玩法与高级技巧。
