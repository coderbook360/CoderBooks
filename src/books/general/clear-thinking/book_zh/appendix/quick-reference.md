## 问题定义

### 5个为什么

```
问题表象
    │
    └── 为什么？ → 直接原因
            │
            └── 为什么？ → 间接原因
                    │
                    └── 为什么？ → 深层原因
                            │
                            └── 为什么？ → 系统原因
                                    │
                                    └── 为什么？ → 根本原因
```

### 问题定义检查

- [ ] 这是真正的问题还是表象？
- [ ] 这是问题还是解决方案？
- [ ] 边界清晰吗？
- [ ] 可以采取行动吗？

### 期望值计算

```
期望值 = Σ (结果i × 概率i)

示例：
选项A：70%概率赚100万，30%概率亏50万
期望值 = 100 × 0.7 + (-50) × 0.3 = 55万
```

## 检验工具

### 预演验尸

```
1. 假设项目已失败
2. 头脑风暴失败原因
3. 识别之前忽视的风险
4. 制定预防措施
```

### 红队问题

- 如果我要推翻这个方案，会怎么攻击？
- 最薄弱的假设是什么？
- 竞争对手会如何反应？
- 执行中最可能出问题的是什么？

### 常见逻辑谬误速查

| 谬误 | 识别 |
|------|------|
| 虚假二分 | "只能A或B" |
| 滑坡谬误 | "A会导致B会导致C..." |
| 诉诸权威 | "专家说所以对" |
| 人身攻击 | 攻击人不攻击论点 |
| 稻草人 | 歪曲对方观点再攻击 |
| 循环论证 | 用结论证明结论 |

## 快速决策流程

### 5分钟决策

```
1. 问题是什么？（1分钟）
2. 有哪些选项？（1分钟）
3. 最佳选择是？（1分钟）
4. 主要风险是？（1分钟）
5. 下一步行动？（1分钟）
```

### 重大决策检查清单

- [ ] 问题定义清晰吗？
- [ ] 信息充分吗？
- [ ] 考虑了足够多选项吗？
- [ ] 用决策矩阵评估了吗？
- [ ] 做了预演验尸吗？
- [ ] 考虑了反对意见吗？
- [ ] 有备选方案吗？
- [ ] 设定了复盘节点吗？
