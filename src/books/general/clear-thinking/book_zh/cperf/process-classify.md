# 问题分类与工具选择

> "如果你只有锤子，所有问题看起来都像钉子。"

## 四种基本问题类型

不同类型的问题需要不同的思维工具。首先要正确分类问题。

### 类型一：事实性问题

**特征**：有客观答案，可以通过查找或验证来回答

**例子**：
- 这个API的调用频率限制是多少？
- 用户昨天的活跃数是多少？
- 这个库是否支持TypeScript？

**解决策略**：
- 查找文档
- 查询数据
- 验证测试

### 类型二：因果性问题

**特征**：需要分析事物之间的因果关系

**例子**：
- 为什么转化率下降了？
- 什么导致了系统崩溃？
- 用户为什么流失？

**解决策略**：
- 假设检验
- 对照分析
- 根因分析

### 类型三：比较选择问题

**特征**：需要在多个选项中做出选择

**例子**：
- 应该用React还是Vue？
- 是自建还是买服务？
- 先做功能A还是功能B？

**解决策略**：
- 建立评估标准
- 权衡利弊
- 决策矩阵

### 类型四：创造性问题

**特征**：需要产生新的想法或解决方案

**例子**：
- 如何提高用户参与度？
- 怎样设计更好的体验？
- 有什么创新的商业模式？

**解决策略**：
- 头脑风暴
- 类比思考
- 逆向思考

### 问题类型识别流程

```
                    这个问题有客观答案吗？
                           │
              ┌────────────┴────────────┐
              ↓ 是                       ↓ 否
         事实性问题              需要分析因果吗？
                                       │
                          ┌────────────┴────────────┐
                          ↓ 是                       ↓ 否
                     因果性问题              需要在选项中选择吗？
                                                   │
                                      ┌────────────┴────────────┐
                                      ↓ 是                       ↓ 否
                                 选择性问题                 创造性问题
```

## 演绎推理与归纳推理

### 演绎推理

**定义**：从一般到特殊，从前提推出必然结论

**结构**：
```
大前提：所有A都是B
小前提：X是A
结论：所以X是B
```

**特点**：
- 如果前提为真，结论必然为真
- 不增加新知识，只是展开前提中隐含的内容
- 结论的可靠性取决于前提的可靠性

**案例**：
```
大前提：所有内存泄漏都会导致性能下降
小前提：这个模块有内存泄漏
结论：所以这个模块会导致性能下降
```

**适用场景**：
- 应用规则和原则
- 逻辑验证
- 排除不符合条件的选项

### 归纳推理

**定义**：从特殊到一般，从观察推出可能的规律

**结构**：
```
观察1：A1是B
观察2：A2是B
观察3：A3是B
结论：可能所有A都是B
```

**特点**：
- 结论是概率性的，可能为真也可能为假
- 增加新知识，发现新规律
- 观察越多，结论越可靠

**案例**：
```
观察1：用户A投诉了加载慢
观察2：用户B投诉了加载慢
观察3：用户C投诉了加载慢
结论：可能系统整体存在性能问题
```

**适用场景**：
- 发现模式和规律
- 形成假设
- 预测趋势

### 演绎与归纳的结合

```
        归纳                    演绎
         │                       │
观察个案 ────→ 形成假设 ────→ 验证假设 ────→ 得出结论
         │                       │
      发现规律              应用规律
```

## 问题-工具对应矩阵

### 工具速查表

| 问题类型 | 推理方式 | 推荐工具 |
|---------|---------|---------|
| 事实性 | 演绎 | 查找、验证、计算 |
| 因果性 | 归纳+演绎 | 5Why、因果图、假设检验 |
| 选择性 | 演绎 | 决策矩阵、利弊分析、加权评估 |
| 创造性 | 归纳+发散 | 头脑风暴、SCAMPER、六顶思帽 |

### 工具详解

**因果分析工具**：
- **5Why分析**：连续追问为什么，找到根因
- **因果图（鱼骨图）**：系统梳理可能的原因
- **假设检验**：提出假设，设计验证方法

**选择决策工具**：
- **决策矩阵**：多维度评估各选项
- **利弊分析**：列出优缺点
- **加权评估**：根据重要性加权

**创造性工具**：
- **头脑风暴**：大量产生想法
- **SCAMPER**：系统性地改造现有方案
- **逆向思考**：从相反方向思考

## 【检查清单】问题类型识别检验

### 类型确认

- [ ] 这个问题属于哪种类型？
- [ ] 是否可能是复合问题？
- [ ] 需要拆分成更小的问题吗？

### 工具匹配

- [ ] 选择了适合这种问题的工具吗？
- [ ] 是否需要组合多种工具？
- [ ] 工具的使用方法清楚吗？

### 推理方式

- [ ] 需要演绎推理还是归纳推理？
- [ ] 前提/观察是否充分？
- [ ] 结论的确定性程度是多少？

### 复杂度评估

- [ ] 问题的复杂度如何？
- [ ] 需要多深入的分析？
- [ ] 时间和资源是否允许？

## 本章小结

- 问题分为事实性、因果性、选择性和创造性四类
- 演绎推理从一般到特殊，结论必然
- 归纳推理从特殊到一般，结论概率性
- 不同类型的问题需要不同的分析工具
- 使用检查清单确保问题分类正确
