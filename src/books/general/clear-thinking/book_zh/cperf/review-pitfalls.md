# 复盘中的常见陷阱

> "复盘不是为了证明自己是对的，而是为了下次做得更好。"

## 自我欺骗的形式

### 结果导向的归因

**表现**：
- 结果好 → 我的决策是对的
- 结果差 → 运气不好 / 别人的错

**真相**：
- 好决策可能有坏结果（坏运气）
- 坏决策可能有好结果（好运气）

**正确做法**：
评估决策时，关注决策过程而非结果

```
              ┌────────────────────────────┐
              │      结果好     结果差      │
              ├─────────────┬──────────────┤
    过程好    │  继续保持   │ 检查运气因素 │
              ├─────────────┼──────────────┤
    过程差    │  侥幸成功   │ 需要改进     │
              └─────────────┴──────────────┘
```

### 事后诸葛亮

**表现**：
- "我早就知道会这样"
- "这是显而易见的"
- "任何人都能看出来"

**问题**：
事后觉得"显然"的事，事前往往并不清晰

**对策**：
- 事前写下预测和理由
- 事后对比预测与实际
- 诚实评估预测准确性

### 选择性记忆

**表现**：
- 只记住支持自己的证据
- 忘记自己犯过的错
- 美化自己的决策过程

**对策**：
- 保留决策记录
- 定期回顾记录
- 接受他人反馈

## 归因偏误

### 基本归因错误

**表现**：
- 别人犯错 → 是他的问题（能力、态度）
- 自己犯错 → 是情境的问题（条件、运气）

**示例**：
- 同事delay → "他工作不上心"
- 自己delay → "需求变动太频繁"

**对策**：
- 评价他人时，多考虑情境因素
- 评价自己时，多考虑自身因素

### 过度归因于个人

**问题**：忽视系统性因素

**示例**：
项目失败归咎于某个人，而忽视：
- 流程设计不合理
- 资源配置不足
- 沟通机制缺失
- 组织激励问题

**对策**：
```
问题发生
    │
    ↓
问5个为什么
    │
    ├── 为什么发生？→ 直接原因
    │       │
    │       ↓
    ├── 为什么会有直接原因？→ 间接原因
    │       │
    │       ↓
    ├── 为什么会有间接原因？→ 系统原因
    │       │
    │       ↓
    ├── 为什么系统允许这种情况？→ 制度原因
    │       │
    │       ↓
    └── 为什么制度没有预防？→ 根本原因
```

## 确认偏误的陷阱

### 只找支持自己的证据

**表现**：
- 复盘时只关注验证自己对的地方
- 把反面证据解释成支持证据
- 避免接触不同意见

**对策**：
主动寻找反面证据
- "有什么证据表明我的判断是错的？"
- "如果结论相反，我应该看到什么？"

### 事实 vs 解释

**区分**：
```
事实：可观察、可验证的客观描述
     "这个功能上线后DAU下降了5%"

解释：对事实的主观理解
     "用户不喜欢这个功能"
     "新用户不会用"
     "有bug影响了体验"
```

**对策**：
- 先确认事实
- 列出多种可能的解释
- 寻找证据验证各种解释

## 情绪干扰

### 自我防卫

**表现**：
- 面对批评时本能反驳
- 把批评当成人身攻击
- 急于证明自己是对的

**后果**：
- 错过学习机会
- 无法获得真实反馈
- 重复同样的错误

### 过度自责

**表现**：
- 把所有问题都归咎于自己
- 陷入负面情绪
- 失去改进的动力

**对策**：
区分责任和学习
- 责任：谁应该为此负责？
- 学习：我能从中学到什么？

### 情绪管理策略

| 情绪 | 陷阱 | 策略 |
|-----|------|------|
| 愤怒 | 责怪他人 | 冷却后再复盘 |
| 羞耻 | 逃避问题 | 区分行为和身份 |
| 恐惧 | 避免风险 | 关注可控因素 |
| 骄傲 | 忽视问题 | 主动寻找改进点 |

## 复盘的正确姿势

### 结构化复盘

```
1. 描述事实
   - 发生了什么？
   - 时间线是怎样的？
   - 涉及哪些人？

2. 分析原因
   - 直接原因是什么？
   - 根本原因是什么？
   - 系统因素有哪些？

3. 评估决策
   - 当时的信息充分吗？
   - 推理过程合理吗？
   - 还有什么选项？

4. 提取教训
   - 学到了什么？
   - 下次会怎么做？
   - 如何防止再犯？

5. 行动计划
   - 具体改进措施是什么？
   - 谁负责执行？
   - 如何追踪效果？
```

### 保持开放心态

**心态调整**：
- 复盘是学习，不是审判
- 错误是成长的机会
- 承认不知道是智慧的开始

**实践方法**：
- 邀请不同视角参与
- 先听取他人意见
- 对不同解释保持开放

## 【检查清单】避免复盘陷阱

### 归因检验

- [ ] 区分了过程和结果吗？
- [ ] 考虑了运气因素吗？
- [ ] 同等标准评价自己和他人了吗？
- [ ] 追问了根本原因吗？

### 证据检验

- [ ] 区分了事实和解释吗？
- [ ] 寻找了反面证据吗？
- [ ] 考虑了多种可能的解释吗？
- [ ] 避免了选择性记忆吗？

### 情绪检验

- [ ] 冷静下来再复盘了吗？
- [ ] 控制了防卫心理吗？
- [ ] 避免了过度自责吗？
- [ ] 保持了开放心态吗？

### 学习检验

- [ ] 明确了具体教训吗？
- [ ] 制定了改进计划吗？
- [ ] 有人负责追踪执行吗？
- [ ] 设定了验证时间点吗？

## 本章小结

- 警惕结果导向和事后诸葛亮
- 区分事实和解释，避免确认偏误
- 管理情绪，保持开放心态
- 结构化复盘，提取有价值的教训
- 复盘的目的是学习，不是证明自己对
