# 收集可靠的信息

> "基于错误信息的正确推理，比基于正确信息的错误推理更危险。"

## 信息的三种类型

在收集信息时，首先要区分信息的类型：

### 1. 事实（Facts）

**定义**：可以被验证的客观陈述

**特征**：
- 可以被证实或证伪
- 不依赖于个人观点
- 可重复观察

**例子**：
- "服务器昨天宕机了3小时" ✓
- "用户投诉量上升了50%" ✓
- "代码有1000行" ✓

### 2. 观点（Opinions）

**定义**：基于价值判断的主观陈述

**特征**：
- 无法被证实或证伪
- 反映个人的价值观或偏好
- 因人而异

**例子**：
- "这个设计很丑" — 观点
- "Go语言比Java好" — 观点
- "这个团队效率太低" — 观点（除非有明确标准）

### 3. 推断（Inferences）

**定义**：基于事实得出的结论

**特征**：
- 建立在事实之上
- 包含逻辑推理
- 可能正确也可能错误

**例子**：
- "销售下降是因为价格太高"（推断，可能对也可能错）
- "用户流失是因为竞品更好"（推断，需要验证）

### 区分三种类型的重要性

```
┌─────────────────────────────────────────────────────────┐
│  混淆信息类型导致的问题：                                  │
│                                                         │
│  • 把观点当事实 → 基于主观判断做决策                      │
│  • 把推断当事实 → 未经验证就采取行动                      │
│  • 把事实当观点 → 忽视重要的客观信息                      │
└─────────────────────────────────────────────────────────┘
```

## 信源可信度评估

### CRAAP评估法

**C - Currency（时效性）**
- 信息是什么时候发布的？
- 是否已经过时？
- 领域是否变化很快？

**R - Relevance（相关性）**
- 与你的问题相关吗？
- 信息的受众是谁？
- 深度是否足够？

**A - Authority（权威性）**
- 作者是谁？有什么资质？
- 发布机构的信誉如何？
- 有没有联系方式？

**A - Accuracy（准确性）**
- 信息来源于哪里？
- 有没有引用和证据？
- 能否被其他来源验证？

**P - Purpose（目的性）**
- 发布这个信息的目的是什么？
- 是否有商业或政治倾向？
- 是事实还是观点？

### 信源等级

```
可信度高                                    可信度低
   │                                           │
   ↓                                           ↓
一手数据 → 学术研究 → 官方统计 → 专家分析 → 媒体报道 → 社交媒体
   │          │          │          │          │          │
自己收集   同行评审    政府发布   行业专家   新闻机构   个人发布
```

## 警惕信息茧房

### 什么是信息茧房

信息茧房指的是你只接触与自己观点一致的信息，形成封闭的信息环境。

**形成原因**：
- 算法推荐：平台推送你喜欢的内容
- 社交圈层：朋友往往观点相似
- 确认偏误：主动寻找支持自己的信息
- 舒适区：回避让自己不舒服的观点

### 信息茧房的危害

- 视野狭窄，看不到全貌
- 过度自信，低估其他可能性
- 极端化，无法理解不同观点
- 决策偏差，遗漏重要信息

### 打破信息茧房的方法

**1. 主动接触不同观点**
- 关注与你观点不同的人
- 阅读不同立场的媒体
- 参与多元化的社群

**2. 使用多种信息源**
- 不要只依赖一个平台
- 混合使用传统媒体和社交媒体
- 直接获取一手信息

**3. 建立信息收集系统**
- 定义关键信息源清单
- 定期审视信息源的多样性
- 记录并反思信息获取模式

## 主动寻找反面证据

### 为什么要找反面证据？

**确认偏误的陷阱**：
我们天生倾向于寻找支持自己观点的证据，忽视反面证据。

**例子**：
你认为React比Vue好，于是搜索"React优点"，找到很多支持文章，更加确信自己是对的。但你没有搜索"React缺点"或"Vue优点"。

### 反面证据搜索策略

**1. 对立搜索法**
搜索正面的同时，搜索反面：
- "XX的优点" + "XX的缺点"
- "为什么应该XX" + "为什么不应该XX"
- "XX成功案例" + "XX失败案例"

**2. 魔鬼代言人**
假设你的观点是错的，寻找证据：
- 如果我是反对者，我会提出什么论据？
- 有什么证据能证明我是错的？
- 历史上有没有类似情况结果相反的？

**3. 概率校准**
问自己：
- 我有多大把握？50%？80%？99%？
- 如果我错了，最可能错在哪里？
- 需要什么证据才能改变我的看法？

## 【检查清单】信息充足性与可靠性检验

### 信息充足性检查

- [ ] 收集了足够做决策的信息吗？
- [ ] 关键问题都有了答案吗？
- [ ] 还有什么信息可以帮助决策？
- [ ] 信息遗漏会造成什么风险？

### 信息可靠性检查

- [ ] 每条关键信息的来源是什么？
- [ ] 信源通过了CRAAP评估吗？
- [ ] 关键信息有多个来源验证吗？
- [ ] 是否区分了事实、观点和推断？

### 信息平衡性检查

- [ ] 收集了不同立场的信息吗？
- [ ] 主动寻找了反面证据吗？
- [ ] 是否存在信息茧房风险？
- [ ] 咨询了持不同观点的人吗？

### 信息时效性检查

- [ ] 信息是最新的吗？
- [ ] 情况是否已经发生变化？
- [ ] 是否需要获取最新数据？
- [ ] 信息的有效期是多长？

## 本章小结

- 区分事实、观点和推断三种信息类型
- 使用CRAAP法评估信源可信度
- 警惕信息茧房，主动接触不同观点
- 刻意寻找反面证据，对抗确认偏误
- 使用检查清单验证信息质量
