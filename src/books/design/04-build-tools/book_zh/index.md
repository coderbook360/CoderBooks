# 序言

## 工具背后的原理

作为前端开发者，我们每天都在使用构建工具：Webpack 打包代码，Vite 提供开发服务器，Babel 转译 ES6+，ESLint 检查代码质量……

但你是否想过：这些工具是如何工作的？

理解工具的原理，不仅能帮助你更高效地使用它们，更能让你在遇到问题时快速定位原因，在需要时进行定制化配置，甚至开发自己的构建插件。

## 从模块化说起

现代前端构建的一切，都建立在模块化的基础之上。

从早期的立即执行函数（IIFE）到 CommonJS，再到今天的 ES Modules——模块化规范的演进推动了构建工具的发展。理解模块化，是理解构建工具的前提。

本书将从模块化规范开始，深入探讨各种构建工具的核心原理。

## 本书的定位

这是一本"原理书"而非"使用手册"。

我们不会详细讲解每个配置选项的用法（那是官方文档的工作），而是聚焦于：
- **原理剖析**：工具内部是如何实现的？
- **设计思想**：为什么要这样设计？
- **插件开发**：如何扩展工具的能力？
- **性能优化**：如何让构建更快？

## 本书的结构

**模块系统**：深入理解 CommonJS 和 ES Modules 的原理与差异。

**包管理**：npm、yarn、pnpm 的内部机制与最佳实践。

**构建工具**：Webpack、Vite、Rollup、esbuild、SWC 的核心原理。

**编译转译**：Babel、TypeScript、PostCSS 的工作机制与插件开发。

**开发体验**：热更新、Source Map、代理等开发工具链。

让我们一起深入构建工具的内核世界。
