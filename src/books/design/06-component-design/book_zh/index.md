# 序言

## 组件是现代前端的基石

从 jQuery 时代的 DOM 操作，到 React/Vue 时代的组件化开发，前端 UI 的构建方式发生了根本性变革。

组件化不仅仅是把 UI 拆成小块，更是一种新的思维方式：封装、复用、组合。一个设计良好的组件，应该像一个精心打造的工具——使用简单、扩展灵活、行为可预测。

## API 设计是核心

组件的 API 设计直接决定了使用体验。

Props 应该如何命名？哪些应该是必选，哪些应该是可选？事件回调应该传递什么参数？如何支持自定义内容的注入？如何在保持灵活性的同时不增加使用复杂度？

这些问题的答案，构成了组件 API 设计的核心知识。

## 本书的定位

本书聚焦于**组件设计原理与实践**，帮助你成为一个能设计高质量组件的开发者。

我们将深入探讨：
- 组件分类与职责划分
- Props、事件、插槽的设计原则
- Headless 组件与多态组件
- 受控与非受控的设计选择
- 状态机驱动的组件设计
- 自定义 Hooks 的设计模式
- 表单组件与复杂组件设计

这些知识将帮助你设计出既易用又灵活的组件 API。
