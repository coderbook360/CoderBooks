# 序言

## 两种思维方式的融合

面向对象和函数式编程，是软件开发中两大核心范式。

长期以来，它们被视为对立的两极。面向对象强调封装、继承、多态；函数式强调纯函数、不可变性、组合。但在现代前端开发中，这两种范式正在深度融合。

React Hooks 的出现是一个标志性事件——它用函数式的方式解决了组件逻辑复用的问题，同时保留了组件化的架构思想。Redux 将函数式的 reducer 概念引入状态管理，RxJS 用响应式流重新定义了异步处理……

## 函数式不是银弹

函数式编程有其独特的优势：纯函数易于测试、不可变数据避免副作用、函数组合提高代码复用性。但它也有局限性：过度抽象可能降低可读性、某些场景下性能开销较大、学习曲线相对陡峭。

本书的目标不是让你成为"函数式原教旨主义者"，而是让你理解函数式思维，并在合适的场景中应用它。

## 架构模式的演进

从 MVC 到 MVVM，从 Flux 到 Redux，从 Clean Architecture 到 DDD——前端架构模式经历了持续的演进。理解这些模式的来龙去脉，比单纯学习某一种模式更加重要。

本书将带你走过架构模式的演进之路，理解每种模式要解决的问题、它的设计理念，以及它的适用场景和局限性。

## 本书的结构

**函数式编程部分**：从纯函数、不可变性等基础概念开始，逐步深入到柯里化、函数组合、Monad 等高级话题。

**架构模式部分**：系统性地讲解 MVC、MVP、MVVM、Flux、Redux、响应式架构、Clean Architecture、DDD 等架构模式。

**代码组织部分**：探讨目录结构设计、模块划分、分层架构、依赖注入等代码组织方法。

让我们一起探索架构设计的更多可能性。
