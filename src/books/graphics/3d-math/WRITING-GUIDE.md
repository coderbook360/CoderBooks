## 一、书籍定位与目标

### 1.1 核心定位
- **系列位置**: 图形学系列 Book 2（Canvas → 3D数学 → WebGL → Three.js）
- **目标读者**: 完成2D图形编程学习，准备进入3D领域的开发者
- **学习周期**: 3-4 周，每天 1-2 小时
- **核心价值**: 用可视化和代码实现消除3D数学的抽象感

### 1.2 学习目标
完成本书学习后，读者应能够：
- ✅ 深刻理解向量、矩阵、四元数的几何意义

- ✅ 独立实现相机控制、碰撞检测等实际功能

### 1.3 与其他资料的差异化
- **对比教科书**: 更注重实现和可视化，少理论证明
- **对比 API 文档**: 解释"为什么"而不仅是"怎么用"
- **对比视频教程**: 系统性更强，可随时查阅
- **独特优势**: JavaScript实现 + Three.js对齐 + 交互式可视化

## 二、写作原则

### 2.1 核心原则（必须遵守）

#### 📐 数学不可怕原则
- **可视化优先**: 每个数学概念都必须有几何图示
- **代码验证**: 每个公式都有可运行的JavaScript代码
- **渐进式难度**: 从具体实例到一般规律，不要直接抛出抽象定义
- **实用导向**: 始终关联实际应用场景（Three.js中如何使用）

#### 🎯 读者中心原则
- **前置知识假设**: 只假设读者懂JavaScript和基础三角函数
- **避免数学术语堆砌**: 用"方向"代替"单位向量"，用"旋转"代替"正交变换"
- **提供多角度理解**: 几何直觉 + 代数计算 + 代码实现
- **及时答疑**: 预判读者困惑点，主动解答常见误解

#### 💻 代码质量原则
- **生产级代码**: 不是演示代码，而是可直接使用的实现
- **Three.js 风格**: API设计、命名、参数顺序都对齐Three.js
- **完整性**: 包含边界检查、错误处理、类型验证
- **可测试性**: 每个方法都可以独立测试

### 2.2 禁止事项（❌ 不要做）
- ❌ 不要直接给出公式而不解释推导过程
- ❌ 不要使用复杂的数学符号而不提供通俗解释
- ❌ 不要假设读者已经理解上一章内容（需要回顾提示）
- ❌ 不要只给出2D例子而不扩展到3D
- ❌ 不要写成API文档风格（罗列方法）
- ❌ 不要使用过时的JavaScript语法（必须ES6+）

## 四、代码规范

### 4.1 JavaScript 代码标准

#### 语法规范
```javascript
// ✅ 使用 ES6+ class 语法
class Vector3 {
  constructor(x = 0, y = 0, z = 0) {
    this.x = x;
    this.y = y;
    this.z = z;
  }
}

// ✅ 使用箭头函数（适当场景）
const normalize = (v) => {
  const len = Math.sqrt(v.x * v.x + v.y * v.y + v.z * v.z);
  return new Vector3(v.x / len, v.y / len, v.z / len);
};

// ✅ 使用解构赋值
const { x, y, z } = vector;

// ❌ 不要使用 var
// ❌ 不要使用 function 构造函数
```

#### 命名规范
```javascript
// 类名: PascalCase
class Matrix4 {}
class Quaternion {}

// 方法名和变量名: camelCase
const rotationAngle = Math.PI / 4;
vector.normalize();

// 常量: UPPER_SNAKE_CASE
const MAX_FLOAT_VALUE = 3.402823466e+38;
const EPSILON = 1e-6;

// 私有属性: 下划线前缀（可选）
class Camera {
  _updateProjectionMatrix() {}
}
```

#### 注释规范
```javascript
/**
 * 计算两个向量的点积
 * 
 * 点积的几何意义：
 * - 如果结果为正，两向量夹角小于90度
 * - 如果结果为零，两向量垂直
 * - 如果结果为负，两向量夹角大于90度
 * 
 * @param {Vector3} v - 另一个向量
 * @returns {number} 点积结果
 * 
 * @example
 * const v1 = new Vector3(1, 0, 0);
 * const v2 = new Vector3(0, 1, 0);
 * const dot = v1.dot(v2); // 0 (垂直)
 */
dot(v) {
  return this.x * v.x + this.y * v.y + this.z * v.z;
}
```

### 4.2 Three.js API 对齐规则

#### 类设计对齐
```javascript
// Three.js 风格：支持链式调用
class Vector3 {
  add(v) {
    this.x += v.x;
    this.y += v.y;
    this.z += v.z;
    return this; // 返回 this 支持链式调用
  }
  
  // 提供不修改原对象的版本
  clone() {
    return new Vector3(this.x, this.y, this.z);
  }
}

// 使用示例
const v = new Vector3(1, 2, 3)
  .add(new Vector3(1, 0, 0))
  .normalize();
```

#### 方法名对齐
```javascript
// ✅ 与 Three.js 保持一致的方法名
vector.normalize()        // 不是 normalized()
vector.length()          // 不是 magnitude()
matrix.multiply(m)       // 不是 mult(m)
quaternion.slerp(q, t)   // 不是 sphericalLerp(q, t)
```

### 4.3 错误处理
```javascript
class Vector3 {
  normalize() {
    const len = this.length();
    
    // ✅ 对异常情况进行检查和处理
    if (len === 0) {
      console.warn('Cannot normalize zero-length vector');
      return this;
    }
    
    return this.divideScalar(len);
  }
  
  divideScalar(scalar) {
    // ✅ 输入验证
    if (scalar === 0) {
      throw new Error('Division by zero');
    }
    
    return this.multiplyScalar(1 / scalar);
  }
}
```

### 6.1 公式呈现

#### 行内公式

#### 独立公式
```markdown
$$
\begin{bmatrix}
x' \\
y' \\
z' \\
1
\end{bmatrix}
=
\begin{bmatrix}
1 & 0 & 0 & t_x \\
0 & 1 & 0 & t_y \\
0 & 0 & 1 & t_z \\
0 & 0 & 0 & 1
\end{bmatrix}
\begin{bmatrix}
x \\
y \\
z \\

\end{bmatrix}
$$
```

### 6.2 公式说明模板

每个公式必须附带说明：

```markdown
**点积公式**：

$$
\vec{a} \cdot \vec{b} = a_x b_x + a_y b_y + a_z b_z = |\vec{a}| |\vec{b}| \cos\theta
$$

**符号说明**：
- $\vec{a} \cdot \vec{b}$: 向量 $\vec{a}$ 和 $\vec{b}$ 的点积
- $a_x, a_y, a_z$: 向量 $\vec{a}$ 的三个分量
- $|\vec{a}|$: 向量 $\vec{a}$ 的长度
- $\theta$: 两向量之间的夹角

**几何意义**：
点积表示一个向量在另一个向量方向上的投影长度，再乘以另一个向量的长度。

**应用场景**：
- 判断两向量是否垂直（点积为0）
- 计算向量夹角
- 光照计算（漫反射）
```

## 九、常见问题(FAQ)

### Q1: 什么时候需要证明数学公式？
**A**: 只在以下情况证明：
- 公式推导过程有教学价值（如投影矩阵推导）
- 帮助理解概念（如点积的几何意义）
- 澄清常见误解

大部分情况下，给出公式、说明含义、展示代码即可。

### Q2: 代码示例应该多详细？
**A**: 
- **教学代码**: 简化版，突出核心逻辑
- **实用代码**: 完整版，包含错误处理和优化
- **建议**: 先给教学版，再给实用版

### Q3: 如何处理不同难度读者的需求？
**A**: 采用分层呈现：
- **基础层**: 核心概念和最简实现（所有人必读）
- **进阶层**: 优化技巧和高级应用（用折叠区或独立章节）
- **参考层**: 详细证明和外部链接（放在章节末尾）

### Q4: 可视化工具选择建议？
**A**: 
- **简单2D图**: 使用Canvas 2D或SVG
- **3D场景**: 使用Three.js
- **数学函数图**: 使用图表库
- **交互式参数调整**: 使用滑块+实时渲染

### Q5: 如何确保代码与Three.js保持一致？
**A**:
1. 对照Three.js源码检查API设计
2. 方法名、参数顺序、返回值类型保持一致
3. 在"与Three.js对比"章节明确说明差异
4. 定期更新以适配Three.js新版本
