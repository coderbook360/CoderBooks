# 渲染管线概述

> "理解渲染管线是掌握 GPU 编程的基础。"

## 什么是渲染管线

### 定义

渲染管线（Rendering Pipeline）是将 3D 场景数据转换为 2D 屏幕图像的一系列有序处理阶段。

```
┌─────────────────────────────────────────────────────────┐
│                     渲染管线                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  顶点数据 → 顶点处理 → 图元装配 → 光栅化 → 片元处理 → 输出 │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 可编程与固定功能

| 阶段 | 类型 | 说明 |
|------|------|------|
| 顶点着色器 | 可编程 | 自定义顶点处理 |
| 图元装配 | 固定功能 | GPU 自动执行 |
| 光栅化 | 固定功能 | GPU 自动执行 |
| 片元着色器 | 可编程 | 自定义像素处理 |
| 深度测试 | 可配置 | 可开关和配置 |
| 混合 | 可配置 | 可开关和配置 |

## 完整管线流程

### 管线阶段详解

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  ┌─────────────────┐                                    │
│  │   应用程序阶段   │  CPU：准备数据、发起绘制调用        │
│  └────────┬────────┘                                    │
│           │                                             │
│           ▼                                             │
│  ┌─────────────────┐                                    │
│  │   顶点着色器    │  GPU：变换、动画、光照预计算         │
│  └────────┬────────┘                                    │
│           │                                             │
│           ▼                                             │
│  ┌─────────────────┐                                    │
│  │   图元装配      │  GPU：组装三角形、裁剪              │
│  └────────┬────────┘                                    │
│           │                                             │
│           ▼                                             │
│  ┌─────────────────┐                                    │
│  │   光栅化        │  GPU：确定覆盖的像素               │
│  └────────┬────────┘                                    │
│           │                                             │
│           ▼                                             │
│  ┌─────────────────┐                                    │
│  │   片元着色器    │  GPU：计算每个像素的颜色            │
│  └────────┬────────┘                                    │
│           │                                             │
│           ▼                                             │
│  ┌─────────────────┐                                    │
│  │   输出合并      │  GPU：深度测试、混合、写入帧缓冲    │
│  └─────────────────┘                                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 各阶段详解

### 1. 应用程序阶段（CPU）

**职责**：
- 准备顶点数据
- 设置渲染状态
- 上传 Uniform 数据
- 发起绘制调用

```javascript
// 准备数据
gl.bindBuffer(gl.ARRAY_BUFFER, vbo);
gl.bufferData(gl.ARRAY_BUFFER, vertices, gl.STATIC_DRAW);

// 设置状态
gl.enable(gl.DEPTH_TEST);
gl.enable(gl.CULL_FACE);

// 上传 Uniform
gl.uniformMatrix4fv(u_mvp, false, mvpMatrix);

// 发起绘制
gl.drawArrays(gl.TRIANGLES, 0, vertexCount);
```

### 2. 顶点处理阶段（GPU）

**顶点着色器执行**：
- 每个顶点独立执行一次
- 不能访问其他顶点的数据
- 输出裁剪空间坐标

```glsl
#version 300 es

in vec3 a_position;
in vec3 a_normal;

uniform mat4 u_mvpMatrix;

out vec3 v_normal;

void main() {
  gl_Position = u_mvpMatrix * vec4(a_position, 1.0);
  v_normal = a_normal;
}
```

### 3. 图元装配阶段（GPU）

**职责**：
- 将顶点组装成图元（点、线、三角形）
- 执行裁剪（Clipping）
- 透视除法

```
顶点数据：v0, v1, v2, v3, v4, v5
                │
                ▼
          图元装配
                │
    ┌───────────┼───────────┐
    ▼           ▼           ▼
  (v0,v1,v2) (v3,v4,v5)   ...
  
  三角形1    三角形2     ...
```

**裁剪**：
- 完全在视锥体外 → 丢弃
- 完全在视锥体内 → 保留
- 部分在内 → 裁剪产生新顶点

### 4. 光栅化阶段（GPU）

**职责**：
- 确定图元覆盖的像素
- 生成片元
- 插值顶点属性

```
           三角形
          ┌──────┐
         /        \
        /   ████   \
       /  ████████  \
      /  ██████████  \
     └────────────────┘
     
     ████ = 生成的片元
```

**插值**：
- 使用重心坐标插值
- varying 变量自动插值

### 5. 片元处理阶段（GPU）

**片元着色器执行**：
- 每个片元独立执行一次
- 接收插值后的 varying 数据
- 输出片元颜色

```glsl
#version 300 es
precision highp float;

in vec3 v_normal;
out vec4 fragColor;

void main() {
  vec3 color = v_normal * 0.5 + 0.5;
  fragColor = vec4(color, 1.0);
}
```

### 6. 输出合并阶段（GPU）

**职责**：
- 深度测试
- 模板测试
- Alpha 混合
- 写入帧缓冲

```
片元 → 深度测试 → 模板测试 → 混合 → 帧缓冲
         │           │        │
         ↓           ↓        ↓
       通过?       通过?    计算颜色
```

## 数据流

### 数据类型

```
┌────────────────────────────────────────────────────┐
│                    数据流向                        │
├────────────────────────────────────────────────────┤
│                                                    │
│  Attribute (每顶点) ──────→ 顶点着色器             │
│                                  │                 │
│  Uniform (全局) ────────────────┼→ 顶点着色器      │
│                                  │  片元着色器     │
│                                  │                 │
│  Varying (插值) ←───────────────┘                 │
│       │                                           │
│       └──→ 光栅化 ──→ 片元着色器                   │
│                                                    │
│  Sampler (纹理) ────────────→ 片元着色器           │
│                                                    │
└────────────────────────────────────────────────────┘
```

### 内存带宽考虑

| 数据类型 | 频率 | 带宽影响 |
|---------|------|---------|
| Attribute | 每顶点 | 高 |
| Uniform | 每绘制调用 | 低 |
| Texture | 每片元采样 | 非常高 |

## 并行处理

### GPU 并行架构

```
┌─────────────────────────────────────────────────────┐
│                      GPU                            │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐              │
│  │ Core │ │ Core │ │ Core │ │ Core │  ...         │
│  └──────┘ └──────┘ └──────┘ └──────┘              │
│      │        │        │        │                  │
│      └────────┴────────┴────────┘                  │
│                    │                               │
│              同时处理多个                           │
│            顶点/片元                               │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 并行执行特点

- 顶点着色器：多个顶点并行处理
- 片元着色器：多个片元并行处理
- 着色器内不能访问其他顶点/片元的数据

## 状态机模型

### 状态影响管线

```javascript
// 这些状态影响管线的行为
gl.enable(gl.DEPTH_TEST);      // 影响深度测试阶段
gl.depthFunc(gl.LESS);

gl.enable(gl.BLEND);           // 影响混合阶段
gl.blendFunc(gl.SRC_ALPHA, gl.ONE_MINUS_SRC_ALPHA);

gl.enable(gl.CULL_FACE);       // 影响图元装配阶段
gl.cullFace(gl.BACK);

gl.viewport(0, 0, w, h);       // 影响光栅化阶段
```

## 本章小结

- 渲染管线是 3D 到 2D 的转换流程
- 顶点着色器和片元着色器是可编程阶段
- 图元装配、光栅化是固定功能阶段
- 输出合并阶段执行深度测试和混合
- GPU 并行处理多个顶点和片元
- 状态机模型控制管线行为
