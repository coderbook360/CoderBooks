# 冗余连接（并查集）

LeetCode 684. Redundant Connection

## 题目描述

树可以看成是一个连通且无环的无向图。

给定一个有 `n` 个节点（节点值从 `1` 到 `n`）的树，再加上一条边。添加的边的两个顶点包含在 `1` 到 `n` 中，且这条附加的边不属于树中已存在的边。

图由数组 `edges` 表示，其中 `edges[i] = [ai, bi]` 表示图中在 `ai` 和 `bi` 之间存在一条边。

请找出那条**冗余的边**，该边删除后可使得结果图是一棵有 `n` 个节点的树。如果有多个答案，返回数组 `edges` 中**最后出现**的那条边。

## 示例

```
输入：edges = [[1,2],[1,3],[2,3]]
输出：[2,3]
解释：
    1
   / \
  2 - 3
边 [2,3] 是多余的，删除后变成树
```

## 思路分析

**关键洞察**：树有 n 个节点和 n-1 条边。添加一条边后有 n 条边，必然形成环。

**冗余边的特征**：连接两个**已经连通**的节点。

**并查集策略**：
1. 依次处理每条边
2. 如果边的两端已经连通，这条边就是冗余边
3. 否则，合并两端

## 代码实现

```typescript
function findRedundantConnection(edges: number[][]): number[] {
  const n = edges.length;
  const parent = Array.from({ length: n + 1 }, (_, i) => i);
  
  function find(x: number): number {
    if (parent[x] !== x) {
      parent[x] = find(parent[x]);
    }
    return parent[x];
  }
  
  function union(x: number, y: number): boolean {
    const rootX = find(x);
    const rootY = find(y);
    
    if (rootX === rootY) {
      return false;  // 已经连通，这条边是冗余的
    }
    
    parent[rootX] = rootY;
    return true;
  }
  
  for (const [a, b] of edges) {
    if (!union(a, b)) {
      return [a, b];  // 发现冗余边
    }
  }
  
  return [];  // 不会执行到这里
}
```

## 执行过程

```
edges = [[1,2],[1,3],[2,3]]

初始：parent = [0, 1, 2, 3]

处理 [1,2]：
  find(1) = 1, find(2) = 2
  不相同，union：parent = [0, 2, 2, 3]

处理 [1,3]：
  find(1) = 2, find(3) = 3
  不相同，union：parent = [0, 2, 2, 2]

处理 [2,3]：
  find(2) = 2, find(3) = 2
  相同！返回 [2,3]
```

## 可视化理解

```
初始状态：
1   2   3  （三个独立节点）

添加边 [1,2]：
1 - 2   3

添加边 [1,3]：
1 - 2
 \
  3

添加边 [2,3]：
1 - 2
 \ |   ← 形成环，[2,3] 是冗余边
  3
```

## 为什么返回最后出现的边？

题目要求"最后出现"的冗余边。我们按顺序处理，第一次发现"已连通"时返回的就是最后一条导致环的边。

```
如果有多条边能形成环：
edges = [[1,2],[2,3],[3,4],[4,2]]

处理顺序：
[1,2]: 合并 1,2
[2,3]: 合并 2,3
[3,4]: 合并 3,4
[4,2]: 4 和 2 已连通，返回 [4,2]

这正是"最后出现"的那条。
```

## 边界情况

```typescript
// 最简单的环
findRedundantConnection([[1,2],[2,1]]);  
// 返回 [2,1]

// 自环（题目不会出现）
// 如果有 [1,1]，第一条就是冗余边
```

- **时间复杂度**：O(n · α(n)) ≈ O(n)
- **空间复杂度**：O(n)
