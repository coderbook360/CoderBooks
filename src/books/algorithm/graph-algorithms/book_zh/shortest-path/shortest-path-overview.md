# 最短路径问题概述

最短路径问题是图论中最经典、应用最广泛的问题之一。从导航软件规划路线，到网络数据包路由，再到社交网络中的六度分隔理论，最短路径算法无处不在。

## 什么是最短路径问题？

给定一个带权图 G = (V, E)，其中每条边 (u, v) 都有一个权重 w(u, v)，最短路径问题要求找出从源点 s 到目标点 t 的一条路径，使得路径上所有边的权重之和最小。

**问题的几种变体**：

- **单源最短路径（SSSP）**：从一个源点出发，求到所有其他顶点的最短路径
- **单目标最短路径**：所有顶点到某一目标点的最短路径（反转边方向即可转化为 SSSP）
- **单对最短路径**：给定源点和目标点，求它们之间的最短路径
- **全源最短路径（APSP）**：求所有顶点对之间的最短路径

## 权重的多样性

边的权重可以表示多种含义：

- **距离**：两点之间的物理距离
- **时间**：通过某条边所需的时间
- **成本**：经过某条边的费用
- **概率**：某条边可通行的概率（取对数后转化为加法）

## 经典算法一览

| 算法 | 适用场景 | 时间复杂度 | 特点 |
|------|---------|-----------|------|
| **BFS** | 无权图/边权相等 | O(V + E) | 最简单高效 |
| **Dijkstra** | 非负权边 | O((V + E) log V) | 最常用的单源算法 |
| **Bellman-Ford** | 可有负权边 | O(VE) | 能检测负环 |
| **SPFA** | 可有负权边 | 平均 O(E)，最差 O(VE) | Bellman-Ford 优化版 |
| **Floyd-Warshall** | 全源最短路径 | O(V³) | 求所有点对间最短路径 |
| **Johnson** | 全源（稀疏图优化） | O(V² log V + VE) | 稀疏图优于 Floyd |

## 最短路径的核心思想：松弛操作

所有最短路径算法的核心都是**松弛（Relaxation）**操作：

```python
def relax(u, v, weight):
    """尝试通过边 (u, v) 更新 v 的最短距离"""
    if dist[u] + weight < dist[v]:
        dist[v] = dist[u] + weight
        parent[v] = u  # 记录路径
        return True
    return False
```

**松弛的含义**：如果从源点经过 u 再到 v 的路径比当前已知的从源点到 v 的路径更短，则更新 v 的最短距离。

## 如何选择算法？

**决策树**：

1. **边权是否全部相等或无权？**
   - 是 → 使用 **BFS**
   - 否 → 继续判断

2. **是否存在负权边？**
   - 否 → 使用 **Dijkstra**
   - 是 → 继续判断

3. **是否需要检测负环？**
   - 是 → 使用 **Bellman-Ford**
   - 否 → 使用 **SPFA**（实际中需谨慎，可能被卡）

4. **需要所有点对的最短路径？**
   - 稠密图 → 使用 **Floyd-Warshall**
   - 稀疏图 → 使用 **Johnson**

## 负权边与负环

**负权边**：边的权重为负数。例如：走某条路可以获得奖励（收益为负成本）。

**负环**：环上所有边权之和为负。存在负环时，最短路径不存在（可以无限走负环使路径长度趋向负无穷）。

```
    A --(-5)--> B
    ^           |
    |          (2)
    |           v
    +---(-3)--- C

环 A→B→C→A 的权重 = -5 + 2 + (-3) = -6 < 0
这是一个负环，不存在最短路径
```

## 小结

- 最短路径问题是图论中的经典问题，有多种变体
- 不同场景选择不同算法：BFS（无权）→ Dijkstra（非负权）→ Bellman-Ford/SPFA（负权）
- 所有算法的核心是**松弛操作**
- 负环会导致最短路径不存在，需要特别处理
