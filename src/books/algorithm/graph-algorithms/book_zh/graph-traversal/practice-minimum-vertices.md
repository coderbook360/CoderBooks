# 可以到达所有点的最少点数目

LeetCode 1557. Minimum Number of Vertices to Reach All Nodes

## 题目描述

给你一个有向无环图（DAG），包含 `n` 个节点，节点编号从 `0` 到 `n-1`。

找到最小的顶点集合，使得从这些顶点出发可以到达图中所有节点。

## 示例

```
输入：n = 6, edges = [[0,1],[0,2],[2,5],[3,4],[4,2]]

0 → 1
↓
2 → 5
↑
4
↑
3

输出：[0,3]
解释：从 0 可以到达 1, 2, 5；从 3 可以到达 4, 2, 5
```

## 思路分析

关键洞察：哪些节点**必须**被选中？

答案：**入度为 0 的节点**。

为什么？
- 入度为 0 的节点没有任何节点可以到达它
- 所以必须把它作为起点
- 因为是 DAG（无环），所以入度为 0 的节点一定可以到达后续所有节点

## 代码实现

```typescript
function findSmallestSetOfVertices(n: number, edges: number[][]): number[] {
  const hasIncoming = new Set<number>();
  
  for (const [_, v] of edges) {
    hasIncoming.add(v);
  }
  
  const result: number[] = [];
  for (let i = 0; i < n; i++) {
    if (!hasIncoming.has(i)) {
      result.push(i);
    }
  }
  
  return result;
}
```

## 为什么这是对的？

**证明：入度为 0 的节点必须选**

假设节点 v 入度为 0，但我们不选它。那么我们需要从某个其他节点到达 v。但因为 v 入度为 0，没有边指向它，所以无法到达。矛盾。

**证明：只选入度为 0 的节点就够了**

因为是 DAG，从入度为 0 的节点出发，可以遍历到所有可达的节点。如果某个节点 u 不可达，说明没有从任何入度为 0 的节点到 u 的路径。但 u 要么自己入度为 0（应该被选中），要么有入边。如果有入边，向前追溯，最终会追溯到一个入度为 0 的节点（DAG 无环保证了这一点）。矛盾。

## 执行过程

```
n = 6, edges = [[0,1],[0,2],[2,5],[3,4],[4,2]]

统计有入边的节点：
edge [0,1]: 1 有入边
edge [0,2]: 2 有入边
edge [2,5]: 5 有入边
edge [3,4]: 4 有入边
edge [4,2]: 2 有入边（已标记）

hasIncoming = {1, 2, 5, 4}

检查 0~5：
0: 不在 hasIncoming → 选中
1: 在 hasIncoming
2: 在 hasIncoming
3: 不在 hasIncoming → 选中
4: 在 hasIncoming
5: 在 hasIncoming

结果：[0, 3]
```

## 注意

这道题的条件是 DAG（有向无环图）。如果有环，情况会更复杂：

```
带环的情况：
0 ⇌ 1

两个节点都有入边，但按照我们的算法不会选任何节点。
但实际上需要选至少一个。
```

所以这个简单的算法只适用于 DAG。

- **空间复杂度**：O(V)
