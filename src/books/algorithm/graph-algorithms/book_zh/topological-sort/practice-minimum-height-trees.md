# 最小高度树

LeetCode 310. Minimum Height Trees

## 题目描述

树是一个无向图，其中任何两个顶点只通过一条路径连接。换句话说，一个没有简单环路的连通图就是一棵树。

给你一棵包含 `n` 个节点的树，标记为 `0` 到 `n - 1`。给定数字 `n` 和一个有 `n - 1` 条边的 `edges` 列表，其中 `edges[i] = [ai, bi]` 表示树中节点 `ai` 和 `bi` 之间存在一条无向边。

可选择树中任何一个节点作为根。当选择节点 `x` 作为根节点时，设结果树的高度为 `h`。在所有可能的树中，具有最小高度的树被称为最小高度树。

返回所有最小高度树的根节点标签列表。

## 示例

```
输入：n = 6, edges = [[3,0],[3,1],[3,2],[3,4],[5,4]]
输出：[3,4]
解释：
    0
    |
3---1    高度为 2
    |
    2
    |
    4
    |
    5

以 3 为根：高度 = 2
以 4 为根：高度 = 2
其他节点为根的高度都 > 2
```

## 思路分析

**暴力思路**：枚举每个节点作为根，计算树高度，时间复杂度 O(n²)。

**优化思路**：从外向内"剥洋葱"。

**核心观察**：
- 叶子节点（度为 1）不可能是最优根
- 最优根一定在树的"中心"位置
- 类似拓扑排序，不断删除叶子节点

```
层层剥离叶子：
0-1-2-3-4-5

第一轮删除叶子 0, 5：
  1-2-3-4

第二轮删除叶子 1, 4：
  2-3

第三轮：2, 3 都是叶子，返回 [2, 3]
```

## 代码实现

```typescript
function findMinHeightTrees(n: number, edges: number[][]): number[] {
  // 特殊情况
  if (n === 1) return [0];
  if (n === 2) return [0, 1];
  
  // 建图 + 计算度数
  const graph: Set<number>[] = Array.from({ length: n }, () => new Set());
  const degree = new Array(n).fill(0);
  
  for (const [a, b] of edges) {
    graph[a].add(b);
    graph[b].add(a);
    degree[a]++;
    degree[b]++;
  }
  
  // 找出所有叶子节点（度为 1）
  let leaves: number[] = [];
  for (let i = 0; i < n; i++) {
    if (degree[i] === 1) {
      leaves.push(i);
    }
  }
  
  let remaining = n;
  
  // 不断剥离叶子，直到剩余节点 ≤ 2
  while (remaining > 2) {
    remaining -= leaves.length;
    const newLeaves: number[] = [];
    
    for (const leaf of leaves) {
      // 删除叶子节点
      for (const neighbor of graph[leaf]) {
        graph[neighbor].delete(leaf);
        degree[neighbor]--;
        
        // 如果邻居变成叶子
        if (degree[neighbor] === 1) {
          newLeaves.push(neighbor);
        }
      }
    }
    
    leaves = newLeaves;
  }
  
  return leaves;
}
```

## 执行过程

```
n = 6, edges = [[3,0],[3,1],[3,2],[3,4],[5,4]]

图结构：
0 - 3 - 1
    |
    2
    |
    4
    |
    5

初始度数：[1, 1, 1, 4, 2, 1]
初始叶子：[0, 1, 2, 5]
remaining = 6

第一轮：
- 删除 0: 3 的度数 4→3
- 删除 1: 3 的度数 3→2
- 删除 2: 3 的度数 2→1，3 变成叶子
- 删除 5: 4 的度数 2→1，4 变成叶子
remaining = 6 - 4 = 2

leaves = [3, 4]
remaining = 2 ≤ 2，退出循环

返回 [3, 4]
```

## 为什么最多剩 2 个节点？

树的结构决定了：
- **奇数直径**：中心是 1 个节点
- **偶数直径**：中心是 2 个相邻节点

```
直径为 4（偶数）：
0-1-2-3-4
中心：[2]

直径为 5（奇数）：
0-1-2-3-4-5
中心：[2, 3]
```

## 与拓扑排序的关系

| 特性 | 拓扑排序 | 最小高度树 |
|------|----------|------------|
| 图类型 | 有向图 | 无向图 |
| 起点 | 入度为 0 | 度为 1（叶子） |
| 终止条件 | 所有节点处理完 | 剩余 ≤ 2 个节点 |
| 目的 | 得到拓扑序 | 找到树的中心 |

## 复杂度分析

- **时间复杂度**：O(n)，每个节点和边最多处理一次
- **空间复杂度**：O(n)，存储图和度数
