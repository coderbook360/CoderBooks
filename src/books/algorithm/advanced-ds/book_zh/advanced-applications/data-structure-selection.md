# 数据结构选型指南

选择合适的数据结构是解决问题的关键。本章提供系统化的选型方法论。

---

## 选型决策框架

### 第一步：分析问题特征

| 维度 | 问题 | 影响选择 |
|------|------|---------|
| 数据规模 | n 是多少？ | 复杂度要求 |
| 操作类型 | 查询/修改比例？ | 静态 vs 动态结构 |
| 维度 | 一维/二维/多维？ | 结构复杂度 |
| 在线/离线 | 查询预知否？ | 预处理空间 |
| 时间限制 | 毫秒级/秒级？ | 常数因子考量 |

### 第二步：匹配数据结构

```python
def select_data_structure(problem):
    """
    数据结构选型决策树
    """
```

## 区间问题选型

### 静态区间查询

| 查询类型 | 推荐结构 | 预处理 | 查询 | 备注 |
|---------|---------|--------|------|------|
| 区间和 | 前缀和 | O(n) | O(1) | 最简单 |
| 区间最值 | ST 表 | O(n log n) | O(1) | 经典 |
| 区间 GCD | ST 表 | O(n log n) | O(1) | 同上 |
| 区间第 K 小 | 主席树 | O(n log n) | O(log n) | 需离散化 |
| 区间不同元素数 | 离线 + BIT | O(n log n) | O(log n) | 需离线 |

### 动态区间查询

| 操作类型 | 推荐结构 | 修改 | 查询 | 备注 |
|---------|---------|------|------|------|
| 单点修改 + 区间和 | 树状数组 | O(log n) | O(log n) | 首选 |
| 单点修改 + 区间最值 | 线段树 | O(log n) | O(log n) | 标准 |
| 区间修改 + 单点查询 | 差分 + BIT | O(log n) | O(log n) | 差分思想 |
| 区间修改 + 区间和 | 线段树 + 懒标记 | O(log n) | O(log n) | 需要懒标记 |
| 动态第 K 小 | 树套树 | O(log² n) | O(log³ n) | 复杂 |

### 决策流程图

```
区间问题
    │
    ├── 需要修改？
    │   ├── 否 → 静态结构
    │   │        ├── 求和 → 前缀和
    │   │        ├── 最值 → ST 表
    │   │        └── 第K小 → 主席树
    │   │
    │   └── 是 → 动态结构
    │            ├── 单点修改
    │            │   ├── 区间和 → 树状数组
    │            │   └── 区间最值 → 线段树
    │            │
    │            └── 区间修改
    │                ├── 单点查询 → 差分数组
    │                └── 区间查询 → 线段树+懒标记
```

## 树上问题选型

| 问题类型 | 推荐结构 | 复杂度 | 备注 |
|---------|---------|--------|------|
| LCA 查询（在线） | 倍增 | O(log n) | 标准方法 |
| LCA 查询（离线） | Tarjan | O(α(n)) | 更快 |
| 路径查询/修改 | 树链剖分 | O(log² n) | 重链思想 |
| 子树统计 | DFS 序 + BIT | O(log n) | 转化为区间 |
| 子树颜色统计 | DSU on Tree | O(n log n) | 启发式合并 |
| 稀疏关键点 | 虚树 | O(k log k) | 减少规模 |

```
树上问题
    │
    ├── LCA
    │   ├── 在线 → 倍增
    │   └── 离线 → Tarjan
    │
    ├── 路径操作
    │   ├── 单次路径 → 倍增
    │   └── 多次路径 → 树链剖分
    │
    └── 子树操作
        ├── 简单统计 → DFS 序
        └── 颜色统计 → DSU on Tree
```

## 实战选型案例

### 案例 1：区间众数

**问题**：区间内出现次数最多的元素

**分析**：
- 众数不满足区间可加性
- 无法用线段树直接合并

**选型**：分块 + 预处理
- 预处理任意两个整块之间的众数
- 查询时处理边界散块

### 案例 2：带修改的区间第 K 小

**问题**：支持单点修改，查询区间第 K 小

**选型方案对比**：

| 方案 | 修改 | 查询 | 实现难度 |
|------|------|------|---------|
| 树状数组套主席树 | O(log² n) | O(log² n) | 高 |
| 线段树套平衡树 | O(log² n) | O(log³ n) | 高 |
| 整体二分（离线） | - | O(n log² n) | 中 |
| 分块 | O(1) | O(√n log n) | 低 |

**建议**：如果可以离线，用整体二分；必须在线，用分块（n ≤ 10⁵）。

### 案例 3：树上路径第 K 大

**问题**：查询树上路径的第 K 大值

**选型**：主席树 + LCA
- 对每个节点维护到根的主席树
- 查询时用 LCA 合并四棵树

## 选型原则总结

### 优先原则

1. **简单优先**：能用前缀和就不用线段树
2. **常数优先**：树状数组优于线段树（功能相同时）
3. **空间优先**：动态开点优于完全分配
4. **离线优先**：能离线就离线

### 复杂度平衡

```python
def acceptable_complexity(n, time_limit_ms=1000):
    """
    估算可接受的复杂度
    假设 10^8 次简单操作/秒
    """
    ops_per_sec = 10 ** 8
    max_ops = ops_per_sec * time_limit_ms / 1000
    
    complexities = {
        "O(n²)": n * n,
        "O(n√n)": n * int(n ** 0.5),
        "O(n log² n)": n * (n.bit_length() ** 2),
        "O(n log n)": n * n.bit_length(),
        "O(n)": n
    }
    
    for name, ops in complexities.items():
        if ops <= max_ops:
            print(f"{name}: 可行")
        else:
            print(f"{name}: 可能超时")
```

## 本章小结

1. **分析问题特征**
