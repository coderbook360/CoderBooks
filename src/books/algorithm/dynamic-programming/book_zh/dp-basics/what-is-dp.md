# 动态规划是什么

## 从一个问题说起

假设你要爬一个 10 级台阶的楼梯，每次可以爬 1 级或 2 级。问：有多少种不同的爬法？

这是一道经典的入门问题。让我们用不同的思维方式来解决它。

### 暴力递归：直觉但低效

最直观的想法是递归：要到达第 10 级，要么从第 9 级爬 1 步，要么从第 8 级爬 2 步。

```typescript
function climbStairs(n: number): number {
  // 终止条件
  if (n === 1) return 1;  // 只有 1 种方式
  if (n === 2) return 2;  // 有 2 种方式：1+1 或 2
  
  // 递归关系
  return climbStairs(n - 1) + climbStairs(n - 2);
}
```

看起来很简洁，对吧？但这个解法有致命问题——它**太慢了**。

计算 `climbStairs(10)` 需要多少次递归调用？我们画出递归树：

```
                    f(10)
                   /     \
               f(9)       f(8)
              /    \     /    \
           f(8)  f(7)  f(7)  f(6)
           ...   ...   ...   ...
```

你会发现 `f(8)` 被计算了 2 次，`f(7)` 被计算了 3 次...这就是问题所在：**大量的重复计算**。

时间复杂度是 O(2^n)，计算 `f(50)` 需要超过 10^15 次操作，现代计算机也要跑好几天。

### 核心问题：重复计算

让我们数一数，计算 `f(10)` 时，每个子问题被计算了多少次：

| 子问题 | 计算次数 |
|-------|---------|
| f(10) | 1 |
| f(9)  | 1 |
| f(8)  | 2 |
| f(7)  | 3 |
| f(6)  | 5 |
| f(5)  | 8 |
| ...   | ... |

这个规律你可能认出来了——斐波那契数列！计算次数呈指数增长。

**如果我们把每个子问题的答案记下来，下次直接用，不就避免了重复计算吗？**

这就是动态规划的核心思想。

## 动态规划的本质

**动态规划（Dynamic Programming，简称 DP）是一种通过存储子问题的解来避免重复计算的算法设计技术。**

关键词有两个：
1. **子问题**：把大问题分解成小问题
2. **存储**：记住已解决的子问题的答案

### 为什么叫"动态规划"？

这个名字确实有点让人困惑。它的发明者 Richard Bellman 在 1950 年代提出这个概念时，故意起了一个听起来很"高大上"的名字，以便向不懂技术的政府官员申请研究经费。

实际上，"动态"指的是问题可以分阶段求解，"规划"指的是优化决策。你可以简单理解为："**分阶段记忆式求解**"。

### DP 与递归的关系

动态规划和递归是解决问题的两种视角：

| 特点 | 递归（自顶向下） | 递推（自底向上） |
|-----|----------------|----------------|
| 思考方向 | 从大问题到小问题 | 从小问题到大问题 |
| 实现方式 | 函数调用自身 | 循环迭代 |
| 存储方式 | 备忘录（哈希表） | DP 数组 |
| 栈空间 | 需要调用栈 | 不需要 |

两种方式本质上是等价的，但递推通常更高效（没有函数调用开销）。

## 爬楼梯问题的 DP 解法

### 方法一：记忆化搜索（自顶向下）

在递归基础上加一个"备忘录"：

```typescript
function climbStairs(n: number): number {
  const memo: Map<number, number> = new Map();
  
  function dp(n: number): number {
    // 查备忘录
    if (memo.has(n)) return memo.get(n)!;
    
    // 终止条件
    if (n === 1) return 1;
    if (n === 2) return 2;
    
    // 计算并存入备忘录
    const result = dp(n - 1) + dp(n - 2);
    memo.set(n, result);
    return result;
  }
  
  return dp(n);
}
```

每个子问题只计算一次，时间复杂度降为 O(n)。

### 方法二：递推（自底向上）

从最小的子问题开始，逐步构建更大问题的解：

```typescript
function climbStairs(n: number): number {
  if (n <= 2) return n;
  
  // dp[i] 表示爬到第 i 级台阶的方法数
  const dp: number[] = new Array(n + 1);
  dp[1] = 1;
  dp[2] = 2;
  
  for (let i = 3; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2];
  }
  
  return dp[n];
}
```

这就是动态规划的标准形式。

### 方法三：空间优化

观察递推公式 `dp[i] = dp[i-1] + dp[i-2]`，每次只用到前两个值，不需要存整个数组：

```typescript
function climbStairs(n: number): number {
  if (n <= 2) return n;
  
  let prev2 = 1;  // dp[i-2]
  let prev1 = 2;  // dp[i-1]
  
  for (let i = 3; i <= n; i++) {
    const curr = prev1 + prev2;
    prev2 = prev1;
    prev1 = curr;
  }
  
  return prev1;
}
```

空间复杂度从 O(n) 优化到 O(1)。

## 什么问题适合用 DP？

动态规划适用于具有以下两个特征的问题：

### 特征一：重叠子问题（Overlapping Subproblems）

问题可以分解成子问题，且子问题会被重复求解。

**有重叠**：爬楼梯、斐波那契、最短路径
**无重叠**：归并排序、快速排序（分治问题，子问题不重复）

### 特征二：最优子结构（Optimal Substructure）

大问题的最优解可以由小问题的最优解推导出来。

**有最优子结构**：
- 最短路径：A→C 的最短路径 = A→B 的最短路径 + B→C 的最短路径
- 最长公共子序列：`LCS(X,Y)` 可由 `LCS(X',Y')` 推导

**无最优子结构**：
- 最长简单路径：A→C 的最长路径 ≠ A→B 最长路径 + B→C 最长路径（可能有重复节点）

## DP 与其他算法的关系

| 算法 | 子问题关系 | 是否存储 | 典型应用 |
|-----|-----------|---------|---------|
| **递归** | 分解 | ❌ | 树的遍历 |
| **分治** | 分解且不重叠 | ❌ | 归并排序 |
| **记忆化搜索** | 分解且重叠 | ✅ | DP 的递归写法 |
| **动态规划** | 分解且重叠 | ✅ | 最优化问题 |
| **贪心** | 局部最优 | ❌ | 活动选择 |

**DP vs 贪心**：
- 贪心：每步选当前最优，不回头
- DP：考虑所有可能，选全局最优

**DP vs 分治**：
- 分治：子问题独立，不存储
- DP：子问题重叠，必须存储

## 动态规划的解题框架

解决 DP 问题的通用步骤：

```
1. 定义状态
   - dp[i] 表示什么？
   - dp[i][j] 表示什么？

2. 确定状态转移方程
   - dp[i] 如何由 dp[j] (j < i) 推导？
   - 枚举所有可能的前一个状态

3. 确定边界条件
   - dp[0] 是什么？
   - dp[0][0] 是什么？

4. 确定计算顺序
   - 保证计算 dp[i] 时，dp[j] (j < i) 已经计算好

5. 优化空间（可选）
   - 能否用滚动数组？
   - 能否降维？
```

后续章节会详细讲解每个步骤。

## 为什么要学动态规划？

### 面试必考

在 LeetCode 上，动态规划相关题目超过 300 道，占比约 15%。在 FAANG 等大厂面试中，DP 几乎是必考内容。

### 培养算法思维

DP 训练的是"状态思维"和"递推思维"，这是算法能力的核心。掌握 DP 后，你会发现很多问题都可以用类似的思维框架解决。

### 实际应用广泛

- **路径规划**：地图软件的最短路径
- **文本处理**：拼写检查、diff 算法
- **资源分配**：背包问题的各种变体
- **金融分析**：股票买卖策略
- **生物信息**：DNA 序列比对

## 本章小结

1. **动态规划是什么**：通过存储子问题的解来避免重复计算的算法技术
2. **核心思想**：把大问题分解成重叠的子问题，记住每个子问题的解
3. **两大特征**：重叠子问题 + 最优子结构
4. **两种实现**：记忆化搜索（自顶向下）和递推（自底向上）
5. **解题框架**：定义状态 → 状态转移 → 边界条件 → 计算顺序 → 空间优化

动态规划的难点在于"定义状态"和"推导转移方程"。下一章，我们将深入探讨重叠子问题与最优子结构的识别方法。
