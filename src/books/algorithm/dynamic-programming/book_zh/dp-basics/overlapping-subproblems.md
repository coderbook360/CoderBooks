# 重叠子问题与最优子结构

上一章我们了解了动态规划的基本概念，本章深入探讨 DP 的两大核心特征：**重叠子问题**和**最优子结构**。理解这两个概念，是判断一个问题能否用 DP 解决的关键。

## 重叠子问题（Overlapping Subproblems）

### 什么是重叠子问题？

当一个问题可以被分解成子问题，且这些子问题**会被多次求解**时，我们说这个问题具有重叠子问题的性质。

用一个公式表达：

```
如果求解 P(n) 需要求解 P(k)，且 P(k) 在整个计算过程中被多次需要，
那么问题 P 具有重叠子问题的性质。
```

### 经典案例：斐波那契数列

斐波那契数列的定义：

```
F(0) = 0
F(1) = 1
F(n) = F(n-1) + F(n-2), n >= 2
```

计算 `F(5)` 的递归树：

```
                    F(5)
                   /    \
                F(4)    F(3)
               /   \    /   \
            F(3)  F(2) F(2) F(1)
           /  \   / \   / \
        F(2) F(1) ... ... ...
        / \
      F(1) F(0)
```

观察这棵树：
- `F(3)` 被计算了 2 次
- `F(2)` 被计算了 3 次
- `F(1)` 被计算了 5 次

**子问题严重重叠！**

### 重叠子问题的判断方法

如何判断一个问题是否有重叠子问题？

**方法一：画递归树**

画出递归树，观察是否有相同的节点出现多次。

```typescript
// 在递归中加日志，观察重复调用
function fib(n: number): number {
  console.log(`Computing fib(${n})`);  // 加这行观察
  if (n <= 1) return n;
  return fib(n - 1) + fib(n - 2);
}
```

运行 `fib(5)` 会看到 `Computing fib(2)` 出现 3 次。

**方法二：分析递归公式**

如果递归公式中同一个子问题被多个更大的问题依赖，就有重叠。

```
F(n) = F(n-1) + F(n-2)
F(n-1) = F(n-2) + F(n-3)
         ↑
      F(n-2) 被 F(n) 和 F(n-1) 都需要
```

### 有重叠 vs 无重叠

| 问题类型 | 子问题关系 | 适合的算法 |
|---------|-----------|-----------|
| 斐波那契、爬楼梯 | 子问题重叠 | 动态规划 |
| 归并排序 | 子问题独立 | 分治 |
| 二分查找 | 子问题独立 | 分治 |
| 快速排序 | 子问题独立 | 分治 |

**归并排序为什么不是 DP？**

```
MergeSort([1,5,2,4])
├── MergeSort([1,5])     // 子问题 A
│   ├── MergeSort([1])
│   └── MergeSort([5])
└── MergeSort([2,4])     // 子问题 B
    ├── MergeSort([2])
    └── MergeSort([4])
```

子问题 A 和子问题 B 完全独立，没有重叠。这是**分治**，不是 DP。

## 最优子结构（Optimal Substructure）

### 什么是最优子结构？

如果一个问题的**最优解可以由子问题的最优解构造出来**，那么这个问题具有最优子结构的性质。

换句话说：**大问题的最优解包含小问题的最优解。**

### 经典案例：最短路径

从城市 A 到城市 C 的最短路径问题，假设必须经过城市 B：

```
A ────────→ B ────────→ C
   最短路径    最短路径
```

**最优子结构**：
- A→C 的最短路径 = A→B 的最短路径 + B→C 的最短路径
- 如果 A→C 的最短路径经过 B，那么 A→B 这段一定是 A 到 B 的最短路径

**证明**（反证法）：
假设 A→C 的最短路径是 A→B→C，且 A→B 不是最短的。那么存在更短的 A→B'→B，将原路径中的 A→B 替换为 A→B'→B，得到更短的 A→C 路径。矛盾！

### 反例：最长简单路径

从 A 到 C 的**最长简单路径**（不重复经过节点）问题：

```
    ┌───→ B ───┐
    │          │
A ──┤          ├──→ C
    │          │
    └───→ D ───┘
```

假设图中有两条路径：
- A → B → C（长度 2）
- A → D → C（长度 2）

**没有最优子结构**：
- A→C 的最长路径可能是 A→B→D→C（如果有 B→D 边）
- 但 A→B 的最长路径是 A→D→B
- 我们不能简单地把 A→B 的最长路径和 B→C 的最长路径拼起来，因为可能会重复经过节点！

### 最优子结构的判断方法

**方法一：验证子问题最优性**

问自己："如果我已经知道所有子问题的最优解，能否直接构造出大问题的最优解？"

**方法二：检查决策独立性**

子问题的解是否相互独立？如果一个子问题的决策会影响另一个子问题的可选范围，可能就没有最优子结构。

### 有最优子结构的问题

| 问题 | 最优子结构表达 |
|-----|---------------|
| 最短路径 | 最短路径的子路径也是最短路径 |
| 最长公共子序列 | LCS 的前缀也是 LCS |
| 矩阵链乘法 | 最优括号化的子表达式也是最优的 |
| 背包问题 | 最优装法的子集也是子背包的最优装法 |

## 两个特征的协同作用

**只有同时具备两个特征，才适合用动态规划：**

```
           有重叠子问题？
              /      \
            是        否
            |          |
     有最优子结构？    分治算法
        /      \
       是       否
       |        |
   动态规划   记忆化搜索
            （无最优解保证）
```

### 案例分析

**案例一：零钱兑换**

> 给定不同面额的硬币和一个总金额，计算凑成总金额所需的最少硬币数。

**重叠子问题？** ✅
- 凑 11 元需要凑 10 元（用 1 元）、凑 9 元（用 2 元）、凑 6 元（用 5 元）
- 凑 10 元需要凑 9 元、凑 8 元、凑 5 元
- `凑 9 元` 被重复需要

**最优子结构？** ✅
- 凑 11 元的最优解 = min(凑 10 元的最优解, 凑 9 元的最优解, 凑 6 元的最优解) + 1
- 子问题的最优解可以构成大问题的最优解

**结论**：适合 DP！

```typescript
function coinChange(coins: number[], amount: number): number {
  // dp[i] = 凑成金额 i 的最少硬币数
  const dp = new Array(amount + 1).fill(Infinity);
  dp[0] = 0;
  
  for (let i = 1; i <= amount; i++) {
    for (const coin of coins) {
      if (i >= coin) {
        dp[i] = Math.min(dp[i], dp[i - coin] + 1);
      }
    }
  }
  
  return dp[amount] === Infinity ? -1 : dp[amount];
}
```

**案例二：最长递增子序列（LIS）**

> 给定一个整数数组，找出最长严格递增子序列的长度。

**重叠子问题？** ✅
- 以 `nums[5]` 结尾的 LIS 需要知道以 `nums[0]` 到 `nums[4]` 结尾的 LIS
- 以 `nums[6]` 结尾的 LIS 也需要知道以 `nums[0]` 到 `nums[5]` 结尾的 LIS
- 子问题被重复依赖

**最优子结构？** ✅
- 以 `nums[i]` 结尾的 LIS = max(以 `nums[j]` 结尾的 LIS) + 1，其中 j < i 且 nums[j] < nums[i]
- 子问题的最优解可以构成大问题的最优解

**结论**：适合 DP！

```typescript
function lengthOfLIS(nums: number[]): number {
  const n = nums.length;
  // dp[i] = 以 nums[i] 结尾的 LIS 长度
  const dp = new Array(n).fill(1);
  
  for (let i = 1; i < n; i++) {
    for (let j = 0; j < i; j++) {
      if (nums[j] < nums[i]) {
        dp[i] = Math.max(dp[i], dp[j] + 1);
      }
    }
  }
  
  return Math.max(...dp);
}
```

## 常见误区

### 误区一：有递归就是 DP

不是所有递归都是 DP。如果子问题不重叠，那只是普通的分治。

```typescript
// 这是分治，不是 DP
function mergeSort(arr: number[]): number[] {
  if (arr.length <= 1) return arr;
  const mid = Math.floor(arr.length / 2);
  const left = mergeSort(arr.slice(0, mid));   // 子问题独立
  const right = mergeSort(arr.slice(mid));      // 子问题独立
  return merge(left, right);
}
```

### 误区二：用了备忘录就是 DP

备忘录只是优化手段。如果问题本身不具有重叠子问题，加备忘录也没用。

### 误区三：DP 一定比暴力快

DP 的时间复杂度是 O(子问题数量 × 每个子问题的时间)。如果子问题数量本身就是指数级的，DP 也快不了。

## 如何识别 DP 问题？

### 关键词提示

看到这些关键词，很可能是 DP 问题：

| 关键词 | 对应 DP 类型 |
|-------|-------------|
| 最长、最短、最大、最小 | 最优化 DP |
| 方案数、有多少种 | 计数 DP |
| 是否可能 | 可行性 DP |
| 第 K 个 | 区间 DP 或状态压缩 |

### 问题特征

1. **可分解性**：问题可以分解成规模更小的子问题
2. **子问题重复**：不同的计算路径会遇到相同的子问题
3. **最优性传递**：子问题的最优解可以组合成大问题的最优解

### 自问清单

遇到一个问题时，问自己：

1. 这个问题能分解成子问题吗？
2. 子问题之间有什么关系？
3. 子问题会被重复求解吗？
4. 子问题的最优解能推出大问题的最优解吗？

## 本章小结

1. **重叠子问题**：子问题被多次求解，是 DP 优化的基础
2. **最优子结构**：大问题的最优解由子问题的最优解构成，是 DP 正确性的保证
3. **两者缺一不可**：只有重叠没最优子结构→记忆化但无最优保证；只有最优子结构没重叠→分治即可
4. **判断方法**：画递归树看重叠，验证子问题最优性看结构
5. **常见陷阱**：不是所有递归都是 DP，不是所有最优化问题都有最优子结构
