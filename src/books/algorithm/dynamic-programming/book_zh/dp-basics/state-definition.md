# 状态定义的艺术

动态规划最难的部分不是写代码，而是**定义状态**。状态定义决定了整个解题思路，一个好的状态定义可以让问题迎刃而解，而一个糟糕的状态定义可能让你陷入死胡同。

本章将系统讲解状态定义的方法论，帮你建立正确的思维框架。

## 什么是"状态"？

在 DP 中，**状态是描述子问题的变量组合**。

以爬楼梯问题为例：
- 子问题："爬到第 i 级台阶有多少种方法？"
- 状态：`dp[i]` = 爬到第 i 级台阶的方法数
- 变量：只有一个——台阶数 `i`

状态定义要回答的核心问题是：**用什么变量可以唯一确定一个子问题？**

## 状态定义的基本原则

### 原则一：状态要能完整描述子问题

状态必须包含求解子问题所需的全部信息，不多也不少。

**反例：不完整的状态**

问题：给定一个数组，求两个不相交子数组的最大和。

错误定义：`dp[i]` = 前 i 个元素的最大子数组和

这个状态不够！因为我们还需要知道"是否已经选了一个子数组"，否则无法判断下一个元素是接着选还是开始新的子数组。

正确定义：`dp[i][j]` 
- `j = 0`：前 i 个元素中，还没选完第一个子数组
- `j = 1`：前 i 个元素中，已选完第一个子数组，还没开始第二个
- `j = 2`：前 i 个元素中，正在选第二个子数组

### 原则二：状态维度要尽量少

维度越多，时间和空间复杂度越高。在满足"完整描述"的前提下，维度越少越好。

**如何降维？**

1. **合并状态**：如果两个状态总是一起出现，可以合并
2. **消除冗余**：如果某个维度可以由其他维度推导，就删掉
3. **滚动数组**：如果只依赖前几个状态，可以压缩空间

### 原则三：状态要便于转移

好的状态定义应该让状态转移方程简洁、自然。如果状态转移异常复杂，可能是状态定义不合适。

## 一维状态：基础模式

### 模式一：以位置结尾

**定义**：`dp[i]` = 以第 i 个元素**结尾**的最优解/方案数

**适用问题**：
- 最长递增子序列
- 最大子数组和
- 以某元素结尾的计数问题

**案例：最长递增子序列（LIS）**

```typescript
// dp[i] = 以 nums[i] 结尾的 LIS 长度
function lengthOfLIS(nums: number[]): number {
  const n = nums.length;
  const dp = new Array(n).fill(1);
  
  for (let i = 1; i < n; i++) {
    for (let j = 0; j < i; j++) {
      // 如果 nums[j] < nums[i]，可以接在 nums[j] 后面
      if (nums[j] < nums[i]) {
        dp[i] = Math.max(dp[i], dp[j] + 1);
      }
    }
  }
  
  return Math.max(...dp);
}
```

**为什么用"以 i 结尾"而不是"前 i 个元素"？**

如果定义 `dp[i]` = 前 i 个元素的 LIS 长度：
- 无法判断 `nums[i]` 能否接在前面的 LIS 后面
- 因为不知道前面 LIS 的最后一个元素是什么

"以 i 结尾"的定义隐含了最后一个元素的信息，便于转移。

### 模式二：前 i 个元素

**定义**：`dp[i]` = 前 i 个元素的最优解/方案数（不要求用到第 i 个）

**适用问题**：
- 打家劫舍
- 爬楼梯
- 前缀类问题

**案例：打家劫舍**

```typescript
// dp[i] = 前 i 个房屋能偷到的最大金额
function rob(nums: number[]): number {
  const n = nums.length;
  if (n === 0) return 0;
  if (n === 1) return nums[0];
  
  const dp = new Array(n);
  dp[0] = nums[0];
  dp[1] = Math.max(nums[0], nums[1]);
  
  for (let i = 2; i < n; i++) {
    // 偷第 i 个：dp[i-2] + nums[i]
    // 不偷第 i 个：dp[i-1]
    dp[i] = Math.max(dp[i - 1], dp[i - 2] + nums[i]);
  }
  
  return dp[n - 1];
}
```

### 两种模式的选择

| 特征 | 以位置结尾 | 前 i 个元素 |
|-----|-----------|-------------|
| 最终答案 | max(dp[0...n-1]) | dp[n-1] 或 dp[n] |
| 状态转移 | 需要枚举前面所有位置 | 通常只看前 1-2 个状态 |
| 典型问题 | LIS、最大子数组和 | 爬楼梯、打家劫舍 |

**选择建议**：
- 如果问题强调"连续性"或"结尾位置"，用"以位置结尾"
- 如果问题是"累积"性质，用"前 i 个元素"

## 二维状态：进阶模式

当一个维度不足以描述子问题时，需要增加维度。

### 模式一：区间 [i, j]

**定义**：`dp[i][j]` = 区间 [i, j] 的最优解

**适用问题**：
- 最长回文子串
- 戳气球
- 矩阵链乘法

**案例：最长回文子串**

```typescript
// dp[i][j] = s[i..j] 是否是回文串
function longestPalindrome(s: string): string {
  const n = s.length;
  const dp: boolean[][] = Array.from({ length: n }, () => 
    new Array(n).fill(false)
  );
  
  let start = 0, maxLen = 1;
  
  // 初始化：单个字符是回文
  for (let i = 0; i < n; i++) {
    dp[i][i] = true;
  }
  
  // 枚举长度
  for (let len = 2; len <= n; len++) {
    for (let i = 0; i + len - 1 < n; i++) {
      const j = i + len - 1;
      if (s[i] === s[j]) {
        if (len === 2) {
          dp[i][j] = true;
        } else {
          dp[i][j] = dp[i + 1][j - 1];
        }
      }
      if (dp[i][j] && len > maxLen) {
        start = i;
        maxLen = len;
      }
    }
  }
  
  return s.substring(start, start + maxLen);
}
```

### 模式二：两个序列

**定义**：`dp[i][j]` = 第一个序列前 i 个元素与第二个序列前 j 个元素的最优解

**适用问题**：
- 最长公共子序列（LCS）
- 编辑距离
- 字符串匹配

**案例：最长公共子序列**

```typescript
// dp[i][j] = text1[0..i-1] 和 text2[0..j-1] 的 LCS 长度
function longestCommonSubsequence(text1: string, text2: string): number {
  const m = text1.length, n = text2.length;
  const dp: number[][] = Array.from({ length: m + 1 }, () => 
    new Array(n + 1).fill(0)
  );
  
  for (let i = 1; i <= m; i++) {
    for (let j = 1; j <= n; j++) {
      if (text1[i - 1] === text2[j - 1]) {
        dp[i][j] = dp[i - 1][j - 1] + 1;
      } else {
        dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
      }
    }
  }
  
  return dp[m][n];
}
```

### 模式三：位置 + 状态

**定义**：`dp[i][s]` = 在位置 i，处于状态 s 时的最优解

**适用问题**：
- 买卖股票（持有/不持有）
- 打家劫舍 III（选/不选）
- 粉刷房子（颜色状态）

**案例：买卖股票（含冷冻期）**

```typescript
// dp[i][0] = 第 i 天不持有股票的最大收益
// dp[i][1] = 第 i 天持有股票的最大收益
// dp[i][2] = 第 i 天处于冷冻期的最大收益
function maxProfit(prices: number[]): number {
  const n = prices.length;
  if (n === 0) return 0;
  
  const dp = Array.from({ length: n }, () => [0, 0, 0]);
  dp[0][0] = 0;
  dp[0][1] = -prices[0];
  dp[0][2] = 0;
  
  for (let i = 1; i < n; i++) {
    // 不持有：前一天不持有 或 前一天冷冻期
    dp[i][0] = Math.max(dp[i - 1][0], dp[i - 1][2]);
    // 持有：前一天持有 或 前一天不持有今天买入
    dp[i][1] = Math.max(dp[i - 1][1], dp[i - 1][0] - prices[i]);
    // 冷冻期：前一天持有今天卖出
    dp[i][2] = dp[i - 1][1] + prices[i];
  }
  
  return Math.max(dp[n - 1][0], dp[n - 1][2]);
}
```

## 状态定义的思维训练

### 训练一：从问题描述提取状态

**问题**：给定 m×n 网格，从左上角走到右下角，每次只能向右或向下，有多少种走法？

**分析**：
- 最终目标：到达 (m-1, n-1)
- 子问题：到达 (i, j) 有多少种走法
- 状态变量：位置 (i, j)
- 状态定义：`dp[i][j]` = 到达位置 (i, j) 的走法数

### 训练二：识别隐藏状态

**问题**：给定整数数组，选择一些数，使得任意两个选中的数不相邻，求最大和。

**分析**：
- 表面上只有一个变量：当前位置 i
- 但是！我们需要知道"前一个数选没选"
- 隐藏状态：选/不选

**状态定义方案一**：
- `dp[i][0]` = 前 i 个数，第 i 个不选的最大和
- `dp[i][1]` = 前 i 个数，第 i 个选的最大和

**状态定义方案二**（更简洁）：
- `dp[i]` = 前 i 个数的最大和（不管第 i 个选没选）
- 转移时自动处理：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`

### 训练三：状态压缩的时机

**问题**：有 n 个物品，选择一些物品，总重量不超过 W，求最大价值。

**朴素状态**：`dp[i][j]` = 前 i 个物品，容量为 j 时的最大价值

观察转移方程：
```
dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])
```

`dp[i]` 只依赖 `dp[i-1]`，可以用滚动数组压缩为一维：

```typescript
// 01 背包的空间优化
function knapsack(weights: number[], values: number[], W: number): number {
  const dp = new Array(W + 1).fill(0);
  
  for (let i = 0; i < weights.length; i++) {
    // 倒序遍历，保证每个物品只用一次
    for (let j = W; j >= weights[i]; j--) {
      dp[j] = Math.max(dp[j], dp[j - weights[i]] + values[i]);
    }
  }
  
  return dp[W];
}
```

## 常见状态设计模式总结

| 模式 | 状态定义 | 典型问题 |
|-----|---------|---------|
| 线性单序列 | `dp[i]` | 爬楼梯、打家劫舍 |
| 以位置结尾 | `dp[i]` = 以 i 结尾的... | LIS、最大子数组和 |
| 区间 | `dp[i][j]` = 区间 [i,j] 的... | 回文子串、戳气球 |
| 双序列 | `dp[i][j]` = 序列1前i个与序列2前j个的... | LCS、编辑距离 |
| 背包 | `dp[i][j]` = 前i个物品、容量j的... | 01背包、完全背包 |
| 状态机 | `dp[i][s]` = 位置i、状态s时的... | 股票买卖 |
| 状态压缩 | `dp[mask]` = 状态为 mask 时的... | TSP、子集问题 |

## 本章小结

1. **状态是什么**：描述子问题的变量组合
2. **三个原则**：完整描述、维度最少、便于转移
3. **一维模式**：以位置结尾 vs 前 i 个元素
4. **二维模式**：区间、双序列、位置+状态
5. **关键技巧**：识别隐藏状态、适时压缩空间

状态定义没有固定公式，需要大量练习培养直觉。下一章，我们将学习如何从状态定义推导出**状态转移方程**。
