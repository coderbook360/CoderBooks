# 实战：整数拆分

## 问题描述

**LeetCode 343: 整数拆分**

给定一个正整数 `n`，将其拆分为至少两个正整数的和，并使这些整数的乘积最大化。返回你可以获得的最大乘积。

**示例1**：
```
输入：n = 2
输出：1
解释：2 = 1 + 1, 1 × 1 = 1
```

**示例2**：
```
输入：n = 10
输出：36
解释：10 = 3 + 3 + 4, 3 × 3 × 4 = 36
```

**约束**：
- `2 <= n <= 58`

## 问题分析

### 朴素思路

**枚举拆分方式**：
- `n = a₁ + a₂ + ... + aₖ`（至少 2 项）
- 求 `max(a₁ × a₂ × ... × aₖ)`

**关键观察**：
1. 拆分成**多少份**？
2. 每份的**大小**是多少？

### 数学推导

**均值不等式**：
```
对于和为定值的正数，当它们相等时，乘积最大
```

**证明**：
```
设 a₁ + a₂ + ... + aₖ = n
则 a₁ × a₂ × ... × aₖ <= [(a₁ + a₂ + ... + aₖ) / k]^k = (n / k)^k
```

**最优拆分策略**：
- 尽量平均拆分
- 拆分成接近 `e ≈ 2.718` 的数字时，乘积最大
- 实际中，拆成 **3** 和 **2** 最优

**规律**：
- 优先拆成 **3**
- 如果余数是 **1**，改成拆两个 **2**（因为 3×3×1 = 9 < 3×2×2 = 12）
- 如果余数是 **2**，直接加一个 **2**

## 解法一：动态规划

### 状态定义

```
dp[i] = 将正整数 i 拆分后获得的最大乘积
```

### 状态转移

```python
dp[i] = max(
    max(j × (i - j), j × dp[i - j]) for j in range(1, i)
)
```

**含义**：
- `j × (i - j)`：拆成两部分 `j` 和 `i-j`
- `j × dp[i - j]`：拆成 `j` 和 `(i-j)` 的继续拆分

### 边界条件

```python
dp[1] = 1  # 虽然题目要求至少拆成 2 个，但作为子问题需要定义
dp[2] = 1  # 2 = 1 + 1, 1 × 1 = 1
```

### 代码实现

```python
def integerBreak(n):
    """
    整数拆分（动态规划）
    """
    if n <= 3:
        return n - 1  # 特殊处理：2->1, 3->2
    
    # 初始化
    dp = [0] * (n + 1)
    dp[1] = 1
    dp[2] = 2  # 注意：这里是 2，而不是 1
    dp[3] = 3  # 注意：这里是 3，而不是 2
    
    # 状态转移
    for i in range(4, n + 1):
        for j in range(1, i // 2 + 1):
            dp[i] = max(dp[i], j * dp[i - j])
    
    return dp[n]
```

**为什么 `dp[2] = 2` 而不是 `1`？**
```
当 n = 2 时，必须拆分：2 = 1 + 1，乘积 = 1
但作为子问题，当 i = 4 时：
  dp[4] = max(2 × dp[2], ...)
  如果 dp[2] = 1，那么 2 × 1 = 2
  但实际上，2 不拆分比拆分更好：2 × 2 = 4 > 2 × 1 = 2
  
所以 dp[2] = 2 表示"2 本身的值"，用于子问题计算
```

### 复杂度分析

- **时间复杂度**：O(n²)
- **空间复杂度**：O(n)

## 解法二：数学优化（贪心）

### 核心思想

**最优策略**：
1. 尽量拆成 **3**
2. 如果 `n % 3 == 1`，拆成 `3^(n//3 - 1) × 4`
3. 如果 `n % 3 == 2`，拆成 `3^(n//3) × 2`
4. 如果 `n % 3 == 0`，拆成 `3^(n//3)`

### 代码实现

```python
def integerBreak_math(n):
    """
    整数拆分（数学优化）
    """
    if n == 2:
        return 1
    if n == 3:
        return 2
    
    # n >= 4
    quotient, remainder = divmod(n, 3)
    
    if remainder == 0:
        return 3 ** quotient
    elif remainder == 1:
        return 3 ** (quotient - 1) * 4
    else:  # remainder == 2
        return 3 ** quotient * 2
```

### 复杂度分析

- **时间复杂度**：O(log n)（幂运算）
- **空间复杂度**：O(1)

### 示例2：n = 10（数学方法）

```
10 % 3 = 1

验证：10 = 3 + 3 + 4
     3 × 3 × 4 = 36 ✓
```

## 数学证明

### 为什么是 3？

**引理**：拆分成尽量多的 3，乘积最大

**证明**：
```
1. 不拆 1：1 × (n-1) < n
2. 尽量少拆 2：2 × (n-2) < n (当 n >= 4)
3. 比较 2 和 3：
   - 6 = 2 + 2 + 2 = 3 + 3, 2³ = 8 < 3² = 9
   - 一般地：3^k > 2^(1.5k) (当 k >= 2)
4. 比较 3 和 4：
   - 4 = 3 + 1，但 3 × 1 = 3 < 4
   - 4 = 2 + 2，2 × 2 = 4 = 4（等价）
   - 所以拆成 3 和 2 都可以，优先 3
```

### 余数处理

```
n % 3 = 0: 全部拆成 3
  例如：9 = 3 + 3 + 3, 3³ = 27

n % 3 = 1: 最后一个 3 和 1 合并成 4
  例如：10 = 3 + 3 + 4, 3² × 4 = 36
  而不是：10 = 3 + 3 + 3 + 1, 3³ × 1 = 27（更小）

n % 3 = 2: 最后加一个 2
  例如：11 = 3 + 3 + 3 + 2, 3³ × 2 = 54
```

## 常见错误

### 错误1：dp 初始化错误

```python
# 错误：dp[2] = 1, dp[3] = 2
dp = [0] * (n + 1)
dp[2] = 1  # 错误！作为子问题应该是 2
dp[3] = 2  # 错误！作为子问题应该是 3

# 正确：
dp[2] = 2  # 2 本身的值
dp[3] = 3  # 3 本身的值

# 特殊处理：
if n == 2: return 1
if n == 3: return 2
```

### 错误2：数学方法边界错误

```python
# 错误：没有处理 n = 2 和 n = 3
def wrong_solution(n):
    quotient, remainder = divmod(n, 3)
    if remainder == 1:
        return 3 ** (quotient - 1) * 4
    # ...

# 当 n = 2 时，quotient = 0, remainder = 2
# 返回 3^0 × 2 = 2，错误！应该返回 1

# 当 n = 3 时，quotient = 1, remainder = 0
# 返回 3^1 = 3，错误！应该返回 2
```

### 错误3：理解错误

```python
# 错误：以为 dp[i] 是必须拆分的结果
dp[2] = 1  # 2 必须拆成 1 + 1
dp[3] = 2  # 3 必须拆成 1 + 2 或 1 + 1 + 1

# 正确：dp[i] 是"最优策略"，可能是不拆分
# 对于 n = 2, 3，必须拆分，所以返回 n - 1
# 对于 i >= 4 作为子问题，不拆分更优
```

## 扩展问题

### 扩展1：拆分成恰好 k 个数

```python
def integerBreak_k_parts(n, k):
    """
    将 n 拆分成恰好 k 个正整数，使乘积最大
    """
    # 平均分配
    quotient, remainder = divmod(n, k)
    
    # remainder 个数分配 quotient + 1
    # k - remainder 个数分配 quotient
    return (quotient + 1) ** remainder * quotient ** (k - remainder)
```

### 扩展2：拆分成至少 k 个数

```python
def integerBreak_at_least_k(n, k):
    """
    拆分成至少 k 个数
    """
    if n < k:
        return 0  # 无法拆分
    
    # 策略：先保证有 k 个数（都是 1）
    # 剩余 n - k 继续拆分成 3 和 2
    remaining = n - k
    
    # 贪心拆分 remaining
    quotient, remainder = divmod(remaining, 3)
    
    if remainder == 0:
        base = 3 ** quotient
    elif remainder == 1:
        base = 3 ** (quotient - 1) * 4 if quotient > 0 else 1
    else:
        base = 3 ** quotient * 2
    
    return base
```

## 小结

### 核心思想
- **DP 方法**：`dp[i] = max(j × (i-j), j × dp[i-j])`
- **数学方法**：尽量拆成 3，余数特殊处理

### 关键步骤
1. **特殊处理**：`n = 2` 返回 `1`，`n = 3` 返回 `2`
2. **DP 初始化**：`dp[2] = 2`, `dp[3] = 3`（作为子问题）
3. **数学策略**：
   - `n % 3 == 0`: 全部拆 3
   - `n % 3 == 1`: 拆成 `3^(k-1) × 4`
   - `n % 3 == 2`: 拆成 `3^k × 2`

### 易错点
- ✓ `dp[2]` 和 `dp[3]` 初始化为 2 和 3
- ✓ 特殊处理 `n = 2, 3`
- ✗ 混淆"必须拆分"和"子问题最优"
- ✗ 数学方法忘记边界

### 优化建议
- **DP 方法**：适合理解问题本质
- **数学方法**：适合性能优化
- **实际应用**：`n <= 10` 用 DP，`n > 10` 用数学

这道题展示了**动态规划**与**数学优化**的结合：DP 保证正确性，数学分析提供最优策略。
