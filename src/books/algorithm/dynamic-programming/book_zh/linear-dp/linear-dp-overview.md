# 线性 DP 概述

线性 DP 是动态规划中最常见、最基础的类型。它的特点是状态沿着一维或二维的方向线性传播，通常只需一次遍历就能完成计算。

## 什么是线性 DP？

**线性 DP** 是指状态转移具有明确方向性的动态规划，状态从一端向另一端逐步推进，形成一条"线性"的计算流程。

### 典型特征

1. **状态线性排列**：状态可以用一维数组 `dp[i]` 或二维数组 `dp[i][j]` 表示
2. **计算顺序明确**：从前往后或从后往前依次计算
3. **依赖关系简单**：当前状态只依赖于之前的若干状态
4. **一次遍历完成**：时间复杂度通常是 O(n) 或 O(n²)

### 与其他 DP 类型的区别

| 类型 | 状态结构 | 转移特点 | 典型问题 |
|-----|---------|---------|---------|
| 线性 DP | 一维/二维 | 顺序推进 | 最大子数组和 |
| 区间 DP | 区间 [l, r] | 由小区间合成大区间 | 戳气球 |
| 树形 DP | 树结构 | 从子节点到父节点 | 打家劫舍 III |
| 状态压缩 DP | 集合（位） | 子集转移 | 旅行商问题 |

## 线性 DP 的分类

### 1. 单序列线性 DP

只涉及一个数组，状态是 `dp[i]`，表示以第 i 个元素结尾（或前 i 个元素）的某种性质。

**典型问题**：
- 最大子数组和
- 最长递增子序列
- 打家劫舍

**状态转移模式**：
```typescript
dp[i] = f(dp[i-1], dp[i-2], ..., nums[i])
```

### 2. 双序列线性 DP

涉及两个数组（或两个字符串），状态是 `dp[i][j]`，表示第一个序列前 i 个元素与第二个序列前 j 个元素的某种关系。

**典型问题**：
- 最长公共子序列
- 编辑距离
- 正则表达式匹配

**状态转移模式**：
```typescript
dp[i][j] = f(dp[i-1][j], dp[i][j-1], dp[i-1][j-1], s1[i], s2[j])
```

### 3. 带额外维度的线性 DP

状态是 `dp[i][k]`，其中 i 是位置，k 是额外的状态维度（如交易次数、颜色等）。

**典型问题**：
- 买卖股票的最佳时机
- 粉刷房子

**状态转移模式**：
```typescript
dp[i][k] = f(dp[i-1][k], dp[i-1][k-1], ...)
```

## 线性 DP 的解题模板

### 模板一：单序列（以 i 结尾）

```typescript
function linearDP(nums: number[]): number {
  const n = nums.length;
  const dp = new Array(n);
  
  // 初始化
  dp[0] = /* 基于第一个元素 */;
  
  // 状态转移
  for (let i = 1; i < n; i++) {
    dp[i] = /* 基于 dp[j] (j < i) 和 nums[i] */;
  }
  
  // 返回结果
  return Math.max(...dp);  // 或 dp[n-1]
}
```

### 模板二：单序列（前 i 个）

```typescript
function linearDP(nums: number[]): number {
  const n = nums.length;
  const dp = new Array(n + 1);
  
  // 初始化：前 0 个元素
  dp[0] = /* 空序列的值 */;
  
  // 状态转移：前 i 个元素
  for (let i = 1; i <= n; i++) {
    dp[i] = /* 基于 dp[j] (j < i) 和 nums[i-1] */;
  }
  
  return dp[n];
}
```

### 模板三：双序列

```typescript
function linearDP(s1: string, s2: string): number {
  const m = s1.length;
  const n = s2.length;
  const dp = Array.from({ length: m + 1 }, () => new Array(n + 1).fill(0));
  
  // 初始化边界
  for (let i = 0; i <= m; i++) dp[i][0] = /* ... */;
  for (let j = 0; j <= n; j++) dp[0][j] = /* ... */;
  
  // 状态转移
  for (let i = 1; i <= m; i++) {
    for (let j = 1; j <= n; j++) {
      if (s1[i - 1] === s2[j - 1]) {
        dp[i][j] = dp[i - 1][j - 1] + /* ... */;
      } else {
        dp[i][j] = /* 基于 dp[i-1][j], dp[i][j-1], dp[i-1][j-1] */;
      }
    }
  }
  
  return dp[m][n];
}
```

## 线性 DP 的常见技巧

### 技巧一：状态定义的两种方式

**以 i 结尾** vs **前 i 个**

| 方式 | 状态含义 | 适用场景 | 答案位置 |
|-----|---------|---------|---------|
| 以 i 结尾 | 必须包含第 i 个元素 | 最大子数组和、LIS | 遍历所有 dp[i] |
| 前 i 个 | 可选择是否包含 | 背包、打家劫舍 | dp[n] |

### 技巧二：空间优化

**滚动数组**：当 `dp[i]` 只依赖 `dp[i-1]` 和 `dp[i-2]` 时，可以用两个变量代替数组。

```typescript
// 优化前
const dp = new Array(n);
for (let i = 2; i < n; i++) {
  dp[i] = dp[i - 1] + dp[i - 2];
}

// 优化后
let prev2 = dp[0], prev1 = dp[1];
for (let i = 2; i < n; i++) {
  const curr = prev1 + prev2;
  prev2 = prev1;
  prev1 = curr;
}
```

### 技巧三：前缀和优化

当转移需要区间和时，用前缀和将 O(n²) 优化到 O(n)。

```typescript
// 优化前：O(n²)
for (let i = 0; i < n; i++) {
  for (let j = 0; j < i; j++) {
    // 需要 sum(j, i)
  }
}

// 优化后：O(n)
const prefix = [0];
for (const num of nums) {
  prefix.push(prefix[prefix.length - 1] + num);
}
// sum(j, i) = prefix[i+1] - prefix[j]
```
