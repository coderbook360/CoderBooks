# 状态机模型的建立

状态机 DP 的核心在于正确识别问题中的状态，并建立状态之间的转移关系。

## 什么是状态机

**状态机（State Machine）** 是一种数学模型，由以下要素组成：

- **状态集合**：系统可能处于的所有状态
- **初始状态**：系统的起始状态
- **转移规则**：从一个状态到另一个状态的条件
- **终止状态**：问题的目标状态

## 状态机 DP 的建模步骤

### 第一步：识别状态

问自己：**在任意时刻，问题有哪些互斥的情况？**

以股票问题为例：
- 持有股票 vs 不持有股票
- 在冷冻期 vs 不在冷冻期
- 已完成 k 次交易 vs 未完成 k 次交易

### 第二步：画状态图

把状态作为节点，转移作为边，画出状态转移图：

```
        ┌─────────┐
        │ 状态 A  │
        └────┬────┘
             │ 条件1
             ↓
        ┌─────────┐
        │ 状态 B  │
        └────┬────┘
             │ 条件2
             ↓
        ┌─────────┐
        │ 状态 C  │
        └─────────┘
```

### 第三步：定义转移方程

对于每个状态，写出它可以从哪些状态转移过来：

```
dp[状态A] = max(
    dp[状态A前一时刻],      // 保持
    dp[状态B前一时刻] + 收益  // 从 B 转移
)
```

### 第四步：确定边界和答案

- **边界**：初始时刻各状态的值
- **答案**：最终时刻哪个（些）状态是目标

## 经典状态机模式

### 模式一：二状态切换

最简单的状态机，两个状态互相切换：

```
     ┌────────────┐
     │   切换条件  │
     ↓            ↑
┌─────────┐  ┌─────────┐
│ 状态 A  │──│ 状态 B  │
└─────────┘  └─────────┘
```

**典型问题**：
- 买卖股票 II（持有/不持有）
- 打家劫舍（偷/不偷）
- 粉刷房子（颜色 A/颜色 B/颜色 C）

### 模式二：带约束的切换

某些转移有额外限制：

```
     ┌────────────┐
     │  可以转移   │
     ↓            │
┌─────────┐  ┌────┴────┐
│ 状态 A  │──│ 状态 B  │
└─────────┘  └────┬────┘
                  │ 冷冻
                  ↓
             ┌─────────┐
             │ 状态 C  │
             └─────────┘
```

**典型问题**：
- 买卖股票含冷冻期
- 买卖股票含手续费

### 模式三：计数状态

需要记录某个计数器：

```
┌──────────┐   ┌──────────┐   ┌──────────┐
│ 计数=0   │──→│ 计数=1   │──→│ 计数=2   │
└──────────┘   └──────────┘   └──────────┘
```

**典型问题**：
- 买卖股票 III/IV（交易次数）
- 学生出勤记录（缺勤/迟到次数）

### 模式四：环形状态

状态之间形成环：

```
    ┌────────────────────┐
    │                    │
    ↓                    │
┌─────────┐  ┌─────────┐─┘
│ 状态 A  │──│ 状态 B  │
└─────────┘  └─────────┘
```

**典型问题**：
- 粉刷房子（环形街道）
- 打家劫舍 II（环形房屋）

## 状态机 DP 模板

```typescript
/**
 * 状态机 DP 通用模板
 */
function stateMachineDP(input: any[]): number {
  // 1. 定义状态变量
  const states = initializeStates();
  
  // 2. 遍历每个决策点
  for (const item of input) {
    // 保存前一时刻的状态
    const prevStates = [...states];
    
    // 3. 更新每个状态
    for (let s = 0; s < states.length; s++) {
      states[s] = computeTransition(prevStates, s, item);
    }
  }
  
  // 4. 返回目标状态的值
  return getAnswer(states);
}
```

## 如何判断是否适用状态机 DP

**适用场景**：
1. 问题涉及多个互斥的状态
2. 决策会导致状态切换
3. 不同状态下的收益/成本不同
4. 需要优化某个目标函数

**不适用场景**：
1. 状态之间没有明确的切换关系
2. 状态数量过多（考虑状态压缩）
3. 转移关系过于复杂

## 状态机 vs 普通 DP

| 特点 | 普通 DP | 状态机 DP |
|-----|--------|----------|
| 状态定义 | 通常是位置/数量 | 互斥的模式/阶段 |
| 转移方式 | 从子问题组合 | 状态之间切换 |
| 可视化 | 表格/DAG | 状态转移图 |
| 思维模式 | 分解子问题 | 识别状态模式 |

## 实战：打家劫舍（状态机视角）

用状态机视角重新理解打家劫舍：

**状态定义**：
- `rob`：上一家被偷了
- `notRob`：上一家没被偷

**状态转移**：
```
         偷当前房
    ┌──────────────┐
    │              ↓
┌───┴───┐     ┌────────┐
│notRob │     │  rob   │
│(可以偷)│     │(不能偷) │
└───┬───┘     └────┬───┘
    ↑              │
    └──────────────┘
         不偷当前房
```

**代码**：

```typescript
function rob(nums: number[]): number {
  let robbed = 0;     // 上一家偷了
  let notRobbed = 0;  // 上一家没偷
  
  for (const money of nums) {
    const prevRobbed = robbed;
    const prevNotRobbed = notRobbed;
    
    // 当前偷：上一家必须没偷
    robbed = prevNotRobbed + money;
    
    // 当前不偷：上一家偷或不偷都行
    notRobbed = Math.max(prevRobbed, prevNotRobbed);
  }
  
  return Math.max(robbed, notRobbed);
}
```

## 状态设计的技巧

### 技巧一：最小化状态数

只保留影响决策的信息：

```typescript
// 不好：记录太多无关信息
dp[i][j][k][l] = ...

// 好：只记录必要信息
dp[i][state] = ...
```

### 技巧二：状态等价合并

如果多个状态对后续决策影响相同，合并它们：

```typescript
// 买卖股票：不关心赚了多少次，只关心当前收益
// 不需要：dp[i][交易次数][持有状态]
// 可以用：dp[j][持有状态]，其中 j = 第几轮
```

### 技巧三：用元组表示复合状态

```typescript
// 复合状态：(持有状态, 交易次数)
type State = [boolean, number];

// 或者用数字编码
// 0 = 不持有+0次, 1 = 持有+0次, 2 = 不持有+1次, ...
```

## 本章小结

1. **状态机的要素**：状态集合、初始状态、转移规则、终止状态
2. **建模步骤**：识别状态 → 画状态图 → 定义转移 → 确定边界
3. **经典模式**：二状态切换、带约束、计数状态、环形状态
4. **核心思维**：把问题抽象为状态之间的转换
