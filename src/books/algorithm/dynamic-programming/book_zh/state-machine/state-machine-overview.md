# 状态机 DP 概述

状态机 DP 是动态规划中一类重要的问题类型，通过显式定义"状态"和"状态转移"来解决有约束条件的决策问题。

## 什么是状态机 DP？

**状态机 DP** 是将问题建模为有限状态自动机（FSM），每个位置可以处于不同的状态，状态之间按照特定规则转移。

### 典型特征

1. **多状态**：每个位置存在多种可能的状态
2. **状态约束**：某些状态之间的转移受到限制
3. **状态转移图**：可以画出清晰的状态转移图
4. **最优子结构**：当前状态的最优值依赖于之前状态的最优值

### 与普通线性 DP 的区别

| 特点 | 普通线性 DP | 状态机 DP |
|-----|-----------|----------|
| 状态数 | 单一 | 多个 |
| 转移来源 | 前一个位置 | 多个状态 |
| 状态定义 | dp[i] | dp[i][state] |
| 典型问题 | 最大子数组和 | 买卖股票 |

## 状态机 DP 的经典问题：股票买卖

股票买卖系列是状态机 DP 的完美案例，通过不同的约束条件，展示了状态机 DP 的各种技巧。

### 状态定义

在股票问题中，每天可能处于以下状态：
- **持有股票**：当天结束时手里有股票
- **不持有股票**：当天结束时手里没有股票

### 状态转移图

```
       买入
  ┌──────────────┐
  │              ↓
不持有  ─────→  持有
  ↑     卖出     │
  └──────────────┘
       持有
```

### 状态转移方程

```
hold[i]    = max(hold[i-1], notHold[i-1] - prices[i])
notHold[i] = max(notHold[i-1], hold[i-1] + prices[i])
```

## 状态机 DP 的解题模板

### 步骤一：识别状态

分析问题中有哪些不同的"状态"，每个状态代表什么含义。

### 步骤二：画状态转移图

明确状态之间如何转移，哪些转移是允许的，哪些是禁止的。

### 步骤三：定义状态数组

`dp[i][state]` = 第 i 个位置处于 state 状态时的最优值

### 步骤四：写转移方程

根据状态转移图，写出每个状态的转移方程。

### 步骤五：确定初始状态和答案

- 初始状态：第一个位置各状态的值
- 答案：最后一个位置中哪些状态是有效的

## 股票买卖系列一览

| 题目 | 约束 | 关键点 |
|-----|------|-------|
| 买卖股票 I | 最多 1 次交易 | 找最大差值 |
| 买卖股票 II | 无限次交易 | 基础状态机 |
| 买卖股票 III | 最多 2 次交易 | 增加交易次数维度 |
| 买卖股票 IV | 最多 k 次交易 | 通用解法 |
| 含冷冻期 | 卖出后等一天 | 增加冷冻状态 |
| 含手续费 | 每次交易收费 | 修改转移方程 |

## 状态机 DP 的通用代码框架

```typescript
function stateMachineDP(nums: number[]): number {
  const n = nums.length;
  
  // 定义状态数组
  const dp: number[][] = Array.from(
    { length: n },
    () => new Array(STATE_COUNT).fill(0)
  );
  
  // 初始化第一个位置的状态
  dp[0][STATE_A] = /* 初始值 */;
  dp[0][STATE_B] = /* 初始值 */;
  
  // 状态转移
  for (let i = 1; i < n; i++) {
    // 状态 A 可以从哪些状态转移来
    dp[i][STATE_A] = /* 转移方程 */;
    
    // 状态 B 可以从哪些状态转移来
    dp[i][STATE_B] = /* 转移方程 */;
  }
  
  // 返回最终答案（通常是某个特定状态或所有状态的最优值）
  return Math.max(dp[n - 1][STATE_A], dp[n - 1][STATE_B]);
}
```
