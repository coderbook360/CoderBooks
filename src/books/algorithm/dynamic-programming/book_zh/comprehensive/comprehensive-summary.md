# 综合实战总结

## 回顾

本章我们通过四道综合性题目，整合了动态规划的多种技巧：

| 题目 | 核心技巧 | 难度 |
|------|----------|------|
| 鸡蛋掉落 | 二分优化、逆向思维 | ⭐⭐⭐⭐⭐ |
| 编辑距离 | 字符串 DP、路径回溯 | ⭐⭐⭐⭐ |
| 戳气球 | 区间 DP、逆向思维 | ⭐⭐⭐⭐⭐ |
| 正则匹配 | 字符串 DP、模式匹配 | ⭐⭐⭐⭐ |

## 核心思想提炼

### 1. 逆向思维

**鸡蛋掉落**：不问"需要多少次确定答案"，而问"k 次操作最多能确定多少层"。

**戳气球**：不问"先戳哪个"，而问"最后戳哪个"，消除了顺序依赖。

> 当正向思考陷入困境时，尝试逆向。

### 2. 状态设计的艺术

**编辑距离**：dp[i][j] = 前缀匹配的最小操作数

**正则匹配**：dp[i][j] = 前缀是否匹配（布尔值）

**戳气球**：dp[i][j] = 开区间内的最优解

> 好的状态定义能让转移方程自然流畅。

### 3. 边界处理技巧

**戳气球**：添加虚拟边界 [1, ...nums, 1]

**正则匹配**：空串与 x* 模式的匹配

> 边界处理往往决定代码的简洁程度。

### 4. 优化手段

| 优化方法 | 适用场景 | 效果 |
|----------|----------|------|
| 空间压缩 | 只依赖前一行 | O(n²) → O(n) |
| 二分优化 | 单调性 | O(n²) → O(n log n) |
| 逆向重定义 | 转换问题本质 | 降维或简化 |

## DP 问题解决框架

```
1. 明确问题：输入、输出、约束
   ↓
2. 定义状态：dp[...] 表示什么
   ↓
3. 状态转移：如何从子问题得到答案
   ↓
4. 边界条件：初始值是什么
   ↓
5. 计算顺序：从小到大还是区间遍历
   ↓
6. 优化：空间或时间
```

## 综合题的特点

1. **多种技巧融合**：一道题可能涉及多个 DP 范式
2. **需要创造性思维**：不是套模板就能解决
3. **边界情况复杂**：需要仔细处理
4. **优化空间大**：朴素解法往往可以进一步优化

## 学习建议

### 基础打牢

- 熟练掌握基本范式：线性、背包、区间、树形
- 理解每种范式的本质和适用场景

### 多维思考

- 一个问题可能有多种状态定义
- 尝试不同的思考角度
- 逆向思维往往有奇效

### 刻意练习

- 不要只追求"做出来"
- 思考：有没有更优雅的解法？
- 复盘：别人的解法好在哪里？

### 总结归类

建立自己的题目分类体系：
- 按 DP 类型
- 按优化技巧
- 按思维模式

## 结语

动态规划的精髓不在于记忆模板，而在于：

1. **识别子问题结构**：问题能否分解？
2. **设计状态表示**：如何精确描述子问题？
3. **推导转移方程**：子问题如何组合？
4. **优化实现**：如何更高效？

掌握了这些核心能力，面对任何 DP 问题都能从容应对。

> "The art of dynamic programming is to find the right subproblems."
> 
> —— 动态规划的艺术在于找到正确的子问题
