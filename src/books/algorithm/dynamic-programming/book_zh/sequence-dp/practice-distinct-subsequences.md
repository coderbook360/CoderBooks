# 实战：不同的子序列

## 题目描述

给你两个字符串 `s` 和 `t`，统计并返回在 `s` 的子序列中 `t` 出现的个数。

题目数据保证答案在 32 位带符号整数范围内。

📎 [LeetCode 115. 不同的子序列](https://leetcode.cn/problems/distinct-subsequences/)

**示例**：

```
输入：s = "rabbbit", t = "rabbit"
输出：3
解释：有 3 种方法可以从 s 中得到 "rabbit"：
- rabb[b]it（选择第 1、2、3、5、6、7 位）
- rab[b]bit（选择第 1、2、4、5、6、7 位）
- ra[b]bbit（选择第 1、2、3、4、6、7 位）
```

## 思路分析

这是一个**计数型** DP 问题，需要统计方案数而不是求最优解。

**问题转化**：从 s 中选择字符组成 t，有多少种选法？

## 状态定义

```
dp[i][j] = s[0..i-1] 的子序列中 t[0..j-1] 出现的次数
```

## 状态转移

```
如果 s[i-1] === t[j-1]:
    dp[i][j] = dp[i-1][j-1] + dp[i-1][j]
    // 选择 s[i-1] 匹配 t[j-1]，或不选择 s[i-1]

否则:
    dp[i][j] = dp[i-1][j]
    // 不能用 s[i-1]，只能跳过
```

## 边界条件

```
dp[i][0] = 1  // t 为空串，只有一种选法（什么都不选）
dp[0][j] = 0 (j > 0)  // s 为空串但 t 不为空，无法匹配
```

## 代码实现

```typescript
/**
 * 不同的子序列
 * 时间复杂度：O(m × n)
 * 空间复杂度：O(m × n)
 */
function numDistinct(s: string, t: string): number {
  const m = s.length, n = t.length;
  
  // dp[i][j] = s[0..i-1] 的子序列中 t[0..j-1] 出现的次数
  const dp: number[][] = Array.from(
    { length: m + 1 },
    () => new Array(n + 1).fill(0)
  );
  
  // 边界：t 为空串时，只有一种选法
  for (let i = 0; i <= m; i++) {
    dp[i][0] = 1;
  }
  
  for (let i = 1; i <= m; i++) {
    for (let j = 1; j <= n; j++) {
      if (s[i - 1] === t[j - 1]) {
        // 选择 s[i-1] 或不选择
        dp[i][j] = dp[i - 1][j - 1] + dp[i - 1][j];
      } else {
        // 不能用 s[i-1]
        dp[i][j] = dp[i - 1][j];
      }
    }
  }
  
  return dp[m][n];
}
```

## 空间优化

注意：由于依赖 `dp[i-1][j-1]` 和 `dp[i-1][j]`，需要从后往前更新。

```typescript
function numDistinct(s: string, t: string): number {
  const m = s.length, n = t.length;
  const dp = new Array(n + 1).fill(0);
  dp[0] = 1;
  
  for (let i = 1; i <= m; i++) {
    // 从后往前更新，避免覆盖
    for (let j = n; j >= 1; j--) {
      if (s[i - 1] === t[j - 1]) {
        dp[j] = dp[j - 1] + dp[j];
      }
      // 不相等时 dp[j] 保持不变
    }
  }
  
  return dp[n];
}
```

## 示例演算

以 `s = "rabbbit"`, `t = "rabbit"` 为例：

|   | "" | r | a | b | b | i | t |
|---|---|---|---|---|---|---|---|
| "" | 1 | 0 | 0 | 0 | 0 | 0 | 0 |
| r | 1 | 1 | 0 | 0 | 0 | 0 | 0 |
| a | 1 | 1 | 1 | 0 | 0 | 0 | 0 |
| b | 1 | 1 | 1 | 1 | 0 | 0 | 0 |
| b | 1 | 1 | 1 | 2 | 1 | 0 | 0 |
| b | 1 | 1 | 1 | 3 | 3 | 0 | 0 |
| i | 1 | 1 | 1 | 3 | 3 | 3 | 0 |
| t | 1 | 1 | 1 | 3 | 3 | 3 | **3** |

最终答案：3

## 状态转移图解

```
s[i-1] === t[j-1]:
                    ┌─────────────┐
                    │dp[i-1][j-1] │ ─┐
                    └─────────────┘  │
                                     ├──→ dp[i][j] = 两者之和
                    ┌─────────────┐  │
                    │ dp[i-1][j]  │ ─┘
                    └─────────────┘

s[i-1] !== t[j-1]:
                    ┌─────────────┐
                    │ dp[i-1][j]  │ ──→ dp[i][j]
                    └─────────────┘
```

## 理解两种选择

当 `s[i-1] === t[j-1]` 时，有两种选择：

1. **选择 s[i-1]**：用 s[i-1] 匹配 t[j-1]，方案数 = `dp[i-1][j-1]`
2. **不选择 s[i-1]**：跳过 s[i-1]，方案数 = `dp[i-1][j]`

两种选择互不干扰，所以总方案数是两者之和。

## 与 LCS 的对比

| 问题 | 目标 | 转移（相等时） |
|-----|------|--------------|
| LCS | 最长长度 | `max` |
| 不同的子序列 | 方案数 | `+` |

## 注意事项

1. **溢出风险**：如果数据量大，可能需要取模
2. **优化方向**：如果 `n > m`，直接返回 0
3. **空间优化**：从后往前更新，因为依赖左上和正上

## 复杂度分析

| 解法 | 时间复杂度 | 空间复杂度 |
|-----|-----------|-----------|
| 标准 DP | O(m × n) | O(m × n) |
| 空间优化 | O(m × n) | O(n) |

## 本章小结

1. **计数型 DP**：求方案数，用加法累计
2. **两种选择**：匹配时可选或不选，累加两种情况
3. **空间优化**：从后往前更新避免覆盖
4. **与 LCS 区别**：LCS 求最优用 max，这里计数用 +
