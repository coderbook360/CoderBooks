# 回溯算法方法论

回溯（Backtracking）是解决组合、排列、子集问题的核心算法，也是DFS的典型应用。

---

## 什么是回溯？

**定义**：在搜索过程中，尝试所有可能的路径，如果发现当前路径不可行，就回退到上一步，尝试其他路径。

**通俗理解**：
- 像走迷宫，遇到死路就退回来换条路
- 像下棋，走错了就悔棋重来

---

## 回溯的典型问题

**1. 组合问题**：
- 从n个元素中选k个
- LeetCode 77. 组合

**2. 排列问题**：
- 元素的所有排列
- LeetCode 46. 全排列

**3. 子集问题**：
- 所有可能的子集
- LeetCode 78. 子集

**4. 棋盘问题**：
- N皇后
- 数独

**5. 切割问题**：
- 分割回文串
- 复原IP地址

---

## 回溯的核心思想

**三步曲**：

1. **选择**：选择一个元素加入当前路径
2. **递归**：对剩余元素继续搜索
3. **撤销**：回退，尝试其他选择

```
选择 → 递归 → 撤销
  ↓      ↓      ↓
做决策  探索  回退
```

---

## 回溯模板

```typescript
function backtrack(路径, 选择列表) {
  if (满足结束条件) {
    result.push([...路径]);
    return;
  }
  
  for (选择 of 选择列表) {
    // 做选择
    路径.push(选择);
    
    // 递归
    backtrack(路径, 新的选择列表);
    
    // 撤销选择
    路径.pop();
  }
}
```

---

## 示例：全排列

给定 `[1,2,3]`，求所有排列。

```typescript
function permute(nums: number[]): number[][] {
  const result: number[][] = [];
  const path: number[] = [];
  const used = new Array(nums.length).fill(false);
  
  function backtrack() {
    // 结束条件
    if (path.length === nums.length) {
      result.push([...path]);
      return;
    }
    
    // 枚举所有选择
    for (let i = 0; i < nums.length; i++) {
      if (used[i]) continue;  // 剪枝
      
      // 做选择
      path.push(nums[i]);
      used[i] = true;
      
      // 递归
      backtrack();
      
      // 撤销选择
      path.pop();
      used[i] = false;
    }
  }
  
  backtrack();
  return result;
}
```

**执行过程**：
```
[]
├─ [1]
│  ├─ [1,2]
│  │  └─ [1,2,3] ✓
│  └─ [1,3]
│     └─ [1,3,2] ✓
├─ [2]
│  ├─ [2,1]
│  │  └─ [2,1,3] ✓
│  └─ [2,3]
│     └─ [2,3,1] ✓
└─ [3]
   ├─ [3,1]
   │  └─ [3,1,2] ✓
   └─ [3,2]
      └─ [3,2,1] ✓
```

## 回溯 vs DFS

**本质**：回溯是DFS的一种特殊形式。

**区别**：
- **DFS**：遍历所有节点，关注访问每个节点
- **回溯**：寻找所有解，强调"选择-探索-撤销"的过程

**关系**：
```
DFS + 路径记录 + 撤销操作 = 回溯
```

**应用场景对比**：

| 场景 | 适用方法 | 示例 |
|-----|---------|------|
| 遍历图/树 | DFS | 二叉树前序遍历 |
| 寻找所有路径 | 回溯 | 二叉树的所有路径 |
| 判断是否存在 | DFS | 图中是否有环 |
| 枚举所有可能 | 回溯 | 全排列、组合 |

## 时间复杂度分析

回溯算法的时间复杂度分析是一个重要但容易被忽视的话题。

### 通用分析框架

```
时间复杂度 = 搜索树节点数 × 每个节点的处理时间
```

### 常见问题的复杂度

| 问题 | 搜索树节点数 | 每节点时间 | 总复杂度 |
|-----|------------|-----------|---------|
| 全排列 | n! | O(n) | O(n! × n) |
| 组合 C(n,k) | C(n,k) | O(k) | O(C(n,k) × k) |
| 子集 | 2ⁿ | O(n) | O(2ⁿ × n) |
| N皇后 | ≤ n! | O(n) | O(n! × n) |
| 数独 | ≤ 9^81 | O(1) | O(9^81) |

### 为什么"每节点时间"要乘 O(n)？

因为通常需要复制当前路径到结果中：

```typescript
if (条件满足) {
  result.push([...path]);  // 复制路径，O(n)
  return;
}
```

### 剪枝后的复杂度

剪枝可以显著减少实际运行时间，但**理论上限不变**。

例如 N 皇后：
- 理论上限：O(n!)
- 实际运行：远小于 n!（大量分支被剪掉）
- n=8 时：理论 40320 种排列，实际只有 92 个解

**为什么还要学回溯？**

因为某些问题**本质上就是指数复杂度**（如 NP-完全问题），回溯+剪枝是最实用的解法。

## 优化：剪枝

**剪枝**：提前排除不可能的分支，减少搜索空间。

**示例**：组合总和
```typescript
function combinationSum(candidates: number[], target: number) {
  candidates.sort((a, b) => a - b);  // 排序方便剪枝
  
  function backtrack(start: number, sum: number, path: number[]) {
    if (sum === target) {
      result.push([...path]);
      return;
    }
    
    for (let i = start; i < candidates.length; i++) {
      // 剪枝：sum + candidates[i] > target
      if (sum + candidates[i] > target) break;
      
      path.push(candidates[i]);
      backtrack(i, sum + candidates[i], path);
      path.pop();
    }
  }
}
```

## 回溯的应用场景

1. **组合优化问题**：0-1背包、旅行商问题（TSP）
2. **游戏**：数独、八皇后、迷宫
3. **字符串问题**：括号生成、IP地址复原
4. **图问题**：所有路径、单词搜索

---

## 关键要点

1. **核心思想**：选择 → 递归 → 撤销
2. **结束条件**：路径达到目标
3. **剪枝优化**：提前排除无效分支
4. **路径记录**：保存当前搜索路径
5. **撤销操作**：回退到上一状态
