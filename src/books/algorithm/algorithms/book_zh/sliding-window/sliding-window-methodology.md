# 滑动窗口方法论

滑动窗口是处理**连续子数组/子串**问题的利器。它本质上是双指针的一种特殊应用——用两个指针维护一个"窗口"，通过窗口的滑动来高效遍历所有可能的子区间。

---

## 什么是滑动窗口？

想象你正在用一个可伸缩的相框在一幅长画卷上移动。相框框住的部分就是"窗口"，你可以：
- 向右扩展窗口（右边界右移）
- 从左收缩窗口（左边界右移）

```
数组: [a, b, c, d, e, f, g, h]
           ├─────────┤
          left     right
          └── 窗口 ──┘
```

---

## 为什么需要滑动窗口？

以"找最长无重复子串"为例：

**暴力解法**：枚举所有子串，检查是否有重复。
- 子串数量：O(n²)
- 每次检查：O(n)
- 总时间：O(n³)

**滑动窗口**：维护一个无重复的窗口，不断扩展和收缩。
- 每个元素最多进窗口一次，出窗口一次
- 总时间：O(n)

---

## 滑动窗口的核心思想

滑动窗口之所以高效，关键在于**增量更新**：
- 窗口扩展时，只需要处理新加入的元素
- 窗口收缩时，只需要处理移出的元素

不需要每次都重新计算整个窗口的状态。

---

## 滑动窗口的两种类型

### 1. 固定窗口

窗口大小固定为 k，每次整体向右滑动一格。

```
大小为 3 的窗口：
[a, b, c] d, e, f → 窗口 1
a, [b, c, d] e, f → 窗口 2
a, b, [c, d, e] f → 窗口 3
```

**适用场景**：
- 求大小为 k 的子数组的最大和
- 求大小为 k 的子数组的平均值

### 2. 可变窗口

窗口大小不固定，根据某个条件动态调整。

```
找满足条件的最长子串：
- 条件满足 → 扩展右边界，尝试更长
- 条件不满足 → 收缩左边界，直到满足
```

**适用场景**：
- 最长无重复子串
- 最小覆盖子串
- 满足条件的最长/最短子数组

## 滑动窗口的通用框架

```typescript
/**
 * 滑动窗口通用框架
 * 
 * 【核心思想】
 * 用两个指针 left 和 right 维护一个"窗口"
 * right 负责扩展窗口，left 负责收缩窗口
 * 通过窗口的滑动来遍历所有可能的子区间
 * 
 * 【为什么滑动窗口高效？】
 * 关键在于"增量更新"：
 * - 扩展窗口时，只处理新加入的一个元素
 * - 收缩窗口时，只处理移出的一个元素
 * - 不需要每次都重新计算整个窗口的状态
 * 
 * 【时间复杂度分析】
 * 虽然有两层循环（for + while），但每个元素最多：
 * - 进入窗口一次（right 扫描时）
 * - 离开窗口一次（left 收缩时）
 * 所以总复杂度是 O(n)，不是 O(n²)！
 */
function slidingWindow(arr: any[]) {
  let left = 0;  // 窗口左边界
  
  // 窗口状态：根据问题选择合适的数据结构
  // - 字符频率 → Map 或数组
  // - 不重复元素 → Set
  // - 子数组和 → 数值变量
  // - 最值 → 单调队列
  const window = new Map();
  
  // right 作为窗口右边界，从左到右扫描
  for (let right = 0; right < arr.length; right++) {
    // ========================================
    // 第一步：扩展窗口（右边界右移）
    // ========================================
    const entering = arr[right];
    // 更新窗口状态...
    // 例如：window.set(entering, (window.get(entering) || 0) + 1);
    
    // ========================================
    // 第二步：收缩窗口（左边界右移）
    // ========================================
    // 当窗口不满足某个条件时，收缩左边界
    // 注意：有些问题用 if，有些用 while
    // - while：收缩直到满足条件
    // - if：只收缩一次（固定窗口大小）
    while (需要收缩的条件) {
      const leaving = arr[left];
      // 更新窗口状态...
      // 例如：window.set(leaving, window.get(leaving)! - 1);
      left++;  // 左边界右移
    }
    
    // ========================================
    // 第三步：更新结果
    // ========================================
    // 根据题目要求更新答案
    // - 求最长：在收缩后更新（窗口满足条件时）
    // - 求最短：在收缩前更新（刚好满足条件时）
  }
}
```

---

## 滑动窗口适用的问题特征

- 问题涉及**连续**的子数组或子串
- 需要求满足某条件的**最长**或**最短**子区间
- 可以用某种数据结构高效维护窗口状态
- 窗口的扩展和收缩具有**单调性**

---

## 常见的窗口状态维护方式

| 问题类型 | 维护方式 |
|---------|---------|
| 子数组和 | 变量累加 |
| 字符频率 | HashMap/数组 |
| 不重复元素 | Set |
| 最大/最小值 | 单调队列 |
