# 双指针方法论

暴力解法是所有算法的起点，但绝不是终点。当你面对一个数组问题，发现自己写下了两层嵌套循环时，是时候问一个问题：**有没有办法用更少的计算解决同样的问题？**

双指针，正是回答这个问题的利器。

---

## 从暴力到优雅：一个经典问题

**问题**：给定一个**有序**数组和一个目标值，找出数组中和为目标值的两个数。

最直接的想法是什么？枚举所有可能的组合：

```typescript
function twoSumBruteForce(nums: number[], target: number): number[] {
  for (let i = 0; i < nums.length; i++) {
    for (let j = i + 1; j < nums.length; j++) {
      if (nums[i] + nums[j] === target) {
        return [i, j];
      }
    }
  }
  return [];
}
```

这个解法的时间复杂度是 O(n²)。对于长度为 10000 的数组，需要检查近 5000 万对组合。

但等一下，我们忽略了一个重要信息：**数组是有序的**。

有序意味着什么？意味着如果当前两数之和太大，增大任何一个数只会让和更大；如果太小，减小任何一个数只会让和更小。

基于这个观察，我们可以用两个指针，一个从最左边开始，一个从最右边开始：

```typescript
/**
 * 有序数组两数之和 - 对撞指针
 * 
 * 【关键洞察】
 * 数组有序意味着：
 * - 左指针指向最小值，右指针指向最大值
 * - 如果和太小，只能增大左指针（让和变大）
 * - 如果和太大，只能减小右指针（让和变小）
 * 
 * 【为什么这样不会漏解？】
 * 每次移动都排除了一整行/列的可能性
 * 例如 sum < target 时，left 与任何 right 的组合都不行
 * 所以可以放心地 left++
 * 
 * 时间复杂度：O(n)
 * 空间复杂度：O(1)
 */
function twoSumTwoPointers(nums: number[], target: number): number[] {
  let left = 0;                    // 左指针从开头开始
  let right = nums.length - 1;     // 右指针从末尾开始
  
  while (left < right) {
    const sum = nums[left] + nums[right];
    
    if (sum === target) {
      return [left, right];  // 找到答案
    }
    
    if (sum < target) {
      // 和太小，需要更大的数
      // 左指针右移，选择一个更大的数
      left++;
    } else {
      // 和太大，需要更小的数
      // 右指针左移，选择一个更小的数
      right--;
    }
  }
  
  return [];  // 未找到
}
```

时间复杂度从 O(n²) 降到了 O(n)。同样是 10000 个元素，现在最多只需要 10000 次操作。

**这就是双指针的魔力：利用问题的结构，用两个指针的协作，一次排除大量不可能的情况。**

---

## 双指针的核心思想

双指针不是一种复杂的数据结构，而是一种**思维方式**：

**用两个指针的移动，代替两层循环的枚举。**

它之所以有效，是因为问题本身存在某种**单调性**或**对称性**：
- 数组有序时，两数之和随指针移动单调变化
- 判断回文时，首尾字符存在对称关系
- 链表找中点时，快指针走完慢指针正好走一半

双指针的关键不在于"指针"这个概念，而在于**识别问题中的结构**，然后设计**指针移动的规则**。

---

## 双指针的两大模式

几乎所有双指针问题，都可以归入两种模式。

### 模式一：对撞指针

两个指针从**两端**出发，向**中间**移动，直到相遇。

```
有序数组：[1, 2, 4, 6, 8, 9, 14, 15]
          ↑                        ↑
        left                     right
```

**适用场景**：
- 有序数组的配对问题（两数之和、三数之和）
- 回文串判断
- 区间类问题（盛水容器、接雨水）

**核心逻辑**：每次移动一个指针，根据当前状态决定移动哪个。

```typescript
// 对撞指针模板
function collisionPointers<T>(arr: T[]): void {
  let left = 0;
  let right = arr.length - 1;
  
  while (left < right) {
    // 根据条件决定移动哪个指针
    if (shouldMoveLeft(arr, left, right)) {
      left++;
    } else {
      right--;
    }
  }
}
```

### 模式二：快慢指针

两个指针从**同一位置**出发，以**不同速度**前进。

```
链表：1 → 2 → 3 → 4 → 5 → null
      ↑
   slow/fast

快指针每次走两步，慢指针每次走一步
```

**适用场景**：
- 链表找中点
- 链表判环
- 找链表倒数第 k 个节点
- 追及问题（如判断快乐数）

**核心逻辑**：利用速度差，让两个指针在特定位置相遇或产生特定距离。

```typescript
// 快慢指针模板（链表找中点）
function findMiddle(head: ListNode | null): ListNode | null {
  let slow = head;
  let fast = head;
  
  while (fast !== null && fast.next !== null) {
    slow = slow!.next;
    fast = fast.next.next;
  }
  
  return slow; // slow 指向中点
}
```

## 什么时候想到双指针

在解题时，以下特征是使用双指针的强烈信号：

| 信号 | 对应模式 | 典型问题 |
|-----|---------|---------|
| **有序数组** + 配对/求和 | 对撞指针 | 两数之和、三数之和 |
| **回文**判断 | 对撞指针 | 验证回文串 |
| **区间**两端相关 | 对撞指针 | 盛水容器、接雨水 |
| **链表**中点/环 | 快慢指针 | 链表中点、环形链表 |
| **原地修改**数组 | 快慢指针 | 移除元素、删除重复项 |

**注意**：双指针通常要求 **O(1) 额外空间**，如果允许使用哈希表，问题往往有其他解法。

---

## 双指针与其他技巧的关系

双指针是一个基础技巧，它与其他算法思想有紧密联系：

- **滑动窗口**：本质是双指针的变体，左右指针维护一个"窗口"
- **二分查找**：也利用有序性，但关注点在"折半"而非"双端"
- **归并排序**：合并阶段使用双指针遍历两个有序数组

掌握双指针后，学习滑动窗口会更加自然。

---

## 本章小结

双指针的核心是**用指针的协作代替暴力枚举**。它有效的前提是问题存在可利用的结构（有序性、对称性、周期性等）。

**两大模式**：
- **对撞指针**：两端向中间，适合有序数组和区间问题
- **快慢指针**：同起点不同速，适合链表和追及问题

**使用信号**：
- 看到有序数组 + 配对 → 对撞指针
- 看到链表 + 中点/环 → 快慢指针
- 看到原地修改 + O(1) 空间 → 快慢指针
