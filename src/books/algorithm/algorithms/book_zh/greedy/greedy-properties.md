# 贪心选择性质与最优子结构

贪心算法之所以能work，背后有两个关键性质支撑。理解它们，是判断"能不能用贪心"的理论基础。

---

## 贪心选择性质

**定义**：通过做出局部最优选择，能够导致全局最优解。

换句话说，**每一步选当下最好的，最终结果也是最好的**。

这听起来很美好，但现实是大多数问题做不到。比如：

```
问题：从 A 到 E，找最短路径

图：A --5-- B --3-- E
    |              |
    3              2
    |              |
    C -----7------ D

贪心：A → C（选最短边 3），然后 C → D → E，总长度 3+7+2=12
最优：A → B → E，总长度 5+3=8

贪心失败了！
```

所以，贪心选择性质不是所有问题都具备的，需要证明。

---

## 最优子结构

**定义**：问题的最优解包含其子问题的最优解。

举例：活动选择问题
- 选择了活动 i 后，剩余问题是"在 i 结束后的活动中继续选择"
- 如果原问题有最优解，那子问题也必须是最优的

最优子结构是贪心和动态规划的共同特征，但：
- **贪心**：直接选最优，不回溯
- **动态规划**：比较所有可能，选最优

---

## 如何证明贪心算法的正确性

### 方法一：交换论证

**核心思想**：假设存在更优解，通过交换操作，证明贪心解不会更差。

**示例：活动选择问题**

```
问题：给定若干活动，每个活动有开始和结束时间，选择最多不重叠的活动

贪心策略：按结束时间排序，每次选最早结束的
```

证明：
1. 假设最优解是 `OPT = {a1, a2, ..., ak}`（按结束时间排序）
2. 贪心解是 `GREEDY = {g1, g2, ..., gm}`
3. 如果 `g1 ≠ a1`，那 `g1` 结束时间 ≤ `a1` 结束时间（贪心选最早结束的）
4. 将 `OPT` 中的 `a1` 替换为 `g1`，得到 `OPT' = {g1, a2, ..., ak}`
5. `OPT'` 仍然合法（g1 更早结束），且活动数不变
6. 重复此过程，最终 `OPT'` = `GREEDY`，所以贪心解也是最优解

### 方法二：反证法

假设贪心解不是最优解，推导出矛盾。

**示例：哈夫曼编码**

```
贪心策略：每次合并频率最小的两个节点

假设：存在更优编码树 T'，其中某两个低频字符 x, y 不在最深层
矛盾：将 x, y 移到最深层，总加权路径长度会减小
结论：贪心构造的树是最优的
```

### 方法三：归纳法

证明每一步贪心选择后，剩余问题仍然满足最优性。

**示例：找零钱问题（特定币值）**

```
币值：[25, 10, 5, 1]
贪心：每次选最大面值

归纳证明：
基础：找零 0 元，最优解是 0 枚硬币
假设：找零 < n 元时，贪心最优
证明：找零 n 元时
  - 如果 n >= 25，贪心选 25，剩余 n-25（归纳假设，最优）
  - 如果 25 > n >= 10，贪心选 10，剩余 n-10（归纳假设，最优）
  - ...
结论：贪心解最优
```

## 两个性质的关系

```
贪心算法可行的充要条件：
= 贪心选择性质 + 最优子结构

贪心选择性质：保证局部最优 → 全局最优
最优子结构：保证子问题也能贪心求解
```

如果只有最优子结构，没有贪心选择性质：
- **动态规划**：需要比较所有可能

如果有贪心选择性质：
- **贪心算法**：直接选局部最优

## 常见的贪心证明模式

| 问题类型 | 证明方法 | 典型问题 |
|---------|---------|---------|
| **区间调度** | 交换论证 | 活动选择、无重叠区间 |
| **哈夫曼树** | 反证法 | 哈夫曼编码 |
| **找零钱（特殊币值）** | 归纳法 | 硬币找零 |
| **排序贪心** | 交换论证 | 分发饼干、分发糖果 |

## 何时怀疑贪心不可行

以下情况，贪心很可能失败：

1. **依赖未来信息**：当前选择需要知道后面的情况
2. **多维度约束**：多个相互影响的目标
3. **后效性**：当前选择会影响后续选择的可行性

示例：
```
问题：0-1 背包

为什么贪心不行？
- 按价值排序：可能装满了高价值但大体积的物品，浪费空间
- 按体积排序：可能装满了小体积但低价值的物品
- 按性价比排序：仍然不行，因为有"后效性"
  
正确做法：动态规划
```

---

## 本章小结

1. **贪心选择性质**：局部最优 → 全局最优
2. **最优子结构**：问题的最优解包含子问题的最优解
3. **证明方法**：交换论证、反证法、归纳法
4. **判断技巧**：
   - 能明确定义"局部最优"吗？
   - 选择后的子问题是否独立？
   - 能否构造反例？

贪心算法是一把双刃剑：用对了事半功倍，用错了南辕北辙。掌握这两个性质，是驾驭贪心的关键。
