# 实战：单调递增的数字

> LeetCode 738. 单调递增的数字 | 难度：中等

通过贪心策略找到不大于 N 的最大单调递增数。

---

## 题目描述

当且仅当每个相邻位数上的数字 `x` 和 `y` 满足 `x <= y` 时，我们称这个整数是**单调递增**的。

给定一个整数 `n`，返回**小于或等于** `n` 的最大单调递增整数。

**示例**：
```
输入：n = 10
输出：9
解释：10 不是单调递增的，9 是

输入：n = 1234
输出：1234
解释：1234 本身就是单调递增的

输入：n = 332
输出：299
解释：332 → 329 → 299
```

---

## 思路分析

**贪心策略**：从右向左扫描，遇到逆序则：
1. 当前位 -1
2. 右边所有位变成 9

**关键洞察**：
- 如果 `digits[i] > digits[i+1]`，则 `digits[i]` 必须减 1
- 减 1 后，右边应该尽可能大（全变成 9）

---

## 代码实现

```typescript
function monotoneIncreasingDigits(n: number): number {
  const digits = n.toString().split('').map(Number);
  const len = digits.length;
  
  // mark: 从这个位置开始，右边都变成 9
  let mark = len;
  
  // 从右向左扫描
  for (let i = len - 1; i > 0; i--) {
    if (digits[i - 1] > digits[i]) {
      // 发现逆序：前一位大于当前位
      mark = i;
      digits[i - 1]--;  // 前一位减 1
    }
  }
  
  // 从 mark 位置开始，全部变成 9
  for (let i = mark; i < len; i++) {
    digits[i] = 9;
  }
  
  return parseInt(digits.join(''), 10);
}
```

## 为什么从右向左扫描？

从左向右会有问题：

```
n = 332

错误的从左向右：
i=0: 3 <= 3? 是，继续
i=1: 3 > 2?  是！digits[1]-- → [3, 2, 2]
     但是现在 3 > 2，又产生了逆序！

正确的从右向左：
i=2: 3 > 2 → digits[1]-- → [3, 2, 2]
i=1: 3 > 2 → digits[0]-- → [2, 2, 2]
然后变 9 → [2, 9, 9]
```

从右向左可以处理**连锁反应**：一位减 1 可能导致与前一位的逆序。

---

## 贪心正确性

**证明**：我们找到的是不大于 n 的最大单调递增数。

1. **不大于 n**：
   - 要么等于 n（本身单调）
   - 要么某位减 1，后面全 9，必然小于 n

2. **最大**：
   - 只在必要时减 1（发现逆序）
   - 减 1 后，后面尽可能大（全 9）
   - 没有其他更大的单调递增数

```
例：332
  331? → 不单调
  329? → 不单调
  299 → 单调，且是最大的
```

## 常见错误

### 错误 1：只处理一次逆序

```typescript
// 错误代码
for (let i = 0; i < len - 1; i++) {
  if (digits[i] > digits[i + 1]) {
    digits[i]--;
    // 只处理一处，可能产生新的逆序
    break;
  }
}
```

### 错误 2：忘记记录 mark 位置

```typescript
// 错误代码
for (let i = len - 1; i > 0; i--) {
  if (digits[i - 1] > digits[i]) {
    digits[i - 1]--;
    digits[i] = 9;  // 只改当前位，忘了右边
  }
}
```

## 小结

本题的贪心策略：

1. **从右向左扫描**：处理连锁反应
2. **遇逆序则减 1**：保证单调性
3. **右边全变 9**：保证最大化

核心理解：**找到第一个需要减小的位置，然后让右边尽可能大**
