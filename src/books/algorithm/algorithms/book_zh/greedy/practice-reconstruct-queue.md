# 实战：根据身高重建队列

> LeetCode 406. 根据身高重建队列 | 难度：中等

通过贪心策略重建队列的经典题目。

---

## 题目描述

假设有打乱顺序的一群人站成一个队列，数组 `people` 表示队列中一些人的属性（不一定按顺序）。

每个 `people[i] = [hi, ki]` 表示第 `i` 个人的身高为 `hi`，前面**恰好有** `ki` 个身高大于或等于 `hi` 的人。

请你重新构造并返回输入数组 `people` 所表示的队列。

**示例**：
```
输入：people = [[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]]
输出：[[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]]
```

---

## 思路分析

**关键洞察**：先安排高个子，再安排矮个子。

- 高个子不会受矮个子的影响（矮的在前面不算）
- 矮个子只需要根据 `k` 值插入到正确位置

**贪心策略**：
1. 按身高降序排列，身高相同则按 `k` 升序
2. 依次将每个人插入到索引 `k` 的位置

---

## 代码实现

```typescript
function reconstructQueue(people: number[][]): number[][] {
  // 1. 排序：身高降序，k 升序
  people.sort((a, b) => {
    if (a[0] !== b[0]) {
      return b[0] - a[0];  // 身高降序
    }
    return a[1] - b[1];    // k 升序
  });
  
  // 2. 依次插入
  const result: number[][] = [];
  
  for (const person of people) {
    // 插入到索引 k 的位置
    result.splice(person[1], 0, person);
  }
  
  return result;
}
```

## 执行过程详解

```

处理 [7,0]：k=0
  result.splice(0, 0, [7,0])
  result = [[7,0]]

处理 [7,1]：k=1
  result.splice(1, 0, [7,1])
  result = [[7,0], [7,1]]

处理 [6,1]：k=1
  result.splice(1, 0, [6,1])
  result = [[7,0], [6,1], [7,1]]
  
  验证 [6,1]：前面有 [7,0]，1个 >= 6 ✓

处理 [5,0]：k=0
  result.splice(0, 0, [5,0])
  result = [[5,0], [7,0], [6,1], [7,1]]
  
  验证 [5,0]：前面无人 ✓

处理 [5,2]：k=2
  result.splice(2, 0, [5,2])
  result = [[5,0], [7,0], [5,2], [6,1], [7,1]]

处理 [4,4]：k=4
  result.splice(4, 0, [4,4])
  result = [[5,0], [7,0], [5,2], [6,1], [4,4], [7,1]]
```

## 为什么这个贪心策略正确？

**核心观察**：

1. **高个子先排**：当我们按身高降序处理时，已经插入的人都比当前人高或等高
2. **k 值就是位置**：因为前面的人都不矮于当前人，所以 `k` 值直接对应插入位置
3. **矮个子不影响高个子**：后续插入的矮个子不会影响高个子的 `k` 值

```
例如：[6,1] 插入后
  [[7,0], [6,1], [7,1]]
  
再插入 [5,0] 到位置0：
  [[5,0], [7,0], [6,1], [7,1]]
  
[6,1] 的 k 值仍然正确！因为 [5,0] 比 6 矮，不计入 k
```

## 复杂度分析

- **时间复杂度**：O(n²)
  - 排序：O(n log n)
  - 插入：每次 splice 最坏 O(n)，共 n 次
  
- **空间复杂度**：O(n) 或 O(log n)
  - 结果数组：O(n)
  - 排序所需空间：O(log n)

## 优化方案

使用**链表**代替数组可以优化插入操作：

```typescript
// 使用链表优化插入
class ListNode {
  val: number[];
  next: ListNode | null = null;
  constructor(val: number[]) {
    this.val = val;
  }
}

function reconstructQueueOptimized(people: number[][]): number[][] {
  // 排序
  people.sort((a, b) => b[0] - a[0] || a[1] - b[1]);
  
  // 使用链表
  const dummy = new ListNode([-1, -1]);
  
  for (const person of people) {
    let prev = dummy;
    let k = person[1];
    
    // 找到插入位置
    while (k > 0 && prev.next) {
      prev = prev.next;
      k--;
    }
    
    // 插入
    const node = new ListNode(person);
    node.next = prev.next;
    prev.next = node;
  }
  
  // 转换为数组
  const result: number[][] = [];
  let curr = dummy.next;
  while (curr) {
    result.push(curr.val);
    curr = curr.next;
  }
  
  return result;
}
```

## 小结

本题的贪心策略：

1. **高个子优先**：按身高降序排列
2. **k 值定位**：身高相同时 k 小的优先
3. **直接插入**：k 值就是目标位置

关键理解：**矮个子不影响高个子的 k 值**，所以可以后处理矮个子。
