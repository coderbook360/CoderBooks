# 实战：移除 K 位数字

> LeetCode 402. 移除 K 位数字 | 难度：中等

单调栈 + 贪心的经典结合，理解"局部峰值删除"策略。

## 题目描述

给定一个以字符串表示的非负整数 `num`，移除这个数中的 k 位数字，使得剩下的数字最小。

返回的结果不能有前导零。

**示例1**：
```
输入：num = "1432219", k = 3
输出："1219"
解释：移除 4, 3, 2 后得到最小的 1219
```

**示例2**：
```
输入：num = "10200", k = 1
输出："200"
解释：移除 1 后得到 0200，去掉前导零为 200
```

**示例3**：
```
输入：num = "10", k = 2
输出："0"
解释：移除所有数字后为空，返回 "0"
```

## 思路分析

### 问题的本质

从 n 位数字中删除 k 位，使剩余数字最小。

### 关键洞察

**高位影响最大**。对于数字 `1432219`：
- 删除高位的 4 比删除低位的 9 更能减小结果
- 但如果高位已经很小，就不该删除

### 贪心策略

**核心规则**：从左到右扫描，如果当前数字比前一个小，就删除前一个。

**直觉理解**：
- 如果序列是 `...4, 3...`，4 > 3，删除 4 让 3 占据高位，数值变小
- 如果序列是 `...3, 4...`，3 < 4，保留 3，因为它在高位更有价值

### 单调栈实现

使用栈维护一个**单调递增序列**：
- 遇到比栈顶小的数字，弹出栈顶（删除峰值）
- 直到满足单调性或删除次数用完

## 代码实现

```typescript
function removeKdigits(num: string, k: number): string {
  const stack: string[] = [];
  
  for (const digit of num) {
    // 贪心：删除"峰值"（比当前数字大的栈顶元素）
    while (k > 0 && stack.length > 0 && stack[stack.length - 1] > digit) {
      stack.pop();
      k--;
    }
    stack.push(digit);
  }
  
  // 如果还没删够 k 个（说明是递增序列），删除末尾
  while (k > 0) {
    stack.pop();
    k--;
  }
  
  // 去除前导零
  const result = stack.join('').replace(/^0+/, '');
  
  // 处理结果为空的情况
  return result || '0';
}
```

### 代码逻辑详解

1. **遍历入栈**：每个数字都尝试入栈
2. **峰值删除**：入栈前，删除所有比当前大的栈顶元素
3. **末尾删除**：如果遍历完还没删够，说明是递增序列，从末尾删除
4. **前导零处理**：正则去除前导零，空结果返回 "0"

## 执行过程详解

```
num = "1432219", k = 3

初始: stack = [], k = 3

遍历 '1': 
  stack 为空，直接入栈
  stack = ['1']

遍历 '4': 
  '4' > '1'，不需要弹出
  stack = ['1', '4']

遍历 '3': 
  '3' < '4'，弹出 '4'，k = 2
  stack = ['1', '3']

遍历 '2': 
  '2' < '3'，弹出 '3'，k = 1
  stack = ['1', '2']

遍历 '2': 
  '2' = '2'，不弹出
  stack = ['1', '2', '2']

遍历 '1': 
  '1' < '2'，弹出 '2'，k = 0
  stack = ['1', '2', '1']

遍历 '9': 
  k = 0，直接入栈
  stack = ['1', '2', '1', '9']

k = 0，不需要末尾删除
结果: "1219"
```

## 边界情况处理

### 边界1：前导零

```
num = "10200", k = 1

遍历 '1': stack = ['1']
遍历 '0': '0' < '1'，弹出 '1'，k = 0
          stack = ['0']
遍历 '2': stack = ['0', '2']
遍历 '0': stack = ['0', '2', '0']
遍历 '0': stack = ['0', '2', '0', '0']

去除前导零: "0200" → "200"
```

### 边界2：全部删除

```
num = "9", k = 1

遍历 '9': stack = ['9']
末尾删除: k = 1，弹出 '9'
stack = []

结果为空: 返回 "0"
```

### 边界3：递增序列

```
num = "12345", k = 2

遍历后: stack = ['1', '2', '3', '4', '5']
（没有任何弹出，因为序列递增）

末尾删除: k = 2，弹出 '5'、'4'
stack = ['1', '2', '3']

结果: "123"
```

## 贪心正确性分析

### 为什么删除峰值最优？

设有数字序列 `...a, b...`，其中 a > b。

如果我们必须在 a 和后面某个数字 c 之间选择删除一个：

1. **删除 a**：序列变成 `...b...c...`，b 占据 a 的高位，结果变小
2. **删除 c**：序列变成 `...a, b...`，a 仍在高位，结果较大

**结论**：优先删除峰值（局部最大值）是最优策略。

### 单调栈的作用

单调栈确保我们能高效地找到并删除所有峰值：
- 每个元素最多入栈出栈各一次
- 总时间复杂度 O(n)

---

## 复杂度分析

- **时间复杂度**：O(n)
  - 每个字符最多入栈出栈各一次
  - 字符串拼接和正则替换也是 O(n)

- **空间复杂度**：O(n)
  - 栈存储最多 n 个字符

## 常见错误

### 错误1：忘记末尾删除

```typescript
// ❌ 错误：递增序列时不会删除任何元素
for (const digit of num) {
  while (k > 0 && stack.length && stack[stack.length - 1] > digit) {
    stack.pop();
    k--;
  }
  stack.push(digit);
}
// 缺少：while (k > 0) stack.pop();
```

### 错误2：忘记处理前导零

```typescript
// ❌ 错误：结果可能有前导零
return stack.join('');
// 应该：return stack.join('').replace(/^0+/, '') || '0';
```

### 错误3：字符比较错误

```typescript
// ❌ 错误：字符比较应该直接用 '>'
while (stack[stack.length - 1] - digit > 0)
// 正确：while (stack[stack.length - 1] > digit)
```

### LeetCode 316. 去除重复字母

同样使用单调栈，但有额外约束：每个字母只能出现一次。

```typescript
function removeDuplicateLetters(s: string): string {
  const lastIndex = new Map<string, number>();
  for (let i = 0; i < s.length; i++) {
    lastIndex.set(s[i], i);
  }
  
  const stack: string[] = [];
  const inStack = new Set<string>();
  
  for (let i = 0; i < s.length; i++) {
    const c = s[i];
    if (inStack.has(c)) continue;  // 已在栈中，跳过
    
    // 弹出条件：栈顶更大 且 后面还有栈顶字符
    while (
      stack.length > 0 && 
      stack[stack.length - 1] > c &&
      lastIndex.get(stack[stack.length - 1])! > i
    ) {
      inStack.delete(stack.pop()!);
    }
    
    stack.push(c);
    inStack.add(c);
  }
  
  return stack.join('');
}
```
