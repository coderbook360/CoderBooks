# 排序算法概览与分类

排序算法是计算机科学的基础，也是面试中的高频考点。本章将建立完整的排序算法知识体系。

---

## 排序算法分类

**按比较方式**：
- **比较排序**：通过元素间的比较决定顺序
  - 冒泡、选择、插入、归并、快速、堆排序
  - **时间下界**：Ω(n log n)
  
- **非比较排序**：不依赖元素间比较
  - 计数、桶、基数排序
  - 可突破 O(n log n) 限制

**按时间复杂度**：
- **O(n²)**：冒泡、选择、插入
- **O(n log n)**：归并、快速、堆排序
- **O(n)**：计数、桶、基数（特定条件下）

**按空间复杂度**：
- **原地排序（O(1)）**：冒泡、选择、插入、快排、堆排序
- **非原地（O(n)）**：归并、计数、桶、基数

**按稳定性**：
- **稳定**：冒泡、插入、归并、计数、桶、基数
- **不稳定**：选择、快排、堆排序

## 排序算法对比

| 算法 | 平均 | 最坏 | 空间 | 稳定性 | 特点 |
|-----|------|------|------|--------|------|
| **冒泡** | O(n²) | O(n²) | O(1) | 稳定 | 简单但慢 |
| **选择** | O(n²) | O(n²) | O(1) | 不稳定 | 交换次数少 |
| **插入** | O(n²) | O(n²) | O(1) | 稳定 | 小数据快 |
| **归并** | O(n log n) | O(n log n) | O(n) | 稳定 | 稳定O(n log n) |
| **快排** | O(n log n) | O(n²) | O(log n) | 不稳定 | 平均最快 |
| **堆排序** | O(n log n) | O(n log n) | O(1) | 不稳定 | 原地O(n log n) |
| **计数** | O(n+k) | O(n+k) | O(k) | 稳定 | 数据范围小 |
| **桶** | O(n+k) | O(n²) | O(n+k) | 稳定 | 数据分布均匀 |
| **基数** | O(d(n+k)) | O(d(n+k)) | O(n+k) | 稳定 | 多关键字 |

## 算法选择指南

**小数据（n < 50）**：
- **插入排序** — 简单高效，常数小

**大数据通用**：
- **快速排序** — 平均最快
- **归并排序** — 稳定排序首选

**特殊要求**：
- **需要稳定** → 归并排序
- **空间受限** → 堆排序
- **数据范围小** → 计数排序
- **链表排序** → 归并排序

**工程实践**：
- **Java Arrays.sort()**：
  - 基本类型：双轴快排（Dual-Pivot QuickSort）
  - 对象类型：TimSort（归并 + 插入）
  
- **C++ std::sort()**：
  - 内省排序（IntroSort）= 快排 + 堆排序 + 插入

## 排序稳定性的重要性

**什么是稳定排序？**

相同元素排序后保持原有相对顺序。

**示例**：
```
原始：[(Alice, 25), (Bob, 30), (Charlie, 25)]
按年龄排序：
稳定：  [(Alice, 25), (Charlie, 25), (Bob, 30)]
不稳定：[(Charlie, 25), (Alice, 25), (Bob, 30)]
```

**为什么重要？**

1. **多字段排序**：先按年龄排，再按姓名排
2. **保留信息**：排序前的顺序可能有意义
3. **数据库查询**：ORDER BY 多个字段

## 比较排序的理论下界

**定理**：任何基于比较的排序算法，最坏情况至少需要 Ω(n log n) 次比较。

**证明思路**（决策树）：
- n 个元素有 n! 种排列
- 决策树高度至少为 log₂(n!)
- 根据斯特林公式：log₂(n!) ≈ n log₂ n

**突破方法**：
- **非比较排序**：计数、桶、基数
- **前提条件**：数据范围有限

## 排序算法的实际应用

1. **数据分析**：排序是很多算法的前置步骤
2. **数据库**：索引、查询优化
3. **搜索引擎**：结果排序
4. **操作系统**：任务调度
5. **图形学**：深度排序
