# 实战：插入排序实现

插入排序是最直观的排序算法：像整理扑克牌一样，把每张新牌插入到已排序牌组的正确位置。

---

## 算法思想

插入排序将数组分为两部分：

1. **已排序部分**：从第一个元素开始，逐渐扩大
2. **未排序部分**：待处理的剩余元素

每次从未排序部分取出第一个元素，在已排序部分找到合适位置并插入。

### 执行过程可视化

```
原数组：[5, 2, 4, 6, 1, 3]

初始：[5 | 2, 4, 6, 1, 3]  第一个元素默认已排序

i=1：取出2，与5比较
     5 > 2，5右移 → [_, 5, 4, 6, 1, 3]
     插入2 → [2, 5 | 4, 6, 1, 3]

i=2：取出4，与5比较
     5 > 4，5右移 → [2, _, 5, 6, 1, 3]
     与2比较，2 < 4，停止
     插入4 → [2, 4, 5 | 6, 1, 3]

i=3：取出6，与5比较
     5 < 6，不移动
     插入6（原位置）→ [2, 4, 5, 6 | 1, 3]

i=4：取出1，逐个比较
     6 > 1，6右移
     5 > 1，5右移
     4 > 1，4右移
     2 > 1，2右移
     插入1 → [1, 2, 4, 5, 6 | 3]

i=5：取出3，逐个比较
     6 > 3，6右移
     5 > 3，5右移
     4 > 3，4右移
     2 < 3，停止
     插入3 → [1, 2, 3, 4, 5, 6]

结果：[1, 2, 3, 4, 5, 6] ✓
```

## 代码实现

```typescript
function insertionSort(arr: number[]): void {
  const n = arr.length;
  
  for (let i = 1; i < n; i++) {
    // 取出待插入元素
    const key = arr[i];
    let j = i - 1;
    
    // 从右向左扫描已排序部分，找插入位置
    // 同时将较大元素右移
    while (j >= 0 && arr[j] > key) {
      arr[j + 1] = arr[j];  // 右移
      j--;
    }
    
    // 插入到正确位置（j+1是因为while结束时j多减了1）
    arr[j + 1] = key;
  }
}
```

### 代码逻辑详解

**为什么从 `i = 1` 开始？**
- 第一个元素（`i = 0`）单独成一个已排序数组，无需处理

**为什么用 `arr[j] > key` 而不是 `>=`？**
- 使用 `>` 保证了稳定性：相等元素不移动，维持原有顺序

**`arr[j + 1] = key` 的理解**：
- while循环结束时，`j` 指向第一个不大于 `key` 的元素（或 `-1`）
- 所以 `key` 应该插入到 `j` 的后一个位置

---

## 为什么插入排序是稳定的？

```
例：[3a, 3b, 1]（3a和3b值相同）

i=1：取出3b，与3a比较
     3a = 3b，不满足 3a > 3b，while循环不执行
     3b插入到原位置
     → [3a, 3b | 1]

i=2：取出1
     3b > 1，右移
     3a > 1，右移
     → [1, 3a, 3b]

结果：3a仍在3b前面 → 稳定 ✓
```

**关键**：使用 `arr[j] > key` 而非 `arr[j] >= key`，相等元素不移动。

## 优化：二分查找插入位置

既然已排序部分是有序的，可以用二分查找代替线性查找：

```typescript
function insertionSortBinary(arr: number[]): void {
  const n = arr.length;
  
  for (let i = 1; i < n; i++) {
    const key = arr[i];
    
    // 二分查找：在[0, i-1]中找key应该插入的位置
    let left = 0, right = i - 1;
    while (left <= right) {
      const mid = Math.floor((left + right) / 2);
      if (arr[mid] > key) {
        right = mid - 1;  // 在左半边
      } else {
        left = mid + 1;   // 在右半边（包括相等情况，保证稳定性）
      }
    }
    
    // left就是插入位置，将[left, i-1]的元素右移
    for (let j = i - 1; j >= left; j--) {
      arr[j + 1] = arr[j];
    }
    
    arr[left] = key;
  }
}
```

**复杂度分析**：
- 查找位置：O(log n) per insertion
- 移动元素：仍是 O(n) per insertion
- 总体：**仍是 O(n²)**

**结论**：二分优化减少了比较次数，但移动次数不变。对于写入代价高的场景意义不大。

## 复杂度分析

**时间复杂度**：

| 情况 | 复杂度 | 场景 |
|------|--------|------|
| 最好 | O(n) | 数组已排序，每个元素只比较一次 |
| 平均 | O(n²) | 随机排列 |
| 最坏 | O(n²) | 数组逆序 |

**最好情况O(n)详解**：
```
已排序数组：[1, 2, 3, 4, 5]

i=1：取出2，与1比较，1 < 2，不移动 → 1次比较
i=2：取出3，与2比较，2 < 3，不移动 → 1次比较
...每个元素只需1次比较
```

**空间复杂度**：O(1)

## 插入排序的适应性

插入排序具有**适应性**（Adaptive）：输入越接近有序，性能越好。

**逆序对衡量无序程度**：

- **逆序对**：满足 `i < j` 且 `arr[i] > arr[j]` 的数对
- 插入排序的移动次数 = 逆序对数量

```
[4, 3, 2, 1] 的逆序对：(4,3),(4,2),(4,1),(3,2),(3,1),(2,1) = 6对
                     插入排序需要移动6次

[1, 2, 4, 3] 的逆序对：(4,3) = 1对
                     插入排序只需移动1次
```

## 插入排序在工业级排序中的应用

现代排序算法（如 **Tim Sort**）将插入排序作为核心组件：

**Tim Sort的策略**：
1. 将数组分成小块（称为 run，通常 32-64 个元素）
2. 每个小块用插入排序
3. 再用归并排序合并所有小块

**为什么用插入排序处理小数组？**

1. **缓存友好**：小数组在CPU缓存中，访问速度极快
2. **常数因子小**：插入排序代码简单，没有递归开销
3. **对部分有序有效**：分块后每块往往部分有序

```typescript
// 模拟Tim Sort中插入排序的使用
const RUN = 32;

function timSortInsertion(arr: number[]): void {
  const n = arr.length;
  
  // 对每个RUN使用插入排序
  for (let i = 0; i < n; i += RUN) {
    insertionSortRange(arr, i, Math.min(i + RUN - 1, n - 1));
  }
  
  // 然后归并这些RUN...
}

function insertionSortRange(arr: number[], left: number, right: number): void {
  for (let i = left + 1; i <= right; i++) {
    const key = arr[i];
    let j = i - 1;
    while (j >= left && arr[j] > key) {
      arr[j + 1] = arr[j];
      j--;
    }
    arr[j + 1] = key;
  }
}
```

## 常见错误

### 错误1：从0开始遍历

```typescript
// ❌ 错误：i=0时没有已排序部分可比较
for (let i = 0; i < n; i++) {

// ✅ 正确：从第二个元素开始
for (let i = 1; i < n; i++) {
```

### 错误2：边界条件错误

```typescript
// ❌ 错误：忘记检查 j >= 0
while (arr[j] > key) {  // j可能变成-1，arr[-1]访问越界
  arr[j + 1] = arr[j];
  j--;
}

// ✅ 正确：双条件检查
while (j >= 0 && arr[j] > key) {
```

### 错误3：破坏稳定性

```typescript
// ❌ 使用 >= 会破坏稳定性
while (j >= 0 && arr[j] >= key) {  // 相等也移动

// ✅ 使用 > 保持稳定性
while (j >= 0 && arr[j] > key) {   // 相等时不移动
```

## 与其他排序算法的对比

| 特性 | 插入排序 | 选择排序 | 冒泡排序 |
|------|---------|---------|---------|
| 最好时间 | O(n) ✓ | O(n²) | O(n) ✓ |
| 适应性 | 高 | 无 | 有 |
| 稳定性 | ✅ | ❌ | ✅ |
| 在线性 | ✅ | ❌ | ❌ |

**在线算法**：插入排序可以边读入数据边排序，不需要预先知道所有数据。

---

## 关键要点

1. **核心思想**：像整理扑克牌，将新元素插入到已排序部分的正确位置
2. **最好O(n)**：对几乎有序的数据非常高效
3. **稳定排序**：相等元素使用 `>` 比较，不交换位置
4. **工业应用**：Tim Sort等混合排序算法的核心组件
5. **适用场景**：
   - 小数据量（n < 50）
   - 数据几乎有序
   - 需要在线排序
