# 序言：为什么要手写动画引擎

## 从一个疑问说起

你是否曾在使用 GSAP 时产生过这样的疑问：

- 为什么 `gsap.to()` 可以让任何属性平滑过渡？
- Timeline 是如何精确控制多个动画的时序关系的？
- 那些丝滑的缓动曲线背后是什么数学原理？
- 动画引擎是如何做到 60fps 流畅渲染的？

如果你用过 GSAP，一定被它的强大所折服。但作为工程师，我们不应该满足于"会用"，而应该追问"为什么"。

## GSAP：Web 动画的黄金标准

GSAP (GreenSock Animation Platform) 是 Web 动画领域无可争议的王者。它诞生于 Flash 时代，经过十多年的迭代，已经成为：

- **Netflix、Nike、Google** 等一线公司的选择
- **CodePen** 上最受欢迎的动画库
- 被誉为"JavaScript 能做到的最快动画"

它的成功源于几个核心优势：

**性能卓越**：GSAP 的动画性能通常比 CSS 动画和其他 JS 库快 20 倍以上。它通过精心优化的渲染管道和智能的属性缓存，实现了接近原生的性能表现。

**功能全面**：从简单的淡入淡出到复杂的时间轴编排，从 CSS 属性到 SVG 路径，从滚动触发到物理动画，GSAP 几乎覆盖了所有动画需求。

**API 优雅**：声明式的配置、链式调用、直观的位置参数，让复杂动画的编写变得轻松愉快。

## 为什么要自己实现

既然 GSAP 已经这么好了，为什么还要自己造轮子？

**1. 理解原理，而非死记 API**

使用 GSAP 时，我们经常写出这样的代码：

```javascript
gsap.to('.box', {
  x: 100,
  duration: 1,
  ease: 'power2.out'
});
```

但你真的理解这背后发生了什么吗？

- `x: 100` 是如何被解析并应用到 transform 的？
- `duration: 1` 的时间是如何精确控制的？
- `power2.out` 是什么数学函数？

当你亲手实现一遍，这些问题的答案将刻入你的技术基因。

**2. 阅读源码的最佳入口**

GSAP 的源码非常庞大且复杂，直接阅读容易迷失。但如果你先实现一个简化版，理解了核心架构，再去看源码就会豁然开朗——那些看似复杂的代码不过是你已经理解的概念的生产级实现。

**3. 提升工程能力**

实现动画引擎涉及许多高级主题：

- 设计模式（观察者、策略、插件架构）
- 性能优化（批量更新、对象池、脏检查）
- 数学应用（矩阵变换、贝塞尔曲线、物理模拟）
- 系统设计（时间管理、事件系统、生命周期）

这是一次绝佳的综合能力训练。

**4. 定制与扩展**

理解原理后，你可以根据项目需求定制自己的动画解决方案，或者为 GSAP 开发插件。

## 本书的目标

我们不是要"重新发明轮子"，而是通过造轮子来**理解轮子**。

本书将带你实现一个 Mini-GSAP，它包含：

- **Ticker 系统**：驱动动画的心跳
- **Tween 引擎**：属性动画的核心
- **Timeline 时间轴**：复杂动画的编排
- **缓动系统**：丰富的动画曲线
- **插件架构**：可扩展的模块化设计
- **CSS Plugin**：DOM 动画处理
- **ScrollTrigger**：滚动驱动动画

每一章都遵循"问题驱动"的方法：先提出问题，分析需求，再逐步实现，最后总结原理。

## 预备知识
