## 从 Moment.js 到 Day.js

在前端发展的历程中，日期处理一直是个痛点。原生 Date 对象的 API 设计反直觉、月份从 0 开始、时区处理混乱。Moment.js 曾经是救星，但它有一个致命问题：**太大了**。

Day.js 的出现改变了一切：

```javascript
// 仅 2KB，API 几乎与 Moment.js 完全兼容
dayjs('2024-12-25').add(7, 'day').format('YYYY-MM-DD')
```

**如果没有 Day.js，你能自己写一个吗？**

这本书的目标：**带你亲手写一个 Mini-Dayjs**。

## 为什么要手写

### 理解不可变设计

Day.js 最核心的设计决策是**不可变性**。每次操作都返回新实例，原实例不变：

```javascript
const date1 = dayjs('2024-01-01')
const date2 = date1.add(1, 'month')
// date1 仍然是 2024-01-01
// date2 是 2024-02-01
```

这个设计如何实现？性能影响如何？手写一遍你就懂了。

### 掌握插件架构

Day.js 的体积能做到 2KB，秘诀在于插件系统。核心只包含最基础的功能，其他都通过插件扩展：

```javascript
import relativeTime from 'dayjs/plugin/relativeTime'
dayjs.extend(relativeTime)
dayjs('2024-01-01').fromNow() // "11 months ago"
```

如何设计一个可扩展的插件系统？这是架构能力的核心考验。

### 面试的硬通货
