# 序言

## 为什么要「造轮子」？

"不要重复造轮子"——这是软件开发中最著名的格言之一。

但讽刺的是，**最优秀的工程师往往都造过轮子**。

React 的作者在写 React 之前，深入研究过 Backbone 和 Ember。Vue 的尤雨溪在创造 Vue 之前，把 Angular 的源码翻了个底朝天。Linus Torvalds 觉得现有的版本控制系统不够好，于是有了 Git。

造轮子不是目的，**理解轮子为什么这样转**才是目的。

## 为什么是 jQuery？

你可能会问：都 2024 年了，React、Vue 横行天下，还学 jQuery 干什么？

这个问题本身就暴露了一个误解：**我们学的不是 jQuery 的 API，而是 jQuery 的设计智慧**。

jQuery 用不到 90KB 的代码，实现了：

- **优雅的链式调用**：`$('.item').addClass('active').fadeIn().click(...)`
- **强大的选择器引擎**：在没有 `querySelectorAll` 的年代，Sizzle 就是最快的选择器
- **统一的事件系统**：抹平了 IE 和标准浏览器的差异
- **流畅的动画系统**：用 JavaScript 实现 CSS 都做不到的动画效果
- **简洁的 Ajax 封装**：让异步请求变得如此简单

这些设计模式，在今天的 Lodash、Axios、甚至 React 的链式操作中，依然随处可见。

**jQuery 不是过时的技术，而是 DOM 操作的教科书。**

## 这本书会教你什么？

我们将从零开始，一步一步实现一个功能完整的 Mini-jQuery。

**不是抄代码，而是理解每一行代码背后的「为什么」**：

- 为什么 `$()` 不需要 `new` 就能创建实例？
- 为什么 jQuery 对象是「类数组」而不是真正的数组？
- 链式调用的本质是什么？仅仅是返回 `this` 这么简单吗？
- 事件委托是怎么实现的？为什么能提升性能？
- 动画的本质是什么？jQuery 如何让元素「动起来」？

每一章，我们都会先提出问题，然后一步步推导出解决方案。你会发现，这些「高深」的设计模式，其实都是解决实际问题的自然产物。

## 你需要什么基础？

- **熟练的 JavaScript 基础**：了解原型链、闭包、this 指向
- **基本的 DOM 知识**：知道什么是节点、事件、样式
- **一颗好奇心**：愿意刨根问底，而不是只会用 API

如果你已经在使用 React 或 Vue，这本书会帮你理解「框架帮你做了什么」。当你知道原生 DOM 操作有多痛苦，才会真正明白为什么需要虚拟 DOM。

## 我们开始吧

在接下来的旅程中，我们会亲手打造：

- 一个支持链式调用的核心架构
- 一个基于 CSS 选择器的查询引擎
- 完整的 DOM 遍历和操作方法
- 一个支持事件委托和命名空间的事件系统
- 一个基于 requestAnimationFrame 的动画引擎
- 一个 Promise 化的 Ajax 模块

最后，我们会把它打包发布到 npm，让它成为一个真正可用的开源库。

**不要只是读，要动手写**。每一章的代码都可以运行。建议你边读边敲，在浏览器控制台里亲眼看到效果。

造轮子的快乐，只有造过的人才懂。

让我们开始吧。
