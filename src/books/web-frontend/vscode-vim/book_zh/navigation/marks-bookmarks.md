# 标记与书签：自定义跳转锚点

跳转列表自动记录历史，但有时你需要主动设置"锚点"——一个稳定的位置，无论后续跳转多少次，都能直接返回。这就是标记（Mark）的作用。

## Vim 标记基础

### 设置标记

```
m{a-z}    在当前位置设置标记
```

例如：`ma` 在当前位置设置标记 `a`。

### 跳转到标记

```
'{a-z}    跳转到标记所在行的行首
`{a-z}    跳转到标记的精确位置（行和列）
```

例如：`` `a `` 跳转到标记 `a` 的精确位置。

### 26 个标记

小写字母 `a-z` 提供 26 个文件内标记。每个文件可以有自己的一套标记。

## 标记的使用场景

### 场景 1：长文件中的固定锚点

在 500 行的文件中工作，需要频繁在两个位置间切换：

```
1. 在位置 A（比如组件定义）设置 ma
2. 在位置 B（比如状态定义）设置 mb
3. 编辑过程中，'a 跳到 A，'b 跳到 B
4. 无论中间跳转多少次，标记始终有效
```

### 场景 2：临时标记

调试或阅读代码时，临时标记感兴趣的位置：

```
1. 遇到可疑代码，mm 设置临时标记
2. 继续阅读、跳转
3. 想回来看看，`m 返回
4. 不再需要，覆盖或忽略
```

### 场景 3：编辑前保存位置

要进行大范围修改，先保存当前位置：

```
1. ma 保存当前位置
2. 进行多处修改
3. 完成后 `a 返回原位
```

## 特殊标记

Vim 自动设置一些特殊标记：

| 标记 | 含义 |
|------|------|
| `` `. `` | 上次修改的位置 |
| `` `" `` | 上次退出时的光标位置 |
| `` `[ `` | 上次修改或复制的起始位置 |
| `` `] `` | 上次修改或复制的结束位置 |
| `` `< `` | 上次可视选择的起始位置 |
| `` `> `` | 上次可视选择的结束位置 |

`` `. `` 特别有用——"我刚才改了哪里？"

## 大写标记（跨文件）

小写标记 `a-z` 是文件内的。大写标记 `A-Z` 是全局的，可以跨文件跳转。

```
mA    设置全局标记 A
`A    从任何文件跳转到标记 A
```

### 使用场景

项目中有几个核心文件需要频繁访问：

```
mA    → 在入口文件 App.tsx 设置 A
mB    → 在路由配置 routes.ts 设置 B
mC    → 在全局状态 store.ts 设置 C
```

现在无论在哪个文件，`` `A `` `` `B `` `` `C `` 直接跳转。

## 查看标记

在 Vim 中，`:marks` 显示所有标记。在 VSCode Vim 中，这个命令有限支持。

不过你通常不需要查看——记住自己设置的标记就行。如果忘记了，直接覆盖。

## VSCode 书签扩展

VSCode Vim 的标记功能有一些限制。如果需要更强大的书签功能，可以使用 **Bookmarks** 扩展。

### 安装和配置

安装 "Bookmarks" 扩展后，配置 Vim 键位：

```json
{
  "vim.normalModeKeyBindingsNonRecursive": [
    {
      "before": ["<leader>", "m", "m"],
      "commands": ["bookmarks.toggle"]
    },
    {
      "before": ["<leader>", "m", "n"],
      "commands": ["bookmarks.jumpToNext"]
    },
    {
      "before": ["<leader>", "m", "p"],
      "commands": ["bookmarks.jumpToPrevious"]
    },
    {
      "before": ["<leader>", "m", "l"],
      "commands": ["bookmarks.list"]
    },
    {
      "before": ["<leader>", "m", "c"],
      "commands": ["bookmarks.clear"]
    }
  ]
}
```

### Bookmarks vs Vim Marks

| 特性 | Vim 标记 | Bookmarks 扩展 |
|------|----------|---------------|
| 可见性 | 不可见 | 编辑器边栏显示图标 |
| 数量 | a-z (26个) + A-Z (26个全局) | 无限 |
| 持久化 | 会话内 | 可持久化 |
| 标签 | 无 | 可添加标签 |

对于临时跳转，Vim 标记更快。对于长期项目导航，Bookmarks 扩展更合适。

## 配合其他导航使用

### 标记 + 跳转列表

标记用于主动设置锚点，跳转列表用于自动历史。两者互补：

```
1. ma 设置重要位置
2. 开始探索，gd 跳转多次
3. Ctrl+O 返回上一步
4. `a 直接返回标记位置（不用多次 Ctrl+O）
```

### 标记 + 可视模式

标记可以用于定义操作范围：

```
1. 在范围起始位置 ma
2. 移动到范围结束
3. d`a 删除从当前位置到标记 a 的内容
```

或者：

```
1. ma 在起始
2. 移动到结束
3. v`a 选中从当前到标记 a
```

## 实用技巧

### 技巧 1：使用固定标记约定

建立个人约定：
- `a`：当前工作位置
- `b`：对比位置
- `z`：临时标记

这样不用记忆，始终知道每个标记的用途。

### 技巧 2：结合跳转使用

```
`a    跳转到 a 的精确位置
'a    跳转到 a 所在行的行首
```

`'a` 更常用，因为你通常关心的是那一行，不是精确列位置。

### 技巧 3：`` `` （两个反引号）

连续按两次反引号跳转到上一次跳转前的位置：

```
`a    跳转到标记 a
``    跳回跳转前的位置
```

这类似于 `Ctrl+O`，但只能跳一步。
