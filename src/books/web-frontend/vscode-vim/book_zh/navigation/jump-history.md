# 跳转历史：Ctrl+O、Ctrl+I 返回机制

每次使用 `gd` 跳转、`/` 搜索或其他大范围移动后，你需要能够轻松返回。Vim 的跳转列表让这变得极其简单。

## 跳转列表概念

Vim 维护一个"跳转列表"（Jump List），记录你的移动历史。每次"大跳转"都会在列表中添加一个位置。

**触发跳转记录的操作**：
- `gd`：跳转到定义
- `gf`：跳转到文件
- `/` 或 `?`：搜索跳转
- `G` 或 `gg`：跳转到文件首尾
- `{n}G`：跳转到指定行
- `{` 或 `}`：段落跳转
- `'` 或 `` ` ``：标记跳转
- `Ctrl+]`：标签跳转

**不触发跳转记录的操作**：
- `j/k`：上下移动
- `w/b/e`：词移动
- `h/l`：左右移动
- `f/t`：行内跳转

简单规则：**小范围移动不记录，大范围跳转记录**。

## 基本导航

| 命令 | 效果 |
|------|------|
| `Ctrl+O` | 返回上一个跳转位置（Older） |
| `Ctrl+I` | 前进到下一个跳转位置 |

### 典型工作流

```
1. 在 App.tsx 第 20 行编辑
2. gd 跳转到 Button.tsx 第 50 行查看定义
3. gd 继续跳转到 types.ts 查看类型
4. Ctrl+O → 返回 Button.tsx 第 50 行
5. Ctrl+O → 返回 App.tsx 第 20 行
6. Ctrl+I → 前进到 Button.tsx 第 50 行
```

## 跨文件跳转

跳转列表是跨文件的。无论你跳转了多少个文件，`Ctrl+O` 都能带你一步步返回。

这解决了一个常见问题："我刚才在哪里来着？"

## 与 Tab 切换的区别

| 功能 | 跳转列表 (Ctrl+O/I) | Tab 切换 (gt/gT) |
|------|---------------------|------------------|
| 维度 | 位置（文件+行号） | 文件（Tab） |
| 记忆 | 精确到光标位置 | 只记住文件 |
| 范围 | 只记录大跳转 | 所有打开的文件 |
| 用途 | 跳转后返回 | 在文件间切换 |

`Ctrl+O` 返回的是**精确位置**，包括行号和列号。而 Tab 切换只是切换到另一个文件的当前状态。

## 查看跳转列表

在 Vim 中可以用 `:jumps` 查看跳转历史。在 VSCode Vim 中，这个命令支持有限。

不过你通常不需要查看列表——直接用 `Ctrl+O` 往回跳就行。

## 上一个编辑位置

除了跳转列表，Vim 还记录编辑位置：

| 命令 | 效果 |
|------|------|
| `` `." `` | 跳转到上次修改的位置 |
| `gi` | 跳转到上次编辑位置并进入 Insert 模式 |

`gi` 特别实用——你在某处编辑，跳转去别处查看，然后 `gi` 直接回到编辑位置继续输入。

在 VSCode Vim 中，`gi` 的支持可能有限。可以配置：

```json
{
  "before": ["g", "i"],
  "commands": ["workbench.action.navigateToLastEditLocation"]
}
```

注意：这会覆盖之前的 `gi`（跳转到实现）配置。根据你的使用频率选择。

## 处理冲突

`Ctrl+I` 可能与 VSCode 的某些功能冲突。如果不生效，检查：

1. 键位是否被其他扩展占用
2. 在 keybindings.json 中明确设置：

```json
{
  "key": "ctrl+i",
  "command": "extension.vim_ctrl+i",
  "when": "editorTextFocus && vim.active && vim.use<C-i>"
}
```

## 最佳实践

### 1. 大胆跳转

知道可以轻松返回后，你应该更大胆地跳转查看代码。看不懂？跳转看定义。想知道谁调用？跳转看引用。`Ctrl+O` 永远能带你回来。

### 2. 少用鼠标滚动

鼠标滚动不会记录位置。如果你滚动了很远，没法用 `Ctrl+O` 返回。改用 `Ctrl+D/U` 翻页，或 `{n}G` 跳转。

### 3. 配合标记使用

如果你知道会多次返回某个位置，设置一个标记：

```
1. ma       # 在当前位置设置标记 a
2. 跳转到别处...
3. 'a       # 返回标记 a
```

标记比跳转列表更稳定——即使中间有很多跳转，标记位置不会丢失。

## 与浏览器后退按钮的比喻

如果你觉得跳转列表难理解，可以类比浏览器：

- `gd` = 点击链接
- `Ctrl+O` = 后退按钮
- `Ctrl+I` = 前进按钮

网页浏览的逻辑完全适用于代码跳转。

## 配置建议

确保 Vim 接管 `Ctrl+O` 和 `Ctrl+I`：

```json
{
  "vim.handleKeys": {
    "<C-o>": true,
    "<C-i>": true
  }
}
```
