# Visual Block 模式批量操作

## 为什么需要 Visual Block 模式？

Visual Block（可视块）模式是 Vim 独有的强大功能，可以对矩形区域进行批量编辑。常见需求：

**问题 1：批量添加前缀/后缀**
- 需要在多行开头添加注释符 `//`
- 需要在多行末尾添加分号 `;`
- 需要在多行前添加缩进

**问题 2：列编辑**
- 修改表格中的某一列
- 调整代码对齐
- 删除多行的相同位置字符

**问题 3：批量输入**
- 在多行相同位置插入相同内容
- 在多行创建相同的变量声明
- 在多行添加相同的属性

**使用 Visual Block 后的效果**：
- 批量操作从**几分钟**缩短到**几秒**
- 精确控制编辑区域（矩形选择）
- 一次操作应用到所有选中行
- **效率提升：10-50 倍**

## 进入 Visual Block 模式

### 基本进入方式

```
Ctrl+v    进入可视块模式
```

在 Windows 上，`Ctrl+v` 默认是粘贴快捷键，VSCode Vim 需要配置：

```json
{
  "vim.handleKeys": {
    "<C-v>": true  // 让 Vim 处理 Ctrl+v
  }
}
```

**替代方案**（如果 Ctrl+v 冲突）：
```json
{
  "vim.normalModeKeyBindingsNonRecursive": [
    {
      "before": ["<leader>", "v"],
      "commands": ["extension.vim.visualBlockMode"]
    }
  ]
}
```

然后使用 `Space` `v` 进入 Visual Block 模式。

### 确认已进入 Visual Block 模式

进入后，VSCode 状态栏会显示：
```
-- VISUAL BLOCK --
```

## 基本操作

### 选择矩形区域

```
Ctrl+v    进入可视块模式
j/k       上下扩展选区
h/l       左右扩展选区
w/b       按单词扩展
$         扩展到行尾
```

**示例**：选择 3 行 × 5 列的矩形区域
```
Ctrl+v    进入可视块模式
j j       向下选择 2 行（共 3 行）
l l l l   向右选择 4 列（共 5 列）
```

### 选择整列

```
Ctrl+v    进入可视块模式
G         选择到文件末尾（当前列）
```

**效果**：选中从当前位置到文件末尾的同一列

## 批量插入

### 在每行开头插入

**任务**：在多行开头添加注释符 `//`

**原代码**：
```javascript
line1
line2
line3
```

**操作序列**：
```
Ctrl+v    可视块模式
j j       选择 3 行
I         插入模式（行首）
//        输入内容
Esc       应用到所有行
```

**结果**：
```javascript
// line1
// line2
// line3
```

**用时**：**3-5 秒**

**对比传统方式**：**15-30 秒**（逐行添加）

**效率提升**：**5-10 倍**

### 在每行相同位置插入

**任务**：在多行的第 10 个字符后插入 `: Type`

**原代码**：
```typescript
const name = 'John';
const email = 'john@example.com';
const age = 30;
```

**操作序列**：
```
f=            跳到第一个 = 号
Ctrl+v        可视块模式
j j           选择 3 行
I             插入模式
: string      输入类型（示例）
Esc           应用到所有行
```

**结果**：
```typescript
const name: string = 'John';
const email: string = 'john@example.com';
const age: string = 30;
```

**注意**：这个示例需要手动调整 `age` 的类型为 `number`。

**用时**：**5-8 秒**

### 在每行末尾插入

**任务**：在多行末尾添加分号 `;`

**原代码**：
```javascript
const a = 1
const b = 2
const c = 3
```

**操作序列**：
```
Ctrl+v    可视块模式
j j       选择 3 行
$         扩展到行尾
A         追加模式（行尾）
;         输入内容
Esc       应用到所有行
```

**结果**：
```javascript
const a = 1;
const b = 2;
const c = 3;
```

**用时**：**3-5 秒**

**对比传统方式**：**10-20 秒**（逐行添加）

**效率提升**：**3-6 倍**

## 批量删除

### 删除选中的矩形区域

**任务**：删除多行相同位置的字符

**原代码**：
```javascript
//    line1
//    line2
//    line3
```

**操作序列**：
```
Ctrl+v    可视块模式
j j       选择 3 行
l l l l   选择 5 列（包括 // 和空格）
d         删除
```

**结果**：
```javascript
line1
line2
line3
```

**用时**：**3-5 秒**

### 删除每行前 N 个字符

**任务**：删除每行前 4 个字符

**原代码**：
```python
    print('a')
    print('b')
    print('c')
```

**操作序列**：
```
Ctrl+v    可视块模式
j j       选择 3 行
l l l     选择 4 列
d         删除
```

**结果**：
```python
print('a')
print('b')
print('c')
```

**用时**：**3-5 秒**

**对比传统方式**：**20-40 秒**（逐行调整缩进）

**效率提升**：**6-13 倍**

### 删除每行特定列

**任务**：删除表格中的某一列

**原数据**：
```
| Name  | Age | City     |
| John  | 30  | New York |
| Alice | 25  | London   |
```

**假设要删除 "Age" 列**：

**操作序列**：
```
移动光标到 "Age" 列的 |
Ctrl+v    可视块模式
j j       选择 3 行
f|        扩展到下一个 |
d         删除
```

**结果**：
```
| Name  | City     |
| John  | New York |
| Alice | London   |
```

**用时**：**5-8 秒**

**用时**：**3-5 秒**

**用时**：**3-5 秒**

**取消注释**：
```
Ctrl+v    可视块模式
9j        向下选择 10 行

```

**对比传统方式**：**30-60 秒**（逐行注释）

**效率提升**：**6-20 倍**

### 场景 2：对齐代码

**任务**：对齐等号

**原代码**：
```javascript
const a = 1;
const longName = 2;
const x = 3;
```

**目标**：
```javascript
const a        = 1;
const longName = 2;
const x        = 3;
```

**操作序列**：
```
1. 找到最长的变量名（longName）
2. 在其他行的对应位置插入空格

f=            跳到第一行的 =
Ctrl+v        可视块模式
j j           选择 3 行
I             插入模式
       输入多个空格（对齐）
Esc           应用
```

**用时**：**10-15 秒**

**对比手动对齐**：**60-120 秒**

**效率提升**：**4-12 倍**

### 场景 3：批量创建变量

**任务**：创建多个相似的变量声明

**需求**：创建 5 个 `const item1`, `const item2`, ..., `const item5`

**操作序列**：
```
i             进入插入模式
const item    输入前缀
Esc           退出
yy            复制行
4p            粘贴 4 次（共 5 行）
Ctrl+v        可视块模式
4j            选择 5 行
$             扩展到行尾
A             追加模式
1             输入数字 1
Esc           应用到所有行
```

**中间结果**：
```javascript
const item1
const item1
const item1
const item1
const item1
```

**继续修改**（使用递增）：
```
Ctrl+v        可视块模式
4j            选择 5 行
g Ctrl+a      递增数字
```

**最终结果**：
```javascript
const item1
const item2
const item3
const item4
const item5
```

**用时**：**15-20 秒**

**对比手动输入**：**60-90 秒**

**效率提升**：**3-6 倍**

### 场景 4：批量修改对象属性

**任务**：为多个对象添加相同的属性 `, age: 0`

**操作**：`f}` → `Ctrl+v` → `jj` → `I` → `, age: 0` → `Esc`

**用时**：**5-8 秒** vs 手动 **30-60 秒**，效率提升 **4-12 倍**

## 高级技巧

### 技巧 1：结合数字递增

在多行插入递增的数字：

```
Ctrl+v        可视块模式
j j j         选择 4 行
I             插入模式
1             输入数字 1
Esc           应用
gv            重新选择
g Ctrl+a      递增数字
```

**结果**：
```

2. item

4. item
```

**用时**：**8-12 秒**

### 技巧 2：与正则搜索结合

先搜索定位，再使用 Visual Block：

```
/pattern      搜索
n n n         跳到目标位置
Ctrl+v        可视块模式
选择区域
操作（I/A/d/c）
```

**用时**：**10-15 秒**

### 技巧 3：跨文件复制列

从一个文件复制列到另一个文件：

```
# 文件 A
Ctrl+v        可视块模式
选择列
y             复制

# 文件 B
Ctrl+Tab      切换文件
移动到目标位置
p             粘贴
```

**用时**：**8-12 秒**

## 技巧 4：使用宏批量操作

录制宏，结合 Visual Block：

```
qa            开始录制宏 a
Ctrl+v        可视块模式
选择操作
q             结束录制

100@a         重复执行 100 次
```

**用时**：**15-25 秒**（录制） + **2-3 秒**（执行）

## 常见问题与解决方案

### 问题 1：Ctrl+v 不进入 Visual Block 模式

**症状**：按 `Ctrl+v` 后触发了粘贴操作，而非进入 Visual Block 模式。

**原因**：Windows 上 `Ctrl+v` 默认是粘贴快捷键，VSCode Vim 未接管。

**解决方案**：
```json
{
  "vim.handleKeys": {
    "<C-v>": true  // 让 Vim 处理 Ctrl+v
  }
}
```

**替代方案**：使用自定义快捷键
```json
{
  "vim.normalModeKeyBindingsNonRecursive": [
    {
      "before": ["<leader>", "v"],
      "commands": ["extension.vim.visualBlockMode"]
    }
  ]
}
```

### 问题 2：插入内容后没有应用到所有行

**症状**：按 `I` 输入内容，按 `Esc` 后只在第一行生效。

**原因**：
1. 未正确进入 Visual Block 模式（可能是 Visual Line 模式）
2. 过早按了 `Esc`（需要等待 1 秒）

**解决方案**：
- 确认状态栏显示 `-- VISUAL BLOCK --`
- 按 `Esc` 后等待 1 秒，让 VSCode Vim 应用到所有行

### 问题 3：选择区域不符合预期

**症状**：选择的矩形区域形状不对。

**原因**：
1. 文件使用了 Tab 缩进（Tab 宽度不统一）
2. 行长度不一致

**解决方案**：
- 将 Tab 转换为空格：`:%s/\t/    /g`
- 使用 `$` 扩展到行尾（自动适应不同长度）

### 问题 4：无法删除行尾空白

**症状**：选择到行尾后，删除操作没有删除行尾空白。

**原因**：行尾空白不可见。

**解决方案**：
- 显示空白字符：VSCode 设置中启用 `editor.renderWhitespace`
- 使用 `$` 确保选择到真正的行尾

### 最佳实践

1. **确认进入 Visual Block 模式**：
   - 检查状态栏显示 `-- VISUAL BLOCK --`
   - 如果不确定，退出重新进入

2. **使用 $ 处理不同长度的行**：
   - 行长度不一致时，用 `$` 扩展到行尾
   - 避免手动计算列数

3. **耐心等待 Esc 生效**：
   - 按 `Esc` 后等待 1 秒
   - VSCode Vim 需要时间将操作应用到所有行

4. **结合其他 Vim 功能**：
   - 搜索 + Visual Block（定位后批量编辑）
   - 宏 + Visual Block（录制复杂操作）
   - 多光标 + Visual Block（更复杂的场景）

5. **处理 Tab 缩进**：
   - 统一使用空格缩进
   - 或在操作前转换 Tab 为空格

6. **善用递增/递减数字**：
   - `g Ctrl+a`：递增数字
   - `g Ctrl+x`：递减数字
   - 快速生成序号

7. **分步操作复杂任务**：
   - 先处理行首
   - 再处理行尾
   - 避免一次性完成所有操作
