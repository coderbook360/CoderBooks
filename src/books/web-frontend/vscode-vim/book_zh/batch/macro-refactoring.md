# 宏录制与批量重构

宏是 Vim 最强大的自动化工具之一，可以录制一系列操作并重复执行，实现复杂的批量重构。

## 宏基础

### 录制宏

```
qa        开始录制到寄存器 a
...       执行操作
q         停止录制
```

### 播放宏

```
@a        执行寄存器 a 中的宏
@@        重复上次执行的宏
10@a      执行 10 次
```

## 宏录制技巧

### 确保可重复性

录制宏时，使用可重复的移动命令：

**好的做法**：
```
0         移到行首
w         移到下一个单词
f"        跳到引号
```

**避免**：
```
10l       移动固定列数（不同行可能不适用）
```

### 以结束位置结尾

宏结束时，光标应该在下一次执行的起始位置：

```
qa        开始录制
0         行首
操作...
j         移到下一行（为下次准备）
q         停止
```

## 实战案例

### 案例 1：批量添加导出

将：
```javascript
const a = 1
const b = 2
const c = 3
```
改为：
```javascript
export const a = 1
export const b = 2
export const c = 3
```

录制：
```
qa            开始录制
0             行首
iexport Esc   插入 export
j             下一行
q             停止
2@a           执行 2 次
```

### 案例 2：转换为箭头函数

将：
```javascript
function add(a, b) {
  return a + b
}
```
改为箭头函数。

录制：
```
qa
/function Enter    搜索 function
cgnconst Esc      替换为 const
f(               跳到括号
i= Esc           添加等号
/)Enter          跳到右括号
a => Esc         添加箭头
dd               删除 return（如果需要）
q
```

### 案例 3：格式化 JSON 键

将：
```javascript
{
  name: "John",
  age: 25,
}
```
改为：
```javascript
{
  "name": "John",
  "age": 25,
}
```

录制：
```
qa
0             行首
f:            跳到冒号
bi"Esc        在键名前加引号
ea"Esc        在键名后加引号
j             下一行
q
```

### 案例 4：批量重命名属性

将 `this.name` 改为 `this._name`：

```
qa
/this\.\w Enter   搜索 this.xxx
f.                跳到点
a_Esc             添加下划线
n                 下一个匹配
q
@a                继续执行
```

## 递归宏

宏可以调用自身，实现递归执行：

```
qa
操作...
@a            在宏中调用自己
q
```

**警告**：确保有退出条件，否则会无限循环。

通常让宏在搜索失败时自然停止：
```
qa
n             下一个搜索匹配（无匹配时停止）
操作...
@a
q
```

## 宏编辑

### 查看宏内容

```
:reg a        查看寄存器 a 的内容
```

### 编辑宏

```
"ap           粘贴宏内容
编辑...
"ayy          重新复制到寄存器
```

## 宏与其他功能配合

### 配合搜索

```
/pattern      先搜索
qa
n             跳到下一个
修改...
q
```

### 配合可视模式

```
qa
V             行选择
修改...
j             下一行
q
```

### 配合文本对象

```
qa
ci"           修改引号内
新内容
Esc
j0            下一行开头
q
```

## 宏 vs 其他方法

| 场景 | 推荐方法 |
|------|----------|
| 简单重复 | `.` 命令 |
| 相同修改多处 | `cgn` + `n.` |
| 复杂多步操作 | 宏 |
| 同时编辑多处 | 多光标 |
| 基于模式替换 | 正则替换 |

## 最佳实践

1. **先手动做一遍**：确认步骤正确
2. **使用可靠的移动**：避免固定数字
3. **小步验证**：录制后先执行一次确认
4. **准备好位置**：宏结束时在正确位置
5. **备份代码**：复杂重构前提交

## 键位总结

| 键位 | 操作 |
|------|------|
| `qa` | 录制宏到 a |
| `q` | 停止录制 |
| `@a` | 执行宏 a |
| `@@` | 重复上次宏 |
| `10@a` | 执行 10 次 |
| `:reg a` | 查看宏内容 |

---

**效率提升**：宏是 Vim 的自动化利器，复杂的重复性重构任务可以在几秒内完成。
