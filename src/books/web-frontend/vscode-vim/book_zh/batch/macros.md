# 宏录制：复杂操作自动化

宏是 Vim 的超级武器——录制一次操作，无限次重放。对于复杂的重复编辑，宏比多光标更强大。

## 宏基础

### 录制宏

| 命令 | 效果 |
|------|------|
| `q{a-z}` | 开始录制宏到寄存器 a-z |
| `q` | 停止录制 |

例如：`qa` 开始录制到寄存器 `a`，操作完成后按 `q` 停止。

### 播放宏

| 命令 | 效果 |
|------|------|
| `@{a-z}` | 播放寄存器中的宏 |
| `@@` | 重复上次播放的宏 |
| `{n}@{a-z}` | 播放宏 n 次 |

例如：`@a` 播放宏 a，`10@a` 播放 10 次。

## 录制宏的步骤

1. 将光标移到第一个要处理的位置
2. `qa` 开始录制
3. 执行你的编辑操作
4. 移动到下一个要处理的位置（重要！）
5. `q` 停止录制
6. `@a` 播放，或 `10@a` 批量播放

### 关键点：结束时移动到下一个位置

宏录制要包含"移动到下一个处理位置"的操作。这样重复播放时才能连续处理。

## 实战示例

### 示例 1：格式化多行

将多行变量声明转换为数组：

```javascript
// 之前
const a = 1;
const b = 2;
const c = 3;
const d = 4;

// 之后
const arr = [
  1,
  2,
  3,
  4
];
```

录制宏：

```
1. 光标在第一行
2. qa 开始录制
3. f= 跳到等号
4. lldw 删除空格和变量值前的空格
5. dt; 删除到分号前（保留值）
6. A, 在行尾加逗号
7. j0 跳到下一行开头
8. q 停止录制
9. 3@a 对剩余 3 行执行
```

### 示例 2：添加引号

给多行文本加引号：

```
// 之前
apple
banana
cherry

// 之后
"apple",
"banana",
"cherry",
```

```
1. qa 开始录制
2. I" 行首加引号
3. A", 行尾加引号和逗号
4. j0 下一行
5. q 停止
6. 2@a 处理剩余行
```

### 示例 3：转换数据格式

将 CSV 转换为 JSON：

```
// 之前
name,age,city
John,25,NYC
Jane,30,LA

// 之后
{ "name": "John", "age": "25", "city": "NYC" },
{ "name": "Jane", "age": "30", "city": "LA" },
```

这需要更复杂的宏，分步处理。

## 宏的技巧

### 技巧 1：使用可靠的移动

不要用 `j` 假设要移动一行。使用搜索或更精确的移动：

```
/pattern    搜索到下一个模式
n           跳到下一个匹配
```

### 技巧 2：先测试再批量执行

```
1. 录制宏
2. @a 执行一次，检查结果
3. 确认正确后，{n}@a 批量执行
```

### 技巧 3：从行首开始

宏开始前用 `0` 或 `^` 移到行首，确保每次执行位置一致。

### 技巧 4：使用寄存器

不同任务用不同寄存器：

- `qa`：当前任务
- `qb`：备用
- `qz`：临时测试

## 宏的持久化

VSCode Vim 的宏在会话结束后丢失。如果有常用的宏，可以：

1. 在配置中定义键位映射实现类似效果
2. 使用代码片段
3. 每次需要时重新录制（通常很快）

## 宏 vs 多光标

| 场景 | 宏 | 多光标 |
|------|----|----|
| 相同位置相同操作 | ✗ | ✓ |
| 不同位置复杂操作 | ✓ | ✗ |
| 跨行不规则处理 | ✓ | ✗ |
| 需要搜索跳转 | ✓ | ✗ |

**多光标**：同时编辑，适合位置整齐的批量修改。

**宏**：顺序执行，适合复杂的、需要多步的重复操作。

## 宏 vs 搜索替换

| 场景 | 宏 | 搜索替换 |
|------|----|----|
| 简单文本替换 | ✗ | ✓ |
| 需要逻辑判断 | ✓ | 有限 |
| 需要多步编辑 | ✓ | ✗ |
| 正则表达式替换 | ✗ | ✓ |

## 递归宏

宏可以调用自身，形成递归：

```
1. qa 开始录制
2. 执行操作
3. @a 在宏中调用自己
4. q 停止录制
5. @a 启动，会递归执行直到出错
```

**危险**：如果没有终止条件，会无限执行。通常通过"搜索失败"或"移动到文件末尾"自然终止。

## VSCode Vim 宏的限制

- 宏不持久化
- 部分 Vim 命令可能不支持
- 与 VSCode 异步操作可能有冲突

对于非常复杂的批量处理，考虑使用脚本或 VSCode 的查找替换功能。

## 实战工作流

### 批量修改模板

```
1. 识别重复模式
2. 在第一个位置录制宏
3. 测试执行一次
4. 批量执行
```

### 数据转换

```
1. 分析源格式和目标格式
2. 找出转换步骤
3. 录制宏实现一行的转换
4. 批量执行
```
