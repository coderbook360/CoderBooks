# 序言：从零实现现代化测试框架

## 为什么要手写测试框架？

在前端开发领域，测试是保障代码质量的关键环节。从最早的 Jasmine，到 Mocha + Chai 的组合，再到 Jest 一统江湖，测试框架的演进反映了 JavaScript 生态的发展轨迹。而 Vitest 的出现，标志着测试框架进入了 ESM 原生时代。

作为一名追求技术深度的开发者，你或许已经熟练使用 Vitest 编写测试用例。但你是否思考过这些问题：

- **`test()` 和 `it()` 是如何收集和执行测试用例的？**
- **`expect().toBe()` 链式调用的背后是怎样的设计？**
- **`vi.mock()` 是如何实现模块级别的替换的？**
- **快照测试如何知道哪些快照需要更新？**
- **并发测试是如何保证隔离性的？**
- **代码覆盖率是如何在运行时收集的？**

这些问题的答案，正是本书要带你探索的领域。

## 本书的核心价值

### 1. 深入理解测试原理

测试框架是连接开发者意图与代码验证的桥梁。通过手写 mini-vitest，你将深入理解：

- **测试运行器原理**：如何发现、收集、调度和执行测试用例
- **断言系统设计**：Chai 风格与 Jest 风格断言的统一
- **Mock 机制**：函数 Mock、模块 Mock、定时器 Mock 的实现
- **快照测试**：快照的生成、比对、更新策略
- **覆盖率收集**：V8 原生覆盖率与 Istanbul 代码插桩的差异

### 2. 掌握 Vite 生态核心技术

Vitest 的「Vite 原生」设计是其最大亮点。本书将深入探讨：

- **Vite 配置复用**：如何与 Vite 共享 resolve.alias、plugins 等配置
- **ESM 优先加载**：告别 CommonJS 的历史包袱
- **HMR 式 Watch Mode**：像 Vite 开发服务器一样智能的测试重跑
- **Transform Pipeline**：TypeScript、JSX 的无缝转换

### 3. 构建生产级工程思维

实现一个完整的测试框架，你将面对诸多工程挑战：

- **并发控制**：Worker 进程管理、资源竞争、测试隔离
- **性能优化**：启动时间、内存占用、分片执行
- **可扩展设计**：Reporter 插件、自定义匹配器、Coverage Provider

## 本书特色

### 📐 架构驱动

每个模块的实现都从架构设计开始，先理解「为什么这样设计」，再动手「如何实现」。

### 🔬 源码级深度

不满足于「能用就行」，我们追求对每个 API 背后原理的完整理解。

### 🎯 渐进式实现

从最小可运行版本开始，逐步增强功能，让你清晰看到每一步的动机与收益。

### ⚖️ 权衡与取舍

真实世界的工程问题往往没有完美解，我们会分析每个设计决策背后的权衡。

## 书籍结构

本书共分为 **32 个部分，239 章**，按照从基础到高级的顺序组织：

| 部分 | 主题 | 核心内容 |
|------|------|---------|
| 第 1-4 部分 | **基础架构** | 配置系统、文件发现、模块解析 |

| 第 20-21 部分 | **快照与覆盖率** | 快照测试、代码覆盖率收集 |
| 第 22-25 部分 | **辅助系统** | Reporter、Watch Mode、CLI、Benchmark |
| 第 26-28 部分 | **高级特性** | 类型测试、浏览器模式、In-Source Testing |
| 第 29-32 部分 | **生态与迁移** | Vitest UI、性能优化、生态集成、Jest 迁移 |
