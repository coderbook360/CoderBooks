## 🔄 通用优化原则

### 1. 删除"打印类"反模式
**识别标志**：
```javascript
class SomethingExplanation {
  static demonstrate() {
    console.log('=== 标题 ===');
    console.log('说明文字');
  }
}
```

**替换为**：
- 直接写讲解文字
- 或用真实可运行代码演示概念

---

### 2. 代码长度控制
**规则**：
- 单个代码块 ≤ 30 行
- 超过 30 行必须：
  - 分段讲解，或
  - 分多个代码块展示，或
  - 先简化版，再完整版

---

### 3. 渐进式增强模板
**标准流程**：
```
1. 问题引入
   ↓
2. 最简实现（暴露不足）
   ↓
3. 问题分析
   ↓
4. 优化版本
   ↓
5. 边界讨论
   ↓
6. 最佳实践
```

---

### 4. 代码前后必有讲解
**代码前**：
- 这段代码要解决什么问题？
- 核心思路是什么？

**代码中**：
- 关键行必须有注释
- 复杂逻辑要标注

**代码后**：
- 这段代码展示了什么原理？
- 实际意义是什么？
- 有哪些注意事项？

---

### 5. 权衡思维
每个技术方案必须讲：
- ✅ 优点：解决了什么问题
- ⚠️ 代价：牺牲了什么
- 🎯 适用场景：何时使用
