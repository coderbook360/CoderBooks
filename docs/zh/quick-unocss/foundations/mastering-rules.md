# 深入规则：静态与动态规则的艺术

在前面的章节中，我们已经体验了 UnoCSS 结合预设带来的便捷开发。你可能会好奇，当我们写下 `p-4`、`text-red-500` 或 `flex` 这些类名时，它们究竟是如何被魔法般地转换成浏览器可以理解的 CSS 的？如果我们想创造一个预设中不存在的、独一无二的样式，又该如何实现呢？

答案就藏在 UnoCSS 的核心引擎之中，而驱动这个引擎的燃料，便是本章的主角——**规则（Rules）**。

理解规则，是真正掌握 UnoCSS 的关键。它能让你从一个单纯的“使用者”转变为“创造者”，让你跳出预设的“舒适区”，随心所欲地定制和扩展你的样式工具集。

## 第一个自定义规则：静态规则

让我们从一个具体的问题开始。假设在你的设计稿中，有一个非常特殊的 `margin` 值：`5.5px`。`preset-wind` 提供的 `m-1` (`4px`) 或 `m-2` (`8px`) 都无法满足需求。这时，最直接的方式就是定义一条属于我们自己的规则。

最简单的规则是**静态规则**，它是一个工具类名和一个 CSS 声明对象的直接映射。

我们可以通过修改 `uno.config.ts` 文件，在 `rules` 配置数组中添加我们的第一条静态规则：

```typescript
// uno.config.ts
import { defineConfig } from 'unocss'

export default defineConfig({
  // ... 这里可以有 presets
  rules: [
    // 这就是一条静态规则
    // 当 UnoCSS 遇到 'm-5.5' 这个类名时，
    // 它会生成一个包含 { margin: '5.5px' } 的 CSS 规则。
    ['m-5.5', { margin: '5.5px' }]
  ]
})
```

现在，你可以在你的 HTML 中使用这个全新的工具类了：

```html
<div class="m-5.5">
  这个元素拥有 5.5px 的外边距。
</div>
```

打开浏览器的开发者工具，你会清晰地看到 UnoCSS 为你生成的 CSS：

```css
.m-5-5 {
  margin: 5.5px;
}
```

> 注意：UnoCSS 会对类名中的 `.` 进行转义，所以 `m-5.5` 变成了 `.m-5-5`。

这非常简单直接！`rules` 数组中的每一项都是一个元组 `[matcher, handler]`。对于静态规则，`matcher` 就是一个普通的字符串，`handler` 则是一个包含 CSS 属性和值的对象。

## 让规则“动”起来：动态规则

静态规则虽然简单，但它的局限性也显而易见。如果我们还需要 `m-10.5px`、`m-22.5px` 呢？难道要为每一个值都手动添加一条规则吗？这显然是不可接受的。

为了解决这个问题，我们需要让规则“动”起来。这就是**动态规则**的舞台，它使用正则表达式作为 `matcher`，从而实现模式匹配和动态生成。

让我们升级一下配置，用一条动态规则来替代刚才的静态规则，实现所有 `m-<数值>` 的匹配：

```typescript
// uno.config.ts
import { defineConfig } from 'unocss'

export default defineConfig({
  rules: [
    // 这是一条动态规则
    // Matcher: /^m-([\.\d]+)$/ 是一个正则表达式
    // Handler: ([_, num]) => ({ margin: `${num}px` }) 是一个处理器函数
    [/^m-([\.\d]+)$/, ([_, num]) => ({ margin: `${num}px` })],

    // 我们还可以举一反三，添加一个 padding 的动态规则
    [/^p-([\.\d]+)$/, ([_, num]) => ({ padding: `${num}px` })]
  ]
})
```

让我们来剖析这条动态规则的精髓：

1.  **Matcher (匹配器)**: ` /^m-([\.\d]+)$/`
    *   `^m-`：匹配以 `m-` 开头的字符串。
    *   `([\.\d]+)`：这是一个**捕获组**。`\d` 代表数字，`\.` 代表小数点，`+` 代表一个或多个。所以它能匹配像 `10`、`5.5` 这样的数字字符串，并将其“捕获”。
    *   `$`：匹配字符串的结尾。

2.  **Handler (处理器)**: `([_, num]) => ({ margin: `${num}px` })`
    *   这是一个函数，它接收正则表达式匹配的结果作为参数。
    *   `_` (下划线) 通常用来表示第一个参数，即整个匹配到的字符串（例如 `m-5.5`），我们这里用不到它。
    *   `num` 正是我们在 `matcher` 中用括号 `()` 定义的那个**捕获组**所捕获到的内容。如果类名是 `m-5.5`，那么 `num` 的值就是字符串 `'5.5'`。
    *   函数返回一个 CSS 对象 `{ margin: `${num}px` }`，动态地将捕获到的数字拼接成最终的 CSS 值。

有了这条动态规则，我们现在可以生成无穷无尽的 `margin` 工具类了：

```html
<div class="m-10">外边距 10px</div>
<div class="p-20">内边距 20px</div>
<div class="m-3.14">外边距 3.14px</div>
```

仅仅两条动态规则，就赋予了我们创建无限 `margin` 和 `padding` 工具类的能力。这就是 UnoCSS 强大扩展性的基石。几乎所有官方预设（如 `preset-wind`）中的工具类，其背后都是由这样一系列精心设计的动态规则构成的。

## 总结

在本章，我们揭开了 UnoCSS 工作的核心秘密——规则。

- **静态规则**提供了一对一的映射，简单直接，适合处理固定的、特殊的样式需求。
- **动态规则**则通过正则表达式和处理器函数，提供了强大的模式匹配和动态生成能力，是 UnoCSS 实现原子化和可扩展性的关键。

现在，你已经掌握了创建自定义规则的能力。在下一章节，我们将学习如何将这些规则、以及变体、快捷方式等，组织和封装成一个可复用的单元——**预设**。
