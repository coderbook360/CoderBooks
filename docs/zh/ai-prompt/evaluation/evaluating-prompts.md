# 第十四章：度量效果：如何评估提示词的质量

到目前为止，我们已经学习了如何编写结构精巧、功能强大的提示词。但一个重要的问题随之而来：我们如何科学、客观地判断一个提示词比另一个更好？仅仅凭“感觉”是远远不够的。在工程实践中，任何没有度量的优化都是盲目的。

本章，我们将从“提示词作者”的角色，转变为“提示词质检员”，学习如何建立一套系统的评估体系，从而量化提示词的效果，驱动持续的迭代和优化。

## 什么是“好”的输出？定义评估标准

“这个 AI 的回答好不好？”——这个模糊的问题，是我们评估工作的起点。要让它变得可以衡量，我们需要将其分解为一系列具体的、可观测的评估指标。这些指标通常分为两大类：**定性指标**和**定量指标**。

让我们以一个贯穿全章的案例——开发一个“食谱生成器”AI——来具体说明。

### 定性指标 (Qualitative Metrics)

定性指标侧重于人类专家的主观判断，评估的是输出的“质量”和“体验”。它们虽然无法完全自动化，但对于衡量与人类感受密切相关的方面至关重要。我们可以设计一个评分卡（Rubric）来进行评估。

**食谱生成器 - 定性评估评分卡**

| 维度 | 1分 (差) | 3分 (中) | 5分 (优) |
| :--- | :--- | :--- | :--- |
| **相关性 (Relevance)** | 生成的食谱与用户输入的食材或菜系完全无关。 | 食谱沾点边，但不是最佳选择。 | 食谱精准匹配用户需求。 |
| **可行性 (Feasibility)** | 步骤混乱，材料用量不合理，几乎无法照做。 | 步骤基本清晰，但有少量不明确之处。 | 步骤清晰，材料精准，新手也能轻松上手。 |
| **创意性 (Creativity)** | 食谱平淡无奇，缺乏想象力。 | 食谱有一些新意，但仍在预料之中。 | 食谱搭配巧妙，令人耳目一新。 |
| **安全性 (Safety)** | 包含危险的烹饪建议或不安全的食材搭配。 | 无安全问题，但缺乏必要的安全提醒。 | 不仅安全，还主动提示了潜在的烹饪风险。 |
| **风格一致性 (Style)** | 语言风格与设定的“美食家”角色完全不符。 | 风格基本一致，但偶尔出戏。 | 完美扮演了美食家角色，语言生动诱人。 |

通过这样一张评分卡，不同的评估者就可以基于相对统一的标准，对同一个输出进行打分，从而量化那些原本模糊的“感觉”。

### 定量指标 (Quantitative Metrics)

定量指标是可以通过程序自动计算的、完全客观的数值指标。它们是实现自动化评估和大规模测试的基础。

-   **格式遵循度 (Format Compliance)**：假设我们要求食谱必须以特定的 JSON 格式输出，以便于前端页面解析。我们可以编写一个简单的脚本来验证输出是否 100% 遵循了预定义的 schema。这是最基本也最重要的定量指标之一。

    ```python
    import json

    def check_format(generated_output):
        try:
            data = json.loads(generated_output)
            # 检查是否包含必要的字段
            if "title" in data and "ingredients" in data and "steps" in data:
                return 1  # 格式正确
            else:
                return 0  # 缺少字段
        except json.JSONDecodeError:
            return 0  # 非法JSON格式
    ```

-   **关键词覆盖率 (Keyword Coverage)**：如果用户要求食谱必须包含“牛肉”和“西红柿”，我们可以计算输出中是否包含了这些必需的关键词。

-   **执行成功率 (Execution Success Rate)**：如果我们的提示词会生成需要调用的函数（例如，一个用于查询卡路里的 API），我们可以度量生成的函数调用参数有多大比例能够被系统成功执行而没有报错。

## 构建黄金评估集 (Golden Set)

定义了评估指标后，我们需要一套“标准考卷”来测试我们的提示词。这套考卷就是**黄金评估集（Golden Set）**，它是一组精心挑选的、具有代表性的“输入-理想输出”对。

黄金评估集是整个评估体系的基石，它确保了我们对不同版本提示词的评估是在一个公平、一致的基准上进行的。

**构建步骤：**

1.  **收集代表性输入**：从真实的用户场景中，收集各种有代表性的、典型的、甚至是边缘情况下的用户请求。例如，对于食谱生成器，可能包括“用冰箱里剩下的鸡蛋和培根做个快手早餐”、“给我一个低脂的素食晚餐建议”、“我想学做川菜，有什么入门推荐？”等等。
2.  **人工撰写理想输出**：由领域专家（比如，一位真正的厨师）为每一个输入，精心撰写一个“完美的”输出。这个输出就是我们评估时参照的“标准答案”。
3.  **标注元数据**：为每个样本添加标签，如任务类型（早餐、正餐、烘焙）、难度（简单、中等、复杂）、涉及的菜系等。这有助于我们对提示词在不同场景下的表现进行细分分析。

**食谱生成器 - 黄金评估集示例 (JSON 格式):**

## 持续的评估循环

拥有了评估指标和黄金评估集后，提示词的优化就不再是凭空想象，而是一个科学的、持续迭代的循环：

**提示 -> 评估 -> 分析 -> 优化 -> 再提示**

1.  **提示 (Prompt)**：设计或修改你的提示词。
2.  **评估 (Evaluate)**：使用黄金评估集作为输入，运行模型，得到一批输出。然后，运用定性和定量指标，对这批输出进行打分。
3.  **分析 (Analyze)**：分析评估结果。在哪些类型的任务上表现不佳？是相关性不够，还是格式总出错？找到短板所在。
4.  **优化 (Optimize)**：针对分析出的问题，回头修改你的提示词。是需要更明确的指令，还是需要提供更丰富的示例？
5.  **循环往复**：带着优化后的提示词，重新进入评估环节，直到各项指标达到你的目标为止。

> **重要提示：警惕评估集过拟合**
>
> 黄金评估集是你的“模拟考卷”，而不是“正式考卷”。切勿只针对评估集中的样本来过度优化你的提示词，这可能会导致它在模拟考中得分很高，但在真实、多变的生产环境中表现不佳。评估集应作为方向指引，而非优化的终点。
