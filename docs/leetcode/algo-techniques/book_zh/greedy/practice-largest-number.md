# 实战：最大数

> LeetCode 179. 最大数 | 难度：中等

自定义排序的贪心应用，理解"比较器设计"的经典案例。

📎 [LeetCode 179. 最大数](https://leetcode.cn/problems/largest-number/)

---

## 题目描述

给定一组非负整数 `nums`，重新排列它们的顺序，使之组成一个最大的整数。

由于结果可能很大，返回字符串形式。

**示例1**：
```
输入：nums = [10, 2]
输出："210"
```

**示例2**：
```
输入：nums = [3, 30, 34, 5, 9]
输出："9534330"
```

**示例3**：
```
输入：nums = [0, 0]
输出："0"
```

---

## 思路分析

### 问题的本质

这是一个**排序问题**，但排序规则不是简单的大小比较。

**关键问题**：给定两个数字，谁应该排在前面？

### 贪心策略

对于两个数 a 和 b，比较拼接后的结果：
- 如果 `ab > ba`，则 a 应该排在 b 前面
- 如果 `ab < ba`，则 b 应该排在 a 前面

```
比较 3 和 30:
  拼接方案1: "330"
  拼接方案2: "303"
  "330" > "303" → 3 应该在 30 前面
```

### 直觉理解

"3" 比 "30" 在高位更有价值，因为：
- 3 开头能得到 "3..."
- 30 开头只能得到 "30..."

但这种直觉不够精确，拼接比较才是正确的判断方法。

## 代码实现

```typescript
function largestNumber(nums: number[]): string {
  // 1. 转换为字符串（方便拼接比较）
  const strs = nums.map(String);
  
  // 2. 自定义排序：ab > ba 则 a 在前
  strs.sort((a, b) => {
    const ab = a + b;
    const ba = b + a;
    // 降序排列（大的在前）
    return ba.localeCompare(ab);
  });
  
  // 3. 处理全零的情况
  // 如果最大的数是 '0'，说明所有数都是 0
  if (strs[0] === '0') {
    return '0';
  }
  
  // 4. 拼接结果
  return strs.join('');
}
```

### 代码逻辑详解

**为什么用 `localeCompare`？**

`localeCompare` 返回负数、0或正数，正好符合 `sort` 的要求。这里用 `ba.localeCompare(ab)` 实现降序。

**为什么检查 `strs[0] === '0'`？**

如果排序后最大的还是 '0'，说明输入全是 0，应该返回 "0" 而不是 "000..."。

```
nums = [3, 30, 34, 5, 9]
strs = ["3", "30", "34", "5", "9"]

排序比较过程（部分示例）：

"3" vs "5":  "35" vs "53"   → "35" < "53"   → "5" 在前
"3" vs "9":  "39" vs "93"   → "39" < "93"   → "9" 在前
"34" vs "3": "343" vs "334" → "343" > "334" → "34" 在前

完整排序结果: ["9", "5", "34", "3", "30"]

拼接: "9534330"
```

## 比较器的正确性证明

### 需要证明什么？

自定义比较器必须满足**传递性**：
- 如果 a 应该在 b 前面，b 应该在 c 前面
- 那么 a 应该在 c 前面

### 传递性证明

设 a, b, c 是三个数字的字符串表示。

**已知**：
- `ab ≥ ba`（a 应该在 b 前面）
- `bc ≥ cb`（b 应该在 c 前面）

**证明** `ac ≥ ca`：

把字符串拼接看作数学运算。设 $|x|$ 表示 x 的位数。

$ab = a \cdot 10^{|b|} + b$
$ba = b \cdot 10^{|a|} + a$

从 $ab \geq ba$ 可得：
$a \cdot 10^{|b|} + b \geq b \cdot 10^{|a|} + a$

类似地处理 $bc \geq cb$，可以推导出 $ac \geq ca$。

（完整证明需要代数运算，但核心思想是：这个比较关系确实满足传递性。）

## 边界情况处理

### 边界1：全零数组

```
输入：nums = [0, 0, 0]
排序后：["0", "0", "0"]
拼接："000"
正确输出："0"

所以需要检查 strs[0] === '0'
```

### 边界2：单元素数组

```
输入：nums = [1]
输出："1"

代码自然处理，无需特殊逻辑
```

### 边界3：包含大数

```
输入：nums = [999999998, 999999997, 999999999]
排序后：["999999999", "999999998", "999999997"]
输出："999999999999999998999999997"

字符串比较自然处理大数，无溢出问题
```

## 复杂度分析

- **时间复杂度**：O(n log n × k)
  - 排序：O(n log n) 次比较
  - 每次比较：O(k)，k 是数字的平均位数

- **空间复杂度**：O(n × k)
  - 存储字符串数组

## 常见错误

### 错误1：直接比较数字大小

```typescript
// ❌ 错误：直接比较数值
nums.sort((a, b) => b - a);
// 输入 [3, 30]
// 排序后 [30, 3]
// 输出 "303" ← 错误！应该是 "330"
```

### 错误2：忘记处理前导零

```typescript
// ❌ 错误：没有检查全零情况
return strs.join('');
// 输入 [0, 0]
// 输出 "00" ← 错误！应该是 "0"
```

### 错误3：使用数值比较可能溢出

```typescript
// ❌ 有溢出风险
strs.sort((a, b) => Number(b + a) - Number(a + b));
// 当数字很大时会溢出
```

## 相关问题

### 变体：最小数

LeetCode 面试题 45：把数组排成最小的数

只需把比较方向反过来：

```typescript
function minNumber(nums: number[]): string {
  const strs = nums.map(String);
  
  // 升序排列
  strs.sort((a, b) => {
    const ab = a + b;
    const ba = b + a;
    return ab.localeCompare(ba);  // 改为升序
  });
  
  return strs.join('');
}
```

## 总结

最大数问题展示了**自定义比较器**的强大：

1. **核心思想**：通过拼接比较决定顺序
2. **数学保证**：比较器满足传递性，排序结果正确
3. **边界处理**：全零情况需要特判

这道题的思路可以推广到任何"组合最优化"问题：当你需要决定多个元素的排列顺序时，考虑设计一个比较器来表达"谁在前更好"。
