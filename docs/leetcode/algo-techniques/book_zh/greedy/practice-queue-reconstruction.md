# 实战：根据身高重建队列

> LeetCode 406. 根据身高重建队列 | 难度：中等

排序 + 贪心插入的经典案例，理解"先处理谁"的决策艺术。

📎 [LeetCode 406. 根据身高重建队列](https://leetcode.cn/problems/queue-reconstruction-by-height/)

---

## 题目描述

假设有打乱顺序的一群人站成一个队列，数组 `people` 表示队列中一些人的属性。

每个 `people[i] = [hi, ki]` 表示：
- `hi`：第 i 个人的身高
- `ki`：前面有多少人身高 **大于或等于** hi

请根据这些信息重建队列。

**示例**：
```
输入：people = [[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]]
输出：[[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]]

解释：
[5,0]: 前面 0 个人 >= 5 ✓
[7,0]: 前面 0 个人 >= 7 ✓
[5,2]: 前面 2 个人 >= 5 (有 7,7) ✓
[6,1]: 前面 1 个人 >= 6 (有 7) ✓
[4,4]: 前面 4 个人 >= 4 ✓
[7,1]: 前面 1 个人 >= 7 ✓
```

---

## 思路分析

### 问题的难点

每个人的位置受两个因素约束：
1. 自己的身高 h
2. 前面有多少人比自己高（k 值）

直接处理很复杂，因为插入一个人可能影响其他人的 k 值。

### 关键洞察

**核心发现**：矮个子的插入不会影响高个子的 k 值！

因为 k 值只统计"身高 >= 自己"的人，矮个子对高个子来说是"透明的"。

### 贪心策略

1. **排序**：按身高降序排列（身高相同时按 k 升序）
2. **插入**：依次将每个人插入到位置 k

**为什么这样有效？**
- 先插入高个子，他们的相对位置就确定了
- 后插入矮个子时，只需考虑"前面有多少高个子"
- 矮个子的 k 值恰好告诉我们该插入哪个位置

## 代码实现

```typescript
function reconstructQueue(people: number[][]): number[][] {
  // 1. 排序：身高降序，k 值升序
  people.sort((a, b) => {
    if (a[0] !== b[0]) {
      return b[0] - a[0];  // 身高高的在前
    }
    return a[1] - b[1];    // 同身高时，k 小的在前
  });
  
  // 2. 依次插入：在位置 k 插入
  const result: number[][] = [];
  
  for (const person of people) {
    // splice(k, 0, person) 在位置 k 插入 person
    result.splice(person[1], 0, person);
  }
  
  return result;
}
```

### 为什么身高相同时 k 小的在前？

身高相同的人互相"可见"。如果 [7,0] 和 [7,1] 同时存在：
- [7,0] 前面需要 0 个人 >= 7
- [7,1] 前面需要 1 个人 >= 7

所以 [7,0] 必须先插入，[7,1] 后插入才能保证正确。

## 执行过程详解

```
输入: [[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]]

Step 1: 排序
[[7,0], [7,1], [6,1], [5,0], [5,2], [4,4]]

Step 2: 依次插入
插入 [7,0] 到位置 0:
  result = [[7,0]]

插入 [7,1] 到位置 1:
  result = [[7,0], [7,1]]

插入 [6,1] 到位置 1:
  result = [[7,0], [6,1], [7,1]]
  
插入 [5,0] 到位置 0:
  result = [[5,0], [7,0], [6,1], [7,1]]
  
插入 [5,2] 到位置 2:
  result = [[5,0], [7,0], [5,2], [6,1], [7,1]]
  
插入 [4,4] 到位置 4:
  result = [[5,0], [7,0], [5,2], [6,1], [4,4], [7,1]]

验证结果:
[5,0]: 前面 0 个 >= 5 ✓
[7,0]: 前面 0 个 >= 7 ✓
[5,2]: 前面 [5,7] 中 2 个 >= 5 ✓
[6,1]: 前面 [5,7,5] 中 1 个 >= 6 ✓
[4,4]: 前面 [5,7,5,6] 中 4 个 >= 4 ✓
[7,1]: 前面 [5,7,5,6,4] 中 1 个 >= 7 ✓
```

## 贪心正确性分析

### 为什么高个子先处理？

1. **高个子只关心高个子**：身高 7 的人只关心前面有几个身高 >= 7 的人
2. **矮个子是透明的**：身高 5 的人插入不会改变身高 7 的人的 k 值
3. **k 值即位置**：处理高个子时，k 值直接对应最终位置

### 为什么插入位置是 k？

当我们插入一个人时，result 数组里都是比他高（或同高）的人。

他的 k 值 = 他前面需要的"高个子数量" = 他在当前数组中的位置。

## 复杂度分析

- **时间复杂度**：O(n²)
  - 排序：O(n log n)
  - 每次插入：O(n)，共 n 次
  - 总计：O(n²)

- **空间复杂度**：O(n)
  - 结果数组

## 另一种思路：从矮到高

也可以从矮个子开始处理：

```typescript
function reconstructQueue(people: number[][]): number[][] {
  // 按身高升序，k 值降序
  people.sort((a, b) => {
    if (a[0] !== b[0]) return a[0] - b[0];
    return b[1] - a[1];
  });
  
  // 准备 n 个空位
  const result: (number[] | null)[] = new Array(people.length).fill(null);
  
  for (const [h, k] of people) {
    // 找第 k+1 个空位
    let emptyCount = 0;
    for (let i = 0; i < result.length; i++) {
      if (result[i] === null) {
        if (emptyCount === k) {
          result[i] = [h, k];
          break;
        }
        emptyCount++;
      }
    }
  }
  
  return result as number[][];
}
```

**思路**：
- 矮个子只能看到空位和已填的矮个子
- 空位将来会填更高的人，都算"大于等于"
- 所以找第 k+1 个空位放入

这个方法时间复杂度也是 O(n²)。

## 常见错误

### 错误1：排序顺序错误

```typescript
// ❌ 错误：身高相同时 k 应该升序
people.sort((a, b) => {
  if (a[0] !== b[0]) return b[0] - a[0];
  return b[1] - a[1];  // 应该是 a[1] - b[1]
});
```

### 错误2：混淆插入逻辑

```typescript
// ❌ 错误：位置计算错误
result.splice(person[1] + 1, 0, person);
```

## 优化思路

标准解法的插入操作是 O(n)，总时间 O(n²)。

可以使用更高级的数据结构优化：
- **树状数组 + 二分**：O(n log² n)
- **线段树**：O(n log n)

但在面试中，O(n²) 解法通常已经足够。

## 总结

根据身高重建队列展示了贪心的精髓：**选择正确的处理顺序**。

1. **核心洞察**：矮个子对高个子"透明"
2. **策略制定**：高个子先处理，确定相对位置
3. **位置计算**：k 值直接对应插入位置

这道题的思想可以推广：当多个因素相互影响时，找到一个因素是"单向影响"的，先处理那一边。
