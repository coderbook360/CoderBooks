# Form固化层——从刻意到自动

## 知道方法不等于会用

到这里，你已经学完了CPERF的核心方法。

但有个问题：**知道和做到是两回事。**

```
刚学完：刻意用，很费劲
用熟了：自然用，不费劲
内化了：自动用，变直觉
```

Form固化层要解决的问题：**如何把方法变成习惯？**

## 元认知：思考自己的思考

### 什么是元认知

元认知就是**"思考自己的思考"**。

```
认知：我在思考这个问题
元认知：我在观察自己怎么思考这个问题
```

### 为什么元认知重要

如果你不知道自己是怎么思考的，就没法改进。

```
无元认知：
遇到问题 → 凭感觉做 → 结果不好 → 不知道为什么

有元认知：
遇到问题 → 观察自己怎么做 → 发现哪步有问题 → 针对性改进
```

### 怎么练习元认知

**方法一：实时自问**

在思考的过程中问自己：

```
"我现在在做什么？"
"我为什么这么想？"
"我漏掉了什么？"
"我的假设是什么？"
```

**方法二：事后回顾**

做完一件事后回顾：

```
"我是怎么做这个决定的？"
"哪步做得好？哪步可以改进？"
"如果重来，我会怎么做？"
```

**方法三：写思考日志**

把思考过程写下来：

```
今天做了什么决定？
当时是怎么想的？
结果怎么样？
下次可以怎么改进？
```

## 建立个人的逻辑检查清单

### 为什么需要检查清单

人脑有容量限制，靠记忆容易漏。

检查清单是**外部记忆**，帮你确保不遗漏。

### CPERF核心检查清单

```markdown
## 遇到问题时的快速检查

### C - Capture 捕获
□ 问题定义清楚了吗？
□ 信息收集充分了吗？
□ 信息可靠吗？

### P - Process 处理
□ 问题分类了吗？知道用什么工具吗？
□ 结构化分析了吗？
□ 决策有依据吗？

### E - Express 表达
□ 表达有结构吗？
□ 结论先行了吗？
□ 受众适配了吗？

### R - Review 复盘
□ 假设验证了吗？
□ 找过反例吗？
□ 需要更新判断吗？

### F - Form 固化
□ 学到什么了？
□ 下次怎么做得更好？
```

### 根据个人情况定制

每个人的弱项不同，检查清单要针对自己的弱项定制。

```
如果你经常"问题定义不清"：
□ 我能用一句话说清楚问题吗？
□ 成功标准是什么？
□ 边界条件是什么？

如果你经常"信息收集不全"：
□ 还有什么我不知道的？
□ 反对意见是什么？
□ 需要找谁确认？

如果你经常"表达混乱"：
□ 我的核心观点是什么？
□ 有几个理由支持？
□ 能用一分钟说完吗？
```

## 习惯形成的认知科学

### 习惯的结构

```
触发 → 行为 → 奖励

触发：什么情况下会做
行为：具体做什么
奖励：做完有什么好处
```

### 形成CPERF习惯

**设置触发条件**

```
触发1：遇到问题时
→ 先问自己："问题是什么？"

触发2：要做决定时
→ 先列选项

触发3：要表达时
→ 先想结论是什么
```

**简化行为**

一开始不要追求完美，先追求"做了"。

```
❌ 每次都走完整CPERF流程
✅ 先养成"问题是什么"的习惯

❌ 写完整的复盘报告
✅ 先养成"30秒快速回顾"的习惯
```

**及时奖励**

感受到方法的好处，就会更愿意用。

```
用了方法，问题解决得更顺利 → 强化
用了方法，表达更清晰 → 强化
用了方法，决策更有把握 → 强化
```

### 21天/66天法则

```
研究表明：

21天：开始形成习惯雏形
66天：习惯基本稳定

关键是：连续练习，不要断
```

## 从刻意到自动的三个阶段

### 阶段一：刻意练习（0-21天）

**特点**：
- 需要刻意提醒自己用方法
- 用起来比较费劲
- 经常忘记用

**策略**：
- 设置提醒
- 用检查清单
- 每天复盘有没有用

```
每天问自己：
"今天遇到问题时，有用CPERF吗？"
"哪个环节用了？哪个没用？"
"明天怎么做得更好？"
```

### 阶段二：熟练应用（21-66天）

**特点**：
- 想起来就能用
- 用得比较顺
- 但还不是自动的

**策略**：
- 增加应用场景
- 尝试更复杂的问题
- 总结自己的使用模式

```
扩展应用：
- 工作中的问题用
- 生活中的问题也用
- 大问题用完整版，小问题用快速版
```

### 阶段三：内化为直觉（66天以后）

**特点**：
- 遇到问题自然用
- 不需要刻意提醒
- 变成思维方式的一部分

**标志**：
```
- 遇到问题，自然先想"问题是什么"
- 做决定前，自然列选项
- 表达时，自然先说结论
- 做完事，自然回顾一下
```

## 固化的检查清单

```markdown
## 阶段一检查（0-21天）

每日检查：
□ 今天有遇到问题吗？
□ 有用CPERF吗？哪个环节？
□ 用得怎么样？哪里可以改进？
□ 明天怎么做得更好？

每周检查：
□ 这周用了几次？
□ 哪个环节用得最多？
□ 哪个环节还不熟？
□ 下周重点练哪个？
```

```markdown
## 阶段二检查（21-66天）

每周检查：
□ 这周应用了哪些场景？
□ 有没有更复杂的问题用上了？
□ 哪些情况下用得顺？哪些还不顺？
□ 有没有形成自己的使用模式？

每月检查：
□ 比上个月进步在哪里？
□ 还有哪些弱项需要加强？
□ 方法有没有根据个人情况调整？
```

```markdown
## 阶段三检查（66天以后）

每月检查：
□ 遇到问题会自然用方法吗？
□ 有没有形成自己的思维习惯？
□ 方法需要更新吗？
□ 有什么新的收获和体会？
```

## 常见的固化障碍

### 障碍一：太忙，没时间用

**应对**：先用快速版，30秒也能用。

```
没时间走完整流程 → 用30秒决策法
没时间写复盘报告 → 30秒快速回顾
```

### 障碍二：用了几天就忘了

**应对**：设置外部提醒。

```
手机提醒："今天用CPERF了吗？"
便利贴提醒：贴在显眼的地方
找人监督：和同事互相提醒
```

### 障碍三：感觉没什么用

**应对**：记录使用效果，看到进步。

```
记录：用了方法的情况 vs 没用方法的情况
对比：哪种情况结果更好？
感受：用了方法是不是更有把握？
```

## 检查点：固化进度的检验

```markdown
□ 触发建立
  - 有明确的触发条件吗？
  - 什么时候会想起用？

□ 行为简化
  - 方法简化到能坚持吗？
  - 不追求完美，先追求"做了"？

□ 奖励感知
  - 感受到方法的好处了吗？
  - 用了之后结果更好吗？

□ 阶段评估
  - 现在处于哪个阶段？
  - 下一阶段的目标是什么？

□ 持续改进
  - 有定期检查吗？
  - 有根据情况调整吗？
```
