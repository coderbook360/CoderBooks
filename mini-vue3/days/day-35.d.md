    -   `computed.ts`: 负责计算属性。

## 3. 性能思考

虽然我们实现了功能，但我们的实现还有优化空间。
比如在 `track` 中，我们每次都 `new Set()`。
在 Vue 3 源码中，为了极致性能，使用了位运算（Bitwise）来标记依赖状态，使用了对象池来复用 Effect 实例。
这些优化虽然增加了代码复杂度，但对于一个被数百万应用使用的框架来说，是值得的。
作为学习者，我们首先追求**逻辑正确**和**代码清晰**，其次才是极致性能。

## 4. 下一站：Runtime Core

响应式系统（Reactivity）是 Vue 3 的**数据驱动引擎**。
现在，引擎已经造好了，轰鸣作响。
但是，车轮（DOM）还没装上，方向盘（组件）还没装上。

从下周开始，我们将进入 **Runtime Core（运行时核心）** 阶段。
我们将利用这套响应式系统，去构建一个真正的 UI 框架：
-   实现 `h` 函数（创建虚拟 DOM）。
-   实现 `renderer`（渲染器）。
-   实现 `diff` 算法（高效更新）。
-   实现组件系统（Props, Slots, Lifecycle）。

这将是一个全新的领域，会有更多关于算法和架构的挑战。
