# 序言

## 为什么写这本书

在 Vue3 的众多特性中，响应式系统无疑是最核心、最重要的部分。它是整个框架的"心脏"，驱动着数据的变化自动反映到视图更新。

作为一名前端开发者，你可能已经熟练使用 `ref`、`reactive`、`computed`、`watch` 等 API，但你是否真正理解：

- 为什么 `ref` 需要通过 `.value` 访问值？
- `reactive` 是如何让对象变成"响应式"的？
- 依赖收集的 `track` 和触发更新的 `trigger` 是如何工作的？
- `computed` 的惰性求值是如何实现的？
- `effectScope` 为什么能统一管理副作用的生命周期？

这些问题的答案，隐藏在 Vue3 响应式系统的设计之中。本书将带你深入 `@vue/reactivity` 的源码，一层层揭开响应式的神秘面纱。

## 本书特色

本书采用 **"设计思想 + 源码解析"** 二合一的架构：

### 第一部分：设计思想

我们不急于直接看代码，而是先理解"为什么"。

- 为什么选择 Proxy 而不是 Object.defineProperty？
- 依赖收集的核心思想是什么？
- 调度器解决了什么问题？

理解设计思想，才能真正读懂源码。

### 第二部分：源码解析

逐行解读 `@vue/reactivity` 的核心实现：

- `reactive` 和 `readonly` 的 Proxy 拦截
- `effect` 副作用函数与依赖收集
- `ref` 的实现与自动解包
- `computed` 的惰性计算与缓存
- `watch` 的实现原理

每一行代码都有其存在的理由，我们会深入探讨每个设计决策。
