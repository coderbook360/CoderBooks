# 回调模式的本质与问题

## 章节定位

本章是第二部分的开篇，系统讲解回调模式。虽然现代 Node.js 更多使用 Promise 和 async/await，但理解回调是理解 Node.js 异步机制的基础，也是阅读大量老代码的必备知识。

## 学习目标

读完本章，读者应该能够：

1. 理解回调函数的本质和设计理念
2. 掌握 Node.js 错误优先回调的约定
3. 理解回调地狱问题及其产生原因
4. 了解回调模式的局限性
5. 为学习 Promise 打下基础

## 核心知识点

### 1. 回调的本质

- 回调是"将函数作为参数传递给另一个函数"
- 异步回调：函数不立即执行，而是在某个事件发生后执行
- 控制反转：你不调用函数，而是让别人调用

### 2. 错误优先回调

Node.js 的约定：回调函数第一个参数是 error

```javascript
fs.readFile('file.txt', (err, data) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log(data);
});
```

### 3. 回调地狱

嵌套回调导致的代码难以维护

```javascript
getUser(userId, (err, user) => {
  getPosts(user.id, (err, posts) => {
    getComments(posts[0].id, (err, comments) => {
      // 金字塔形代码
    });
  });
});
```

### 4. 回调的问题

- 错误处理分散
- 无法使用 try/catch
- 无法使用 return
- 代码难以组合和复用
- 回调可能被多次调用
- 回调可能不被调用

### 5. 何时仍然使用回调

- 事件监听器
- 流式处理
- 简单的一次性操作
- 性能关键场景
