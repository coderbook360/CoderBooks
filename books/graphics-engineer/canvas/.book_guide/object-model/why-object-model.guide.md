# 章节写作指导：为什么需要对象模型

## 1. 章节信息（强制性基础信息）
- **章节标题**: 为什么需要对象模型
- **文件名**: why-object-model.md
- **所属部分**: 第七部分：对象模型设计
- **预计阅读时间**: 15分钟
- **难度等级**: 中级

## 2. 学习目标（验收清单）

### 知识目标
- 理解直接使用 Canvas API 绘制图形的局限性和痛点
- 掌握对象模型在图形编辑器中的核心作用
- 了解 Fabric.js 等图形库采用对象模型的设计动机
- 认识对象模型带来的架构优势和功能可能性

### 技能目标
- 能够识别何时需要引入对象模型
- 能够对比分析"直接绘制"与"对象模型"两种架构
- 为后续章节的对象模型设计打好理论基础

## 3. 内容要点（内容清单）

### 核心概念（必须全部讲解）
- **Canvas 即时模式（Immediate Mode）**: 解释 Canvas 的即时绘制特性，绘制后不保留对象状态
- **保留模式（Retained Mode）**: 对比 SVG 的保留模式，说明保留对象状态的优势
- **对象模型（Object Model）**: 在 Canvas 之上构建的逻辑对象层，用于管理图形对象
- **图形编辑器的需求**: 选择、拖拽、变换、删除等交互操作对对象管理的需求

### 关键知识点（必须全部覆盖）
- 直接使用 Canvas API 的典型场景和代码示例
- 直接绘制模式在实现交互功能时遇到的具体问题
- 对象模型解决的核心痛点（交互、状态管理、重绘优化等）
- Fabric.js 对象模型的简要介绍（预告后续深入学习）
- 对象模型的架构优势与代价权衡

## 4. 写作要求（结构规范）

- **开篇方式**: 
  - 从一个实际场景切入：比如"你想在画布上绘制三个可拖拽的矩形"

  - 然后提出问题："如果要删除其中一个矩形怎么办？"引发思考

- **结构组织**: 
  1. **引入问题**：从实际场景出发，展示直接绘制模式的痛点

  3. **对比 SVG**：通过对比 SVG 的保留模式，说明保留对象状态的价值
  4. **引入对象模型**：说明对象模型的核心思想和架构
  5. **优势总结**：列举对象模型带来的具体好处
  6. **代价与权衡**：客观说明引入对象模型的复杂度成本
  7. **章节小结**：承上启下，为后续章节铺垫

- **代码示例**: 
  - 必须包含"直接绘制三个矩形"的完整代码示例（约 20-30 行）

## 5. 技术细节（技术规范）

- **源码参考**: 
  - 可参考 Fabric.js 的 `fabric.Object` 基类设计思想
  - 可参考 Konva.js、Paper.js 等库的架构介绍文档

- **实现要点**: 
  - 清楚说明 Canvas 绘制后不保留对象引用的特性
  - 解释为什么交互操作（选择、拖拽、变换）需要对象引用
  - 说明如何通过 JavaScript 对象来"保留"图形状态
  - 强调对象模型的核心职责：状态管理、渲染管理、事件管理

- **常见问题**: 
  - **问题 1**: "为什么不能直接在 Canvas 上删除某个图形？"
    - **解答**: Canvas 是位图，绘制后只是像素，没有"图形对象"的概念
  - **问题 2**: "对象模型会影响性能吗？"
    - **解答**: 会增加一定开销，但通过脏矩形、离屏渲染等优化可以达到良好性能
  - **问题 3**: "什么时候应该使用对象模型？"
    - **解答**: 当需要交互、动画、状态管理时；纯展示型绘制可以不用

## 6. 风格指导（表达规范）

- **语气语调**: 
  - 采用"提出问题 → 分析问题 → 解决问题"的叙事结构
  - 语气亲切，多用"你"来与读者互动
  - 用"我们一起来看看"、"让我们思考一个问题"等引导性语言

- **类比方向**: 
  - 类比 1：Canvas 像是"画在墙上的画"，画完就固定了；对象模型像是"可以移动的便利贴"
  - 类比 2：Canvas 即时模式像"水彩画"，无法撤销；对象模型像"拼图"，可以随时调整
  - 类比 3：对象模型像是"图层管理器"，每个图形对象都是一个可管理的图层

## 8. 写作建议与注意事项

### 重点强调
- 这一章是"理念篇"，重点在于建立正确的思维模型，而不是技术细节
- 务必让读者理解对象模型的核心价值：**将 Canvas 的"像素绘制"抽象为"对象管理"**
- 为后续章节（对象基类设计、属性系统等）做好概念铺垫

### 常见误区
- 不要过早引入复杂的实现细节（如观察者模式、序列化等），留给后续章节
- 不要让读者觉得"对象模型"过于抽象，务必用具体的代码示例说明
- 不要忽视性能讨论，要客观说明对象模型的开销

### 推荐示例场景
- **场景 1**: 绘制可拖拽的矩形（最基础）
- **场景 2**: 删除画布上的某个图形（引出管理需求）
