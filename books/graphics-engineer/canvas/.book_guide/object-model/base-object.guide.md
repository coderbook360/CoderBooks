## 3. 内容要点（内容清单）

### 核心概念（必须全部讲解）
- **图形对象基类（Base Object）**: 所有图形对象的抽象父类，定义共同属性和行为
- **对象标识（ID/UID）**: 每个对象的唯一标识符，用于引用和查找
- **几何属性**: 位置（x, y）、尺寸（width, height）、旋转角度等
- **样式属性**: 填充色、边框色、边框宽度、透明度等
- **变换矩阵**: 对象的变换状态（平移、旋转、缩放）
- **渲染方法**: 将对象绘制到 Canvas 上的核心方法
- **生命周期钩子**: 对象创建、更新、销毁时的回调机制

### 关键知识点（必须全部覆盖）
- 基类应包含的核心属性列表及其用途
- 基类应提供的核心方法（render、transform、clone 等）
- 如何实现对象的坐标变换（应用变换矩阵）
- 如何实现对象的碰撞检测（点击检测的基础）
- 继承体系的设计：如何从基类派生矩形、圆形等具体类
- 对象与 Canvas 上下文的交互方式

## 5. 技术细节（技术规范）

- **源码参考**: 
  - Fabric.js 的 `fabric.Object` 基类实现
  - Konva.js 的 `Konva.Node` 基类
  - Paper.js 的 `paper.Item` 基类

- **实现要点**: 
  1. **对象标识**: 使用 UUID 或递增 ID 生成唯一标识符
  2. **几何属性**: 
     - 位置：`x, y`（左上角或中心点，需明确说明）
     - 尺寸：`width, height`
     - 旋转：`rotation`（角度或弧度，需明确说明）
     - 缩放：`scaleX, scaleY`
  3. **样式属性**: 
     - 填充：`fill`（颜色字符串）
     - 边框：`stroke`（颜色字符串）、`strokeWidth`
     - 透明度：`opacity`（0-1 之间）
  4. **变换矩阵**: 使用 `DOMMatrix` 或自定义矩阵类存储变换状态
  5. **渲染方法**: 
     - 抽象方法：`render(ctx)` 由子类实现具体绘制逻辑
     - 模板方法：`draw(ctx)` 封装完整的渲染流程
  6. **生命周期**: 
     - 构造时初始化默认属性
     - 提供 `dispose()` 方法清理资源

- **常见问题**: 
  - **问题 1**: "对象的坐标原点应该在哪里？"
    - **解答**: 可以选择左上角或中心点，Fabric.js 使用中心点，更便于旋转和缩放
  - **问题 2**: "如何应用变换矩阵？"
    - **解答**: 使用 `ctx.save()` → `ctx.transform()` → 绘制 → `ctx.restore()`
  - **问题 3**: "基类的 render 方法应该是抽象的还是有默认实现？"
    - **解答**: 可以设计为抽象方法（子类必须实现），也可以提供默认实现（抛出错误）

## 8. 写作建议与注意事项

### 重点强调
- 基类设计是整个对象模型的核心，决定了后续功能的扩展性
- 务必让读者理解"模板方法模式"：基类提供流程框架，子类填充具体实现
- 强调属性设计的合理性：既要覆盖常用需求，又不能过度冗余

### 常见误区
- 不要在基类中放置过多的属性和方法，保持基类的简洁性
- 不要过早引入复杂的特性（如事件系统、动画等），留给后续章节
- 不要忽视性能考虑，说明哪些操作是昂贵的（如矩阵计算）

### 推荐实现细节
1. **对象标识生成**: 
   ```javascript
   let objectIdCounter = 0;
   class BaseObject {
     constructor() {
       this.id = `object_${++objectIdCounter}`;
     }
   }
   ```

2. **渲染流程封装**: 
   ```javascript
   draw(ctx) {
     ctx.save();
     this.applyTransform(ctx);
     this.render(ctx);
     ctx.restore();
   }
   ```

3. **抽象方法声明**: 
   ```javascript
   render(ctx) {
     throw new Error('render() must be implemented by subclass');
   }
   ```
