## 3. 内容要点

### 核心概念（必须全部讲解）

#### 3.1 三种遮罩类型对比
| 类型 | 实现原理 | 边缘效果 | 性能 | 适用场景 |
|------|----------|----------|------|----------|
| **Stencil Mask** | GPU 模板缓冲区 | 硬边 | ★★★★★ | 矩形/多边形裁剪 |
| **Alpha Mask** | 纹理 Alpha 通道 | 软边、渐变 | ★★★ | 不规则形状、羽化 |
| **Color Mask** | 写入颜色通道控制 | 无 | ★★★★ | 特殊渲染效果 |

#### 3.2 MaskSystem 核心机制
```typescript
class MaskSystem {
  // 遮罩栈
  maskStack: MaskData[];
  
  // 推入遮罩
  push(target: Container, maskData: MaskData): void {
    // 1. 检测遮罩类型
    // 2. 切换到对应的遮罩模式
    // 3. 渲染遮罩形状到 Stencil / 创建 Alpha 纹理
    // 4. 入栈
  }
  
  // 弹出遮罩
  pop(target: Container): void {
    // 1. 出栈
    // 2. 恢复之前的遮罩状态
  }
}
```

#### 3.3 Stencil Mask 工作流程
```
渲染遮罩形状（写入 Stencil = 1）
    ↓
设置 Stencil 测试：GL_EQUAL, ref = 1
    ↓
渲染被遮罩内容（只有 Stencil = 1 的像素通过）
    ↓
清除 Stencil 或恢复上一级
```

### 关键知识点（必须全部覆盖）
1. **遮罩类型检测**: 自动识别最优遮罩方式
2. **Stencil 引用值**: 多层嵌套的引用值管理
3. **Alpha 遮罩实现**: RenderTexture + 着色器采样
4. **遮罩栈维护**: push/pop 的平衡性
5. **性能考量**: Stencil 无额外纹理，Alpha 需要离屏渲染
6. **常见问题**: 遮罩不生效的排查方法
7. **与 Filter 的区别**: Mask 是裁剪，Filter 是后处理
