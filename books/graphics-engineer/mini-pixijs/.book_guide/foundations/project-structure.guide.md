## 3. 内容要点

### 核心概念（必须全部讲解）
- **Monorepo 架构**: 使用 pnpm workspace 管理多包
- **核心包 (Core Packages)**: @pixi/core, @pixi/app, @pixi/scene 等
- **功能包 (Feature Packages)**: graphics, text, filters 等
- **入口包 (Entry Package)**: pixi.js 统一导出

### 关键知识点（必须全部覆盖）
- packages 目录结构与命名规范
- 核心包与功能包的划分原则
- 包之间的依赖关系与分层设计
- TypeScript 配置与类型导出
- 构建工具链：rollup/esbuild 配置
- 版本管理与发布策略

### 前置知识
- 基本的 npm/pnpm 包管理概念
- ES Module 导入导出语法
- TypeScript 基础

## 5. 技术细节

### 源码参考
- `package.json` - 根目录配置
- `pnpm-workspace.yaml` - workspace 配置
- `packages/*/package.json` - 各包配置
- `rollup.config.js` 或 `tsconfig.json` - 构建配置

### 实现要点
- pnpm workspace 的工作原理
- TypeScript project references 的使用
- 如何实现按需导入（Tree Shaking）
- 类型定义文件的生成与导出

### 常见问题
- Q: 为什么使用 monorepo 而不是单一仓库？
  A: 更好的模块化、独立版本控制、按需安装
- Q: 如何只安装需要的模块？
  A: 直接安装子包如 `@pixi/sprite`，配合 bundler tree shaking
