## 3. 内容要点
### 核心概念（必须全部讲解）
- **向量长度**: 向量的大小，即从起点到终点的距离
- **计算公式**: $\|\mathbf{v}\| = \sqrt{x^2 + y^2 + z^2}$（勾股定理的 3D 扩展）
- **单位向量**: 长度为 1 的向量（方向相同，长度标准化）
- **归一化**: 将向量转换为单位向量的过程
- **平方长度**: 避免开方运算的性能优化技巧

### 关键知识点（必须全部覆盖）
- 从 2D 勾股定理到 3D 的推广
- 为什么需要单位向量（在方向计算中的应用）
- 归一化的计算方法：$\hat{\mathbf{v}} = \frac{\mathbf{v}}{\|\mathbf{v}\|}$
- 零向量的归一化问题（边界情况）
- lengthSquared() 在比较长度时的应用
- 性能优化：避免不必要的开方运算

## 5. 技术细节
- **实现要点**: 
  ```javascript
  length() {
    return Math.sqrt(this.x * this.x + this.y * this.y + this.z * this.z);
  }
  
  lengthSquared() {
    return this.x * this.x + this.y * this.y + this.z * this.z;
  }
  
  normalize() {
    const len = this.length();
    if (len === 0) {
      console.warn('Cannot normalize zero vector');
      return this;
    }
    return this.multiplyScalar(1 / len);
  }
  ```
- **设计考虑**: 
  - 零向量归一化的处理策略（警告、返回零向量、抛出异常）
  - 数值精度问题（长度接近 0 的情况）
  - 是否需要 normalized() vs normalize()（不可变 vs 可变）
- **性能优化**: 
  - 何时使用 lengthSquared()（比较长度、范围检测）
  - 开方运算的性能成本
