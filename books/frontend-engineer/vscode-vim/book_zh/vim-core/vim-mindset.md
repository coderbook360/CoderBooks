# Vim 思维模式：动词-文本对象模型

想象你正在编辑代码，需要修改一个函数名。传统思路是：用鼠标选中函数名，删除，输入新名称。三步操作，手在键盘和鼠标之间来回移动。

Vim 的思路完全不同。你只需说出意图：**"修改一个词"**——`ciw`，三个键完成。光标停在函数名上，按下 `ciw`，整个词被删除，你直接开始输入新名称。

这就是 Vim 的核心思维模式：**用语言描述编辑意图**。

## Vim 编辑语法

Vim 有自己的"语法"：

```
操作符 + 动作/文本对象 = 编辑命令
```

以 `ciw` 为例：
- `c`：change（修改），这是**操作符**
- `i`：inner（内部）
- `w`：word（词），这是**文本对象**
- 组合起来：change inner word = 修改词内部内容

再看几个例子：

| 命令 | 分解 | 效果 |
|------|------|------|
| `dw` | d(delete) + w(word) | 删除到下一个词 |
| `d$` | d(delete) + $(行尾) | 删除到行尾 |
| `ci"` | c(change) + i(inner) + "(引号) | 修改引号内内容 |
| `ya(` | y(yank/复制) + a(around) + ((括号) | 复制整个括号（含括号） |

这种语法的威力在于**组合性**。你不需要记忆 100 个独立命令，只需要掌握：
- 10 个操作符
- 10 个动作
- 10 个文本对象

就能组合出 10 × 10 = 100 种操作。

## 操作符（Operator）

操作符定义**做什么**：

| 操作符 | 全称 | 功能 |
|--------|------|------|
| `d` | delete | 删除 |
| `c` | change | 修改（删除后进入 Insert 模式） |
| `y` | yank | 复制 |
| `v` | visual | 选择（进入 Visual 模式） |
| `>` | indent | 增加缩进 |
| `<` | outdent | 减少缩进 |
| `=` | format | 自动格式化缩进 |
| `gu` | lowercase | 转小写 |
| `gU` | uppercase | 转大写 |
| `g~` | toggle case | 切换大小写 |

操作符单独按没有作用，必须跟一个动作或文本对象组合。

### 重复操作符

操作符按两次对当前行操作：

| 命令 | 效果 |
|------|------|
| `dd` | 删除当前行 |
| `cc` | 修改当前行 |
| `yy` | 复制当前行 |
| `>>` | 当前行增加缩进 |
| `<<` | 当前行减少缩进 |
| `==` | 自动格式化当前行 |

这是一个便捷约定，因为"对当前行操作"太常用了。

## 动作（Motion）

动作定义**影响范围**：

**字符级动作**：
| 动作 | 效果 |
|------|------|
| `h` | 左移一个字符 |
| `l` | 右移一个字符 |

**词级动作**：
| 动作 | 效果 |
|------|------|
| `w` | 到下一个词首 |
| `b` | 到上一个词首 |
| `e` | 到当前词尾 |
| `W/B/E` | 按空格分隔的大词 |

**行内动作**：
| 动作 | 效果 |
|------|------|
| `0` | 行首 |
| `^` | 第一个非空字符 |
| `$` | 行尾 |
| `f{char}` | 跳到字符 |
| `t{char}` | 跳到字符前 |

**跨行动作**：
| 动作 | 效果 |
|------|------|
| `j` | 下一行 |
| `k` | 上一行 |
| `gg` | 文件首 |
| `G` | 文件尾 |
| `{n}G` | 第 n 行 |

### 操作符 + 动作示例

| 命令 | 分解 | 效果 |
|------|------|------|
| `dw` | d + w | 删除到下一个词首 |
| `de` | d + e | 删除到词尾 |
| `d$` | d + $ | 删除到行尾 |
| `d0` | d + 0 | 删除到行首 |
| `dgg` | d + gg | 删除到文件首 |
| `dG` | d + G | 删除到文件尾 |
| `df"` | d + f" | 删除到下一个引号（含引号） |
| `dt"` | d + t" | 删除到下一个引号前 |
| `cw` | c + w | 修改到下一个词首 |
| `c$` | c + $ | 修改到行尾 |

## 文本对象（Text Object）

文本对象是 Vim 最强大的概念之一。它定义**逻辑边界**，而不是光标位置。

文本对象格式：`i/a` + 对象类型

- `i`：inner，不含边界
- `a`：around/a，含边界

| 文本对象 | inner 版本 | around 版本 |
|----------|------------|-------------|
| word | `iw` | `aw` |
| WORD | `iW` | `aW` |
| 句子 | `is` | `as` |
| 段落 | `ip` | `ap` |
| 引号 " | `i"` | `a"` |
| 引号 ' | `i'` | `a'` |
| 反引号 | `` i` `` | `` a` `` |
| 圆括号 | `i(` 或 `ib` | `a(` 或 `ab` |
| 方括号 | `i[` | `a[` |
| 花括号 | `i{` 或 `iB` | `a{` 或 `aB` |
| 尖括号 | `i<` | `a<` |
| 标签 | `it` | `at` |

### 文本对象实战

看一个真实例子：

```javascript
const message = "Hello World";
```

光标在 `Hello` 上：

| 命令 | 选中/影响范围 | 效果 |
|------|--------------|------|
| `ciw` | `Hello` | 修改 Hello |
| `caw` | `Hello ` | 修改 Hello（含后面空格）|
| `ci"` | `Hello World` | 修改引号内全部内容 |
| `ca"` | `"Hello World"` | 修改整个字符串（含引号）|
| `di"` | `Hello World` | 删除引号内容 |
| `da"` | `"Hello World"` | 删除整个字符串 |

再看 HTML 场景：

```html
<div class="container">Content</div>
```

光标在 Content 上：

| 命令 | 效果 |
|------|------|
| `cit` | 修改 Content |
| `cat` | 修改整个 `<div>...</div>` |
| `dit` | 删除 Content |
| `dat` | 删除整个标签 |

### inner vs around 的选择

什么时候用 `i`，什么时候用 `a`？

**使用 `i`（inner）的场景**：
- 修改内容但保留结构：`ci"` 修改字符串值，引号保留
- 删除内容但保留结构：`di(` 清空括号内容

**使用 `a`（around）的场景**：
- 完全删除：`da"` 删除整个字符串
- 剪切后粘贴到别处：`daw` + `p`

经验法则：
- **修改用 i**：你通常想保留结构
- **删除用 a**：你通常想完全移除

## 组合的威力

现在你理解了为什么 Vim 高效。传统编辑器的命令是"原子"的，每个功能一个快捷键。Vim 的命令是"分子"的，由操作符和对象组合而成。

这带来几个优势：

**1. 学习效率高**

掌握 10 个操作符 + 10 个文本对象 = 100 种操作能力

**2. 可预测性强**

一旦理解语法，你可以"说出"任何操作：
- "删除段落"→ 想：delete + around + paragraph → `dap`
- "复制括号内"→ 想：yank + inner + parenthesis → `yi(`

**3. 可扩展**

学新的操作符或文本对象，立即与已有的组合生效。比如学会 `it`（inner tag），立即可以 `cit`、`dit`、`yit`。

## 常见误区

**误区 1：记命令而不理解语法**

不要死记 `ci"` 是"修改引号内容"。要理解 `c` + `i"` 的组合逻辑。

**误区 2：只用少数几个命令**

很多人只用 `dd`、`yy`、`p`。这就像学了英语只会说 "Hello"。花时间掌握更多文本对象，回报巨大。

**误区 3：期望立即熟练**

Vim 思维需要时间内化。最初你需要"想"命令，慢慢变成本能反应。给自己 2 周时间。

## 实战对比

最后，用数据说话：

**任务：将 "Hello World" 修改为 "Hi Vim"**

传统方式：
1. 鼠标双击选中 "Hello"（2 秒）
2. 输入 "Hi"（0.5 秒）
3. 鼠标双击选中 "World"（2 秒）
4. 输入 "Vim"（0.5 秒）

总计约 **5 秒**，手在鼠标和键盘间移动 2 次。

Vim 方式：
1. `ciw` Hi `Esc`（修改 Hello）
2. `w`（移动到 World）
3. `ciw` Vim `Esc`

总计约 **2 秒**，手完全不离开键盘。

或者更快：
1. `ci"` Hi Vim（直接修改引号内全部内容）

总计约 **1 秒**。
