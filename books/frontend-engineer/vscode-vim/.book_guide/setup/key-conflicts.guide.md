# 章节写作指导：解决快捷键冲突

## 1. 章节信息（强制性基础信息）
- **章节标题**: 解决快捷键冲突：翻页、窗口、系统键
- **文件名**: setup/key-conflicts.md
- **所属部分**: 第一部分：VSCode Vim 基础与配置优化
- **预计阅读时间**: 20 分钟
- **难度等级**: 中级

## 2. 学习目标（验收清单）
### 知识目标
- 理解 VSCode Vim 快捷键冲突的根本原因
- 掌握冲突解决的三种策略（重映射、禁用、融合）
- 了解常见冲突键位及其优先级

### 技能目标
- 能够识别并解决翻页键冲突（Ctrl+D/U/F/B）
- 能够配置窗口管理键位（Ctrl+W）
- 能够处理系统键冲突（Ctrl+C/V/A/X）
- 能够自定义键位映射策略

## 3. 内容要点（内容清单）
### 核心概念（必须全部讲解）
- **键位冲突本质**: VSCode 和 Vim 两套系统的快捷键同时生效时的优先级问题
- **三种解决策略**:
  1. 重映射 Vim 键位（让 Vim 适应 VSCode）
  2. 禁用 VSCode 快捷键（让 VSCode 适应 Vim）
  3. 融合使用（混合模式，根据场景选择）

### 关键知识点（必须全部覆盖）
- **翻页键冲突**（最高频问题）:
  - `Ctrl+D`: Vim 向下翻半页 vs VSCode 添加光标
  - `Ctrl+U`: Vim 向上翻半页 vs VSCode 撤销
  - `Ctrl+F`: Vim 向下翻页 vs VSCode 查找
  - `Ctrl+B`: Vim 向上翻页 vs VSCode 切换侧边栏
  
- **窗口管理冲突**:
  - `Ctrl+W`: Vim 窗口命令前缀 vs VSCode 关闭标签
  - `Ctrl+W H/J/K/L`: Vim 移动窗口焦点 vs VSCode 无冲突
  
- **系统键冲突**:
  - `Ctrl+C`: Vim 退出 Insert 模式 vs 系统复制
  - `Ctrl+V`: Vim Visual Block 模式 vs 系统粘贴
  - `Ctrl+A`: Vim 数字递增 vs 全选
  - `Ctrl+X`: Vim 数字递减 vs 剪切

## 5. 技术细节（技术规范）
- **settings.json 配置示例**:
```json
{
  "vim.handleKeys": {
    "<C-d>": true,      // Vim 接管 Ctrl+D（翻页）
    "<C-u>": true,      // Vim 接管 Ctrl+U
    "<C-f>": false,     // VSCode 接管 Ctrl+F（查找）
    "<C-c>": false,     // VSCode 接管 Ctrl+C（复制）
    "<C-v>": false,     // VSCode 接管 Ctrl+V（粘贴）
    "<C-a>": false,     // VSCode 接管 Ctrl+A（全选）
    "<C-w>": true       // Vim 接管 Ctrl+W（窗口管理）
  }
}
```

- **keybindings.json 重映射示例**:
```json
[
  {
    "key": "ctrl+w",
    "command": "-workbench.action.closeActiveEditor",
    "when": "editorTextFocus && vim.active"
  },
  {
    "key": "alt+w",
    "command": "workbench.action.closeActiveEditor",
    "when": "editorTextFocus"
  }
]
```

- **常见问题**: 
  - 配置修改后需要重启 VSCode 或重新加载窗口
  - `vim.handleKeys` 中 `true` 表示 Vim 接管，`false` 表示 VSCode 接管
  - 某些键位在特定上下文（when 条件）下才会冲突
