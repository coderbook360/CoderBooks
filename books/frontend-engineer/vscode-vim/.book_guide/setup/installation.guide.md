# 章节写作指导：VSCode Vim 插件安装与环境配置

## 1. 章节信息（强制性基础信息）
- **章节标题**: VSCode Vim 插件安装与环境配置
- **文件名**: setup/installation.md
- **所属部分**: 第一部分：VSCode Vim 基础与配置优化
- **预计阅读时间**: 15 分钟
- **难度等级**: 初级

## 2. 学习目标（验收清单）
### 知识目标
- 理解 VSCode Vim 插件的功能范围和限制
- 了解 VSCode Vim 与纯 Vim 的差异
- 掌握插件的基本配置原理

### 技能目标
- 能够成功安装 VSCode Vim 插件
- 能够验证插件是否正常工作
- 能够进行基础的环境检查

## 3. 内容要点（内容清单）
### 核心概念（必须全部讲解）
- **VSCode Vim 插件定位**: 在 VSCode 环境中模拟 Vim 编辑体验，而非完整的 Vim
- **模式系统**: Normal、Insert、Visual、Command-line 四种模式的基本介绍
- **键位映射原理**: VSCode 的键绑定与 Vim 键位的协调机制

### 关键知识点（必须全部覆盖）
- VSCode Vim 插件的官方名称和维护者
- 插件的核心功能支持列表
- **明确告知不支持的功能**（设置正确预期）
- 安装步骤的详细说明
- 首次启动后的验证方法
- 常见安装问题与排查

## 4. 写作要求（结构规范）
- **开篇方式**: 以一个痛点场景开始："你是否习惯了 Vim 的高效编辑，却在使用 VSCode 时不得不切换回鼠标？"
- **结构组织**: 
  1. 引言：为什么需要 VSCode Vim
  2. 了解 VSCode Vim：功能与限制
  3. 安装步骤（图文并茂）
  4. 验证与测试
  5. 疑难排查
  6. 下一步建议

- **代码示例**: 必须包含：
  - 插件搜索关键词截图说明
  - 基本测试命令（如 `dd`、`yy`、`p` 等）
  - settings.json 的初始配置示例
  
- **图表需求**: 
  - 必须包含：VSCode 扩展市场搜索界面
  - 建议包含：模式切换状态栏截图
  - 功能对比表：VSCode Vim vs 纯 Vim

## 5. 技术细节（技术规范）
- **插件信息**: 
  - 扩展 ID: `vscodevim.vim`
  - GitHub 仓库: https://github.com/VSCodeVim/Vim
  - 当前版本建议（写作时核实最新稳定版本）
  
- **实现要点**: 
  - VSCode Vim 通过 VSCode 的 KeyBinding API 实现键位拦截
  - 某些快捷键可能与 VSCode 原生快捷键冲突（引导下一章）
  - 插件启动性能考虑（首次加载可能较慢）

- **常见问题**: 
  - 中文输入法切换问题（引导第 5 章）
  - CapsLock 键建议重映射为 Ctrl（预告附录内容）
  - Ctrl+C/Ctrl+V 等系统键位处理（引导第 2 章）

## 6. 风格指导（表达规范）
- **语气语调**: 友好、实用、避免说教。使用"你"而非"您"，增强亲切感
- **类比方向**: 
  - 将 VSCode Vim 比作"在现代汽车里安装手动挡"——保留驾驶乐趣，同时享受现代便利
  - 避免"纯 Vim 原教旨主义"，强调"融合"而非"替代"
