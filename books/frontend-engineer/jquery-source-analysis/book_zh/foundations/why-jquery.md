# 为什么要学习jQuery源码

2024年了，还在谈jQuery？

这可能是很多读者看到这本书时的第一反应。在React、Vue、Svelte主导的时代，jQuery似乎成了"上古技术"的代名词，学习它的源码看起来像是在浪费时间。

但我想先问你一个问题：**你真的了解jQuery吗？**

不是会用 `$('.btn').click()`，而是理解它为什么能在浏览器兼容性地狱的年代一统江湖，为什么至今仍有超过77%的网站在使用它（来自W3Techs 2024年统计），为什么它的设计理念深刻影响了整个前端生态？

## 源码学习的真正价值

首先要问一个问题：我们为什么要读源码？

很多人的答案是"面试要考"。这个理由太功利了，也太脆弱了——靠死记硬背来的知识，用完就忘。

真正的价值在于**思维方式的升级**。

当你读到jQuery如何用一个简单的 `extend` 函数实现深浅拷贝、插件扩展、默认参数合并时，你学到的不是这个函数本身，而是**一个函数如何通过精巧设计承担多重职责**。

当你看到Sizzle选择器引擎如何通过编译缓存将选择器性能提升数十倍时，你理解的是**空间换时间的工程权衡**。

当你分析事件系统如何用数据分离实现事件委托时，你掌握的是**关注点分离的架构思想**。

这些思维方式，才是真正值钱的东西。

## jQuery解决了什么问题

现在我要问第二个问题：在jQuery诞生之前，前端开发是什么样子？

让我们回到2006年。那时候，写一个简单的DOM操作是这样的：

```javascript
// 获取元素
var element = document.getElementById('myId');
if (!element) {
  element = document.getElementsByClassName('myClass')[0]; // IE8不支持
}

// 添加事件监听
if (element.addEventListener) {
  element.addEventListener('click', handler, false);
} else if (element.attachEvent) {
  element.attachEvent('onclick', handler); // IE专属
}

// 设置样式
element.style.cssText = 'color: red; font-size: 14px;';
// 或者
element.style.color = 'red';
element.style.fontSize = '14px';

// 发送Ajax请求
var xhr;
if (window.XMLHttpRequest) {
  xhr = new XMLHttpRequest();
} else {
  xhr = new ActiveXObject('Microsoft.XMLHTTP'); // IE6
}
xhr.open('GET', '/api/data', true);
xhr.onreadystatechange = function() {
  if (xhr.readyState === 4 && xhr.status === 200) {
    var data = JSON.parse(xhr.responseText);
    // 处理数据
  }
};
xhr.send();
```

每一行代码都在和浏览器兼容性作斗争。IE6、IE7、IE8，每个版本都有自己的"脾气"。开发者80%的时间花在处理这些差异上，真正的业务逻辑反而成了配角。

然后，jQuery来了：

```javascript
// 获取元素
$('#myId, .myClass')

// 添加事件监听
$('#myId').on('click', handler)

// 设置样式
$('#myId').css({ color: 'red', fontSize: '14px' })

// 发送Ajax请求
$.get('/api/data', function(data) {
  // 处理数据
})
```

**Write Less, Do More**——这不是营销口号，而是实实在在的开发体验革命。

jQuery在底层处理了所有浏览器差异，开发者只需要关注业务逻辑。这种"屏蔽复杂性，暴露简洁接口"的设计思想，至今仍是优秀库设计的黄金法则。

## 为什么jQuery源码值得深入学习

可能很多人会问：既然jQuery解决的是浏览器兼容性问题，现代浏览器已经标准化了，那学习它的源码还有什么意义？

意义在于**设计智慧是跨时代的**。

### 代码质量极高

jQuery是经过十几年打磨的工业级代码。每一行都经过深思熟虑，每一个边界情况都有处理。这种代码质量，在开源项目中并不多见。

更重要的是，jQuery的注释非常完善。源码中大量的注释解释了"为什么这样做"，而不仅仅是"做了什么"。这对于学习者来说是极其宝贵的。

### 覆盖前端核心领域

jQuery的源码涵盖了前端开发的几乎所有核心领域：

- **选择器引擎**：词法分析、语法编译、匹配优化
- **DOM操作**：遍历、操作、事件
- **样式处理**：计算样式、尺寸位置
- **Ajax通信**：请求封装、Promise模式
- **动画系统**：补间计算、队列调度

读懂jQuery，相当于把前端核心技术过了一遍。

### 设计模式的活教材

jQuery源码中到处是设计模式的实战应用：

- **工厂模式**：`$()` 函数根据参数类型创建不同对象
- **外观模式**：统一接口屏蔽浏览器差异
- **观察者模式**：事件系统的核心
- **策略模式**：Ajax的prefilter和transport
- **装饰器模式**：插件机制
- **代理模式**：事件委托

这些不是教科书上的干巴巴例子，而是真实解决问题的优雅方案。

### 兼容性处理的艺术

虽然现代浏览器已经标准化，但兼容性问题从未消失。只是战场从浏览器差异转移到了：

- 不同框架版本的API差异
- 不同环境（Node/Browser/Worker）的API差异
- 不同设备（移动端/桌面端）的特性差异

jQuery处理兼容性的思路——特性检测、优雅降级、渐进增强——依然适用。

## 本书的学习路线

本书按照jQuery源码的模块划分，带你逐步深入：

**第一部分：源码阅读准备**

在正式进入源码之前，我们需要做好准备工作。包括了解jQuery的源码结构、搭建调试环境、掌握源码阅读方法论。这些是高效阅读源码的基础。

**第二部分：核心架构**

jQuery的核心设计是整个库的基石。我们会深入分析IIFE封装、构造函数、原型链、链式调用等核心机制。理解这些，才能理解jQuery的"骨架"。

**第三部分：选择器引擎Sizzle**

Sizzle是jQuery最复杂的部分之一，也是最精彩的部分。我们会从词法分析开始，逐步解析选择器编译、匹配策略、性能优化。这部分会让你对编译原理有更直观的认识。

**第四部分：DOM操作**

DOM操作是jQuery最常用的功能。我们会分析遍历、过滤、插入、删除、克隆等操作的实现，理解domManip和buildFragment等核心函数。

**第五部分：属性与样式**

attr与prop的区别是经典面试题，我们会从源码角度彻底讲清楚。还有data缓存系统、css方法、尺寸计算等实用功能的实现。

**第六部分：事件系统**

事件系统是jQuery的精华之一。事件委托、命名空间、特殊事件处理，每一个设计都值得细细品味。

**第七部分：Ajax系统**

Ajax模块展示了如何设计一个灵活可扩展的请求库。Deferred和Promise的实现更是异步编程的经典案例。

**第八部分：动画系统**

动画系统涉及补间计算、队列调度、缓动函数等概念，是理解动画原理的好材料。

**第九部分：高级机制**

Callbacks回调队列、Queue通用队列、Hooks钩子机制，这些是jQuery可扩展性的基础。

**第十部分：实战与总结**

最后，我们会讨论jQuery的设计模式、性能优化策略，以及如何将学到的知识应用到实际工作中。

## 学习建议

在开始之前，有几点建议：

**1. 保持好奇心**

不要抱着"完成任务"的心态，而是带着好奇心去探索。遇到不理解的地方，停下来思考：如果是我，会怎么设计？

**2. 动手调试**

源码不是用来"看"的，而是用来"跑"的。下一章我们会搭建调试环境，之后的每个知识点，都建议你亲自设置断点，观察执行过程。

**3. 做好笔记**

阅读源码的过程中，及时记录你的理解、疑问、收获。这些笔记会成为你最宝贵的学习资料。

**4. 不必追求完美**

jQuery源码有上万行，不可能一次读完读透。先建立整体认识，再逐步深入细节。遇到实在看不懂的地方，可以暂时跳过，以后回来再看。

## 小结

让我们回到开篇的问题：2024年了，为什么还要学jQuery源码？

答案已经很清楚了：

- 不是为了使用jQuery，而是为了**学习设计智慧**
- 不是为了背诵API，而是为了**提升代码阅读能力**
- 不是为了应付面试，而是为了**成为更好的工程师**
