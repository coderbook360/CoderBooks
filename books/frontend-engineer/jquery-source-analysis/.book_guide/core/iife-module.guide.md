# 章节指导：立即执行函数与模块封装

## 章节定位
本章开启核心架构部分，讲解jQuery如何通过IIFE和模块模式实现代码封装和隔离。

## 写作方向调整
- **完全聚焦现代JavaScript**：忽略所有浏览器兼容性讨论
- **专注设计思想**：为什么这样设计？解决什么问题？
- **对比ES Module**：用现代模块系统作为参照理解IIFE的设计意图

## 学习目标
1. 理解IIFE的作用和设计意图
2. 掌握模块模式的实现原理
3. 理解jQuery如何避免全局污染
4. 对比IIFE与ES Module的异同

## 核心内容要求

### 1. IIFE的本质
- 立即执行函数的语法
- 创建独立作用域
- 避免变量泄露到全局

### 2. jQuery的IIFE结构
- 整体封装结构
- 参数传递（window、undefined）
- 返回值暴露

### 3. 模块模式
- 私有变量与公共接口
- 闭包的应用
- 模块间通信

### 4. 与ES Module的对比
- IIFE是ES Module之前的解决方案
- 两者的相似设计理念
- 现代项目如何选择

## 禁止事项
- 禁止讨论IE兼容性
- 禁止讨论undefined被重写问题（现代环境不存在）
- 禁止过度展开历史背景
