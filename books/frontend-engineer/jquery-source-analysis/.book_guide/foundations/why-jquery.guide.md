# 章节指导：为什么要学习jQuery源码

## 章节定位
本章是全书的开篇，需要回答读者心中最核心的问题：在 React/Vue 时代，为什么还要学习一个"过时"的库？

## 学习目标
1. 理解 jQuery 的历史地位和设计智慧
2. 认识到源码阅读对技术成长的价值
3. 建立正确的源码学习心态
4. 明确本书的学习路径和收获预期

## 核心内容要求

### 1. 破除偏见：jQuery 不是"过时"的代名词
- jQuery 解决的问题：浏览器兼容性地狱、DOM 操作繁琐
- jQuery 的设计哲学：Write Less, Do More
- jQuery 对前端生态的影响：选择器引擎、链式调用、插件机制

### 2. 源码学习的真正价值
- 不是为了"面试造火箭"，而是理解设计思维
- 经典代码中的设计模式和最佳实践
- 培养阅读复杂代码的能力

### 3. jQuery 源码的学习价值
- 代码质量极高，注释完善
- 覆盖前端核心领域：DOM、事件、Ajax、动画
- 设计模式的活教材：工厂模式、观察者模式、策略模式等
- 兼容性处理的艺术

### 4. 本书的学习路线图
- 预备知识要求
- 各部分内容概览
- 建议的学习顺序

## 写作风格要求
- 以问题开篇，直击读者痛点
- 用数据和事实说话，不空谈
- 避免说教，激发读者内在动力
- 结尾要有力，让读者迫不及待地开始学习

## 示例代码要求
- 展示 jQuery 之前原生 DOM 操作的繁琐
- 对比 jQuery 的简洁优雅
- 不需要复杂代码，重点在于对比

## 禁止事项
- 禁止贬低现代框架
- 禁止过度吹捧 jQuery
- 禁止技术考古式的无意义历史回顾
