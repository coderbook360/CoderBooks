## 3. 内容要点

### 核心概念

| 概念 | 解释要求 |
|-----|---------|
| **CI (持续集成)** | 解释频繁集成代码并自动验证 |
| **CD (持续交付/部署)** | 解释自动化交付和部署流程 |
| **流水线** | 解释自动化执行的任务序列 |
| **工作流** | 解释 CI/CD 平台的配置文件 |

### 关键知识点

1. **CI/CD 的价值**
   - 快速反馈
   - 降低风险
   - 提高效率
   - 标准化流程

2. **流水线设计**
   - 代码检查阶段
   - 构建阶段
   - 测试阶段
   - 部署阶段

3. **GitHub Actions**
   - 工作流语法
   - 触发器配置
   - 任务和步骤
   - Secrets 管理

4. **其他 CI/CD 平台**
   - GitLab CI
   - Jenkins
   - CircleCI
   - 平台对比

5. **自动化测试集成**
   - 单元测试
   - E2E 测试
   - 视觉回归测试
   - 覆盖率报告

6. **自动化部署**
   - 环境区分
   - 部署策略
   - 回滚机制
   - 通知集成
