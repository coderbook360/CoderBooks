## 2. 学习目标

- 理解监控数据上报的核心挑战

## 3. 内容要点

### 核心概念

| 概念 | 解释要求 |
|-----|---------|
| **数据上报** | 解释将采集的监控数据发送到服务端 |
| **采样 (Sampling)** | 解释只上报部分数据以降低成本 |
| **限流 (Rate Limiting)** | 解释限制上报频率防止服务端过载 |
| **Beacon API** | 解释页面卸载时可靠发送数据的 API |
| **批量上报** | 解释积累多条数据后一次性发送 |

### 关键知识点

1. **上报的核心挑战**
   - 数据量过大
   - 上报丢失
   - 页面卸载时的上报
   - 对用户体验的影响

2. **上报方式对比**
   - XMLHttpRequest/Fetch
   - navigator.sendBeacon
   - Image Beacon
   - WebSocket
   - 各方式优缺点

3. **采样策略设计**
   - 随机采样
   - 用户采样
   - 错误全量 + 正常采样
   - 动态采样率

4. **限流策略**
   - 时间窗口限流
   - 令牌桶算法
   - 滑动窗口计数
   - 动态限流

5. **批量上报**
   - 数据积累策略
   - 上报时机选择
   - 数据压缩

6. **失败处理**
   - 重试机制
   - 本地存储补报
   - 降级策略
