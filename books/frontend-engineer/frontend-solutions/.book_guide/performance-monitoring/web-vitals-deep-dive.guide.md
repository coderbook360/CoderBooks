### 技能目标

## 3. 内容要点

### 核心概念

| 概念 | 解释要求 |
|-----|---------|
| **LCP (Largest Contentful Paint)** | 解释最大内容绘制，测量加载性能 |
| **FID (First Input Delay)** | 解释首次输入延迟，测量交互响应 |
| **CLS (Cumulative Layout Shift)** | 解释累积布局偏移，测量视觉稳定性 |
| **INP (Interaction to Next Paint)** | 解释交互到下一次绘制，FID 的替代指标 |
| **TTFB (Time to First Byte)** | 解释首字节时间，测量服务器响应 |

### 关键知识点

1. **Web Vitals 的设计理念**
   - 以用户为中心的性能指标
   - 核心 vs 辅助指标
   - 指标的演进历史

2. **LCP 深度解析**
   - 什么元素算 LCP
   - LCP 的计算时机
   - 影响 LCP 的因素
   - 优化 LCP 的方向

3. **FID/INP 深度解析**
   - 输入延迟的定义
   - FID vs INP 的区别
   - 影响交互响应的因素
   - 优化策略

4. **CLS 深度解析**
   - 布局偏移的计算
   - 影响分数与距离分数
   - 预期 vs 非预期偏移
   - 优化 CLS 的策略

5. **指标测量方式**
   - Performance Observer API
   - web-vitals 库的使用
   - 实验室数据 vs 真实用户数据

6. **性能目标设定**
   - Google 推荐的阈值
   - P75 为何重要
   - 建立性能预算
