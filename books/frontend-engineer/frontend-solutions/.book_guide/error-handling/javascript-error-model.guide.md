- 理解 JavaScript 错误对象的内部结构

- 能够正确创建和抛出自定义错误

## 3. 内容要点

### 核心概念

| 概念 | 解释要求 |
|-----|---------|
| **Error 对象** | 解释 Error 对象的三大核心属性：message, name, stack |
| **内置错误类型** | 解释 TypeError, ReferenceError, SyntaxError 等的区别 |
| **错误堆栈 (Stack Trace)** | 解释堆栈的生成时机、格式、调用栈的关系 |
| **Error.captureStackTrace** | 解释 V8 特有的堆栈捕获 API |
| **AggregateError** | 解释 ES2021 新增的聚合错误类型 |

### 关键知识点

1. **Error 对象结构**
   - name 属性：错误类型名称
   - message 属性：错误描述信息
   - stack 属性：错误堆栈（非标准但广泛支持）
   - cause 属性：ES2022 新增的错误链

2. **内置错误类型详解**
   - Error：通用错误基类
   - TypeError：类型错误
   - ReferenceError：引用错误
   - SyntaxError：语法错误
   - RangeError：范围错误
   - URIError：URI 编码错误
   - EvalError：eval 相关错误（已废弃）

3. **错误堆栈深度解析**
   - 堆栈格式解析（Chrome vs Firefox vs Safari）
   - 堆栈生成的时机
   - 异步代码中的堆栈问题
   - Error.stackTraceLimit 配置

4. **自定义错误设计**
   - 继承 Error 类的正确方式
   - 自定义错误的命名规范
   - 错误码与错误信息的设计
   - 业务错误 vs 系统错误

5. **V8 错误处理机制**
   - Error.captureStackTrace 的使用
   - Error.prepareStackTrace 的定制
   - 结构化堆栈数据的获取
