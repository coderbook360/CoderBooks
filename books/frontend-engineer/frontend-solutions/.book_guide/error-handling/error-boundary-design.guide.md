## 3. 内容要点

### 核心概念

| 概念 | 解释要求 |
|-----|---------|
| **错误边界 (Error Boundary)** | 解释组件级错误隔离机制，防止错误导致整个应用崩溃 |
| **getDerivedStateFromError** | 解释 React 静态方法，用于渲染降级 UI |
| **componentDidCatch** | 解释 React 生命周期，用于错误上报 |
| **errorHandler (Vue)** | 解释 Vue 全局错误处理器 |
| **renderError (Vue)** | 解释 Vue 组件的 renderError 选项 |

### 关键知识点

1. **错误边界的设计理念**
   - 局部错误不应导致全局崩溃
   - 优雅降级 vs 白屏
   - 错误隔离的粒度设计

2. **React Error Boundary**
   - 类组件实现方式
   - getDerivedStateFromError 的作用
   - componentDidCatch 的作用
   - 无法捕获的错误类型

3. **React Error Boundary 的局限**
   - 事件处理器中的错误
   - 异步代码中的错误
   - 服务端渲染错误
   - 自身抛出的错误

4. **Vue 错误处理机制**
   - app.config.errorHandler
   - 组件的 errorCaptured 钩子
   - 错误传播规则
   - renderError 选项

5. **错误边界的层次设计**
   - 应用级 vs 路由级 vs 组件级
   - 关键区域 vs 非关键区域
   - 错误边界的嵌套

6. **错误恢复与用户体验**
   - 降级 UI 的设计
   - 重试机制的实现
   - 错误信息的展示
   - 用户反馈收集
