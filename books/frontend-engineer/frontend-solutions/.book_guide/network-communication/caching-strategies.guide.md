## 3. 内容要点

### 核心概念

| 概念 | 解释要求 |
|-----|---------|
| **强缓存 (Strong Cache)** | 解释 Cache-Control, Expires 的作用 |
| **协商缓存 (Negotiated Cache)** | 解释 ETag, Last-Modified 的工作机制 |
| **应用级缓存** | 解释前端代码层面的数据缓存 |
| **缓存失效** | 解释缓存何时需要更新 |
| **SWR (Stale-While-Revalidate)** | 解释先返回缓存再后台更新的策略 |

### 关键知识点

1. **HTTP 缓存机制**
   - 缓存头字段详解
   - 强缓存的判断流程
   - 协商缓存的请求流程
   - 缓存优先级

2. **Cache-Control 详解**
   - max-age
   - no-cache vs no-store
   - private vs public
   - must-revalidate
   - immutable

3. **前端应用级缓存**
   - 内存缓存 (Map)
   - localStorage/sessionStorage
   - IndexedDB
   - Cache API

4. **缓存策略设计**
   - Cache First：缓存优先
   - Network First：网络优先
   - Stale While Revalidate
   - Cache Only / Network Only

5. **缓存失效与更新**
   - 时间过期
   - 主动失效
   - 依赖失效
   - 版本号策略

6. **SWR 库设计思想**
   - 数据获取的生命周期
   - 缓存与重新验证
   - 乐观更新
