## 3. 内容要点

### 核心概念

| 概念 | 解释要求 |
|-----|---------|
| **实时通信** | 解释服务端与客户端的双向实时数据传输 |
| **轮询 (Polling)** | 解释定时请求服务器获取最新数据 |
| **长轮询 (Long Polling)** | 解释保持请求直到有新数据才返回 |
| **WebSocket** | 解释全双工的持久连接协议 |
| **SSE (Server-Sent Events)** | 解释服务端向客户端的单向推送 |

### 关键知识点

1. **实时通信的需求场景**
   - 即时消息
   - 实时通知
   - 协作编辑
   - 实时数据仪表盘

2. **轮询方案**
   - 短轮询的实现
   - 轮询间隔设计
   - 轮询的优缺点

3. **长轮询方案**
   - 长轮询的原理
   - 实现方式
   - 与短轮询的对比

4. **WebSocket 详解**
   - 握手过程
   - 数据帧格式
   - 连接管理
   - 心跳机制
   - 重连策略

5. **SSE 详解**
   - EventSource API
   - 数据格式
   - 自动重连
   - 与 WebSocket 的对比

6. **方案选型指南**
   - 各方案对比表
   - 不同场景推荐
   - 兼容性考虑
