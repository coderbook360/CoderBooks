## 2. 学习目标

### 知识目标
- 理解离线优先 (Offline First) 的设计理念
- 掌握网络状态检测的方式
- 了解 Service Worker 的离线缓存能力
- 理解离线数据同步的策略

### 技能目标
- 能够检测和响应网络状态变化
- 能够设计离线可用的应用架构
- 能够实现数据的离线存储与同步
- 能够处理离线冲突

## 3. 内容要点

### 核心概念

| 概念 | 解释要求 |
|-----|---------|
| **离线优先 (Offline First)** | 解释将离线视为常态而非异常的设计理念 |
| **网络状态检测** | 解释 navigator.onLine 和相关事件 |
| **Service Worker** | 解释浏览器后台脚本的离线缓存能力 |
| **离线队列** | 解释离线时将请求入队，联网后重放 |
| **冲突解决** | 解释离线修改与服务端数据冲突的处理 |

### 关键知识点

1. **离线优先的设计理念**
   - 为什么需要离线优先
   - 离线优先 vs 网络优先
   - 适用场景分析

2. **网络状态检测**
   - navigator.onLine API
   - online/offline 事件
   - Network Information API
   - 真实可用性检测

3. **Service Worker 离线缓存**
   - Service Worker 生命周期
   - Cache API 使用
   - 常见缓存策略
   - 预缓存与运行时缓存

4. **离线数据存储**
   - IndexedDB 使用
   - 数据本地持久化
   - 存储容量管理

5. **离线请求队列**
   - 请求的序列化
   - 队列的持久化
   - 联网后的重放

6. **数据同步与冲突**
   - 同步策略设计
   - 冲突检测
   - 冲突解决策略
   - 乐观锁与版本控制
