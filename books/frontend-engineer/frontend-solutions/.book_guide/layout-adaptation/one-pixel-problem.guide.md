## 3. 内容要点

### 核心概念

| 概念 | 解释要求 |
|-----|---------|
| **1px 问题** | 解释问题现象：设置 1px 边框在高 DPR 设备上显示较粗 |
| **CSS 像素** | 解释 CSS 像素是逻辑像素，与设备无关 |
| **物理像素** | 解释设备屏幕的实际像素点 |
| **设备像素比 (DPR)** | 解释 DPR = 物理像素 / CSS 像素的关系 |

### 关键知识点

1. **问题本质分析**
   - DPR 为 2 时，1 CSS 像素 = 2 物理像素
   - 为什么设计师要求的是"物理像素"的 1px
   - 不同 DPR 下的视觉差异

2. **解决方案一：transform scale**
   - 原理：先放大再缩小
   - 伪元素 + scale(0.5) 的实现
   - 适用于直线边框

3. **解决方案二：viewport scale**
   - 原理：整体缩放视口
   - initial-scale = 1/DPR 的配置
   - 对全局布局的影响

4. **解决方案三：border-image**
   - 使用渐变或图片作为边框
   - 实现半透明细边框
   - 兼容性与维护成本

5. **解决方案四：box-shadow**
   - 使用极小的阴影模拟边框
   - box-shadow: 0 0 0 0.5px 的技巧
   - 局限性分析

6. **解决方案五：SVG**
   - 使用 SVG 绘制细边框
   - 矢量图形的优势
   - 复杂形状的处理

7. **圆角边框的特殊处理**
   - transform scale 在圆角上的问题
   - 推荐的圆角 1px 方案
