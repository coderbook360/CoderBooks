# 章节写作指导：生产环境问题诊断

## 1. 章节信息

- **章节标题**: 生产环境问题诊断
- **文件名**: debugging/production-issue-diagnosis.md
- **所属部分**: 第七部分：调试与诊断
- **章节序号**: 50
- **预计阅读时间**: 30 分钟
- **难度等级**: 高级

## 2. 学习目标

### 知识目标
- 理解生产环境调试的特殊性
- 掌握问题诊断的系统方法
- 了解无侵入式调试技术
- 理解用户反馈的收集分析

### 技能目标
- 能够快速定位生产问题
- 能够使用错误监控数据
- 能够进行远程诊断
- 能够建立问题诊断流程

## 3. 内容要点

### 核心概念

| 概念 | 解释要求 |
|-----|---------|
| **生产调试** | 解释在生产环境进行问题诊断 |
| **无侵入式调试** | 解释不修改代码的调试方式 |
| **错误回溯** | 解释根据错误信息定位源头 |
| **复现环境** | 解释模拟生产问题的环境 |

### 关键知识点

1. **生产调试的挑战**
   - 代码经过压缩
   - 无法直接调试
   - 环境难以复现
   - 用户信息有限

2. **错误监控数据分析**
   - 错误聚合分析
   - 堆栈还原
   - 用户行为回放
   - 关联上下文

3. **问题复现策略**
   - 环境模拟
   - 数据模拟
   - 用户行为模拟
   - 灰度环境

4. **无侵入式调试**
   - 浏览器覆盖
   - 代理注入
   - 书签脚本
   - 扩展程序

5. **诊断流程建立**
   - 问题分类
   - 优先级排序
   - 责任分配
   - 复盘总结

6. **用户反馈处理**
   - 收集有效信息
   - 引导用户描述
   - 远程协助

## 4. 写作要求

### 开篇方式
用户反馈线上出问题，但开发环境无法复现。这是前端开发最头疼的场景，需要一套系统的诊断方法。

### 结构组织
```
1. 生产问题的诊断困境 (问题引入)
2. 从错误监控数据入手 (数据分析)
3. 问题复现策略 (复现方法)
4. 无侵入式调试技术 (调试技巧)
5. 诊断流程与规范 (流程建立)
6. 用户反馈处理 (信息收集)
7. 小结与检查清单
```

### 代码示例要求
- **必须包含**：书签脚本示例
- **必须包含**：代理注入示例
- **必须包含**：诊断信息收集代码
- **推荐包含**：完整诊断流程

### 图表需求
- 诊断流程图
- 问题分类矩阵
- 工具选择决策图
