## 3. 内容要点

### 核心概念
- **内部函数**: 不暴露给用户、仅供内部调用的工具函数
- **代码复用**: 通过抽象通用逻辑减少重复代码
- **分层设计**: 底层内部函数 → 中层通用函数 → 顶层公共 API

### 关键内部函数
| 函数名 | 作用 | 被调用场景 |
|--------|------|-----------|
| baseEach | 基础遍历 | forEach, map, filter 等 |
| baseIteratee | 迭代器工厂 | 所有接受 iteratee 参数的函数 |
| baseGet | 路径取值 | get, has, result 等 |
| baseSet | 路径赋值 | set, update 等 |
| baseFlatten | 基础展平 | flatten, flattenDeep 等 |
| baseClone | 基础克隆 | clone, cloneDeep 等 |
| castPath | 路径转换 | 所有路径相关函数 |
| isIndex | 索引判断 | 数组相关操作 |
| isKey | 键判断 | 属性访问相关 |

## 5. 技术细节

### 源码参考
- `.internal/baseEach.js`
- `.internal/baseIteratee.js`
- `.internal/baseGet.js`
- `.internal/baseFlatten.js`
- `.internal/castPath.js`

### 实现要点
- base* 函数通常只做最核心的逻辑，参数校验交给调用者
- 内部函数之间可以相互调用，形成分层
- 命名规范：base* 表示基础实现，cast* 表示类型转换

### 常见问题
- Q: 为什么不直接使用公共函数而要单独抽取内部函数？
- A: 减少参数校验的重复开销，内部调用时可跳过校验
