## 2. 学习目标

## 3. 内容要点

### 核心概念
- **服务端路由**：每个 URL 对应服务器上的一个资源，页面切换需要完整刷新
- **客户端路由**：URL 变化不刷新页面，由 JavaScript 控制视图更新
- **Hash 模式**：利用 `#` 后的哈希值，通过 `hashchange` 事件监听
- **History 模式**：利用 HTML5 History API（`pushState`/`replaceState`）

### 关键知识点
1. 传统多页应用（MPA）的路由方式
2. SPA 兴起与客户端路由的需求
3. Hash 模式的原理与局限
4. History API 的核心方法
5. 两种模式的对比与选择
