# 实战：三数之和

## 题目描述

给定一个整数数组 `nums`，找出所有和为 `0` 的三元组 `[nums[i], nums[j], nums[k]]`，且 `i ≠ j ≠ k`。

答案中不能包含重复的三元组。

📎 [LeetCode 15. 三数之和](https://leetcode.cn/problems/3sum/)

**示例**：

```
输入：nums = [-1, 0, 1, 2, -1, -4]
输出：[[-1, -1, 2], [-1, 0, 1]]
```

## 思路分析

### 这道题在考什么？

这道题的核心难点有两个：
1. 如何高效找到和为 0 的三元组？
2. 如何避免重复的三元组？

### 为什么用双指针？

如果用暴力三重循环，时间复杂度是 O(n³)，太慢了。

观察到：如果数组是**有序的**，我们可以用**对撞指针**将两数之和的查找从 O(n) 降到 O(1)。

**核心思路**：固定第一个数，对剩余部分使用对撞指针找两数之和。

### 解题思路

1. **排序**：先对数组排序，时间 O(n log n)
2. **遍历第一个数**：用 `i` 遍历第一个数
3. **双指针找剩余两数**：在 `[i+1, n-1]` 范围内用对撞指针
4. **去重**：跳过重复的数字

```
排序后：[-4, -1, -1, 0, 1, 2]

i=0, nums[i]=-4, 需要找和为 4 的两数
  left=1, right=5: -1+2=1 < 4, left++
  left=2, right=5: -1+2=1 < 4, left++
  left=3, right=5: 0+2=2 < 4, left++
  left=4, right=5: 1+2=3 < 4, left++
  left=5, 退出

i=1, nums[i]=-1, 需要找和为 1 的两数
  left=2, right=5: -1+2=1 = 1, 找到 [-1,-1,2]!
  ...
```

## 代码实现

```typescript
/**
 * LeetCode 15. 三数之和
 * 难度：中等
 * 技巧：排序 + 双指针
 * 
 * 思路：固定第一个数，对撞指针找剩余两数
 */
function threeSum(nums: number[]): number[][] {
  const result: number[][] = [];
  const n = nums.length;
  
  // 边界条件：少于 3 个数无解
  if (n < 3) return result;
  
  // Step 1: 排序
  nums.sort((a, b) => a - b);
  
  // Step 2: 遍历第一个数
  for (let i = 0; i < n - 2; i++) {
    // 剪枝：最小的数都大于 0，不可能有解
    if (nums[i] > 0) break;
    
    // 去重：跳过重复的第一个数
    if (i > 0 && nums[i] === nums[i - 1]) continue;
    
    // Step 3: 双指针找剩余两数
    let left = i + 1;
    let right = n - 1;
    const target = -nums[i]; // 需要找的两数之和
    
    while (left < right) {
      const sum = nums[left] + nums[right];
      
      if (sum === target) {
        // 找到一组解
        result.push([nums[i], nums[left], nums[right]]);
        
        // 去重：跳过重复的 left 和 right
        while (left < right && nums[left] === nums[left + 1]) left++;
        while (left < right && nums[right] === nums[right - 1]) right--;
        
        // 继续寻找下一组
        left++;
        right--;
      } else if (sum < target) {
        left++; // 和太小，左指针右移
      } else {
        right--; // 和太大，右指针左移
      }
    }
  }
  
  return result;
}
```

## 复杂度分析

- **时间复杂度**：O(n²)
  - 排序 O(n log n)
  - 外层遍历 O(n)，内层双指针 O(n)
  - 总体 O(n log n + n²) = O(n²)

- **空间复杂度**：O(log n)
  - 排序的递归栈空间
  - 不计算输出结果的空间

## 易错点

1. **忘记排序**：双指针的前提是有序数组
2. **去重遗漏**：
   - 第一个数的去重：`if (i > 0 && nums[i] === nums[i-1]) continue`
   - 第二、三个数的去重：找到答案后跳过相同元素
3. **剪枝不完整**：`nums[i] > 0` 时可以提前退出

## 举一反三

这道题的核心模式——**固定一个数，双指针找另外两个数**——可以推广到：

1. **K 数之和**：固定前 K-2 个数，最后用双指针
2. **最接近的 K 数之和**：同样的框架，判断条件改为更新最小差值
3. **有重复元素的组合问题**：排序 + 跳过重复元素的技巧是通用的
