# 实战：书籍分配

> 经典问题

将书籍分配给学生，使阅读量最大的学生阅读量最小。

---

## 题目描述

有 n 本书，第 i 本有 `pages[i]` 页。要将书分配给 k 个学生。

约束：
- 每本书必须完整分配给一个学生
- 每个学生至少分配一本书
- 书必须按顺序分配（连续的书给同一个学生）

求分配方案中，**最大阅读页数的最小值**。

**示例**：
```
输入：pages = [12, 34, 67, 90], k = 2
输出：113
解释：
方案1: [12, 34, 67] 和 [90] → 最大 113
方案2: [12, 34] 和 [67, 90] → 最大 157
最小的最大值是 113
```

---

## 思路分析

这与"分割数组的最大值"完全相同。

- 答案空间：[max(pages), sum(pages)]
- 单调性：允许的最大页数越大，所需学生越少
- check(maxPages)：判断每人最多读 maxPages 页时能否分给 k 人

---

## 代码实现

```typescript
function allocateBooks(pages: number[], k: number): number {
  const n = pages.length;
  
  // 如果学生比书多，无法分配
  if (k > n) return -1;
  
  let left = Math.max(...pages);
  let right = pages.reduce((a, b) => a + b, 0);
  
  while (left < right) {
    const mid = left + Math.floor((right - left) / 2);
    
    if (canAllocate(pages, mid, k)) {
      right = mid;
    } else {
      left = mid + 1;
    }
  }
  
  return left;
}

function canAllocate(pages: number[], maxPages: number, k: number): boolean {
  let students = 1;
  let currentPages = 0;
  
  for (const p of pages) {
    if (currentPages + p > maxPages) {
      students++;
      currentPages = p;
    } else {
      currentPages += p;
    }
  }
  
  return students <= k;
}
```

## 执行过程详解

以 `pages = [12, 34, 67, 90], k = 2` 为例，详细跟踪每一步：

**初始化**：
- left = max(12, 34, 67, 90) = 90
- right = 12 + 34 + 67 + 90 = 203

**第一轮：mid = 146**
```
遍历分配过程：
  学生1: 0页
    - 加12 → 12页 (≤146)
    - 加34 → 46页 (≤146)  
    - 加67 → 113页 (≤146)
    - 加90 → 203页 (>146) → 分配给下一个学生
  学生2: 90页
  
  学生数 = 2 ≤ 2，可行！
  right = 146
```

**第二轮：mid = 118**
```
遍历分配过程：
  学生1: 0页
    - 加12 → 12页 (≤118)
    - 加34 → 46页 (≤118)
    - 加67 → 113页 (≤118)
    - 加90 → 203页 (>118) → 分配给下一个学生
  学生2: 90页
  
  学生数 = 2 ≤ 2，可行！
  right = 118
```

**第三轮：mid = 104**
```
遍历分配过程：
  学生1: 0页
    - 加12 → 12页 (≤104)
    - 加34 → 46页 (≤104)
    - 加67 → 113页 (>104) → 分配给下一个学生
  学生2: 67页
    - 加90 → 157页 (>104) → 分配给下一个学生
  学生3: 90页
  
  学生数 = 3 > 2，不可行！
  left = 105
```

**第四轮：mid = 111**
```
遍历分配过程：
  学生1: 12 + 34 = 46页 (≤111)
    - 加67 → 113页 (>111) → 分配给下一个学生
  学生2: 67页
    - 加90 → 157页 (>111) → 分配给下一个学生
  学生3: 90页
```

**第五轮：mid = 115**
```
学生1: 12 + 34 + 67 = 113页 (≤115)

学生2: 90页

学生数 = 2 ≤ 2，可行！
right = 115
```

**继续二分**，最终收敛到 `left = right = 113`。

## 复杂度分析

- **时间复杂度**：O(n log S)，S = sum(pages)
  - 二分次数：O(log S)
  - 每次 check：O(n)
  
- **空间复杂度**：O(1)，只用常数额外空间

## 常见错误分析

### 错误1：左边界设置错误

```typescript
// ❌ 错误：左边界设为0或1
let left = 0;

// ✅ 正确：左边界是最大单本页数
let left = Math.max(...pages);
```

**原因**：每个学生至少分配一本书，所以最大阅读量至少是最大单本页数。

### 错误2：忘记处理 k > n 的情况

```typescript
// ❌ 遗漏边界检查
function allocateBooks(pages: number[], k: number): number {
  // 直接开始二分...
}

// ✅ 正确：先检查是否可能分配
function allocateBooks(pages: number[], k: number): number {
  if (k > pages.length) return -1;
  // ...
}
```

**原因**：如果学生数超过书本数，每人至少一本就无法满足。

### 错误3：check 函数中学生计数初始值

```typescript
// ❌ 错误：从0开始计数
let students = 0;

// ✅ 正确：从1开始（第一个学生）
let students = 1;
```

**原因**：即使不需要分割，也至少有一个学生。

### 错误4：分割时机判断错误

```typescript
// ❌ 错误：先加再判断会导致越界
currentPages += p;
if (currentPages > maxPages) {
  students++;
  currentPages = p;  // 这里 currentPages 已经包含了 p
}

// ✅ 正确：先判断再加
if (currentPages + p > maxPages) {
  students++;
  currentPages = p;
} else {
  currentPages += p;
}
```

## 与相关问题的对比

### "最小化最大值"问题家族

| 问题 | 答案含义 | 左边界 | 右边界 |
|------|----------|--------|--------|
| 书籍分配 | 最大阅读页数 | max(pages) | sum(pages) |
| 分割数组 | 最大子数组和 | max(nums) | sum(nums) |
| 运送包裹 | 船运载能力 | max(weights) | sum(weights) |
| 切木头 | 木头段数（反向） | 1 | max(lengths) |

**共同模式**：
1. 答案越大，分段/分组越少
2. check 函数贪心地分配
3. 使用"找左边界"的二分模板

### 与动态规划解法对比

```
二分答案：O(n log S) 时间，O(1) 空间
动态规划：O(n²k) 时间，O(nk) 空间

当 n 较大时，二分答案更高效
```

## 变体问题

### 变体1：不要求连续分配

如果书可以不连续分配（任意挑选），问题变成了 NP-hard，需要用贪心近似或回溯。

### 变体2：学生有能力限制

如果每个学生有不同的阅读能力上限，需要修改 check 函数，匹配书本和学生。

---

## 总结

书籍分配问题是"最小化最大值"二分答案的典型代表：

1. **识别模式**：看到"最大值最小"就想到二分
2. **确定边界**：[max(单项), sum(全部)]
3. **设计 check**：贪心分配，统计所需数量
4. **模板选择**：找左边界（第一个可行解）

掌握这道题后，船运包裹、分割数组等问题都能迎刃而解。
