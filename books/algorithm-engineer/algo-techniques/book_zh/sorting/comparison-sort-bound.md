# 比较排序的下界

为什么所有基于比较的排序算法，最坏情况都不能快于 O(n log n)？这是一个深刻的理论问题。

---

## 决策树模型

**比较排序 = 决策树**：
- 每次比较是一个决策节点
- 叶子节点对应一种排列
- 树的高度 = 最坏情况比较次数

**示例**：对 [a, b, c] 排序

```
          a:b
         /   \
       <       ≥
      /         \
   b:c         a:c
   / \         / \
  <   ≥      <   ≥
 /     \    /     \
abc   acb bac    bca
              \
               b:c
               / \
              <   ≥
             /     \
           cab     cba
```

---

## 下界证明

**定理**：任何比较排序算法，最坏情况至少需要 ⌈log₂(n!)⌉ 次比较。

**证明**：

1. **排列数量**：n 个元素有 n! 种排列
2. **叶子节点数**：决策树至少有 n! 个叶子
3. **树的高度**：高度为 h 的二叉树最多有 2ʰ 个叶子
4. **推导**：
   ```
   2ʰ ≥ n!
   h ≥ log₂(n!)
   ```

5. **斯特林公式**：
   ```
   log₂(n!) = log₂(1) + log₂(2) + ... + log₂(n)
            ≈ ∫₁ⁿ log₂(x) dx
            = n log₂(n) - n/ln(2) + O(log n)
            = Θ(n log n)
   ```

**结论**：**任何比较排序最坏情况至少需要 Ω(n log n) 次比较**。

---

## 直观理解

**信息论视角**：
- 排序前：数据处于 n! 种可能状态之一（熵 = log₂(n!)）
- 每次比较：提供 1 bit 信息（二分决策）
- 排序后：确定唯一状态（熵 = 0）
- 所需比较次数：log₂(n!)

**通俗解释**：

你需要猜一个1到n!之间的数字（代表某种排列），每次只能问"是/否"问题（比较），最少需要问多少次？

答案：log₂(n!)  ≈ n log n 次

---

## 常见排序算法的比较次数

| 算法 | 平均 | 最坏 | 是否最优 |
|-----|------|------|---------|
| **归并排序** | n log n | n log n | **是** |
| **快速排序** | n log n | n² | **否**（最坏） |
| **堆排序** | n log n | n log n | **是** |
| **插入排序** | n² | n² | **否** |

**归并和堆排序达到了理论下界！**

---

## 如何突破 O(n log n)？

**答案**：不使用比较！

**非比较排序**：
1. **计数排序**：统计每个值的出现次数
2. **桶排序**：将元素分配到桶中
3. **基数排序**：按位排序

**前提条件**：
- 数据范围有限（计数、桶）
- 数据有特定结构（基数）

**时间复杂度**：
- 计数排序：O(n + k)，k = 数据范围
- 桶排序：O(n)（理想情况）
- 基数排序：O(d · n)，d = 位数

---

## 实际意义

**理论指导实践**：

1. **不要追求 O(n) 的比较排序**：
   - 不存在这样的算法
   - 如果有人声称发明了，一定有问题

2. **优化方向**：
   - 减少常数因子
   - 优化缓存局部性
   - 减少分支预测失败

3. **选择合适算法**：
   - 通用场景：快排（平均快）
   - 需要稳定：归并
   - 数据特殊：非比较排序

---

## 扩展：平均情况下界

**定理**：基于比较的排序，平均情况也需要 Ω(n log n) 次比较。

**证明思路**：
- 考虑所有 n! 种输入排列
- 每种排列的比较次数 ≥ 某个决策树路径长度
- 平均路径长度 ≥ log₂(n!)

---

## 历史趣闻

**Knuth 的《计算机程序设计艺术》**：

> "排序和查找占据了计算机80%的时间。"（1970年代）

虽然现在比例有所下降，但排序依然是基础中的基础。

**排序算法的演进**：
- 1940s: 归并排序（冯·诺依曼）
- 1960s: 快速排序（C.A.R. Hoare）
- 1964: 堆排序（Williams）
- 2002: TimSort（Python的排序算法）

---

## 关键要点

1. **理论下界**：比较排序最坏 Ω(n log n)
2. **证明工具**：决策树、信息论
3. **达到下界**：归并、堆排序
4. **突破方法**：非比较排序
5. **实践意义**：指导算法选择
