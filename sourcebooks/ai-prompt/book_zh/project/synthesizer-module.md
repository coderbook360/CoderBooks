## 24. 综合器模块：从原始信息中提炼行程

至此，我们已经拥有了由规划器绘制的精确“蓝图”，以及由执行器搜集回来的、堆积如山的“原材料”（航班列表、酒店信息、景点介绍等）。这些原材料是原始的、冰冷的、机器可读的数据。如果直接把它们丢给用户，无疑是一场灾难。

现在，舞台将交给我们的“金牌行程规划师”——综合器（Synthesizer）。它的使命，是将这些零散、枯燥的数据，“编织”成一篇有温度、有逻辑、有故事的每日行程。

### 综合器的角色：从数据到故事的“魔术师”

-   **定位**：团队的“主笔”和“故事讲述者”。
-   **输入**：
    1.  **全局旅行计划 (Full Plan)**：规划器生成的原始 JSON 蓝图。
    2.  **执行结果库 (Execution Results)**：一个包含了所有任务执行结果的数据库（通常是一个以 `task_id` 为键的字典或 JSON 对象）。
-   **核心任务**：它不是简单的“复制+粘贴”，而是真正的“综合”与“创作”。具体工作包括：
    -   **提炼关键信息**：从执行器返回的一大堆 JSON 数据中，提取出用户最关心的核心信息，如航班号、酒店名、价格、地址等。
    -   **增加叙事性**：用流畅的语言将一天的活动串联起来，讲述一个连贯的“旅行故事”。
    -   **添加实用建议**：在行程中加入贴心的小提示，如“建议提前半小时出门，避开早高峰”、“这家餐厅的招牌菜是……”，提升行程的附加值。
    -   **统一口吻与风格**：确保最终的文字风格是亲切、诱人、且一致的。
-   **输出**：一份详细的、可读性极强的、Markdown 格式的每日行程描述。

让我们再次通过“输入 vs 输出”的对比，感受综合器的魔力：

**输入 (Input):**

```json
// 全局计划的一部分
{
  "day": 1,
  "city": "巴黎",
  "theme": "抵达与艺术初探",
  "tasks": [
    {"task_id": "day1-flight-search", ...},
    {"task_id": "day1-hotel-search", ...}
  ]
}

// 执行结果库的一部分
{
  "day1-flight-search": {
    "result": [{"airline": "Air France", "flight_no": "AF277", "price": 7800, "duration": "11h"}]
  },
  "day1-hotel-search": {
    "result": [{"name": "Hôtel du Louvre", "rating": 4.8, "address": "Place André Malraux"}]
  }
}
```

**输出 (Output - 有温度的行程):**

```markdown
### **第一天：抵达巴黎，开启艺术之旅**

**主题：抵达与艺术初探**

您将乘坐法国航空的 **AF277** 航班，在阔别已久的蓝天上飞行约11小时后，降落在浪漫之都——巴黎。我们为您精心挑选了位于卢浮宫广场的 **Hôtel du Louvre** 酒店，它本身就是一件艺术品。办理入住后，您可以稍作休整，下午我们将直接步入艺术的殿堂——卢浮宫，与蒙娜丽莎进行一场跨越时空的对话。

**温馨提示**：长途飞行后，建议您在酒店小憩片刻，以最佳状态迎接接下来的艺术盛宴。
```

### 综合器提示词的艺术：赋予数据以灵魂

综合器的提示词设计，重点在于引导模型进行“创造性写作”，而非“信息搬运”。

```
你是一位经验丰富的旅行作家和资深编辑，以优美、引人入胜的文笔而闻名。
你的任务是将一份结构化的旅行数据，转化为一篇生动、详实、充满吸引力的每日行程指南。

**全局旅行计划如下:**
<full_travel_plan>
{{full_plan_json}}
</full_travel_plan>

**所有任务的执行结果（原始数据）如下:**
<execution_results>
{{execution_results_json}}
</execution_results>

**当前需要你撰写的，是第 {{day_number}} 天的行程。**

请你完成以下工作：
1.  **提炼与整合**：仔细阅读第 {{day_number}} 天的计划和所有相关的原始数据，提取出最核心、最吸引人的信息。
2.  **故事化叙述**：将当天的航班、酒店、活动、餐厅等信息，用流畅的语言串联成一个完整的故事。不要只是罗列数据。
3.  **增加亮点**：为行程添加一些贴心的小提示、有趣的背景知识或吸引人的描述，让行程看起来更诱人。
4.  **格式化输出**：使用清晰的 Markdown 格式，包括标题、粗体、列表等，让内容易于阅读。

请直接输出撰写好的第 {{day_number}} 天的 Markdown 行程内容。

```

#### 提示词设计解析

-   **Persona (角色)**: “经验丰富的旅行作家和资深编辑”，强调其“写作”能力而非“数据处理”能力。
-   **Instruction (指令)**: 核心指令是“将结构化的旅行数据，转化为一篇生动的行程指南”。我们明确要求它“故事化叙述”、“增加亮点”，避免它简单地罗列数据。
-   **Context (情境)**: 提示词同时提供了“蓝图”（`full_travel_plan`）、“原材料”（`execution_results`）以及“当前焦点”（`day_number`），为模型提供了最全面的创作背景。

### “垃圾进，垃圾出”原则

综合器虽然强大，但它不是无中生有的魔术师。它的产出质量，高度依赖于前两个环节的质量。

-   如果**规划器**的蓝图逻辑混乱（例如，让用户在一天内往返两个相距甚远的城市）。
-   如果**执行器**获取的数据有误或不完整（例如，未能找到合适的航班，或返回了错误的酒店信息）。

那么，无论综合器的提示词多么精妙，它也无法“烹饪”出一道美味的大餐。这再次印证了，一个高质量的 AI 系统，必然是一个环环相扣、每个环节都足够健壮的整体。

至此，我们已经拥有了一份份精美的每日行程。但它们还像散落的珍珠。在下一章，我们将介绍“报告生成器”，看看它如何将这些珍珠串成一条完整的项链，并以最华丽的方式呈现给用户。
