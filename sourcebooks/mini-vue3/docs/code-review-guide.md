# Code Review 指南

> 每周日进行 Code Review，确保代码质量和深度理解

## 📋 Code Review 检查清单

### 1️⃣ 功能正确性
- [ ] 所有测试用例是否通过？
- [ ] 是否处理了边界情况？
- [ ] 是否有潜在的 Bug？
- [ ] 功能实现是否完整？

### 2️⃣ 代码质量

#### 命名规范
- [ ] 变量名是否语义化？（avoid `a`, `b`, `temp`）
- [ ] 函数名是否清楚表达意图？（动词开头）
- [ ] 常量是否使用大写？（`MAX_SIZE`）
- [ ] 私有变量/函数是否有标识？（`_internal`）

#### 代码风格
- [ ] 是否符合 ESLint 规范？
- [ ] 是否有一致的缩进和格式？
- [ ] 是否有不必要的空行或注释？
- [ ] 是否有 console.log 或调试代码需要清理？

#### DRY 原则（Don't Repeat Yourself）
- [ ] 是否有重复代码可以抽取为函数？
- [ ] 是否有相似逻辑可以统一处理？
- [ ] 是否合理使用了工具函数？

### 3️⃣ 性能考虑
- [ ] 是否有不必要的循环或递归？
- [ ] 是否有 O(n²) 可优化为 O(n)？
- [ ] 是否有重复计算可以缓存？
- [ ] 是否有内存泄漏风险？（事件监听、闭包）

### 4️⃣ 类型安全（TypeScript）
- [ ] 类型定义是否完善？
- [ ] 是否滥用 `any` 类型？
- [ ] 是否有类型推导问题？
- [ ] 是否需要添加泛型约束？

### 5️⃣ 错误处理
- [ ] 是否处理了可能的错误？
- [ ] 错误信息是否友好清晰？
- [ ] 是否有合适的错误恢复机制？
- [ ] 是否需要添加警告日志？

### 6️⃣ 可测试性
- [ ] 函数是否容易测试？
- [ ] 是否有合理的依赖注入？
- [ ] 是否有副作用难以控制？
- [ ] 测试覆盖率是否充分？

### 7️⃣ 文档和注释
- [ ] 复杂逻辑是否有注释说明？
- [ ] 公共 API 是否有 JSDoc 文档？
- [ ] 是否说明了"为什么"而不只是"做什么"？
- [ ] 是否有必要的 TODO 或 FIXME 标记？

### 8️⃣ 架构设计
- [ ] 模块划分是否合理？
- [ ] 单一职责原则是否遵守？
- [ ] 是否有适当的抽象层次？
- [ ] 是否便于未来扩展？

## 🔍 每周 Code Review 流程

### Week 1-3: 基础夯实期

**重点关注**：
- ✅ 理解每一行代码的作用
- ✅ 命名和注释的规范性
- ✅ 测试用例的完整性

**自查问题**：
1. 我能向别人清楚解释这段代码吗？
2. 如果 3 个月后再看，我还能快速理解吗？
3. 如果要添加新功能，现有结构是否支持？

### Week 4-7: 响应式系统期

**重点关注**：
- ✅ 依赖收集的准确性
- ✅ 内存泄漏风险
- ✅ 边界情况的处理
- ✅ 性能优化点

**对比官方源码**：
- 数据结构设计差异
- 算法实现差异
- 优化技巧学习

### Week 8-12: 运行时核心期

**重点关注**：
- ✅ Diff 算法的效率
- ✅ 组件生命周期的完整性
- ✅ 内存管理
- ✅ 复杂场景测试

**性能基准测试**：
- 对比官方 Vue 的性能
- 找出性能瓶颈
- 优化关键路径

### Week 13-16: 编译器期

**重点关注**：
- ✅ AST 转换的正确性
- ✅ 错误提示的友好性
- ✅ 编译优化的有效性
- ✅ 生成代码的质量

### Week 17-20: 高级特性期

**重点关注**：
- ✅ 特性的完整性
- ✅ API 设计的一致性
- ✅ 异常处理的健壮性
- ✅ 与现有系统的集成

### Week 21-24: 优化与工程化期

**重点关注**：
- ✅ 打包体积优化
- ✅ 运行时性能优化
- ✅ 文档的完整性
- ✅ 整体架构的合理性

## 🔧 Code Review 工具

### VS Code 插件推荐
- **ESLint**: 代码规范检查
- **Prettier**: 代码格式化
- **Code Spell Checker**: 拼写检查
- **Better Comments**: 注释高亮
- **TODO Highlight**: TODO 标记高亮

### 在线工具
- **AST Explorer**: 查看 AST 结构
- **TypeScript Playground**: 测试 TS 代码
- **Visualizer**: 可视化数据结构和算法

## 💪 持续改进

### 每周一个小目标
- Week 1: 掌握规范的代码风格
- Week 2: 提高测试覆盖率到 90%+
- Week 3: 优化一个性能瓶颈
- Week 4: 重构一个复杂模块
- ...

### 每月技术分享
- 月末准备一次 20-30 分钟的技术分享
- 可以是对某个模块的深度解析
- 也可以是与官方源码的对比分析
- 或者是踩坑经验总结
