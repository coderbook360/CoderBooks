### 1.1 核心模块回顾

1.  **Reactive / Ref**：
    -   **作用**：数据的守门员。
    -   **原理**：利用 `Proxy` (对象) 和 `get/set` (基本类型) 拦截数据的读写操作。
    -   **关键点**：懒代理、深度响应、Ref 解包。

2.  **Track / Trigger**：
    -   **作用**：系统的神经中枢。
    -   **原理**：`track` 在读取时收集依赖，`trigger` 在修改时触发依赖。
    -   **关键点**：`targetMap` 三层结构 (WeakMap -> Map -> Set)。

3.  **ReactiveEffect**：
    -   **作用**：副作用的执行单元。
    -   **原理**：包装用户函数，维护 `activeEffect` 全局变量，管理依赖的清理 (`cleanup`)。
    -   **关键点**：双向记录、嵌套 Effect 处理、Stop 机制。

4.  **Scheduler**：
    -   **作用**：性能优化器。
    -   **原理**：利用微任务 (`Promise`) 实现批量更新，去重任务。
    -   **关键点**：`queueJob`、`flushJobs`、`nextTick`。

5.  **Computed / Watch**：
    -   **作用**：高级应用。
    -   **原理**：基于 `ReactiveEffect` 的封装。`computed` 利用 `lazy` 和 `dirty` 实现缓存；`watch` 利用 `scheduler` 实现回调。

### Q1: 为什么 Vue 3 用 Proxy 替代了 Object.defineProperty？

<details>
<summary>点击查看答案</summary>

1.  **功能更强**：Proxy 可以拦截 13 种操作（包括 `delete`、`in`、`ownKeys` 等），而 `defineProperty` 只能拦截读写。
2.  **支持数组**：Proxy 可以完美支持数组的索引修改和 `length` 变化，不需要像 Vue 2 那样重写数组方法（虽然为了性能我们还是做了一些优化）。
3.  **性能更好（初始化）**：Proxy 是懒代理，只有访问属性时才代理；`defineProperty` 需要递归遍历所有属性，初始化开销大。
4.  **支持 Map/Set**：Proxy 可以代理 Map、Set 等集合类型。

</details>

### Q2: 为什么 computed 需要 dirty 标记？

<details>
<summary>点击查看答案</summary>

为了实现**缓存**和**惰性计算**。
如果不加 dirty 标记，每次访问 computed 都会重新执行计算函数，这和普通函数没区别。
有了 dirty 标记：
1.  依赖变了，只把 dirty 设为 true，**不计算**（惰性）。
2.  访问时，如果 dirty 为 false，直接返回旧值，**不计算**（缓存）。
只有当 dirty 为 true 且被访问时，才真正计算。

</details>

### Q3: 为什么 watchEffect 需要 onCleanup？

<details>
<summary>点击查看答案</summary>

为了解决**竞态问题**和**资源泄漏**。
比如在 `watchEffect` 里发起网络请求。如果数据变化很快，上一个请求还没回来，下一个请求又发出去。
我们需要在发新请求前，取消掉上一个请求。
`onCleanup` 提供了一个钩子，让我们在副作用重新执行前（或停止时）做一些清理工作。

</details>
