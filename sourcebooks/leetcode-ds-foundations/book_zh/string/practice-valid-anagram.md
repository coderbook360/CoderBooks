# 实战：有效的字母异位词

"anagram" 和 "nagaram" 是不是同一组字母的不同排列？这道题考查的就是**字母异位词**的判断。

首先要问一个问题：什么是字母异位词？

简单说：两个字符串包含**完全相同的字母**，且每个字母出现的**次数也相同**，只是排列顺序不同。

## 题目描述

> **LeetCode 242. 有效的字母异位词**
>
> 给定两个字符串 s 和 t，编写一个函数来判断 t 是否是 s 的字母异位词。
>
> 注意：若 s 和 t 中每个字符出现的次数都相同，则称 s 和 t 互为字母异位词。

**示例**：

```
输入: s = "anagram", t = "nagaram"
输出: true

输入: s = "rat", t = "car"
输出: false
```

**进阶**：如果输入字符串包含 Unicode 字符怎么办？

## 解法一：排序比较

最直观的思路：如果两个字符串是异位词，那么排序后应该**完全相同**。

```javascript
function isAnagram(s, t) {
    if (s.length !== t.length) return false;
    
    return [...s].sort().join('') === [...t].sort().join('');
}
```

简洁，但有代价。

**复杂度分析**：
- **时间复杂度**：O(n log n)，排序的开销
- **空间复杂度**：O(n)，创建了新数组

现在我要问一个问题：能不能做到 O(n)？

## 解法二：字符计数（数组）

核心思路：**统计每个字符出现的次数**。如果两个字符串是异位词，那么每个字符的出现次数应该相同。

对于仅包含小写字母的情况，我们可以用一个长度为 26 的数组来计数：

```javascript
function isAnagram(s, t) {
    if (s.length !== t.length) return false;
    
    // 26 个字母的计数数组
    const count = new Array(26).fill(0);
    
    for (let i = 0; i < s.length; i++) {
        // s 中的字符 +1，t 中的字符 -1
        count[s.charCodeAt(i) - 97]++;
        count[t.charCodeAt(i) - 97]--;
    }
    
    // 如果是异位词，所有计数应该为 0
    return count.every(c => c === 0);
}
```

### 执行过程追踪

以 `s = "anagram", t = "nagaram"` 为例：

```
初始：count = [0, 0, 0, ..., 0]（26 个 0）

i=0: s[0]='a', t[0]='n'
     count['a'-'a']++ → count[0]=1
     count['n'-'a']-- → count[13]=-1

i=1: s[1]='n', t[1]='a'
     count[13]++ → count[13]=0
     count[0]-- → count[0]=0

i=2: s[2]='a', t[2]='g'
     count[0]++ → count[0]=1
     count[6]-- → count[6]=-1

...（继续遍历）

最终 count 数组全为 0，返回 true
```

**技巧解析**：我们用一个数组同时统计 s 和 t。s 中的字符 +1，t 中的字符 -1。如果最终全为 0，说明两边字符完全抵消——即是异位词。

### 复杂度分析

- **时间复杂度**：O(n)，一次遍历
- **空间复杂度**：O(1)，固定 26 个元素的数组

这比排序快多了。但有个限制：只适用于**小写字母**。

## 解法三：字符计数（Map）

如果要支持 Unicode 字符，就不能用固定大小的数组了。改用 Map：

```javascript
function isAnagram(s, t) {
    if (s.length !== t.length) return false;
    
    const count = new Map();
    
    // 统计 s 中每个字符的出现次数
    for (const char of s) {
        count.set(char, (count.get(char) || 0) + 1);
    }
    
    // 用 t 中的字符去抵消
    for (const char of t) {
        if (!count.has(char) || count.get(char) === 0) {
            return false;  // 字符不存在或已用完
        }
        count.set(char, count.get(char) - 1);
    }
    
    return true;
}
```

这个版本的逻辑略有不同：
1. 先统计 s 中所有字符
2. 遍历 t，每遇到一个字符就 -1
3. 如果遇到不存在或已用完的字符，直接返回 false

**为什么不需要最后检查所有计数为 0？** 因为我们已经确保 `s.length === t.length`，如果 t 能完全抵消 s 的计数，那必然是异位词。

### 复杂度分析

- **时间复杂度**：O(n)
- **空间复杂度**：O(k)，k 是字符集大小

## 三种方法对比

| 方法 | 时间复杂度 | 空间复杂度 | Unicode 支持 | 适用场景 |
|-----|-----------|-----------|-------------|---------|
| 排序 | O(n log n) | O(n) | ✅ | 代码最简洁 |
| 数组计数 | O(n) | O(1) | ❌ | 仅小写字母，最高效 |
| Map 计数 | O(n) | O(k) | ✅ | 通用场景 |

**选择建议**：
- 题目明确只有小写字母 → 数组计数
- 需要支持 Unicode → Map 计数
- 追求代码简洁，性能不敏感 → 排序

## 常见错误

1. **忘记检查长度**：长度不同必定不是异位词，先判断可以快速返回
2. **数组下标越界**：`charCodeAt(i) - 97` 只适用于小写字母，大写或其他字符会出错
3. **空字符串**：两个空字符串互为异位词，代码应正确处理

## 本章小结

判断字母异位词的核心是**字符频率统计**：

- 排序后比较是最直观的方法，但效率不是最优
- 数组计数适用于有限字符集（如 26 个字母），时间 O(n)，空间 O(1)
- Map 计数是通用方案，支持任意字符

"统计频率"是字符串处理中的常见模式，后续还会在滑动窗口、分组等问题中频繁使用。
