# 实战：无重叠区间

这是区间贪心的经典问题。

## 问题描述

给定一个区间的集合`intervals`，其中`intervals[i] = [starti, endi]`。

返回需要移除区间的**最小数量**，使剩余区间互不重叠。

## 思路分析

### 问题转化

"移除最少使不重叠" = "保留最多的不重叠区间"

这就是经典的**活动选择问题**。

### 贪心策略

按结束时间排序，优先选择结束时间早的区间。

为什么？结束越早，留给后面区间的空间越大。

## 代码实现

```javascript
function eraseOverlapIntervals(intervals) {
    if (intervals.length === 0) return 0;
    
    // 按结束时间排序
    intervals.sort((a, b) => a[1] - b[1]);
    
    let count = 1;  // 保留的区间数
    let end = intervals[0][1];  // 上一个保留区间的结束时间
    
    for (let i = 1; i < intervals.length; i++) {
        if (intervals[i][0] >= end) {
            // 不重叠，可以保留
            count++;
            end = intervals[i][1];
        }
        // 重叠的区间被"移除"（不更新count和end）
    }
    
    return intervals.length - count;
}
```

## 图解

```
intervals = [[1,2], [2,3], [3,4], [1,3]]

按结束时间排序: [[1,2], [2,3], [1,3], [3,4]]

[1,2]: 保留, end=2
[2,3]: start=2 >= end=2, 保留, end=3
[1,3]: start=1 < end=3, 移除
[3,4]: start=3 >= end=3, 保留, end=4

保留3个，移除1个
```

## 为什么按结束时间排序？

### 交换论证

假设最优解的第一个区间是A，结束时间是`a_end`。

贪心解的第一个区间是B，结束时间是`b_end <= a_end`。

如果把A换成B：
- B结束更早，不会与后面的区间产生新的冲突
- 所以解不会变差

因此，贪心选择结束最早的区间是正确的。

### 直觉理解

结束越早，意味着：
- 占用的时间段越短
- 留给后面的选择越多
- 更有可能容纳更多区间

## 按开始时间排序行吗？

不行。反例：

```
[[1,10], [2,3], [3,4], [4,5]]

按开始时间排序后选[1,10]，后面都冲突，只能选1个。
但最优解是选[2,3], [3,4], [4,5]，共3个。
```

## 边界处理

### 区间端点相等

`[1,2]`和`[2,3]`不算重叠（题目定义）。

代码中用`intervals[i][0] >= end`而不是`>`来判断。

### 空数组

返回0。

## 复杂度分析

**时间复杂度**：O(n log n)，主要是排序

**空间复杂度**：O(log n)，排序的栈空间

## 与活动选择的对比

| 活动选择 | 无重叠区间 |
|---------|-----------|
| 最多能参加几个活动 | 最少移除几个区间 |
| 直接求"保留"数量 | 用总数减去"保留"数量 |

本质上是同一个问题。

## 小结

无重叠区间是区间贪心的代表问题：
1. **按结束时间排序**是关键
2. 贪心选择结束最早的区间
3. 本质是活动选择问题的变形
