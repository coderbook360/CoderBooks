# 实战：加油站

这道题展示了贪心在环形问题中的应用。

## 问题描述

在一条环路上有`n`个加油站，其中第`i`个加油站有汽油`gas[i]`升。

你有一辆油箱容量无限的汽车，从第`i`个加油站开往第`i+1`个加油站需要消耗汽油`cost[i]`升。

你从其中的一个加油站出发，开始时油箱为空。给定两个整数数组`gas`和`cost`，如果你可以绑定环路行驶一周，则返回出发时加油站的编号，否则返回`-1`。

如果题目有解，该答案即为**唯一**答案。

## 思路分析

### 暴力思路

从每个站点出发，模拟跑一圈，O(n²)。

### 贪心洞察1：总量判断

如果`sum(gas) < sum(cost)`，肯定无解——油不够跑完一圈。

如果`sum(gas) >= sum(cost)`，一定有解。

### 贪心洞察2：起点选择

如果从站点`i`出发，在站点`j`油箱变空（无法继续），那么：
- 从`i`到`j`之间的任何站点出发，都无法到达`j`
- 应该从`j+1`重新开始尝试

为什么？因为从`i`能到达`i+1`、`i+2`...直到`j`之前，说明这些中间站点的"净收益"是正的。但即使带着这些正收益，到`j`还是不行，那从中间站点出发（没有之前的正收益）更不可能到达`j`。

## 代码实现

```javascript
function canCompleteCircuit(gas, cost) {
    const n = gas.length;
    let totalTank = 0;
    let currTank = 0;
    let start = 0;
    
    for (let i = 0; i < n; i++) {
        totalTank += gas[i] - cost[i];
        currTank += gas[i] - cost[i];
        
        if (currTank < 0) {
            // 从i出发走不到i+1，重置起点为i+1
            start = i + 1;
            currTank = 0;
        }
    }
    
    return totalTank >= 0 ? start : -1;
}
```

## 图解

```
gas  = [1, 2, 3, 4, 5]
cost = [3, 4, 5, 1, 2]

净收益 = [-2, -2, -2, 3, 3]
总和 = 0，有解

i=0: currTank = -2 < 0, start=1
i=1: currTank = -2 < 0, start=2
i=2: currTank = -2 < 0, start=3
i=3: currTank = 3
i=4: currTank = 3 + 3 = 6

totalTank = 0 >= 0, 返回 start = 3
```

## 正确性证明

### 为什么跳过中间站点是对的？

假设从站点`i`出发，在站点`j`之前油箱都不为负，但到`j`时为负。

对于任意中间站点`k`（i < k < j）：
- 从`i`到`k`，油箱是非负的（否则早就停了）
- 所以从`i`到`k`的净收益是非负的

如果从`k`出发：
- 失去了`i`到`k`的非负净收益
- 后面的路程不变
- 所以更容易变负

因此，从`k`出发也无法到达`j`，可以跳过。

### 为什么从`start`出发一定能成功？

因为`totalTank >= 0`，说明总油量够跑完一圈。

如果最终`start`确定后，说明从`start`到末尾的净收益是非负的。

而`totalTank >= 0`意味着整圈的净收益是非负的，所以从末尾到`start-1`的净收益也必须是非负的（可能靠`start`到末尾的"盈余"补上）。

## 特殊情况

### 只有一个站点

如果`gas[0] >= cost[0]`，返回0，否则返回-1。

### 所有站点净收益相同

如果都是非负的，返回0。

## 复杂度分析

**时间复杂度**：O(n)

**空间复杂度**：O(1)

## 小结

加油站问题的贪心关键在于两点：
1. 总量决定是否有解
2. 失败点决定跳过的范围

这种"失败后跳到下一个候选"的贪心策略很重要，在很多问题中都有应用。
