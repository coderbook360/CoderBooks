# 实战：根据身高重建队列

这道题展示了多维度排序的贪心技巧。

## 问题描述

假设有打乱顺序的一群人站成一个队列，数组`people`表示队列中一些人的属性。每个`people[i] = [hi, ki]`表示第`i`个人的身高为`hi`，前面**正好**有`ki`个身高大于或等于`hi`的人。

请你重新构造并返回这个队列。

## 思路分析

### 问题难点

每个人有两个属性：身高和前面的人数。如何同时满足两个约束？

### 贪心洞察

先处理一个维度，再处理另一个。

**关键策略**：按身高降序排列，身高相同按k升序排列。然后依次将每个人插入到索引k的位置。

### 为什么这样做？

处理一个人时，已经处理过的人都比他高（或等高）。

所以他的k值就是他在当前队列中的正确位置——前面有k个比他高的人。

后面插入的人更矮，不会影响已经插入的人的k值。

## 代码实现

```javascript
function reconstructQueue(people) {
    // 按身高降序，身高相同按k升序
    people.sort((a, b) => {
        if (a[0] !== b[0]) {
            return b[0] - a[0];  // 身高降序
        }
        return a[1] - b[1];  // k升序
    });
    
    const result = [];
    
    for (const person of people) {
        // 插入到索引k的位置
        result.splice(person[1], 0, person);
    }
    
    return result;
}
```

## 图解

```
输入: [[7,0], [4,4], [7,1], [5,0], [6,1], [5,2]]

排序后: [[7,0], [7,1], [6,1], [5,0], [5,2], [4,4]]

插入过程:
[7,0] -> [[7,0]]
[7,1] -> [[7,0], [7,1]]
[6,1] -> [[7,0], [6,1], [7,1]]
[5,0] -> [[5,0], [7,0], [6,1], [7,1]]
[5,2] -> [[5,0], [7,0], [5,2], [6,1], [7,1]]
[4,4] -> [[5,0], [7,0], [5,2], [6,1], [4,4], [7,1]]

输出: [[5,0], [7,0], [5,2], [6,1], [4,4], [7,1]]
```

## 正确性证明

### 归纳证明

**假设**：处理完前i个人后，队列是正确的（每个人的k值都满足）。

**归纳**：处理第i+1个人时：
- 他比队列中所有人都矮（或等高）
- 所以队列中所有人都"算数"
- 把他插入位置k，前面恰好有k个人（都比他高或等高）
- 他的插入不影响已有人的k值（因为他更矮）

**结论**：最终队列正确。

## 为什么身高相同时k要升序？

如果两个人身高相同，比如[5,0]和[5,2]。

按身高降序时，他们的相对顺序由k决定：
- k小的先处理，会被插入更前面
- k大的后处理，插入时前面已经有那个人了

如果k降序（先处理k大的），会出错：
- [5,2]先插入位置2，但此时位置2可能还没有足够的人
- 后来[5,0]插入位置0，[5,2]被挤到后面，位置可能不对

## 优化版本

使用链表可以优化插入效率：

```javascript
function reconstructQueue(people) {
    people.sort((a, b) => {
        if (a[0] !== b[0]) return b[0] - a[0];
        return a[1] - b[1];
    });
    
    // 使用链表节点
    const dummy = { next: null };
    
    for (const person of people) {
        let prev = dummy;
        let count = person[1];
        
        while (count > 0) {
            prev = prev.next;
            count--;
        }
        
        const node = { val: person, next: prev.next };
        prev.next = node;
    }
    
    // 转回数组
    const result = [];
    let curr = dummy.next;
    while (curr) {
        result.push(curr.val);
        curr = curr.next;
    }
    
    return result;
}
```

但实际上，对于这道题数组的splice已经足够。

## 复杂度分析

**时间复杂度**：O(n²)
- 排序O(n log n)
- 插入操作每次O(n)，共n次

**空间复杂度**：O(n)，结果数组

## 小结

身高重建队列展示了多维度贪心的技巧：
1. **降维**：先固定一个维度（身高）
2. **排序**：按高到矮处理
3. **插入**：另一个维度（k）直接决定插入位置

这种"高个子先站队，矮个子往里插"的策略非常巧妙。
