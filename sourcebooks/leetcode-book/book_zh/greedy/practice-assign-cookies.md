# 实战：分发饼干

这是贪心算法最经典的入门题。

## 问题描述

假设你是一位很棒的家长，想要给孩子们一些小饼干。但是，每个孩子最多只能给一块饼干。

每个孩子`i`都有一个胃口值`g[i]`，这是能让孩子满足的饼干最小尺寸。每块饼干`j`都有一个尺寸`s[j]`。如果`s[j] >= g[i]`，我们可以将这个饼干分配给孩子，这个孩子会得到满足。

你的目标是**尽可能满足最多数量的孩子**，并输出这个最大数值。

## 思路分析

### 贪心策略

用最小的饼干满足胃口最小的孩子。

为什么这样做？
- 小饼干只能满足胃口小的孩子
- 大饼干可以满足任何孩子
- 用大饼干满足小胃口是"浪费"

### 实现方法

1. 将孩子的胃口和饼干尺寸都排序
2. 用双指针匹配
3. 如果当前饼干能满足当前孩子，两者都前进
4. 如果不能，饼干前进（找更大的饼干）

## 代码实现

```javascript
function findContentChildren(g, s) {
    g.sort((a, b) => a - b);
    s.sort((a, b) => a - b);
    
    let child = 0;
    let cookie = 0;
    
    while (child < g.length && cookie < s.length) {
        if (s[cookie] >= g[child]) {
            child++;
        }
        cookie++;
    }
    
    return child;
}
```

## 正确性证明

### 交换论证

假设最优解中，饼干`s[i]`分给了孩子`g[j]`，而我们的贪心解把`s[i]`分给了胃口更小的孩子`g[k]`（k < j）。

由于`s[i] >= g[j] >= g[k]`，所以`s[i]`确实能满足`g[k]`。

原来满足`g[j]`的饼干（假设是`s[m]`，m > i），现在可以空出来满足别的孩子，或者仍然满足`g[j]`。

所以我们的贪心选择不会让解变差。

## 示例

```
输入: g = [1, 2, 3], s = [1, 1]
输出: 1

解释:
- 只有两块尺寸为1的饼干
- 只能满足胃口为1的孩子
- 答案是1
```

```
输入: g = [1, 2], s = [1, 2, 3]
输出: 2

解释:
- 饼干1给胃口1的孩子
- 饼干2给胃口2的孩子
- 两个孩子都满足了
```

## 复杂度分析

**时间复杂度**：O(n log n + m log m)，主要是排序

**空间复杂度**：O(log n + log m)，排序的栈空间

## 另一种贪心策略

从大到小也可以：用最大的饼干满足胃口最大的孩子。

```javascript
function findContentChildren(g, s) {
    g.sort((a, b) => b - a);
    s.sort((a, b) => b - a);
    
    let child = 0;
    let cookie = 0;
    
    while (child < g.length && cookie < s.length) {
        if (s[cookie] >= g[child]) {
            cookie++;
        }
        child++;
    }
    
    return cookie;
}
```

这种策略也是正确的，但代码稍微不那么直观。

## 小结

分发饼干展示了贪心的基本模式：
1. **排序**：让比较有序进行
2. **贪心匹配**：用"刚好够"的资源满足需求
3. **双指针**：高效遍历两个序列

这种"排序 + 双指针"的贪心模式在很多题目中都会用到。
