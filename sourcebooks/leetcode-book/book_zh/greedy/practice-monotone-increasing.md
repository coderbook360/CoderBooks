# 实战：单调递增的数字

这道题展示了贪心在数字构造中的应用。

## 问题描述

给定一个非负整数`n`，返回小于或等于`n`的最大整数，且该整数的各位数字是**单调递增**的。

单调递增意味着从左到右每位数字不减。

## 思路分析

### 问题本质

找一个最大的数，满足：
- 不超过n
- 各位数字递增

### 贪心洞察

从高位到低位遍历，找到第一个"下降"的位置。

在下降位置之前的某一位减1，后面全部变成9，就能得到最大的递增数。

### 具体做法

1. 找到第一个`digits[i] > digits[i+1]`的位置
2. 但要注意，可能需要回退：比如332，第一个下降在位置1，但减1后变成323，还是不递增
3. 所以要找最靠前的"峰值"位置

## 代码实现

```javascript
function monotoneIncreasingDigits(n) {
    const digits = String(n).split('').map(Number);
    const len = digits.length;
    
    // 找到需要减1的位置
    let mark = len;  // 从这个位置开始全部变成9
    
    for (let i = len - 1; i > 0; i--) {
        if (digits[i - 1] > digits[i]) {
            mark = i;
            digits[i - 1]--;
        }
    }
    
    // 从mark开始全部变成9
    for (let i = mark; i < len; i++) {
        digits[i] = 9;
    }
    
    return Number(digits.join(''));
}
```

## 图解

```
n = 332

digits = [3, 3, 2]

从右往左遍历:
i=2: digits[1]=3 > digits[2]=2, mark=2, digits[1]-- → [3, 2, 2]
i=1: digits[0]=3 > digits[1]=2, mark=1, digits[0]-- → [2, 2, 2]

从mark=1开始变9: [2, 9, 9]

结果: 299
```

```
n = 1234

digits = [1, 2, 3, 4]

从右往左遍历:
i=3: 3 < 4, 不处理
i=2: 2 < 3, 不处理
i=1: 1 < 2, 不处理

mark = 4, 不需要变9

结果: 1234 (本身就是递增的)
```

## 为什么从右往左遍历？

考虑n=332：

如果从左往右：
- 位置1: 3 > 2, 减1得到 [3, 2, 2]
- 现在3 > 2，还需要处理位置0
- 逻辑变复杂

从右往左：
- 一次遍历，遇到下降就减1
- 减1后可能造成新的下降，继续向左处理
- 最后从某个位置开始全变9

## 贪心选择性质的证明

### 为什么要在最高的下降位置减1？

假设在更低的位置减1，会浪费高位的"价值"。

### 为什么后面全变9？

因为我们已经在某位减1，后面的位数无论怎么变都不会超过原数。

为了最大化结果，后面全部取最大值9。

## 特殊情况

### 本身递增

```
n = 123
结果: 123
```

### 单个数字

```
n = 9
结果: 9
```

### 含0

```
n = 10
digits = [1, 0]
i=1: 1 > 0, mark=1, digits[0]-- → [0, 0]
变9: [0, 9]
结果: 9
```

## 复杂度分析

**时间复杂度**：O(d)，d是数字位数，约为log₁₀(n)

**空间复杂度**：O(d)，存储数字的各位

## 小结

单调递增数字展示了贪心在数字构造中的应用：
1. **从右往左**：处理级联的下降
2. **减1策略**：在最高的不满足位置减1
3. **填9策略**：后面全部取最大值

这种"找关键位置，前面调整，后面填最大"的模式在很多数字构造题中都有用。
