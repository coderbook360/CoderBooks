# 实战：分发糖果

这道题展示了双向遍历的贪心技巧。

## 问题描述

`n`个孩子站成一排。给你一个整数数组`ratings`表示每个孩子的评分。

你需要按照以下要求，给这些孩子分发糖果：
- 每个孩子至少分配到1个糖果
- 相邻两个孩子评分更高的孩子会获得更多的糖果

请你给每个孩子分发糖果，返回**需要准备的最少糖果数目**。

## 思路分析

### 难点

一个孩子的糖果数取决于左右两个邻居，但左右邻居又取决于他们的邻居...

### 贪心洞察

把问题分解：
- 满足"比左边评分高就比左边糖果多"
- 满足"比右边评分高就比右边糖果多"

两个方向分别处理，最后取较大值。

### 两次遍历

1. **从左到右**：如果当前比左边评分高，糖果数 = 左边 + 1
2. **从右到左**：如果当前比右边评分高，糖果数 = max(当前, 右边 + 1)

## 代码实现

```javascript
function candy(ratings) {
    const n = ratings.length;
    const candies = new Array(n).fill(1);
    
    // 从左到右：处理右边比左边大的情况
    for (let i = 1; i < n; i++) {
        if (ratings[i] > ratings[i - 1]) {
            candies[i] = candies[i - 1] + 1;
        }
    }
    
    // 从右到左：处理左边比右边大的情况
    for (let i = n - 2; i >= 0; i--) {
        if (ratings[i] > ratings[i + 1]) {
            candies[i] = Math.max(candies[i], candies[i + 1] + 1);
        }
    }
    
    return candies.reduce((a, b) => a + b, 0);
}
```

## 图解

```
ratings = [1, 0, 2]

初始:    [1, 1, 1]

从左到右:
i=1: 0 < 1, 不变
i=2: 2 > 0, candies[2] = candies[1] + 1 = 2
结果:    [1, 1, 2]

从右到左:
i=1: 0 < 2, 不变
i=0: 1 > 0, candies[0] = max(1, 1+1) = 2
结果:    [2, 1, 2]

总糖果: 5
```

## 正确性证明

### 左边约束

第一次遍历后，对于任意i，如果`ratings[i] > ratings[i-1]`，则`candies[i] > candies[i-1]`。

### 右边约束

第二次遍历后，对于任意i，如果`ratings[i] > ratings[i+1]`，则`candies[i] > candies[i+1]`。

由于第二次遍历用的是max，不会破坏第一次遍历建立的左边约束。

### 最少糖果

每个孩子的糖果数是满足约束的最小值：
- 没有更高评分的邻居，得1个
- 有更高评分的邻居，得邻居的糖果+1

## 特殊情况

### 评分相同

评分相同的相邻孩子，糖果数可以不同。

```
ratings = [1, 1, 1]
candies = [1, 1, 1]  // 都是1个即可
```

### 严格单调

```
ratings = [1, 2, 3, 4, 5]
candies = [1, 2, 3, 4, 5]
```

```
ratings = [5, 4, 3, 2, 1]
candies = [5, 4, 3, 2, 1]
```

### 山峰形状

```
ratings = [1, 3, 2]

从左到右: [1, 2, 1]
从右到左: [1, 2, 1]  // 2 > 1, 但candies[1]=2已经>candies[2]=1

结果: [1, 2, 1]
```

## 优化：O(1)空间

可以用一次遍历+计数来优化空间，但代码复杂度大增，实际中不推荐。

## 复杂度分析

**时间复杂度**：O(n)

**空间复杂度**：O(n)，存储每个孩子的糖果数

## 小结

分发糖果展示了双向遍历的贪心技巧：
1. **分解约束**：左边约束和右边约束分开处理
2. **两次遍历**：分别满足两个方向的约束
3. **取较大值**：第二次遍历用max保持两个约束都满足

这种双向遍历的思想在很多问题中都有应用。
