# 实战：用最少数量的箭引爆气球

另一道经典的区间贪心问题。

## 问题描述

有一些球形气球贴在一堵墙上。墙用XY平面表示，气球用二维数组`points`表示，其中`points[i] = [xstart, xend]`表示气球的直径从`xstart`到`xend`。

一支弓箭可以从X轴沿着Y轴方向射出。弓箭会在`x`处射出，若`xstart <= x <= xend`，则该气球会被引爆。

返回引爆所有气球所需的**最小弓箭数**。

## 思路分析

### 问题转化

这其实是求"最少的点，能覆盖所有区间"。

### 贪心洞察

如果两个气球有重叠，一箭可以同时射爆它们。

贪心策略：按结束位置排序，尽可能让一箭射爆更多气球。

### 具体做法

1. 按气球的结束位置排序
2. 第一箭射在第一个气球的结束位置
3. 遍历后续气球：如果起始位置大于当前箭的位置，需要新箭

## 代码实现

```javascript
function findMinArrowPoints(points) {
    if (points.length === 0) return 0;
    
    // 按结束位置排序
    points.sort((a, b) => a[1] - b[1]);
    
    let arrows = 1;
    let arrowPos = points[0][1];  // 当前箭的位置
    
    for (let i = 1; i < points.length; i++) {
        if (points[i][0] > arrowPos) {
            // 当前气球的起始位置超过了箭的位置，需要新箭
            arrows++;
            arrowPos = points[i][1];
        }
        // 否则，当前箭可以射到这个气球（因为气球的起始 <= arrowPos <= 气球的结束）
    }
    
    return arrows;
}
```

## 图解

```
points = [[10,16], [2,8], [1,6], [7,12]]

按结束位置排序: [[1,6], [2,8], [7,12], [10,16]]

[1,6]: 第1箭, arrowPos=6
[2,8]: start=2 <= 6, 用第1箭
[7,12]: start=7 > 6, 第2箭, arrowPos=12
[10,16]: start=10 <= 12, 用第2箭

总共2箭
```

## 为什么射在结束位置？

假设要射爆气球A，箭可以射在A的任意位置。

如果射在A的结束位置，能覆盖的后续气球最多。因为后续气球按结束位置排序，射得越靠后，越有可能与后面的气球重叠。

## 与无重叠区间的关系

| 无重叠区间 | 射气球 |
|-----------|-------|
| 保留最多不重叠区间 | 用最少的点覆盖所有区间 |
| 每个保留的区间代表一个"分组" | 每支箭代表一组重叠的气球 |

两者答案相同：不重叠区间的数量 = 所需弓箭数

因为每组重叠的气球可以用一支箭射爆，而不同组之间必须用不同的箭。

## 边界条件

### 端点相等

`[1,6]`和`[6,10]`可以用同一支箭（x=6处）。

代码中用`points[i][0] > arrowPos`而不是`>=`。

### 大量重叠

所有气球都重叠的话，只需要1支箭。

## 复杂度分析

**时间复杂度**：O(n log n)，主要是排序

**空间复杂度**：O(log n)，排序的栈空间

## 小结

射气球问题展示了区间贪心的另一面：
- 无重叠区间是"保留最多"
- 射气球是"分最少组"
- 核心都是按结束位置排序的贪心策略
