# 实战：买卖股票的最佳时机II

这道题展示了贪心在序列问题中的应用。

## 问题描述

给你一个整数数组`prices`，其中`prices[i]`表示某支股票第`i`天的价格。

你可以尽可能地完成更多的交易（多次买卖一支股票）。但你不能同时参与多笔交易（必须先卖出才能再次购买）。

返回你能获得的**最大利润**。

## 思路分析

### 贪心洞察

如果明天的价格比今天高，今天就买，明天就卖。

这个策略看起来太简单了，但它是正确的。

### 为什么正确？

考虑连续上涨的情况：
- 第1天价格1，第2天价格2，第3天价格3
- 第1天买第3天卖，利润 = 3 - 1 = 2
- 或者：第1天买第2天卖（利润1），第2天买第3天卖（利润1），总利润也是2

两种方式利润相同！

实际上：`prices[2] - prices[0] = (prices[2] - prices[1]) + (prices[1] - prices[0])`

所以"只抓上涨的部分"等价于"全程持有上涨段"。

## 代码实现

```javascript
function maxProfit(prices) {
    let profit = 0;
    
    for (let i = 1; i < prices.length; i++) {
        if (prices[i] > prices[i - 1]) {
            profit += prices[i] - prices[i - 1];
        }
    }
    
    return profit;
}
```

就这么简单。遍历一遍，累加所有上涨。

## 与动态规划的对比

这道题也可以用动态规划解：

```javascript
function maxProfit(prices) {
    let hold = -prices[0];  // 持有股票的最大利润
    let cash = 0;           // 不持有股票的最大利润
    
    for (let i = 1; i < prices.length; i++) {
        const newHold = Math.max(hold, cash - prices[i]);
        const newCash = Math.max(cash, hold + prices[i]);
        hold = newHold;
        cash = newCash;
    }
    
    return cash;
}
```

DP更通用，但贪心更简洁高效。

## 贪心选择性质的证明

### 交换论证

假设最优解在某一天选择了"不交易"（下跌时卖出），而这一天实际是上涨的。

那么我们可以把"不交易"换成"交易"，获得正收益，解变得更好。

所以，每一个上涨的部分都应该抓住。

## 变种问题

### 有交易次数限制

如果只能交易k次，贪心不再适用，需要用DP。

### 有冷冻期

卖出后要等一天才能买入，也需要DP。

### 有交易手续费

每次交易要扣费，需要修改贪心策略或用DP。

## 复杂度分析

**时间复杂度**：O(n)

**空间复杂度**：O(1)

## 小结

股票II是贪心的经典应用：
- **关键洞察**：连续上涨可以拆分成多次小交易
- **贪心策略**：抓住每一次上涨
- **简洁优雅**：一次遍历，O(1)空间
