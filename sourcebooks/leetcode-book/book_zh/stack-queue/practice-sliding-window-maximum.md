# 实战：滑动窗口最大值

这道题是单调队列的经典应用。单调队列和单调栈类似，但它是一个双端队列，支持两端操作。这使得它能在O(1)时间内获取滑动窗口的最大值。

## 问题描述

给你一个整数数组`nums`，有一个大小为`k`的滑动窗口从数组的最左侧移动到最右侧。你只能看到在滑动窗口内的`k`个数字。滑动窗口每次只向右移动一位。返回滑动窗口中的最大值。

**示例**：
```
输入: nums = [1,3,-1,-3,5,3,6,7], k = 3
输出: [3,3,5,5,6,7]

窗口位置                最大值
---------------        -----
[1  3  -1] -3  5  3  6  7    3
 1 [3  -1  -3] 5  3  6  7    3
 1  3 [-1  -3  5] 3  6  7    5
 1  3  -1 [-3  5  3] 6  7    5
 1  3  -1  -3 [5  3  6] 7    6
 1  3  -1  -3  5 [3  6  7]   7
```

## 思路分析

### 暴力解法：O(nk)

每个窗口都遍历找最大值：

```javascript
function maxSlidingWindow(nums, k) {
    const result = [];
    for (let i = 0; i <= nums.length - k; i++) {
        let max = nums[i];
        for (let j = i + 1; j < i + k; j++) {
            max = Math.max(max, nums[j]);
        }
        result.push(max);
    }
    return result;
}
```

当k很大时，这个方法效率很低。

### 单调队列：O(n)

**核心思想**：维护一个单调递减的双端队列，队首始终是当前窗口的最大值。

**关键操作**：
1. 从队尾移除所有比当前元素小的元素（它们不可能成为最大值了）
2. 从队首移除已滑出窗口的元素
3. 队首就是当前窗口的最大值

## 完整实现

```javascript
/**
 * @param {number[]} nums
 * @param {number} k
 * @return {number[]}
 */
function maxSlidingWindow(nums, k) {
    const result = [];
    const deque = [];  // 存储索引，对应值单调递减
    
    for (let i = 0; i < nums.length; i++) {
        // 1. 移除超出窗口范围的元素（从队首）
        while (deque.length && deque[0] <= i - k) {
            deque.shift();
        }
        
        // 2. 维护单调递减（从队尾移除较小元素）
        while (deque.length && nums[deque[deque.length - 1]] < nums[i]) {
            deque.pop();
        }
        
        // 3. 当前索引入队
        deque.push(i);
        
        // 4. 窗口形成后（i >= k-1），记录最大值
        if (i >= k - 1) {
            result.push(nums[deque[0]]);
        }
    }
    
    return result;
}
```

## 执行过程图解

以`nums = [1,3,-1,-3,5,3,6,7], k = 3`为例：

```

  -3 < -1，直接入队
  deque: [1, 2, 3] (对应值[3, -1, -3])
  result.push(nums[1]=3)
  result: [3, 3]

i=4 (值5):
  检查队首：1 <= 4-3=1，移除
  deque: [2, 3] (对应值[-1, -3])
  5 > -3，移除3

  deque: [4] (对应值[5])
  result.push(nums[4]=5)
  result: [3, 3, 5]

i=5 (值3):
  检查队首：4 > 5-3=2，在窗口内
  3 < 5，直接入队
  deque: [4, 5] (对应值[5, 3])
  result.push(nums[4]=5)
  result: [3, 3, 5, 5]

i=6 (值6):
  检查队首：4 <= 6-3=3，移除
  deque: [5] (对应值[3])
  6 > 3，移除5
  deque: [6] (对应值[6])
  result.push(nums[6]=6)
  result: [3, 3, 5, 5, 6]

i=7 (值7):
  检查队首：6 > 7-3=4，在窗口内
  7 > 6，移除6
```

## 为什么是正确的？

**单调递减的意义**：队列中的元素从前到后递减，队首是最大值。

**为什么可以移除较小元素？**

当一个元素A比后来的元素B小时：
- A在B之前离开窗口
- 在B离开之前，A不可能是最大值

所以A可以被安全移除，不会影响结果。

## 单调队列的本质

**单调队列 = 双端队列 + 单调性维护**

两端操作：
- **队首**：移除超出窗口的元素
- **队尾**：移除违反单调性的元素

这就是为什么需要双端队列而不是普通队列或栈。

## 复杂度分析

**时间复杂度：O(n)**
- 每个元素最多入队一次、出队一次
- 总操作数不超过2n

**空间复杂度：O(k)**
- 双端队列最多存储k个元素

## 单调队列 vs 单调栈

| 数据结构 | 操作端 | 典型问题 |
|---------|-------|---------|
| 单调栈 | 只在栈顶 | 下一个更大元素 |
| 单调队列 | 两端都操作 | 滑动窗口最值 |

**选择依据**：
- 不需要限制范围 → 单调栈
- 需要维护固定范围（窗口） → 单调队列

## 变体：滑动窗口最小值

只需改变比较方向，维护单调递增队列：

```javascript
function minSlidingWindow(nums, k) {
    const result = [];
    const deque = [];  // 单调递增
    
    for (let i = 0; i < nums.length; i++) {
        while (deque.length && deque[0] <= i - k) {
            deque.shift();
        }
        
        // 改为：移除比当前元素大的
        while (deque.length && nums[deque[deque.length - 1]] > nums[i]) {
            deque.pop();
        }
        
        deque.push(i);
        
        if (i >= k - 1) {
            result.push(nums[deque[0]]);
        }
    }
    
    return result;
}
```

## 边界情况

| 输入 | 结果 |
|------|------|
| `nums=[1], k=1` | `[1]` |
| `nums=[1,2,3], k=3` | `[3]` |
| `nums=[3,2,1], k=1` | `[3,2,1]` |
| `nums=[1,1,1], k=2` | `[1,1]` |

## 小结

单调队列是解决滑动窗口最值问题的利器：

**核心思想**：
1. 维护单调递减队列（找最大值）或单调递增队列（找最小值）
2. 队首移除超出窗口的元素
3. 队尾移除违反单调性的元素
4. 队首就是当前窗口的最值

**记忆口诀**：
- 找最大 → 单调递减
- 找最小 → 单调递增
- 两端操作 → 双端队列

单调队列将O(nk)的暴力解法优化到O(n)，是滑动窗口类问题的重要技巧。下一题我们将学习单调栈的另一个经典应用——柱状图中最大的矩形。
