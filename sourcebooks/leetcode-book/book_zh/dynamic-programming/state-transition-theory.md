# 状态定义与状态转移

状态定义是DP最难也是最关键的一步。

## 状态定义的艺术

好的状态定义有以下特点：
1. **能表达子问题**：状态能完整描述子问题
2. **能递推**：状态之间有明确的转移关系
3. **够简洁**：维度尽可能少

## 常见的状态定义模式

### 模式1：以位置结尾

`dp[i]` = 以第i个元素结尾的最优解

适用：最长递增子序列、最大子数组和

```javascript
// 最长递增子序列
// dp[i] = 以nums[i]结尾的LIS长度
dp[i] = max(dp[j] + 1), 其中 j < i 且 nums[j] < nums[i]
```

### 模式2：前i个元素

`dp[i]` = 前i个元素的最优解

适用：打家劫舍、背包问题

```javascript
// 打家劫舍
// dp[i] = 前i个房子能抢到的最大金额
dp[i] = max(dp[i-1], dp[i-2] + nums[i])
```

### 模式3：区间

`dp[i][j]` = 区间[i,j]的最优解

适用：回文串、合并石头

```javascript
// 最长回文子串
// dp[i][j] = s[i..j]是否是回文
dp[i][j] = (s[i] === s[j]) && dp[i+1][j-1]
```

### 模式4：两个序列

`dp[i][j]` = 第一个序列前i个和第二个序列前j个的最优解

适用：最长公共子序列、编辑距离

```javascript
// 最长公共子序列
// dp[i][j] = s1前i个字符和s2前j个字符的LCS长度
if (s1[i-1] === s2[j-1]) {
    dp[i][j] = dp[i-1][j-1] + 1;
} else {
    dp[i][j] = max(dp[i-1][j], dp[i][j-1]);
}
```

### 模式5：选择状态

`dp[i][state]` = 到第i步，处于state状态的最优解

适用：股票买卖、状态机DP

```javascript
// 股票买卖（含冷冻期）
// dp[i][0] = 第i天不持有股票的最大收益
// dp[i][1] = 第i天持有股票的最大收益
dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i]);
dp[i][1] = max(dp[i-1][1], dp[i-2][0] - prices[i]);
```

## 状态转移的思考方法

### 方法1：最后一步分析

思考：达到当前状态的最后一步是什么？

以爬楼梯为例：
- 到达第n阶的最后一步：从n-1爬1步，或从n-2爬2步
- 所以`dp[n] = dp[n-1] + dp[n-2]`

### 方法2：枚举决策

思考：在当前状态，有哪些决策？

以打家劫舍为例：
- 对于第i个房子：抢或不抢
- 抢：`dp[i-2] + nums[i]`
- 不抢：`dp[i-1]`
- 取最大：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`

### 方法3：分类讨论

根据不同情况分别处理。

以编辑距离为例：
- 如果`s1[i] === s2[j]`：不需要操作，`dp[i-1][j-1]`
- 否则三种操作取最小：
  - 插入：`dp[i][j-1] + 1`
  - 删除：`dp[i-1][j] + 1`
  - 替换：`dp[i-1][j-1] + 1`

## 初始条件的确定

初始条件对应最小的子问题：

```javascript
// 爬楼梯
dp[0] = 1;  // 站在地面，一种方法
dp[1] = 1;  // 到第1阶，一种方法

// 最长公共子序列
dp[0][j] = 0;  // 空串和任何串的LCS为0
dp[i][0] = 0;

// 编辑距离
dp[0][j] = j;  // 空串变成长度j的串，需要j次插入
dp[i][0] = i;  // 长度i的串变成空串，需要i次删除
```

## 计算顺序

确保计算`dp[i]`时，依赖的子问题已经算过。

```javascript
// 一维：从小到大
for (let i = 1; i <= n; i++)

// 二维：通常从小到大
for (let i = 1; i <= m; i++)
    for (let j = 1; j <= n; j++)

// 区间DP：按长度从小到大
for (let len = 2; len <= n; len++)
    for (let i = 0; i + len <= n; i++)
```

## 空间优化

很多DP可以优化空间：

### 滚动数组

当`dp[i]`只依赖`dp[i-1]`时：

```javascript
// 原始
dp[i] = dp[i-1] + dp[i-2];

// 优化后
let prev2 = 0, prev1 = 1;
for (let i = 2; i <= n; i++) {
    const curr = prev1 + prev2;
    prev2 = prev1;
    prev1 = curr;
}
```

### 一维化二维

当`dp[i][j]`只依赖上一行时：

```javascript
// 原始：O(mn)空间
dp[i][j] = dp[i-1][j] + dp[i][j-1];

// 优化：O(n)空间
for (let i = 1; i <= m; i++)
    for (let j = 1; j <= n; j++)
        dp[j] = dp[j] + dp[j-1];
```

## 调试技巧

1. **打印DP表**：观察状态值是否符合预期
2. **手算小例子**：验证转移方程正确性
3. **边界检查**：确保初始条件正确
4. **回溯路径**：记录决策，输出具体解

## 小结

状态定义与转移的关键：
1. **状态要完整**：能唯一确定子问题
2. **转移要正确**：覆盖所有情况
3. **初始要准确**：最小子问题的解
4. **顺序要合理**：依赖的子问题先算

DP的难点在于抽象，多练习才能培养直觉。
