# 算法基础理论

从今天起，你将踏上一段有趣的旅程。在这段旅程中，你会学到如何用代码解决各种各样的问题。不过在动手写代码之前，我们需要先理解一个核心概念：**什么是算法？**

## 什么是算法

假设你每天早上要从家去公司。你可以选择不同的路线：走大马路、穿小巷、坐地铁、骑共享单车……每种选择都能让你到达公司，但花费的时间和精力不同。

如果有人问你"怎么去公司"，你会这样描述：

1. 出门左转
2. 走到地铁站
3. 乘坐 2 号线，坐 5 站
4. 从 B 出口出站
5. 步行 200 米到公司

这就是一个算法——**解决特定问题的一系列明确步骤**。

在计算机科学中，算法的定义更加严谨：

> **算法**是为解决特定问题而设计的有限指令序列，每条指令表示一个或多个操作。

一个合格的算法必须具备五个特性：

### 1. 有穷性

算法必须在有限步骤后结束，不能无限循环下去。

```javascript
// ✓ 正确：有穷算法
function countTo(n) {
    for (let i = 1; i <= n; i++) {
        console.log(i);
    }
    // 无论 n 是多少，循环总会结束
}

// ✗ 错误：无穷循环（这不是算法）
function infiniteLoop() {
    while (true) {
        console.log("forever...");
    }
    // 永远不会结束
}
```

### 2. 确定性

算法的每一步都必须有明确的定义，没有歧义。给定相同的输入，必须产生相同的输出。

```javascript
// ✓ 确定性：相同输入，相同输出
function add(a, b) {
    return a + b;
}

console.log(add(2, 3)); // 永远是 5
console.log(add(2, 3)); // 还是 5

// ✗ 不确定性（这不是纯粹的算法）
function randomAdd(a, b) {
    return a + b + Math.random();
}
// 每次结果都不同
```

### 3. 可行性

算法的每一步都必须是可以执行的基本操作，不能是抽象的或无法实现的。

```javascript
// ✓ 可行：每一步都可以执行
function findMax(a, b) {
    if (a > b) {
        return a;
    } else {
        return b;
    }
}

// ✗ 不可行（伪代码）：
// 1. 用意念感知哪个数更大
// 2. 返回那个数
// 这些步骤无法用代码实现
```

### 4. 输入

算法有零个或多个输入，这些输入取自特定的对象集合。

```javascript
// 零个输入
function getPi() {
    return 3.14159;
}

// 两个输入
function multiply(a, b) {
    return a * b;
}

// 多个输入
function sum(...numbers) {
    return numbers.reduce((acc, n) => acc + n, 0);
}
```

### 5. 输出

算法有一个或多个输出，输出是与输入有某种关系的量。

```javascript
// 一个输出
function square(n) {
    return n * n;
}

// 多个输出（返回对象或数组）
function divideAndRemainder(a, b) {
    return {
        quotient: Math.floor(a / b),
        remainder: a % b
    };
}

console.log(divideAndRemainder(10, 3)); // { quotient: 3, remainder: 1 }
```

## 为什么要学习算法

你可能会问："我只是想做个网页/App，为什么要学算法？"

这是一个很好的问题。让我从三个角度来回答。

### 1. 算法决定程序的效率

同样的问题，不同的算法解决起来效率可能相差天壤之别。

假设你有一个包含 100 万个数字的数组，需要找出其中是否存在两个数之和等于目标值。

**暴力方法**：检查每一对数字

```javascript
function twoSumBrute(nums, target) {
    for (let i = 0; i < nums.length; i++) {
        for (let j = i + 1; j < nums.length; j++) {
            if (nums[i] + nums[j] === target) {
                return [i, j];
            }
        }
    }
    return null;
}
```

这需要检查约 5000 亿对数字，可能需要几分钟。

**聪明方法**：使用哈希表

```javascript
function twoSumSmart(nums, target) {
    const seen = new Map();
    for (let i = 0; i < nums.length; i++) {
        const complement = target - nums[i];
        if (seen.has(complement)) {
            return [seen.get(complement), i];
        }
        seen.set(nums[i], i);
    }
    return null;
}
```

这只需要遍历数组一次，不到 1 秒就能完成。

**同样的问题，算法不同，效率差了几十万倍。**

### 2. 算法是技术面试的核心

Google、Meta、Microsoft、字节跳动等顶级科技公司的面试中，算法题占比通常超过 60%。他们相信：**能写出优秀算法的人，能解决复杂的工程问题**。

不仅是面试，日常开发中你也会不断遇到算法问题：
- 如何快速搜索商品？
- 如何推荐用户可能喜欢的内容？
- 如何优化数据库查询？
- 如何压缩传输的数据？

### 3. 算法训练思维能力

学习算法不仅是学习代码技巧，更是训练**问题分解**和**逻辑思维**的能力。

当你面对一个复杂问题时，算法思维会帮助你：
- 把大问题分解成小问题
- 识别问题的模式和规律
- 找到最优的解决方案
- 预判可能出现的边界情况

这些能力在任何技术领域都是宝贵的。

## 如何评价算法的好坏

判断一个算法好不好，我们通常从四个维度来考量。

### 1. 正确性

这是最基本的要求——算法必须能正确解决问题。

```javascript
// 需求：判断一个数是否为偶数

// ✗ 错误的实现
function isEvenWrong(n) {
    return n / 2 === 0; // 这是错的！
}

// ✓ 正确的实现
function isEven(n) {
    return n % 2 === 0;
}

console.log(isEvenWrong(4)); // false（错误）
console.log(isEven(4));      // true（正确）
```

验证正确性的方法：
- **边界测试**：测试最小值、最大值、空输入等特殊情况
- **随机测试**：用大量随机数据验证
- **数学证明**：对于关键算法，最好能证明其正确性

### 2. 可读性

代码是写给人看的，顺便让机器执行。好的算法实现应该易于理解和维护。

```javascript
// ✗ 难以理解
function f(a) {
    let r = 1;
    for (let i = 2; i <= a; i++) r *= i;
    return r;
}

// ✓ 清晰易懂
function factorial(n) {
    let result = 1;
    for (let i = 2; i <= n; i++) {
        result *= i;
    }
    return result;
}
```

提升可读性的方法：
- 使用有意义的变量名
- 添加必要的注释
- 保持代码结构清晰
- 遵循统一的代码风格

### 3. 健壮性

好的算法能处理各种异常情况，而不是轻易崩溃。

```javascript
// ✗ 脆弱的实现
function divideFragile(a, b) {
    return a / b;
}

// ✓ 健壮的实现
function divideRobust(a, b) {
    if (typeof a !== 'number' || typeof b !== 'number') {
        throw new Error('参数必须是数字');
    }
    if (b === 0) {
        throw new Error('除数不能为零');
    }
    return a / b;
}

console.log(divideFragile(10, 0));  // Infinity（可能不是期望的结果）
console.log(divideRobust(10, 0));   // 抛出错误：除数不能为零
```

需要考虑的边界情况：
- 空输入（null、undefined、空数组、空字符串）
- 边界值（最大值、最小值、零）
- 非法输入（类型错误、格式错误）

### 4. 效率

效率包括两个方面：**时间效率**和**空间效率**。

- **时间效率**：算法运行需要多长时间
- **空间效率**：算法运行需要多少内存

这两者经常需要权衡。有时候用更多的内存可以换来更快的速度，有时候为了节省内存需要牺牲一些速度。

下一章我们会详细讲解如何分析算法的时间复杂度和空间复杂度。

## 算法与数据结构

计算机科学家 Niklaus Wirth 有一句名言：

> **程序 = 算法 + 数据结构**

算法描述"如何做"，数据结构描述"数据怎么组织"。两者相辅相成：

- **数组**配合**遍历算法**可以完成搜索
- **树**配合**递归算法**可以实现高效查找
- **图**配合**广度优先搜索**可以找到最短路径
- **哈希表**配合**查找算法**可以实现 O(1) 检索

选择正确的数据结构，往往是设计高效算法的第一步。

## 学习路径

本书将带你系统学习算法，从基础到进阶，循序渐进：

1. **基础篇**：数组、链表、栈、队列等基本数据结构和操作
2. **进阶篇**：二叉树、哈希表、堆、图等高级数据结构
3. **技巧篇**：双指针、滑动窗口、前缀和、二分查找等常用技巧
4. **高级篇**：动态规划、回溯、贪心、分治等算法思想

每个主题都遵循"理论 + 实战"的模式：先讲解核心原理，再通过 LeetCode 经典题目巩固练习。

学习建议：

1. **理解优先**：不要死记代码，重点理解算法背后的思想
2. **动手实践**：每学完一个算法，亲手实现一遍
3. **循序渐进**：先掌握基础，再挑战进阶
4. **持之以恒**：每天学习 1-2 小时，比突击学习更有效

- **算法的五大特性**：有穷性、确定性、可行性、输入、输出
- **为什么学算法**：提升程序效率、应对面试、训练思维
- **评价算法的四个维度**：正确性、可读性、健壮性、效率
- **算法与数据结构的关系**：程序 = 算法 + 数据结构
