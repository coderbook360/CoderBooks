# 合并两个有序数组

> LeetCode 88. Merge Sorted Array

给你两个**有序**整数数组 `nums1` 和 `nums2`，请你将 `nums2` 合并到 `nums1` 中，使 `nums1` 成为一个有序数组。

这道题有一个巧妙的设定：`nums1` 的长度足够大，后面留有空位来容纳 `nums2` 的元素。这个设定暗示了一种优雅的解法——**从后往前合并**。

## 问题描述

```javascript
输入：
nums1 = [1, 2, 3, 0, 0, 0], m = 3
nums2 = [2, 5, 6], n = 3

输出：[1, 2, 2, 3, 5, 6]

说明：
- nums1 前 m 个元素是有效数据
- nums1 后面的 0 是占位符，用于存放合并后的结果
```

## 思路分析

### 为什么不能从前往后？

最直观的想法是：两个指针分别指向两个数组的开头，比较大小，小的先放入结果。

但问题来了：结果要存在哪里？

```javascript
nums1 = [1, 2, 3, 0, 0, 0]
nums2 = [2, 5, 6]

// 假设从前往后合并
// 第一步：比较 nums1[0]=1 和 nums2[0]=2
// 1 更小，应该放在位置 0...但位置 0 本来就是 1，不用动

// 第二步：比较 nums1[1]=2 和 nums2[0]=2
// 相等，取 nums2[0]=2，放在位置 1
// 问题！原来 nums1[1]=2 被覆盖了！

[1, 2, 3, 0, 0, 0]  →  [1, 2, ?, ?, ?, ?]
    ↑                      ↑
    原来的2被覆盖了
```

从前往后合并，会覆盖 `nums1` 中还没处理的元素。

### 逆向双指针思想

换个思路：**从后往前合并**。

`nums1` 后面的位置（那些 0）是空的，从这里开始填充，不会覆盖任何有效数据。

每次比较两个数组的末尾元素，把较大的放到 `nums1` 的最后面：

```
nums1 = [1, 2, 3, 0, 0, 0]
                     ↑
                     从这里开始填充
nums2 = [2, 5, 6]
```

## 解法详解

### 解法一：逆向双指针（推荐）

```javascript
function merge(nums1, m, nums2, n) {
    let p1 = m - 1;      // 指向 nums1 有效部分的末尾
    let p2 = n - 1;      // 指向 nums2 的末尾
    let p = m + n - 1;   // 指向合并后数组的末尾
    
    // 只要 nums2 还有元素没处理完
    while (p2 >= 0) {
        // 如果 nums1 还有元素，且 nums1 的当前元素更大
        if (p1 >= 0 && nums1[p1] > nums2[p2]) {
            nums1[p] = nums1[p1];
            p1--;
        } else {
            // nums1 已用完，或 nums2 的当前元素更大（或相等）
            nums1[p] = nums2[p2];
            p2--;
        }
        p--;
    }
    // 注意：如果 nums2 先用完，nums1 剩余的元素本来就在正确位置，不需要移动
}
```

**为什么循环条件是 `p2 >= 0`**？

- 如果 `nums2` 先用完，`nums1` 剩余的元素已经在正确的位置了
- 如果 `nums1` 先用完，需要把 `nums2` 剩余的元素都放进去

所以只需要确保 `nums2` 的所有元素都被处理。

## 执行过程图解

以 `nums1 = [1, 2, 3, 0, 0, 0]`，`nums2 = [2, 5, 6]` 为例：

```
初始状态：p1=2, p2=2, p=5
nums1: [1, 2, 3, 0, 0, 0]
              ↑        ↑
              p1       p
nums2: [2, 5, 6]
              ↑
              p2

第1步：比较 nums1[2]=3 和 nums2[2]=6
       6 > 3，把 6 放到位置 5
nums1: [1, 2, 3, 0, 0, 6]
              ↑     ↑
              p1    p
       p2--, p--

第2步：比较 nums1[2]=3 和 nums2[1]=5
       5 > 3，把 5 放到位置 4
nums1: [1, 2, 3, 0, 5, 6]
              ↑  ↑
              p1 p
       p2--, p--

第3步：比较 nums1[2]=3 和 nums2[0]=2
       3 > 2，把 3 放到位置 3
nums1: [1, 2, 3, 3, 5, 6]
           ↑  ↑
           p1 p
       p1--, p--

第4步：比较 nums1[1]=2 和 nums2[0]=2
       相等，取 nums2 的 2 放到位置 2
nums1: [1, 2, 2, 3, 5, 6]
           ↑
           p
       p2--, p--

第5步：p2 = -1 < 0，循环结束

最终结果: [1, 2, 2, 3, 5, 6]
```

### 解法二：正向双指针 + 额外空间

如果不要求原地操作，可以用额外空间保存 `nums1` 的副本：

```javascript
function merge(nums1, m, nums2, n) {
    // 复制 nums1 的有效部分
    const copy = nums1.slice(0, m);
    
    let p1 = 0;  // 指向 copy
    let p2 = 0;  // 指向 nums2
    let p = 0;   // 指向 nums1（结果）
    
    // 归并
    while (p1 < m && p2 < n) {
        if (copy[p1] <= nums2[p2]) {
            nums1[p++] = copy[p1++];
        } else {
            nums1[p++] = nums2[p2++];
        }
    }
    
    // 处理剩余元素
    while (p1 < m) nums1[p++] = copy[p1++];
    while (p2 < n) nums1[p++] = nums2[p2++];
}
```

这就是经典的**归并排序**中的合并步骤。

## 复杂度分析

**解法一（逆向双指针）**：
- 时间复杂度：O(m + n)，每个元素最多被处理一次
- 空间复杂度：O(1)，只用了几个指针变量

**解法二（正向 + 额外空间）**：
- 时间复杂度：O(m + n)
- 空间复杂度：O(m)，需要复制 `nums1` 的有效部分

解法一是更优的选择。

## 边界情况

```javascript
// nums2 为空
merge([1, 2, 3], 3, [], 0)
// 结果: [1, 2, 3]，不需要任何操作

// nums1 有效部分为空
merge([0, 0], 0, [1, 2], 2)
// 结果: [1, 2]，直接复制 nums2

// 两个数组无交叉（nums2 全部小于 nums1）
merge([4, 5, 6, 0, 0, 0], 3, [1, 2, 3], 3)
// 结果: [1, 2, 3, 4, 5, 6]

// 两个数组无交叉（nums1 全部小于 nums2）
merge([1, 2, 3, 0, 0, 0], 3, [4, 5, 6], 3)
// 结果: [1, 2, 3, 4, 5, 6]
```

## 常见误区

**误区一：忘记处理 nums1 为空的情况**

```javascript
// ❌ 如果 m=0，p1=-1，第一次比较就会访问 nums1[-1]
while (p2 >= 0) {
    if (nums1[p1] > nums2[p2]) {  // nums1[-1] 是 undefined！
        ...
    }
}

// ✅ 正确做法：先检查 p1 >= 0
if (p1 >= 0 && nums1[p1] > nums2[p2]) {
    ...
}
```

**误区二：循环条件写成 `p1 >= 0 && p2 >= 0`**

```javascript
// ❌ 如果 nums1 先用完，nums2 剩余的元素不会被处理
while (p1 >= 0 && p2 >= 0) {
    ...
}

// ✅ 只要 nums2 还有元素就继续
while (p2 >= 0) {
    ...
}
```

## 小结

这道题的核心技巧：

1. **逆向思维**：从后往前填充，避免覆盖未处理的元素
2. **利用空间**：`nums1` 后面的空位正是为逆向合并准备的
3. **简化终止条件**：只需关注 `nums2` 是否处理完

逆向双指针是一个重要的技巧，当你发现"从前往后会覆盖有效数据"时，试试从后往前。这种思维在很多数组问题中都能派上用场。
