# 找到所有数组中消失的数字

> LeetCode 448. Find All Numbers Disappeared in an Array

给你一个长度为 n 的数组，其中元素的值在 [1, n] 范围内，找出所有在 [1, n] 范围内但没有出现在数组中的数字。

这道题有一个神奇的解法：**不用额外空间**，只通过正负号就能标记数字是否出现过。

## 问题描述

```javascript
输入：nums = [4, 3, 2, 7, 8, 2, 3, 1]
输出：[5, 6]

解释：数组长度为 8，元素范围是 [1, 8]
出现的数字：1, 2, 3, 4, 7, 8
消失的数字：5, 6
```

**要求**：能否在 O(n) 时间、O(1) 额外空间内完成？（返回结果不算额外空间）

## 思路分析

### 暴力解法（哈希表）

最直观的想法：用哈希表记录哪些数字出现过。

```javascript
function findDisappearedNumbers(nums) {
    const seen = new Set(nums);
    const result = [];
    
    for (let i = 1; i <= nums.length; i++) {
        if (!seen.has(i)) {
            result.push(i);
        }
    }
    
    return result;
}
```

时间 O(n)，空间 O(n)。能不能做到 O(1) 空间？

### 原地标记技巧

关键洞察：**数组本身就可以当哈希表用**。

- 数组长度为 n，索引范围是 [0, n-1]
- 元素范围是 [1, n]，正好可以和索引一一对应
- 数字 k 对应索引 k-1

**如何"标记"一个数字出现过**？

把对应位置的数字变成**负数**：
- 遇到数字 k，就把 `nums[k-1]` 变成负数
- 遍历完成后，仍然是正数的位置，对应的数字就没出现过

```
原数组:  [4, 3, 2, 7, 8, 2, 3, 1]
索引:     0  1  2  3  4  5  6  7

数字 4 出现 → nums[3] 变负
数字 3 出现 → nums[2] 变负
...

最终如果 nums[4] 和 nums[5] 是正数
说明 5 和 6 没出现过
```

## 解法详解

### 为什么可以用负数标记？

原数组的元素都是正数（[1, n]）。我们用正/负来表示两种状态：
- **正数**：这个位置对应的数字**没出现过**
- **负数**：这个位置对应的数字**出现过**

负号只是标记，绝对值仍然是原来的数字，不影响后续遍历。

### 代码实现

```javascript
function findDisappearedNumbers(nums) {
    const n = nums.length;
    
    // 第一次遍历：标记出现过的数字
    for (let i = 0; i < n; i++) {
        // 取绝对值！因为 nums[i] 可能已经被标记为负数
        const index = Math.abs(nums[i]) - 1;
        
        // 只有正数才需要标记（避免重复标记）
        if (nums[index] > 0) {
            nums[index] = -nums[index];
        }
    }
    
    // 第二次遍历：找出未被标记的位置
    const result = [];
    for (let i = 0; i < n; i++) {
        if (nums[i] > 0) {
            // 索引 i 对应数字 i+1
            result.push(i + 1);
        }
    }
    
    return result;
}
```

### 执行过程图解

以 `[4, 3, 2, 7, 8, 2, 3, 1]` 为例：

```
初始数组:
索引: 0  1  2  3  4  5  6  7
值:   4  3  2  7  8  2  3  1

第一次遍历（标记阶段）:

i=0: nums[0]=4 → 标记索引3 (4-1=3)
     nums[3]=7 → nums[3]=-7
     [4, 3, 2, -7, 8, 2, 3, 1]

i=1: nums[1]=3 → 标记索引2 (3-1=2)
     nums[2]=2 → nums[2]=-2
     [4, 3, -2, -7, 8, 2, 3, 1]

i=2: |nums[2]|=2 → 标记索引1 (2-1=1)
     nums[1]=3 → nums[1]=-3
     [4, -3, -2, -7, 8, 2, 3, 1]

i=3: |nums[3]|=7 → 标记索引6 (7-1=6)
     nums[6]=3 → nums[6]=-3
     [4, -3, -2, -7, 8, 2, -3, 1]

i=4: nums[4]=8 → 标记索引7 (8-1=7)
     nums[7]=1 → nums[7]=-1
     [4, -3, -2, -7, 8, 2, -3, -1]

i=5: |nums[5]|=2 → 索引1，nums[1]=-3 已经是负数，跳过

i=6: |nums[6]|=3 → 索引2，nums[2]=-2 已经是负数，跳过

i=7: |nums[7]|=1 → 标记索引0 (1-1=0)
     nums[0]=4 → nums[0]=-4
     [-4, -3, -2, -7, 8, 2, -3, -1]

第二次遍历（收集结果）:
索引: 0   1   2   3   4  5   6   7
值:  -4  -3  -2  -7   8  2  -3  -1
      ✗   ✗   ✗   ✗   ✓  ✓   ✗   ✗

索引4: nums[4]=8 > 0 → 数字 5 消失
索引5: nums[5]=2 > 0 → 数字 6 消失

结果: [5, 6]
```

## 复杂度分析

**时间复杂度：O(n)**
- 两次遍历，每次 O(n)

**空间复杂度：O(1)**
- 只用了几个变量
- 结果数组不算额外空间（题目要求返回）

## 边界情况

```javascript
// 没有消失的数字
findDisappearedNumbers([1])     // []

// 全部消失一半
findDisappearedNumbers([1, 1])  // [2]

// 全部都是同一个数字
findDisappearedNumbers([2, 2])  // [1]
```

## 常见误区

**误区一：忘记取绝对值**

```javascript
// ❌ 错误
const index = nums[i] - 1;  // nums[i] 可能已经是负数！

// ✅ 正确
const index = Math.abs(nums[i]) - 1;
```

**误区二：重复标记**

```javascript
// ❌ 可能出问题：nums[index] 已经是负数，再取负会变回正数
nums[index] = -nums[index];

// ✅ 正确：只有正数才标记
if (nums[index] > 0) {
    nums[index] = -nums[index];
}
```

## 小结

这道题的核心技巧：

1. **值与索引的对应**：数字 k 对应索引 k-1
2. **原地标记**：用正负号表示"出现/未出现"
3. **不破坏信息**：绝对值保留原值，只改变符号

这种技巧叫做**原地哈希**，在空间受限的场景下非常有用。核心思想是：**让数组自己记录信息，不借助额外空间**。
