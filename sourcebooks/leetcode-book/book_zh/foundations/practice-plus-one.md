# 加一

> LeetCode 66. Plus One

给定一个由整数组成的数组 `digits`，表示一个大整数，返回这个整数加一后的结果。

这道题模拟的是我们小学学过的加法：从个位开始，逢十进一。关键在于处理进位，特别是全是 9 的情况。

## 问题描述

```javascript
输入：digits = [1, 2, 3]
输出：[1, 2, 4]
解释：123 + 1 = 124

输入：digits = [9, 9, 9]
输出：[1, 0, 0, 0]
解释：999 + 1 = 1000
```

数组中每个元素都是 0 到 9 的数字，最高位在数组开头。

## 思路分析

### 进位逻辑

从最低位（数组末尾）开始：
1. 如果当前位小于 9，直接加 1 返回
2. 如果当前位等于 9，变成 0，继续看前一位

就像我们手算加法一样：
```
  1 2 9
+     1
-------
  1 3 0
```

9 + 1 = 10，写 0 进 1。2 + 1 = 3，不用再进位。

### 特殊情况：全是 9

当所有位都是 9 时，每一位都会变成 0，最后还需要在最前面加一个 1：

```
  9 9 9
+     1
-------
1 0 0 0
```

这种情况需要**扩展数组**。

## 解法详解

```javascript
function plusOne(digits) {
    const n = digits.length;
    
    // 从最低位开始
    for (let i = n - 1; i >= 0; i--) {
        // 如果当前位小于 9，直接加 1 就完成了
        if (digits[i] < 9) {
            digits[i]++;
            return digits;
        }
        // 当前位是 9，变成 0，继续处理前一位
        digits[i] = 0;
    }
    
    // 走到这里，说明所有位都是 9
    // 原数组已经变成 [0, 0, 0, ...]
    // 需要在最前面加 1
    return [1, ...digits];
}
```

**代码的精妙之处**：

1. **提前返回**：只要遇到小于 9 的位，加 1 后立即返回，不需要继续遍历
2. **自然处理进位**：每遇到 9 就变成 0，继续循环就是在处理进位
3. **最后处理全 9**：循环结束还没返回，说明全是 9

### 执行过程

**例1**：`[1, 2, 9]`

```
i=2: digits[2]=9, 变成0 → [1, 2, 0]
i=1: digits[1]=2 < 9, 加1 → [1, 3, 0]
返回 [1, 3, 0]
```

**例2**：`[9, 9, 9]`

```
i=2: digits[2]=9, 变成0 → [9, 9, 0]
i=1: digits[1]=9, 变成0 → [9, 0, 0]
i=0: digits[0]=9, 变成0 → [0, 0, 0]
循环结束，返回 [1, 0, 0, 0]
```

## 复杂度分析

**时间复杂度：O(n)**
- 最坏情况（全是 9）需要遍历所有位
- 平均情况下，很快就会遇到非 9 的位并提前返回

**空间复杂度**：
- 一般情况：O(1)，原地修改
- 最坏情况（全是 9）：O(n)，需要创建新数组

## 边界情况

```javascript
// 单个 0
plusOne([0])      // [1]

// 单个 9
plusOne([9])      // [1, 0]

// 末尾不是 9
plusOne([1, 2, 3])  // [1, 2, 4]

// 末尾是 9，但不全是 9
plusOne([1, 9, 9])  // [2, 0, 0]

// 全是 9
plusOne([9, 9, 9])  // [1, 0, 0, 0]
```

## 另一种写法

有些人喜欢用 `carry` 变量显式表示进位：

```javascript
function plusOne(digits) {
    let carry = 1;  // 初始进位就是要加的 1
    
    for (let i = digits.length - 1; i >= 0 && carry; i--) {
        const sum = digits[i] + carry;
        digits[i] = sum % 10;
        carry = Math.floor(sum / 10);
    }
    
    if (carry) {
        digits.unshift(1);
    }
    
    return digits;
}
```

这种写法更通用，可以扩展为"加任意数"，但对于"加一"这个特定问题，前面的写法更简洁。

## 小结

这道题的关键点：

1. **从低位到高位处理**：模拟真实的加法过程
2. **提前返回优化**：遇到非 9 的位就可以结束
3. **全 9 特殊处理**：需要扩展数组长度

这是一道看似简单但考察细节的题目。在面试中，关键是要考虑到"全是 9"这种边界情况，很多人会忘记处理数组扩展。
