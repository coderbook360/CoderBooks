# 实战：外观数列

外观数列是一道非常有趣的字符串题目。它的规则乍看有些费解，但一旦理解了"读"数字的方式，你会发现这道题其实在考察你对字符串处理的基本功——如何识别并统计连续相同的字符。

## 问题描述

给定一个正整数 `n`，返回外观数列的第 `n` 项。

外观数列的定义如下：
- 第 1 项是 `"1"`
- 从第 2 项开始，每一项都是对前一项的描述

什么是"对前一项的描述"？就是把前一项"读"出来。比如：
- `"1"` 读作"1个1"，所以写成 `"11"`
- `"11"` 读作"2个1"，所以写成 `"21"`
- `"21"` 读作"1个2，1个1"，所以写成 `"1211"`

**示例 1：**
```
输入：n = 1
输出："1"
```

**示例 2：**
```
输入：n = 4
输出："1211"
```

## 规则详解

### 数列生成过程

我们来看看外观数列前几项是如何一步步生成的：

- **第 1 项**：`"1"`（初始值，就是数字 1）
- **第 2 项**：`"11"`（第 1 项有"1个1"）
- **第 3 项**：`"21"`（第 2 项有"2个1"）
- **第 4 项**：`"1211"`（第 3 项有"1个2，1个1"）
- **第 5 项**：`"111221"`（第 4 项有"1个1，1个2，2个1"）
- **第 6 项**：`"312211"`（第 5 项有"3个1，2个2，1个1"）

### "读"的规则

核心规则其实很简单：**从左到右扫描字符串，统计连续相同字符的个数，然后把"个数+字符"拼接起来**。

以第 5 项 `"111221"` 为例：
1. 前 3 个字符是 `111`，读作"3个1"，记为 `31`
2. 接下来 2 个字符是 `22`，读作"2个2"，记为 `22`
3. 最后 1 个字符是 `1`，读作"1个1"，记为 `11`
4. 拼接起来就是 `"312211"`，即第 6 项

理解了这个规则，算法就呼之欲出了：我们需要一个函数，能够把当前字符串"读"成下一个字符串。然后从第 1 项开始，连续调用这个函数 `n-1` 次，就能得到第 `n` 项。

## 解法详解

### 解法一：迭代法（推荐）

迭代法的思路非常直接：用一个循环从第 2 项迭代到第 n 项，每次根据当前字符串生成下一个字符串。

生成下一个字符串的核心是双指针技巧：用一个指针遍历字符串，遇到不同的字符时，就把之前统计的"个数+字符"追加到结果中。

```javascript
/**
 * @param {number} n
 * @return {string}
 */
function countAndSay(n) {
    let result = "1";  // 第 1 项
    
    for (let i = 2; i <= n; i++) {
        let next = "";
        let j = 0;
        
        while (j < result.length) {
            const char = result[j];
            let count = 0;
            
            // 统计连续相同字符的个数
            while (j < result.length && result[j] === char) {
                count++;
                j++;
            }
            
            // 拼接 "个数 + 字符"
            next += count.toString() + char;
        }
        
        result = next;
    }
    
    return result;
}
```

**代码解读：**

- `result` 初始化为 `"1"`，即第 1 项
- 外层循环从 2 到 n，每次根据 `result` 生成下一项
- 内层用双指针统计连续相同字符：`char` 记录当前字符，`count` 统计个数
- 当遇到不同字符时，把 `count + char` 追加到 `next`，然后继续处理下一组

### 解法二：递归法

递归法利用了"第 n 项依赖第 n-1 项"这个递推关系。我们先递归求出第 n-1 项，然后"读"出第 n 项。

```javascript
function countAndSay(n) {
    // 基本情况：第 1 项是 "1"
    if (n === 1) return "1";
    
    // 递归获取前一项
    const prev = countAndSay(n - 1);
    
    // "读"前一项，生成当前项
    let result = "";
    let i = 0;
    
    while (i < prev.length) {
        const char = prev[i];
        let count = 0;
        
        while (i < prev.length && prev[i] === char) {
            count++;
            i++;
        }
        
        result += count + char;
    }
    
    return result;
}
```

递归法的代码结构更清晰，但由于递归调用的开销，实际性能略逊于迭代法。对于 LeetCode 的输入范围（n ≤ 30），两种方法都能通过。

### 解法三：正则表达式

JavaScript 的正则表达式有一个强大的功能：`replace` 方法的第二个参数可以是一个函数，用来处理每个匹配项。我们可以用正则 `/(.)\1*/g` 匹配连续相同的字符，然后替换成"个数+字符"。

```javascript
function countAndSay(n) {
    let result = "1";
    
    for (let i = 2; i <= n; i++) {
        // 匹配连续相同字符，替换成 "个数 + 字符"
        result = result.replace(/(.)\1*/g, (match) => {
            return match.length + match[0];
        });
    }
    
    return result;
}
```

**正则解析：**
- `(.)` 匹配任意一个字符，并捕获为第 1 组
- `\1*` 匹配与第 1 组相同的字符，出现 0 次或多次
- 合起来就是"匹配连续相同的字符"
- `g` 标志表示全局匹配，即匹配所有的连续字符组

这是最简洁的写法，但对于不熟悉正则的同学可能不太直观。面试时建议使用迭代法，更容易解释清楚。

## 复杂度分析

设第 n 项的长度为 L(n)。

**时间复杂度**：O(L(1) + L(2) + ... + L(n))

每一项的生成都需要遍历前一项，所以总时间是所有项长度之和。L(n) 的增长速度大约是指数级的，但由于 n ≤ 30，实际运行时间很短。

**空间复杂度**：O(L(n))

主要是存储结果字符串的空间。递归法还需要 O(n) 的调用栈空间。

## 小结

外观数列这道题的核心在于**理解"读"的规则，并用双指针统计连续相同字符**。一旦理解了规则，无论是迭代、递归还是正则，实现起来都不难。

这道题给我们的启示是：有些题目看起来很复杂，但只要把规则理清楚，拆解成基本操作，就能迎刃而解。字符串处理题往往就是这样——关键不在于算法有多高深，而在于你能否准确理解题意、正确处理边界情况。

**关键技巧回顾：**
- 双指针统计连续相同字符
- 迭代 vs 递归的选择
- 正则表达式的巧妙应用
