# 移动零

> LeetCode 283. Move Zeroes

给定一个数组，将所有的 0 移动到数组末尾，同时保持非零元素的相对顺序。

这道题是双指针的经典应用，和上一题"删除重复项"有异曲同工之妙，但有一个关键区别：不能简单地覆盖，而要保留所有元素。

## 问题描述

```javascript
输入：nums = [0, 1, 0, 3, 12]
输出：[1, 3, 12, 0, 0]
```

**要求**：
- 必须原地操作，不能创建新数组
- 保持非零元素的相对顺序

## 思路分析

先回顾一下上一题"删除重复项"的做法：
- 慢指针维护"结果区域"
- 快指针扫描寻找新元素
- 发现新元素就覆盖到慢指针位置

这道题的区别在于：**我们不能丢弃 0，而是要把它们"移"到后面**。

想象你在整理一排物品，要把空盒子（0）都移到最右边：
- 方法一：先把有东西的盒子往左挤，然后在右边补空盒子
- 方法二：遇到有东西的盒子就和左边的空盒子交换位置

## 解法详解

### 解法一：两次遍历

思路直观：先把非零元素全部移到前面，然后把后面的位置填 0。

```javascript
function moveZeroes(nums) {
    let slow = 0;
    
    // 第一次遍历：将非零元素移到前面
    for (let fast = 0; fast < nums.length; fast++) {
        if (nums[fast] !== 0) {
            nums[slow] = nums[fast];
            slow++;
        }
    }
    
    // slow 现在指向第一个应该放 0 的位置
    // 第二次遍历：填充零
    for (let i = slow; i < nums.length; i++) {
        nums[i] = 0;
    }
}
```

**执行过程**：

```
输入: [0, 1, 0, 3, 12]

第一次遍历（移动非零元素）:
fast=0: nums[0]=0，跳过
fast=1: nums[1]=1≠0，nums[0]=1，slow=1
fast=2: nums[2]=0，跳过
fast=3: nums[3]=3≠0，nums[1]=3，slow=2
fast=4: nums[4]=12≠0，nums[2]=12，slow=3

中间状态: [1, 3, 12, 3, 12]，slow=3

第二次遍历（填充零）:
i=3: nums[3]=0
i=4: nums[4]=0

最终: [1, 3, 12, 0, 0]
```

### 解法二：一次遍历 + 交换（推荐）

更优雅的做法：遇到非零元素，就和 slow 位置交换。

```javascript
function moveZeroes(nums) {
    let slow = 0;
    
    for (let fast = 0; fast < nums.length; fast++) {
        if (nums[fast] !== 0) {
            // 交换 nums[slow] 和 nums[fast]
            [nums[slow], nums[fast]] = [nums[fast], nums[slow]];
            slow++;
        }
    }
}
```

**为什么交换是安全的**？

关键洞察：当 `fast > slow` 时，`nums[slow]` 一定是 0。

- 如果 `slow` 位置原本是非零元素，那 `slow` 早就前进了
- 所以 `slow` 停留的位置，要么是 0，要么就是 `slow === fast`

当 `slow === fast` 时，交换等于自己和自己交换，不影响结果。

## 执行过程图解

以 `[0, 1, 0, 3, 12]` 为例，使用交换法：

```
初始状态：slow=0, fast=0
[0, 1, 0, 3, 12]
 ↑
 s,f

fast=0: nums[0]=0，是零，跳过
        fast++ → fast=1

fast=1: nums[1]=1≠0，交换 nums[0] 和 nums[1]
[1, 0, 0, 3, 12]
 ↑  ↑
 s  f
        slow++ → slow=1
        fast++ → fast=2

fast=2: nums[2]=0，是零，跳过
        fast++ → fast=3

fast=3: nums[3]=3≠0，交换 nums[1] 和 nums[3]
[1, 3, 0, 0, 12]
    ↑     ↑
    s     f
        slow++ → slow=2
        fast++ → fast=4

fast=4: nums[4]=12≠0，交换 nums[2] 和 nums[4]
[1, 3, 12, 0, 0]
       ↑      ↑
       s      f
        slow++ → slow=3

遍历结束，最终结果: [1, 3, 12, 0, 0]
```

交换法的精妙之处在于：**零会自然地"冒泡"到数组末尾**。

## 复杂度分析

**两种解法的复杂度**：

**解法一（两次遍历）**：
- 时间复杂度：O(n) — 两次遍历，常数系数为 2
- 空间复杂度：O(1)

**解法二（一次遍历+交换）**：
- 时间复杂度：O(n) — 只需一次遍历
- 空间复杂度：O(1)

解法二的常数因子更小，是更优的选择。

## 边界情况

```javascript
// 单元素
moveZeroes([0])    // [0]，已经满足条件
moveZeroes([1])    // [1]，已经满足条件

// 全是零
moveZeroes([0, 0, 0])  // [0, 0, 0]，不需要移动

// 没有零
moveZeroes([1, 2, 3])  // [1, 2, 3]，不需要移动

// 零在开头
moveZeroes([0, 0, 1])  // [1, 0, 0]

// 零在结尾
moveZeroes([1, 2, 0])  // [1, 2, 0]，已经满足条件
```

## 两种解法的对比

| 对比维度 | 解法一（两次遍历） | 解法二（一次遍历+交换） |
|---------|------------------|---------------------|
| 遍历次数 | 2 | 1 |
| 写操作次数 | 非零元素数 + 零的数量 | 非零元素数 × 2（交换） |
| 代码简洁性 | 逻辑清晰 | 更简洁 |
| 推荐场景 | 初学者理解 | 实际使用 |

## 小结

这道题的核心技巧：

1. **双指针分离元素**：慢指针维护"已处理区域"，快指针扫描原数组
2. **交换保持完整性**：交换操作让零自然移到后面，不会丢失任何元素
3. **一次遍历更优**：交换法只需一次遍历，常数因子更小

对比"删除重复项"，你会发现双指针有两种模式：
- **覆盖模式**：不需要保留被移除的元素
- **交换模式**：需要保留所有元素，只是改变位置

根据问题需求选择合适的模式，是解决数组问题的关键技能。
