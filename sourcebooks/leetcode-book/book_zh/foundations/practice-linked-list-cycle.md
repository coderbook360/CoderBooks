# 环形链表

> LeetCode 141. Linked List Cycle

给你一个链表的头节点，判断链表中是否有环。

这道题是快慢指针的经典应用——**Floyd 判圈算法**，也叫"龟兔赛跑"算法。

## 问题描述

```javascript
输入：head = [3, 2, 0, -4], pos = 1
输出：true
解释：链表中有一个环，尾节点连接到第 1 个节点（索引从 0 开始）

输入：head = [1, 2], pos = 0
输出：true
解释：链表中有一个环，尾节点连接到第 0 个节点

输入：head = [1], pos = -1
输出：false
解释：链表中没有环
```

**注意**：`pos` 只是说明环的位置，不作为参数传入函数。

## 思路分析

### 哈希表解法（简单但不推荐）

遍历链表，用 Set 记录访问过的节点。如果遇到已访问过的节点，说明有环。

```javascript
function hasCycle(head) {
    const seen = new Set();
    
    while (head) {
        if (seen.has(head)) {
            return true;
        }
        seen.add(head);
        head = head.next;
    }
    
    return false;
}
```

时间 O(n)，空间 O(n)。能不能做到 O(1) 空间？

### 快慢指针解法

用两个指针遍历：
- **慢指针**：每次走 1 步
- **快指针**：每次走 2 步

如果有环，快指针最终会"追上"慢指针。就像在操场跑步，跑得快的人会套圈跑得慢的人。

## 为什么快慢指针能检测环？

### 直观理解

想象两个人在环形跑道上跑步：
- 如果跑道有终点（无环），跑得快的人先到终点
- 如果跑道是环形的，跑得快的人会追上跑得慢的人

在链表中：
- 无环：快指针先到达 `null`
- 有环：快指针会在环内追上慢指针

### 数学证明

假设链表结构如下：

```
非环部分: a 个节点
环的长度: b 个节点

  ●─●─●─●─●─●
  │         │
  └─●─●─●───┘
  ↑         ↑
  环入口    相遇点可能在环内任意位置
```

当两指针相遇时：
- 设慢指针走了 s 步
- 快指针走了 2s 步（速度是慢指针的 2 倍）
- 快指针比慢指针多走了 s 步

因为快指针在环内追慢指针，多走的步数一定是环长度的整数倍：
```
s = k × b  (k 为正整数)
```

这意味着：当 s = b 时（最少），两者就会相遇。

**关键结论**：如果有环，快指针一定会在有限步内追上慢指针。

### 为什么不会"跳过"？

有人担心：快指针每次走 2 步，会不会"跳过"慢指针？

不会。设某一时刻快指针在慢指针后面 d 步：
- 下一步后，快指针比慢指针多走 1 步
- 距离从 d 变成 d-1

每次距离减少 1，所以快指针一定会追上（d = 0），不会跳过。

## 解法详解

### 代码实现

```javascript
function hasCycle(head) {
    // 边界情况
    if (!head || !head.next) return false;
    
    let slow = head;
    let fast = head;
    
    while (fast && fast.next) {
        slow = slow.next;        // 慢指针走 1 步
        fast = fast.next.next;   // 快指针走 2 步
        
        if (slow === fast) {
            return true;  // 相遇，有环
        }
    }
    
    return false;  // fast 到达末尾，无环
}
```

### 执行过程图解

**有环的情况**：

```
链表: 1 → 2 → 3 → 4
              ↑   ↓
              6 ← 5

初始: slow = 1, fast = 1

步骤1: slow = 2, fast = 3
       1 → 2 → 3 → 4
           ↑       ↓
           6 ← 5
           s     f

步骤2: slow = 3, fast = 5
       1 → 2 → 3 → 4
               ↑   ↓
               6 ← 5
               s   f

步骤3: slow = 4, fast = 3
       1 → 2 → 3 → 4
               f   ↓
               6 ← 5
                   s

步骤4: slow = 5, fast = 5
       1 → 2 → 3 → 4
                   ↓
               6 ← 5
                  s,f

slow === fast，返回 true
```

**无环的情况**：

```
链表: 1 → 2 → 3 → null

初始: slow = 1, fast = 1

步骤1: slow = 2, fast = 3
步骤2: slow = 3, fast = null

fast.next 不存在，循环结束
返回 false
```

## 复杂度分析

**时间复杂度：O(n)**
- 无环：fast 最多走 n 步到达末尾
- 有环：慢指针最多走 n 步，快指针就会追上

**空间复杂度：O(1)**
- 只用了两个指针变量

## 边界情况

```javascript
// 空链表
hasCycle(null)  // false

// 单节点，无环
hasCycle(1 → null)  // false

// 单节点，自环
// 1 → 1（next 指向自己）
hasCycle(...)  // true

// 两节点，互相指向
// 1 ⇄ 2
hasCycle(...)  // true
```

## 小结

这道题的核心是 **Floyd 判圈算法**：

**算法思想**：
- 快指针每次走 2 步，慢指针每次走 1 步
- 有环：快指针会追上慢指针
- 无环：快指针先到达末尾

**数学本质**：
- 快指针每次比慢指针多走 1 步
- 如果有环，这个"距离差"最终会累积到环的长度

**应用场景**：
- 判断链表是否有环
- 找环的入口点（进阶）
- 判断两个链表是否相交（转化为判环问题）

快慢指针是链表问题的核心技巧，务必牢牢掌握。
