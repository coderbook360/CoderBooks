# 实战：报数序列

报数序列问题是编程界的一个经典话题，它有一个更广为人知的名字——**约瑟夫环问题**。这个问题据说源自古罗马时代的一个传说，至今仍是算法面试中的常客。

## 问题描述

N 个人围成一圈，从第 1 个人开始报数，报到 M 的人出列。然后从出列者的下一个人重新开始从 1 报数，报到 M 的人再出列。如此循环，直到所有人都出列。

**问题 1**：求出列的顺序。

**问题 2**：求最后一个出列的人是谁（即"幸存者"）。

**示例：**
```
输入：n = 5, m = 3
过程：
- 初始：1, 2, 3, 4, 5
- 报到3，3号出列：1, 2, 4, 5
- 从4开始报数，报到3（即6号，但6不存在，所以是1号）：1号出列
- 从2开始报数，报到3（即5号）：5号出列
- 从2开始报数，报到3（即4号）：4号出列
- 最后剩下2号

出列顺序：[3, 1, 5, 4, 2]
最后幸存者：2
```

## 思路分析

### 模拟法

最直观的想法就是**模拟整个报数过程**：
1. 用一个数组表示还在圈中的人
2. 用一个指针模拟当前报数位置
3. 每次数到 M，就把这个人从数组中移除
4. 重复直到数组为空

模拟法的优点是逻辑清晰、容易理解；缺点是每次移除元素需要 O(n) 时间，总时间复杂度是 O(n²)。

### 数学递推法

约瑟夫环问题有一个漂亮的数学解法。假设我们已经知道了 n-1 个人时幸存者的位置，能否推出 n 个人时幸存者的位置？

设 f(n) 为 n 个人时最后幸存者的编号（0-indexed），则有递推公式：

```
f(1) = 0
f(n) = (f(n-1) + m) % n
```

**为什么？**

当第一个人（编号为 m-1）出列后，剩下 n-1 个人。此时的编号变成了：
```
m, m+1, ..., n-1, 0, 1, ..., m-2
```

如果我们把这些人重新编号为 0, 1, 2, ..., n-2，那么这就是一个 n-1 个人的约瑟夫环问题。设这个子问题的解是 f(n-1)，我们需要把它映射回原来的编号：
```
原编号 = (f(n-1) + m) % n
```

这就是递推公式的由来。用这个公式，我们只需要 O(n) 时间、O(1) 空间就能求出幸存者。

## 解法详解

### 解法一：数组模拟

```javascript
/**
 * 约瑟夫环问题：求出列顺序
 * @param {number} n - 总人数
 * @param {number} m - 报到 m 出列
 * @return {number[]} - 出列顺序（1-indexed）
 */
function josephus(n, m) {
    const result = [];
    // 初始化：1 到 n 号
    const people = Array.from({ length: n }, (_, i) => i + 1);
    
    let index = 0;  // 当前报数位置
    
    while (people.length > 0) {
        // 计算出列位置：从 index 开始数 m 个
        index = (index + m - 1) % people.length;
        
        // 移除并记录
        result.push(people.splice(index, 1)[0]);
        
        // 注意：splice 后 index 不需要调整
        // 因为后面的元素会自动前移，index 已经指向"下一个人"
    }
    
    return result;
}
```

**关键点解释：**

- `(index + m - 1) % people.length`：为什么是 `m - 1` 而不是 `m`？因为当前 `index` 位置的人算第 1 个，所以只需要再数 `m - 1` 个。
- `splice` 之后不需要调整 `index`：因为删除元素后，后面的元素会自动前移。如果删除的是最后一个元素，`index` 会等于新的数组长度，这时取模会自动回到 0。

### 解法二：数学递推（求幸存者）

如果只需要求最后一个出列的人，使用数学递推更高效：

```javascript
/**
 * 约瑟夫环：求最后幸存者的编号
 * @param {number} n - 总人数
 * @param {number} m - 报到 m 出列
 * @return {number} - 幸存者编号（1-indexed）
 */
function lastRemaining(n, m) {
    let pos = 0;  // 1个人时，幸存者位置为 0
    
    // 从 2 个人递推到 n 个人
    for (let i = 2; i <= n; i++) {
        pos = (pos + m) % i;
    }
    
    return pos + 1;  // 转为 1-indexed
}
```

这个解法只用了一个变量 `pos`，空间复杂度 O(1)，时间复杂度 O(n)。

**递推过程示例（n=5, m=3）：**
- i=2：pos = (0 + 3) % 2 = 1
- i=3：pos = (1 + 3) % 3 = 1
- i=4：pos = (1 + 3) % 4 = 0
- i=5：pos = (0 + 3) % 5 = 3

所以 5 个人时，幸存者的 0-indexed 位置是 3，即第 4 号。

### 解法三：递归版数学解

递推公式也可以用递归形式写：

```javascript
function lastRemaining(n, m) {
    if (n === 1) return 1;
    return (lastRemaining(n - 1, m) + m - 1) % n + 1;
}
```

注意这里的公式稍有不同，是因为直接用 1-indexed 计算。递归写法更简洁，但 n 较大时可能栈溢出。

## 边界情况

- **n = 1**：只有一个人，直接出列，返回 `[1]` 或幸存者为 1
- **m = 1**：依次出列，顺序就是 `[1, 2, 3, ..., n]`
- **m > n**：完全合法，`m % n` 就是实际的报数位置

## 复杂度分析

**模拟法：**
- 时间复杂度：O(n²)。每次删除需要 O(n)，共删除 n 次。
- 空间复杂度：O(n)。存储人员数组。

**数学递推法：**
- 时间复杂度：O(n)。只需要一次遍历。
- 空间复杂度：O(1)。只用了常数空间。

## 约瑟夫环的历史背景

约瑟夫环问题得名于古罗马历史学家约瑟夫斯（Flavius Josephus）。据说在公元 67 年，约瑟夫和 40 名士兵被罗马军队围困在洞穴里。他们宁死不屈，决定围成一圈，每隔两人杀一人，直到只剩最后一人自杀。

约瑟夫不想死，于是他迅速计算出自己应该站在哪个位置才能成为最后的幸存者。最终，他和另一个人活了下来，并选择投降而非自杀。

这个故事的真实性已无从考证，但约瑟夫环问题却成为了算法领域的经典题目，考察了模拟、数学推导、递归等多种技能。

## 小结

约瑟夫环问题是一道非常有教育意义的题目：
1. **模拟法**让我们练习了数组操作和循环逻辑
2. **数学递推法**展示了如何通过分析问题结构找到高效解法
3. **历史故事**让这道枯燥的算法题变得有趣

面试时，如果只需要求幸存者，推荐使用数学递推法；如果需要求出列顺序，就老老实实模拟。无论哪种方法，关键是要理解取模运算在"环形"问题中的作用——它能把线性的数组变成一个逻辑上的环。

**关键技巧回顾：**
- 取模运算实现环形遍历
- 数学递推优化时间复杂度
- 理解递推公式的推导过程
