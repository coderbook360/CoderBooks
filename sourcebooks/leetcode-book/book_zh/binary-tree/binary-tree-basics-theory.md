# 二叉树基础理论

二叉树是算法世界中最重要的数据结构之一。它不仅是面试的高频考点，更是理解许多高级数据结构（如堆、BST、红黑树）的基础。掌握二叉树，你就掌握了一把打开算法大门的钥匙。

## 什么是二叉树

二叉树是一种树形数据结构，它的特点是：**每个节点最多有两个子节点**，分别称为左子节点和右子节点。

```javascript
class TreeNode {
    constructor(val = 0, left = null, right = null) {
        this.val = val;
        this.left = left;
        this.right = right;
    }
}
```

这个简单的定义蕴含着巨大的威力。通过递归地连接节点，我们可以构建出各种复杂的层次结构。

## 基本术语

在深入学习之前，我们需要先统一一些术语：

**节点相关：**
- **根节点（Root）**：树的顶层节点，没有父节点
- **叶子节点（Leaf）**：没有子节点的节点
- **内部节点**：至少有一个子节点的节点
- **父节点/子节点**：直接相连的上下级关系
- **兄弟节点**：拥有相同父节点的节点

**结构相关：**
- **深度（Depth）**：从根节点到该节点的边数
- **高度（Height）**：从该节点到最远叶子节点的边数
- **层（Level）**：深度相同的节点在同一层（根节点在第 0 层或第 1 层，取决于定义）
- **子树（Subtree）**：以某节点为根的树

**举例说明：**
```
        1        <- 根节点，深度0，高度2
       / \
      2   3      <- 深度1
     / \
    4   5        <- 叶子节点，深度2，高度0
```

在这棵树中：
- 节点 1 是根节点，高度为 2
- 节点 4 和 5 是叶子节点
- 节点 2 的子树包含节点 2、4、5

## 特殊的二叉树类型

二叉树有几种特殊形态，它们在算法中非常常见：

### 满二叉树（Full Binary Tree）

满二叉树中，**每个节点要么是叶子节点，要么有两个子节点**。也就是说，不存在只有一个子节点的情况。

```
        1
       / \
      2   3
     / \ / \
    4  5 6  7
```

### 完全二叉树（Complete Binary Tree）

完全二叉树的定义是：**除了最后一层，其他每层的节点都是满的；最后一层的节点都靠左排列**。

```
        1
       / \
      2   3
     / \
    4   5
```

完全二叉树的一个重要性质是：可以用数组高效存储。如果节点在数组中的索引是 `i`，那么：
- 左子节点的索引是 `2i + 1`
- 右子节点的索引是 `2i + 2`
- 父节点的索引是 `(i - 1) / 2`（向下取整）

堆就是用这种方式实现的。

### 平衡二叉树（Balanced Binary Tree）

平衡二叉树要求：**任意节点的左右子树高度差不超过 1**。

```
        1
       / \
      2   3
     /
    4
```
这是一棵平衡二叉树，因为每个节点的左右子树高度差都 ≤ 1。

```
        1
       /
      2
     /
    3
```
这**不是**平衡二叉树，因为节点 1 的左子树高度为 2，右子树高度为 0，差值为 2。

平衡二叉树的查找、插入、删除操作都能保持 O(log n) 的时间复杂度，这是 AVL 树、红黑树等高级数据结构的基础。

### 退化的二叉树（Degenerate Tree）

当每个节点都只有一个子节点时，二叉树就退化成了链表：

```
    1
     \
      2
       \
        3
         \
          4
```

这种情况下，所有操作的时间复杂度都退化为 O(n)，是我们需要避免的。

## 二叉树的存储方式

### 链式存储

我们在开头定义的 `TreeNode` 类就是链式存储的实现：

```javascript
const root = new TreeNode(1);
root.left = new TreeNode(2);
root.right = new TreeNode(3);
root.left.left = new TreeNode(4);
root.left.right = new TreeNode(5);
```

优点：灵活，能表示任意形态的二叉树。
缺点：需要额外的指针空间。

### 数组存储

对于完全二叉树，可以用数组高效存储：

```javascript
// 完全二叉树 [1,2,3,4,5] 的数组表示
const tree = [1, 2, 3, 4, 5];
// tree[0] = 1 是根
// tree[1] = 2 是 1 的左子节点
// tree[2] = 3 是 1 的右子节点
// tree[3] = 4 是 2 的左子节点
// tree[4] = 5 是 2 的右子节点
```

LeetCode 题目中经常用这种方式表示二叉树输入，`null` 表示空节点。

## 二叉树的性质

理解这些性质有助于分析算法的复杂度：

**基本性质：**
- 第 i 层最多有 $2^i$ 个节点（假设根节点在第 0 层）
- 深度为 k 的二叉树最多有 $2^{k+1} - 1$ 个节点
- n 个节点的完全二叉树高度为 $\lfloor \log_2 n \rfloor$

**节点数与边数的关系：**
- n 个节点的二叉树有 n - 1 条边
- 设叶子节点数为 $n_0$，度为 2 的节点数为 $n_2$，则 $n_0 = n_2 + 1$

这些性质在分析算法复杂度时非常有用。比如，完全二叉树的高度是 O(log n)，这就是为什么堆的插入和删除操作是 O(log n) 的原因。

## 递归思维：理解二叉树的关键

二叉树天生具有递归结构：**一棵二叉树 = 根节点 + 左子树 + 右子树**。

这意味着，几乎所有二叉树问题都可以用递归来解决。递归的核心思想是：

1. **定义子问题**：左子树和右子树都是规模更小的子问题
2. **确定基本情况**：空节点是递归的终止条件
3. **合并结果**：根据左右子树的结果，计算当前节点的答案

举个例子，计算二叉树的节点总数：

```javascript
function countNodes(root) {
    if (!root) return 0;  // 基本情况：空树有 0 个节点
    // 递归计算左右子树的节点数，加上当前节点
    return 1 + countNodes(root.left) + countNodes(root.right);
}
```

这段代码虽然简单，但体现了二叉树递归的精髓：**相信递归调用会返回正确的结果**，然后在此基础上解决当前问题。
