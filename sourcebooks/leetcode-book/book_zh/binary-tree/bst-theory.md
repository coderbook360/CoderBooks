# 二叉搜索树理论

二叉搜索树（Binary Search Tree，简称BST）是一种特殊的二叉树，它将"有序"和"树结构"完美结合。理解BST，是掌握后续AVL树、红黑树等高级数据结构的基础。

## 什么是二叉搜索树？

BST的定义非常简单，只有一条规则：

**对于树中的任意节点，左子树所有节点的值 < 当前节点的值 < 右子树所有节点的值。**

这个规则递归地适用于所有子树。

```
        8
       / \
      3   10
     / \    \
    1   6   14
       / \  /
      4  7 13
```

在这棵BST中：
- 根节点8的左子树所有节点（1, 3, 4, 6, 7）都小于8
- 根节点8的右子树所有节点（10, 13, 14）都大于8
- 这个规则同样适用于节点3、10等所有子树

## BST的核心特性：中序遍历有序

BST最重要的特性是：**中序遍历的结果是有序的**。

对上面的BST进行中序遍历：
```
左 → 根 → 右
1 → 3 → 4 → 6 → 7 → 8 → 10 → 13 → 14
```

结果是一个升序序列！这个特性在很多算法中都会用到。

### 为什么中序遍历有序？

回顾中序遍历的顺序：先访问左子树，再访问根，最后访问右子树。

结合BST的定义（左 < 根 < 右），中序遍历天然就是从小到大的顺序。

## BST vs 有序数组

你可能会问：有序数组也能快速查找，为什么需要BST？

关键在于**插入和删除的效率**：

| 操作 | 有序数组 | BST（平衡时） |
|------|----------|--------------|
| 查找 | O(log n) | O(log n) |
| 插入 | O(n) | O(log n) |
| 删除 | O(n) | O(log n) |

有序数组插入一个元素，需要移动后续所有元素，这是O(n)操作。而BST只需要找到正确位置，修改指针即可。

## BST的查找操作

在BST中查找一个值，比在无序结构中查找要高效得多：

```javascript
function searchBST(root, val) {
    // 基础情况：空树或找到目标
    if (!root || root.val === val) {
        return root;
    }
    
    // 利用BST性质：小的在左，大的在右
    if (val < root.val) {
        return searchBST(root.left, val);
    } else {
        return searchBST(root.right, val);
    }
}
```

每次比较后，我们都能排除一半的节点，这就是二分查找的思想。

### 迭代版本

```javascript
function searchBST(root, val) {
    let curr = root;
    while (curr) {
        if (curr.val === val) {
            return curr;
        }
        curr = val < curr.val ? curr.left : curr.right;
    }
    return null;
}
```

## BST的时间复杂度分析

BST的操作效率取决于**树的高度**。

**理想情况（平衡BST）**：
```
        4
       / \
      2   6
     / \ / \
    1  3 5  7

高度 = log₂(n)
```

**最坏情况（退化为链表）**：
```
1
 \
  2
   \
    3
     \
      4

高度 = n
```

| 操作 | 平衡BST | 退化BST |
|------|---------|---------|
| 查找 | O(log n) | O(n) |
| 插入 | O(log n) | O(n) |
| 删除 | O(log n) | O(n) |

这就是为什么实际应用中，我们通常使用AVL树或红黑树来保证平衡。

## BST的应用场景

1. **数据库索引**：B树和B+树是BST的扩展
2. **内存中的有序集合**：如Java的TreeMap、TreeSet
3. **优先队列的实现**：虽然堆更常用，但BST也可以
4. **范围查询**：找出某个范围内的所有元素

## BST的局限性

1. **可能退化**：顺序插入会导致退化为链表
2. **不支持重复值**：标准BST不允许重复，需要特殊处理
3. **需要可比较的键**：节点的值必须能够比较大小

## 小结

BST是算法世界中最重要的数据结构之一，理解它的关键点：

- **定义**：左子树 < 根 < 右子树
- **核心特性**：中序遍历有序
- **效率**：平衡时查找、插入、删除都是O(log n)
- **风险**：可能退化为链表，需要平衡机制
