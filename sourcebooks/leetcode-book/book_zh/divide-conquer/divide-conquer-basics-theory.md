# 分治算法基础理论

分治算法是一种优雅而强大的算法设计范式，它将复杂问题分解为更小的子问题来解决。

## 什么是分治？

**分治**（Divide and Conquer）的核心思想：

1. **分解**（Divide）：将问题分解为若干个规模更小的子问题
2. **解决**（Conquer）：递归地解决子问题
3. **合并**（Combine）：将子问题的解合并成原问题的解

## 分治的特点

### 子问题独立

分治要求子问题之间相互独立，不共享状态。

如果子问题有重叠，动态规划可能更合适。

### 子问题同构

子问题的结构与原问题相同，只是规模更小。

这允许使用递归来解决。

### 可以合并

子问题的解可以有效地合并成原问题的解。

合并的效率决定了整个算法的效率。

## 经典例子：归并排序

```javascript
function mergeSort(arr) {
    if (arr.length <= 1) return arr;
    
    // 分解
    const mid = Math.floor(arr.length / 2);
    const left = arr.slice(0, mid);
    const right = arr.slice(mid);
    
    // 解决
    const sortedLeft = mergeSort(left);
    const sortedRight = mergeSort(right);
    
    // 合并
    return merge(sortedLeft, sortedRight);
}

function merge(left, right) {
    const result = [];
    let i = 0, j = 0;
    
    while (i < left.length && j < right.length) {
        if (left[i] <= right[j]) {
            result.push(left[i++]);
        } else {
            result.push(right[j++]);
        }
    }
    
    return result.concat(left.slice(i)).concat(right.slice(j));
}
```

归并排序完美展示了分治的三个步骤：
- 分解：将数组分成两半
- 解决：递归排序两半
- 合并：将两个有序数组合并

## 分治 vs 递归

递归是一种实现技术，分治是一种算法范式。

分治通常用递归实现，但不是所有递归都是分治。

## 分治 vs 动态规划

| 特性 | 分治 | 动态规划 |
|-----|------|---------|
| 子问题 | 独立 | 重叠 |
| 解决顺序 | 自顶向下 | 自底向上（通常） |
| 存储 | 不需要 | 需要存储子问题解 |
| 典型问题 | 排序、搜索 | 最优化问题 |

## 分治的适用场景

### 1. 排序问题

归并排序、快速排序都是分治。

### 2. 搜索问题

二分查找是最简单的分治。

### 3. 数学计算

大数乘法、矩阵乘法、快速幂。

### 4. 组合问题

生成排列、组合、子集。

## 设计分治算法的步骤

1. **定义子问题**：如何把问题分成更小的同类问题？
2. **确定基准情况**：什么时候问题足够小，可以直接解决？
3. **设计合并方法**：如何将子问题的解组合起来？
4. **分析复杂度**：用主定理分析时间复杂度。

## 分治的局限性

### 递归开销

函数调用有开销，递归深度太大可能栈溢出。

### 合并开销

如果合并步骤太复杂，可能抵消分治的优势。

### 子问题重叠

如果子问题重叠严重，分治会重复计算，这时应该用动态规划。

## 小结

分治是一种"化繁为简"的思想：
- 把大问题分成小问题
- 递归解决小问题
- 把小答案合成大答案
