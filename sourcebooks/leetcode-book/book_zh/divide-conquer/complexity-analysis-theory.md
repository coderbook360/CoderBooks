# 分治算法的时间复杂度分析

分治算法的时间复杂度通常用递推关系式表示，然后用主定理求解。

## 递推关系式

分治算法的时间复杂度通常可以表示为：

**T(n) = aT(n/b) + f(n)**

其中：
- **a**：子问题的个数
- **n/b**：每个子问题的规模
- **f(n)**：分解和合并的代价

## 主定理（Master Theorem）

对于递推式 T(n) = aT(n/b) + f(n)，设 d = log_b(a)：

**情况1**：如果 f(n) = O(n^c)，且 c < d
- 则 T(n) = Θ(n^d)

**情况2**：如果 f(n) = Θ(n^d)
- 则 T(n) = Θ(n^d * log n)

**情况3**：如果 f(n) = Ω(n^c)，且 c > d
- 则 T(n) = Θ(f(n))

## 经典例子分析

### 归并排序

```
T(n) = 2T(n/2) + O(n)

a=2, b=2, f(n)=n
d = log_2(2) = 1
f(n) = Θ(n^1) = Θ(n^d)

属于情况2：T(n) = Θ(n log n)
```

### 二分查找

```
T(n) = T(n/2) + O(1)

a=1, b=2, f(n)=1
d = log_2(1) = 0
f(n) = Θ(n^0) = Θ(n^d)

属于情况2：T(n) = Θ(log n)
```

### 快速排序（平均情况）

```
T(n) = 2T(n/2) + O(n)

与归并排序相同：T(n) = Θ(n log n)
```

### Strassen矩阵乘法

```
T(n) = 7T(n/2) + O(n²)

a=7, b=2, f(n)=n²
d = log_2(7) ≈ 2.807
f(n) = O(n²), 2 < 2.807

属于情况1：T(n) = Θ(n^2.807)
```

比朴素的O(n³)更快。

### Karatsuba大数乘法

```
T(n) = 3T(n/2) + O(n)

a=3, b=2, f(n)=n
d = log_2(3) ≈ 1.585
f(n) = O(n^1), 1 < 1.585

属于情况1：T(n) = Θ(n^1.585)
```

比朴素的O(n²)更快。

## 不适用主定理的情况

### 子问题规模不均匀

快速排序最坏情况：T(n) = T(n-1) + O(n)

这不符合T(n/b)的形式，需要用其他方法分析，结果是O(n²)。

### f(n)不是多项式

某些情况下f(n)可能是log n等，需要用主定理的推广形式。

## 递归树法

当主定理不适用时，可以画递归树：

```
                    f(n)
                   /    \
              f(n/2)    f(n/2)
              /   \     /    \
         f(n/4) f(n/4) f(n/4) f(n/4)
         ...
```

每层的代价相加，然后求和。

### 归并排序的递归树

```
层数：log n
每层代价：O(n)
总代价：O(n log n)
```

## 代换法

先猜测答案，然后用数学归纳法证明。

### 例：证明T(n) = 2T(n/2) + n是O(n log n)

**猜测**：T(n) ≤ cn log n

**归纳**：
假设对于n/2成立：T(n/2) ≤ c(n/2)log(n/2)

则：
```
T(n) = 2T(n/2) + n
     ≤ 2 * c(n/2)log(n/2) + n
     = cn(log n - 1) + n
     = cn log n - cn + n
     ≤ cn log n  (当c ≥ 1时)
```

证毕。

## 实战技巧

### 快速判断

- 如果子问题个数a ≤ b^c（c是f(n)的指数），合并主导
- 如果a > b^c，分治主导

### 常见模式

- T(n) = T(n/2) + O(1) → O(log n)
- T(n) = 2T(n/2) + O(n) → O(n log n)
- T(n) = 2T(n/2) + O(1) → O(n)
- T(n) = 2T(n/2) + O(n²) → O(n²)

## 小结

分治算法的复杂度分析：
1. 写出递推关系式T(n) = aT(n/b) + f(n)
2. 用主定理快速求解
3. 主定理不适用时，用递归树或代换法

理解这些工具，能帮助你评估分治算法的效率。
