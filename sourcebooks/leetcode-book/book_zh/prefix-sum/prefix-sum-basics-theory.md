# 前缀和基础理论

假设你是一位超市收银员，每天结束时需要统计各个时段的销售额。老板经常问你："从上午10点到下午3点，一共卖了多少钱？""中午12点到晚上6点呢？"

如果每次都从头加一遍，不仅慢，还容易出错。聪明的你想到一个办法：提前算好从开业到每个时刻的累计销售额，这样任何时段的销售额只需要做一次减法就能得到。

这就是**前缀和**的核心思想——用空间换时间，通过预处理实现快速区间查询。

## 什么是前缀和

前缀和是一种预处理技术。给定一个数组 `nums`，我们构建一个新数组 `prefix`，其中 `prefix[i]` 表示原数组前 `i` 个元素的和。

```
原数组：  nums   = [2,  4,  1,  3,  5]
前缀和：  prefix = [0,  2,  6,  7, 10, 15]
                   ↑   ↑   ↑   ↑   ↑   ↑
                   空 [0] [0,1][0,1,2]...
```

注意，前缀和数组比原数组多一个元素，`prefix[0] = 0` 表示"前0个元素的和"，这个设计让边界处理更简洁。

有了前缀和数组，计算任意区间 `[left, right]` 的和就变成了简单的减法：

```
sum(left, right) = prefix[right + 1] - prefix[left]
```

为什么这个公式成立？让我们用数学来理解：

- `prefix[right + 1]` = `nums[0] + nums[1] + ... + nums[right]`
- `prefix[left]` = `nums[0] + nums[1] + ... + nums[left - 1]`
- 两者相减，正好剩下 `nums[left] + ... + nums[right]`

## 构建前缀和数组

实现非常直观：

```javascript
function buildPrefixSum(nums) {
    const n = nums.length;
    const prefix = new Array(n + 1).fill(0);
    
    for (let i = 0; i < n; i++) {
        prefix[i + 1] = prefix[i] + nums[i];
    }
    
    return prefix;
}
```

每个位置的值等于前一个位置的值加上当前元素，时间复杂度 O(n)，空间复杂度 O(n)。

## 区间求和

有了前缀和数组，区间求和只需 O(1) 时间：

```javascript
function rangeSum(prefix, left, right) {
    return prefix[right + 1] - prefix[left];
}
```

让我们验证一下。对于数组 `[2, 4, 1, 3, 5]`，计算区间 `[1, 3]` 的和：

```
prefix = [0, 2, 6, 7, 10, 15]
rangeSum(1, 3) = prefix[4] - prefix[1] = 10 - 2 = 8
验证：4 + 1 + 3 = 8 ✓
```

## 为什么前缀和有效

前缀和的威力在于**一次预处理，多次 O(1) 查询**。

假设需要处理 m 次区间查询：

- **暴力方法**：每次遍历区间求和，时间复杂度 O(m × n)
- **前缀和**：预处理 O(n)，每次查询 O(1)，总时间复杂度 O(n + m)

当 m 很大时，前缀和的优势非常明显。这种"预处理 + 快速查询"的思想在算法中极为常见。

## 前缀和的变体

前缀和的思想可以推广到很多场景：

**前缀积**：用乘法替代加法
```javascript
prefix[i + 1] = prefix[i] * nums[i]
```

**前缀异或**：用异或替代加法
```javascript
prefix[i + 1] = prefix[i] ^ nums[i]
```

**前缀最大值/最小值**：记录到当前位置的极值
```javascript
prefixMax[i] = Math.max(prefixMax[i - 1], nums[i])
```

这些变体的共同特点是：通过预处理，将区间查询优化为 O(1) 操作。

## 前缀和与哈希表结合

前缀和的一个强大应用是与哈希表结合，解决"和为 K 的子数组"类问题。

核心思想：如果 `prefix[j] - prefix[i] = k`，说明区间 `[i, j-1]` 的和为 k。

我们可以用哈希表记录每个前缀和出现的次数（或位置），然后遍历数组时，检查"当前前缀和 - k"是否在哈希表中出现过。

这个技巧将问题从 O(n²) 的暴力搜索优化到 O(n)，是前缀和最常见的高级应用。

## 复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 |
|-----|-----------|-----------|
| 构建前缀和 | O(n) | O(n) |
| 区间查询 | O(1) | O(1) |

## 适用场景

前缀和适合以下场景：

- **静态数组的多次区间查询**：数组不变，但需要频繁查询区间和
- **子数组和问题**：找满足特定条件的子数组
- **配合哈希表**：解决"和为 K"类问题

不适合的场景：

- 数组频繁修改：每次修改都需要重建前缀和
- 只有一次查询：预处理的开销不值得

对于需要同时支持修改和查询的场景，我们需要更高级的数据结构，如线段树或树状数组。

## 小结

前缀和是一种简单但强大的技术：

- **核心思想**：预处理累积和，空间换时间
- **关键公式**：`sum(l, r) = prefix[r + 1] - prefix[l]`
- **时间优化**：将区间查询从 O(n) 优化到 O(1)
- **扩展应用**：与哈希表结合解决子数组问题
