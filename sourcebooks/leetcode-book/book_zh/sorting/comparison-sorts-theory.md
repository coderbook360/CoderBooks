# 比较排序与非比较排序

理解这两类排序的区别，是掌握排序算法的关键。

## 比较排序

### 定义

通过**比较两个元素**来确定它们的相对顺序。

### 核心操作

```javascript
if (a < b) {
    // a 排在 b 前面
} else {
    // b 排在 a 前面
}
```

### 理论下界

比较排序的时间复杂度**下界**是O(n log n)。

**证明**：

n个元素有n!种排列。每次比较最多排除一半的可能性（二分决策树）。

需要的比较次数至少是log₂(n!)。

由Stirling公式：log₂(n!) ≈ n log₂n - n log₂e ≈ O(n log n)

所以，任何比较排序至少需要O(n log n)次比较。

### 常见比较排序

- **O(n²)**：冒泡、选择、插入
- **O(n log n)**：归并、快速、堆排序

### 适用场景

比较排序适用于**任何可比较的元素**：
- 数字
- 字符串
- 自定义对象（定义比较规则）

## 非比较排序

### 定义

不通过比较元素，而是利用**元素的特性**（如数值范围）来排序。

### 突破下界

非比较排序可以达到O(n)，突破比较排序的O(n log n)下界。

但有**适用限制**。

### 常见非比较排序

**计数排序**：
- 适用于整数，范围不太大
- 时间O(n + k)，k是数值范围

**桶排序**：
- 适用于均匀分布的数据
- 平均O(n)

**基数排序**：
- 适用于可以分解为"位"的数据
- 时间O(nk)，k是位数

## 详细对比

### 计数排序

**原理**：统计每个值出现的次数，然后按次数输出。

**适用条件**：
- 整数
- 范围已知且不太大（如0-1000）

**优点**：
- O(n + k)时间
- 稳定

**缺点**：
- 只能用于整数
- 范围大时空间浪费

### 桶排序

**原理**：把元素分到若干个"桶"中，每个桶单独排序，然后合并。

**适用条件**：
- 数据分布均匀
- 可以映射到桶

**优点**：
- 平均O(n)
- 灵活

**缺点**：
- 最坏O(n²)（所有元素在同一个桶）
- 需要额外空间

### 基数排序

**原理**：按位排序，从低位到高位（或高到低）。

**适用条件**：
- 可以分解为位（整数、字符串）
- 位数有限

**优点**：
- O(nk)时间，k通常很小
- 稳定

**缺点**：
- 实现稍复杂
- 对负数需要特殊处理

## 什么时候用非比较排序？

### 用计数排序

- 整数数组
- 范围较小（如年龄、成绩）

### 用桶排序

- 数据均匀分布
- 浮点数排序

### 用基数排序

- 大整数排序
- 字符串排序（固定长度）

## 稳定性对比

| 排序算法 | 稳定性 |
|---------|--------|
| 归并排序 | 稳定 |
| 快速排序 | 不稳定 |
| 堆排序 | 不稳定 |
| 计数排序 | 稳定 |
| 桶排序 | 取决于内部排序 |
| 基数排序 | 稳定 |

## 空间对比

| 排序算法 | 空间复杂度 |
|---------|-----------|
| 快速排序 | O(log n) |
| 堆排序 | O(1) |
| 归并排序 | O(n) |
| 计数排序 | O(k) |
| 桶排序 | O(n + k) |
| 基数排序 | O(n + k) |

## 实际选择

### 通用场景

用**比较排序**（快排、归并），通用性好。

### 特定场景

如果数据满足条件，**非比较排序**可能更快：
- 成绩排序（0-100）→ 计数排序
- 均匀分布的浮点数 → 桶排序
- 大整数或字符串 → 基数排序

## 小结

- **比较排序**：通用，下界O(n log n)
- **非比较排序**：特定场景可达O(n)

理解两者的区别和适用场景，能帮助你在实际中做出正确的选择。
