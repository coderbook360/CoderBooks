# 排序算法概览

排序是计算机科学中最基础、最重要的算法之一。

## 为什么学习排序？

排序不仅仅是把元素从小到大排列：
- **面试高频**：排序算法是面试的必考内容
- **算法基础**：很多高级算法以排序为基础
- **工程实践**：选择合适的排序对性能至关重要

## 排序算法分类

### 按比较方式分类

**比较排序**：通过比较元素来确定顺序
- 冒泡排序、选择排序、插入排序
- 归并排序、快速排序、堆排序
- 理论下界：O(n log n)

**非比较排序**：不通过比较，利用元素特性
- 计数排序、桶排序、基数排序
- 可以突破O(n log n)，但有适用限制

### 按稳定性分类

**稳定排序**：相等元素的相对顺序不变
- 冒泡排序、插入排序、归并排序
- 应用：多关键字排序

**不稳定排序**：相等元素的相对顺序可能改变
- 选择排序、快速排序、堆排序

### 按空间复杂度分类

**原地排序**：O(1)额外空间
- 冒泡、选择、插入、堆排序、快速排序

**非原地排序**：需要额外空间
- 归并排序O(n)、计数排序O(k)

## 常见排序算法对比

| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定 |
|-----|------|------|------|------|------|
| 冒泡 | O(n) | O(n²) | O(n²) | O(1) | 是 |
| 选择 | O(n²) | O(n²) | O(n²) | O(1) | 否 |
| 插入 | O(n) | O(n²) | O(n²) | O(1) | 是 |
| 希尔 | O(n log n) | O(n^1.3) | O(n²) | O(1) | 否 |
| 归并 | O(n log n) | O(n log n) | O(n log n) | O(n) | 是 |
| 快速 | O(n log n) | O(n log n) | O(n²) | O(log n) | 否 |
| 堆 | O(n log n) | O(n log n) | O(n log n) | O(1) | 否 |
| 计数 | O(n+k) | O(n+k) | O(n+k) | O(k) | 是 |
| 桶 | O(n) | O(n+k) | O(n²) | O(n+k) | 是 |
| 基数 | O(nk) | O(nk) | O(nk) | O(n+k) | 是 |

## 如何选择排序算法？

### 数据规模小（n < 50）

用**插入排序**，简单高效。

### 数据规模中等

用**快速排序**，平均性能最好。

### 需要稳定性

用**归并排序**。

### 数据范围已知且较小

用**计数排序**或**桶排序**。

### 需要O(n log n)最坏保证

用**堆排序**或**归并排序**。

## 实际工程中的选择

JavaScript的`Array.sort()`：
- V8引擎使用Timsort（归并+插入的混合）
- 对小数组用插入排序
- 对大数组用归并排序

Python的`sorted()`和`list.sort()`：
- 使用Timsort
- 稳定、O(n log n)

C++的`std::sort`：
- 通常是Introsort（快排+堆排+插入的混合）
- 不稳定，O(n log n)

## 排序的应用

### 1. 搜索优化

排序后可以用二分查找，O(n) → O(log n)。

### 2. 去重

排序后相同元素相邻，去重只需O(n)。

### 3. 中位数和第K大

排序后直接取，或用快速选择优化。

### 4. 区间问题

排序后区间问题更容易处理。

### 5. 贪心算法

很多贪心算法第一步是排序。

## 小结

排序算法是算法学习的基石：
- 理解各种排序的原理和适用场景
- 掌握时间、空间复杂度和稳定性
- 学会根据场景选择合适的算法
