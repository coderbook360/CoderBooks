# 场景7——会议上临时被点名发言

## 场景描述

技术评审会上，讨论完一个方案后，领导突然转向你：

```
"小李，你负责前端这块，你觉得这个方案怎么样？"
```

你之前一直在听，没准备发言，现在要在10秒内给出一个有价值的回应。

这是**临时发言**场景。

常见的糟糕表现：
- "啊...我觉得还行..."（没内容）
- "我觉得这个方案很好，很全面..."（空洞）
- "我没什么意见..."（没存在感）

你需要在极短时间内组织出有价值的表达。

## CPERF应用

### C - Capture 捕获（3秒）

**快速确认问题**：

被问的是什么？

```
领导问的是："你觉得这个方案怎么样？"
→ 要评价方案
→ 从前端角度评价
```

**快速调取相关信息**：

你刚才听到了什么？有什么印象？

```
刚才听到的：
- 方案要做一个新的数据展示模块
- 后端会提供新接口
- 时间比较紧，两周要完成
```

### P - Process 处理（5秒）

**快速形成观点**：

用三个问题快速形成观点：

```
1. 总体上你是支持还是有疑虑？
   → 方向我支持，但有一个担心

2. 有没有具体的点可以说？
   → 两周时间对前端来说比较紧

3. 有没有建设性建议？
   → 建议分两期做
```

**选择表达结构**：

临时发言用**三段式**：

```
第一段：总体态度
第二段：具体观点
第三段：建议或总结
```

### E - Express 表达

**实际回应**：

```
"我从前端角度说几点。

总体上，这个方案方向我是支持的，
但有一个担心。

具体来说，两周时间对前端来说比较紧。
因为数据展示模块涉及到图表和大量的交互，
如果要做得完善，两周可能做不完。

我的建议是，可以分两期做。
第一期先做核心的数据展示，两周能完成。
第二期再做高级交互功能。

我的意见就这些。"
```

**为什么这个回应好**：

```
1. 有立场："方向支持，但有担心"
2. 有具体："两周时间紧"
3. 有理由："图表和交互需要时间"
4. 有建议："分两期做"
5. 有收尾："我的意见就这些"
```

### R - Review 复盘

**会后反思**：

```
做得好的：
- 给出了明确的态度
- 有具体的观点和理由
- 提出了建设性建议

可以改进的：
- 如果能提前预判可能被问到，准备会更充分
- 可以准备一些通用的发言模板
```

### F - Form 固化

**临时发言模板**：

```markdown
## 被问"你怎么看"时的模板

### 模板1：三段式
"我说三点..."
1. 总体态度
2. 具体观点（带理由）
3. 建议或总结

### 模板2：赞同+补充式
"我同意刚才XX说的，补充一点..."
1. 表示赞同
2. 从自己角度补充

### 模板3：问题+建议式
"我有一个问题/担心..."
1. 提出问题或担心
2. 给出建议
```

## 关键要点

### 要点1：一定要有立场

```
❌ "我觉得还行..."（没有明确立场）
✅ "我支持这个方向，但有一个担心..."（有立场）

没有立场 = 没有价值
```

### 要点2：从自己的角度说

```
你被问到，是因为你有独特的视角。

"我从前端角度说..."
"从技术实现来看..."
"作为之前做过类似功能的人..."

突出你的独特价值。
```

### 要点3：有具体的点

```
❌ "方案很好，很全面"（空洞）
✅ "两周时间对前端来说比较紧"（具体）

说一个具体的点，比说十个空洞的赞美有价值。
```

### 要点4：给出建议

```
不要只提问题，要给建议。

❌ "时间太紧，做不完"
✅ "时间紧，建议分两期做"

提问题+给建议 = 有建设性
```

### 要点5：有明确的收尾

```
不要说着说着就没了。

"我的意见就这些。"
"这是我的一点看法。"

明确的收尾让听众知道你说完了。
```

## 临时发言的三个锚点

当你大脑一片空白时，用这三个锚点：

```
锚点1：我支持/不支持/有疑虑
→ 先给出总体态度

锚点2：具体来说...
→ 找一个具体的点展开

锚点3：我建议...
→ 给出一个建议
```

### 示例

**被问**："你觉得这个排期合理吗？"

**10秒内的回应**：

```
锚点1：我觉得排期整体合理，但有一个点需要关注。
锚点2：具体来说，第三周同时有两个功能上线，
       如果其中一个出问题，可能会影响另一个。
锚点3：建议把两个上线时间错开一天，降低风险。
```

## 本场景的CPERF检查清单

```markdown
□ Capture（3秒）
  - 明确被问的问题了吗？
  - 快速调取相关信息了吗？

□ Process（5秒）
  - 形成了总体态度吗？
  - 找到了具体的点吗？
  - 想到建议了吗？

□ Express
  - 有明确的立场吗？
  - 有具体的观点吗？
  - 有建设性的建议吗？
  - 有明确的收尾吗？

□ Review（会后）
  - 回顾发言效果了吗？
  - 哪些做得好，哪些可以改进？

□ Form
  - 总结了临时发言的模板吗？
  - 下次会用什么模板？
```

## 常见场景的发言模板

### 场景1：被问方案评价

```
"我从XX角度说几点。
总体上支持/有疑虑，具体来说...
建议..."
```

### 场景2：被问有没有问题

```
"我有一个问题/担心...
原因是...
建议..."

或者：
"目前我没有问题。"（如果真的没有）
```

### 场景3：被问同不同意

```
同意："我同意，补充一点..."
不同意："我有不同看法，原因是...建议..."
部分同意："大方向我同意，但有一点建议..."
```

### 场景4：被问进展

```
"目前进展正常/有一些问题。
具体来说...
下一步计划..."
```

## 小结

临时发言的核心方法：
1. 一定要有立场
2. 从自己的角度说
3. 有具体的点
4. 给出建议
5. 有明确的收尾

**三个锚点**：
```
1. 我支持/有疑虑
2. 具体来说...
3. 我建议...
```

记住：**说一个有价值的点，比说一堆空话强。**
