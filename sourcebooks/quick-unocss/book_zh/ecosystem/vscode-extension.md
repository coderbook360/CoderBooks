# 编辑器赋能：玩转 VS Code 插件

一个优秀的工具库，离不开强大的生态支持，尤其是编辑器集成。UnoCSS 的官方 VS Code 插件将“所见即所得”的开发体验提升到了一个全新的高度。它不仅仅是一个简单的代码补全工具，更是一个深度集成到你编码流程中的智能助手。

## 1. 安装插件

在 VS Code 的扩展市场中搜索 “UnoCSS” 并安装。该插件由 UnoCSS 的作者 Anthony Fu 开发，请认准官方出品。

安装完成后，插件会自动检测你项目中的 `uno.config.ts` 文件并被激活。

## 2. 核心功能一览

### 智能自动补全 (IntelliSense)

这是最基础也最重要的功能。当你开始输入一个类名时，插件会提供一个包含所有可用工具类的列表，包括：

-   所有来自你已安装预设的工具类。
-   你自己定义的快捷方式 (shortcuts)。
-   你自己定义的规则 (rules)。

![Autocomplete](https://user-images.githubusercontent.com/11247099/140622333-525a33f6-45e3-48f2-840c-953e7a332d43.gif)

### 悬停样式预览 (Hover Preview)

将鼠标悬停在任何一个 UnoCSS 工具类上，插件会立即显示这个类将生成的完整 CSS 规则。你再也无需去猜测 `p-4` 到底等于多少 `rem`，或者一个复杂的快捷方式 `btn-primary` 包含了哪些样式。

![Hover](https://user-images.githubusercontent.com/11247099/140622342-83a7109a-525a-43a8-a28a-4b52139015b3.gif)

### 颜色可视化

当你的工具类涉及到颜色时（例如 `bg-blue-500` 或 `text-red-600`），插件会在类名的旁边直接渲染出一个对应颜色的方块。这让你对设计稿的还原更加直观和精确。

### 快捷方式与规则的文档链接

如果你使用了某个预设中的工具类，悬停时插件还会提供一个直接跳转到相关 MDN 文档的链接，方便你查阅该 CSS 属性的详细信息。

### 属性化模式支持

插件对属性化模式（Attributify Mode）提供了完美的支持。无论是自动补全还是悬停预览，在 `bg="..."` 或 `p="x-4 y-2"` 这样的属性中都能正常工作。

## 3. 为什么这个插件如此重要？

原子化 CSS 的一个潜在缺点是需要记忆大量的类名。虽然 UnoCSS 的命名非常直观，但在项目初期，你仍然可能需要频繁查阅文档。

VS Code 插件通过以下方式完美地解决了这个问题：

-   **降低记忆负担**：强大的自动补全让你无需记住所有类名。
-   **提供即时反馈**：悬停预览让你对自己正在编写的样式充满信心。
-   **减少上下文切换**：你几乎所有的时间都可以停留在编辑器中，而无需切换到浏览器查文档。

它将 UnoCSS 的文档和引擎无缝地嵌入到了你的开发环境中，使得使用原子化 CSS 变得前所未有的流畅和高效。
