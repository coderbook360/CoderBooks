# 手势库的世界

欢迎来到我们冒险的第一站。在正式动手之前，让我们先花些时间，像鸟儿一样飞向高空，俯瞰一下我们即将探索的这片“手势库大陆”。

## 一切的起点：从触摸事件说起

在智能手机普及的今天，我们早已习惯了用手指与屏幕进行各种交互。这一切的背后，都源于浏览器为我们提供的一套底层 API——**触摸事件（Touch Events）**。

这套 API 主要由四个核心事件构成：

*   `touchstart`: 当手指**接触**到屏幕时触发。
*   `touchmove`: 当手指在屏幕上**滑动**时连续触发。
*   `touchend`: 当手指**离开**屏幕时触发。
*   `touchcancel`: 当触摸过程被系统打断（例如，一个来电）时触发。

通过监听这些事件，我们能够获取到手指触摸的坐标、数量、时间等原始信息。理论上，仅仅依靠这些原生事件，我们就能开发出所有的触摸交互。

## “原始”的烦恼：为何需要手势库？

然而，理论归理论。现实是，直接使用原生触摸事件进行开发，就像是直接用泥土和木材来建造一座宫殿——虽然可行，但过程极其繁琐、低效，且容易出错。

想象一下，如果我们要识别一个最简单的“双击”手势，需要做什么？

1.  监听 `touchstart`，记录下第一次点击的时间和位置。
2.  在 `touchend` 时，判断与 `touchstart` 的时间间隔是否足够短，如果太长，那就只是个单击。
3.  如果时间足够短，我们需要继续等待下一次 `touchstart`。
4.  如果在一定时间窗口内（比如 300ms），第二次 `touchstart` 发生了，并且位置与第一次相差不大，我们才能初步判断这是一个“双击”。

这还只是一个简单的“双击”。如果是更复杂的“捏合缩放”（Pinch），我们需要同时跟踪两个手指的移动轨迹，实时计算它们之间的距离变化；如果是“旋转”（Rotate），我们还需要计算两个手指连线的角度变化。

这其中充满了大量的坐标计算、时间判断、状态管理和边界条件处理。代码会变得异常复杂且难以维护。**这，就是手势库存在的意义。**

## 手势库：从“事件”到“意图”的翻译官

手势库的核心价值，在于它扮演了一个“翻译官”的角色。它接收来自底层的、零散的触摸事件（Events），经过内部复杂的计算和状态机分析，最终输出给我们的是具有明确语义的、高级的**手势（Gestures）**。

换句话说，手势库帮助我们从关心“**用户的手指在做什么**”（触摸事件），上升到关心“**用户的意图是什么**”（手势）。

| 底层事件 (Events) | 上层手势 (Gestures) |
| :--- | :--- |
| 一次短暂的 `touchstart` 和 `touchend` | `Tap` (单击) |
| 一次持续的 `touchmove` | `Pan` (拖动) |
| 一次快速的 `touchmove` | `Swipe` (轻扫) |
| 两个手指距离发生变化 | `Pinch` (捏合) |
| 两个手指角度发生变化 | `Rotate` (旋转) |

有了手势库，我们的代码会变得极其简洁和富有表现力：

```javascript
// 使用原生事件 (伪代码)
let touch1, touch2;
myElement.addEventListener('touchstart', e => { /* ...复杂的计算 */ });
myElement.addEventListener('touchmove', e => { /* ...更复杂的计算 */ });
myElement.addEventListener('touchend', e => { /* ...最终的判断 */ });

// 使用手势库 (以 Hammer.js 为例)
const hammer = new Hammer(myElement);
hammer.on('pinch rotate', e => {
  // 直接使用 scale 和 rotation 属性
  myElement.style.transform = `scale(${e.scale}) rotate(${e.rotation}deg)`;
});
```

## 著名的手势库们

在前端的世界里，涌现出了许多优秀的手势库，它们各有特点：

*   **Hammer.js**: 我们本次旅程的主角。它功能强大、API 设计成熟，是 PC 和移动端手势库的标杆之作。
*   **AlloyFinger**: 由腾讯 AlloyTeam 出品，轻量、灵活，在国内有广泛应用。
*   **ZingTouch**: 功能同样强大，提供了非常丰富的配置项和手势类型。

这些库的设计思想和实现方式各有千秋，但其核心使命都是一致的：**将开发者从繁琐的底层触摸事件中解放出来，专注于实现富有创造力的交互体验。**
